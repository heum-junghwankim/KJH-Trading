# Auto VWAP

트레이딩뷰용 Pine Script 오버레이 지표 설명서입니다.

대상 스크립트:
- [`auto-vwap.pine`](./auto-vwap.pine)

이 지표는 `현재 세션 기준 VWAP`, `최근 N거래일 기준 VWAP`, `최근 N봉 기준 VWAP`를 한 스크립트에서 바꿔 볼 수 있게 만든 버전입니다. 핵심은 `평균 체결 단가가 어디인지`, 그리고 `현재 가격이 그 기준에서 얼마나 멀리 벗어났는지`를 빠르게 읽는 데 있습니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| VWAP | 선택한 기준 구간의 거래량 가중 평균 단가입니다. |
| Band #1 | VWAP 대비 1차 확장 구간입니다. |
| Band #2 | VWAP 대비 2차 확장 구간으로, 과확장 감시에 더 가까운 선입니다. |
| 밴드 계산 타입 | `Log Standard Deviation`, `Absolute Standard Deviation` 중 선택합니다. |
| Session 모드 | 현재 세션만 표시하고 이전 세션 선은 남기지 않습니다. |
| Recent Trading Days 모드 | 최근 `N`거래일 누적 구간을 하나로 묶어 계산합니다. |
| Recent Bars 모드 | 최근 `N`봉 누적 구간을 하나의 창처럼 계산합니다. |
| VWAP 세트(두 세트) | VWAP·밴드·밴드2는 **두 세트**로 계산합니다(CEO 지시: 변곡 전용 VWAP 세트 재도입). (1) **기본 화면용** — window 길이 `Recent Bars = 50`(기본), 화면 plot·밴드·라벨의 기준. (2) **변곡 지점용** — window 길이 `[변곡] Recent Bars = 22`(기본, N선), 기본 화면 세트와 **완전 독립**이며 변곡 지점 마커 판정에만 씁니다. `Recent Bars` 앵커 모드에서만 length가 유효하며, `Session`/`Recent Trading Days` 모드는 앵커가 세션/거래일 기준이라 length 개념이 없습니다(anchored VWAP 특성상의 해석). |
| 선 굵기·가격축 | VWAP 중앙선은 **1px**이며(CEO 지시), **가격축(price scale) 라벨은 VWAP 선만** 표시합니다(밴드는 차트엔 그리되 가격축 라벨 제외). |
| 선 색상 변화 | VWAP 선·`Band #1`은 기본 흰색이고, `Enable Dynamic Line Color`를 켜면 `Band #2` 이탈 시 하단 아래 녹색·상단 위 빨간색으로 읽습니다. |
| 최외곽 선(`Band #2`) 색·두께 | `Upper/Lower Band #2`(±2σ, 최외곽 선)는 **위치가 항상 밴드2 원값(VWAP±2σ)에 고정**되며(이치모쿠 밴드처럼 위치 불변), **색도 항상 기본 단색(`activeLineColor`)으로 고정**입니다. 예전에는 청산선 안으로 들어오면 상단 빨강/하단 녹색으로 색이 바뀌었지만, **이 색 변경은 제거되었습니다(CEO 지시)**. 대신 그 "변곡 지점"은 아래 **변곡 지점 마커**로 표시합니다. 선 두께는 **기본 1px 대시(dashed)**입니다. 위 `Enable Dynamic Line Color`(녹/빨강 이탈 표시)는 VWAP 선·`Band #1`에 적용됩니다. |
| 변곡 지점(Inflection Point) 마커 | **변곡용 VWAP 세트(length=22)의 밴드2** 상/하단 선이 **변곡용 청산선**(`ipLongClear`/`ipShortClear`) 안으로 들어오고, **동시에 베이스라인 조건(AND 게이트)** 이 성립하는 순간을 마커로 표시합니다. **상단(빨강)**: 상단 밴드이 위쪽 청산선 안으로 진입(`ipUpperBandValue2 < ipShortClear`) **AND** `ipBaseLine > ipVwapValue` → 변곡용 상단 밴드2 값 위치에 숏 청산선 색 마커. **하단(녹색)**: 하단 밴드이 아래쪽 청산선 안으로 진입(`ipLowerBandValue2 > ipLongClear`) **AND** `ipBaseLine < ipVwapValue` → 변곡용 하단 밴드2 값 위치에 롱 청산선 색 마커. 마커 색은 **변곡용 청산선 색(`ipLongClearLineColor`/`ipShortClearLineColor`)과 동일**하고, 모양은 옵션(`변곡 지점 마커 모양`, 기본값 십자가)으로 조정합니다. 상단·하단은 각각 독립 판정입니다. 마커는 **변곡용 밴드2 값 좌표(`location.absolute` + `size.tiny`)** 에 그려지며, 화면에 그려지는 밴드2 선의 위치·색은 절대 바꾸지 않습니다(마커만 추가). |
| 변곡점용 베이스 라인(이치모쿠 22선, 화면 미표시) | 이치모쿠 베이스라인(Kijun-sen) `ipBaseLine = (ta.highest(high, len) + ta.lowest(low, len)) / 2`(기본 `len=22`)를 계산합니다. **화면에는 그리지 않고(hidden)**, 오직 위 변곡 마커 AND 게이트 판정에만 씁니다(베이스라인 vs 변곡 VWAP 위치 관계). 목적은 변곡점 마커 표시의 정확성 측정입니다. 기간은 `[변곡] 이치모쿠 베이스라인 기간`(기본 22)으로 조정합니다. |
| 변곡 시 청산선 수평 필터(밴드워킹 억제) | `변곡 시 청산선 수평 필터` 옵션(기본 ON)을 켜면, 변곡 시점 **청산선의 기울기가 근사 수평일 때만** 마커를 표시합니다. 청산선이 추세로 기울어 **밴드워킹 중**(위/아래로 워킹)이면 변곡 조건이 성립해도 마커를 **억제**합니다. 판정은 **최근 N봉 사이 청산선 값 변화량을 ATR로 정규화한 비율**(=ATR 대비 배수)이 임계 이하이면 수평으로 간주합니다(절대 가격차가 아닌 ATR 정규화라 종목·타임프레임 무관). **상단 마커는 변곡용 청산선(`ipShortClear`), 하단 마커는 변곡용 청산선(`ipLongClear`)의 기울기로 각각 독립 판정**하며, ATR은 변곡용 스윕 ATR 길이(`ipCeAtrLen`)에 맞춥니다. `[변곡] 수평 판정 lookback 봉수`(기본 3), `[변곡] 수평 허용 기울기(ATR 대비 배수)`(기본 0.10, 보수적)로 조정합니다. 임계를 키우면 더 기울어도 표시, 줄이면 더 엄격히 수평만 표시. 필터를 끄면 기존처럼 기울기 무관하게 표시합니다. |
| 끝 라벨 | `VWAP = 0`, 각 밴드는 VWAP 대비 `+/- %` 차이를 표시합니다. |

참고:
- 기본값은 `Anchor Mode = Recent Bars`, `Recent Bars = 50`, `Bands Calculation Mode = Log Standard Deviation`입니다. 즉 기본 화면 상태는 **최근 50봉 기준 VWAP + 로그 표준편차 밴드**이며, 변곡 지점 판정은 **변곡용 VWAP 세트(`[변곡] Recent Bars = 22`)의 밴드2**를 씁니다(기본 화면 세트와 독립).
- `Session`은 오늘 평균 단가, `Recent Trading Days`는 최근 비용대, `Recent Bars`는 최근 리듬 기준선에 더 가깝습니다.
- `Band #2` 바깥 자체가 바로 반전은 아니고, 강한 추세에서는 바깥 체류가 이어질 수 있습니다.

## 트레이딩 의미 핵심 4가지

### 1. 가격이 VWAP 위 / 아래에 있는 의미

가격이 VWAP 위에 있으면 `선택한 기준 구간 평균 단가보다 위에서 거래 중`, 아래에 있으면 `평균 단가보다 아래에서 거래 중`이라고 읽습니다.

- 방향 예측선이라기보다 현재 가격의 `비용대 기준 위치`를 보는 도구입니다.
- 실전에서는 `회복 후 유지`, `이탈 후 재진입 실패` 쪽이 더 중요합니다.

요약하면, VWAP은 `평균 단가 기준선`입니다.

### 2. VWAP 근처 왕복 의미

VWAP 근처를 계속 왕복하면 `방향성 추세`보다 `평균 단가 공방`이 진행 중일 가능성이 큽니다.

- 이 구간에서는 추격 진입보다 방향 확정 대기가 더 낫습니다.
- 다른 지표가 좋아 보여도 VWAP 공방이 길면 후속이 약할 수 있습니다.

요약하면, VWAP 공방은 `판단 유보 구간`으로 보는 편이 안전합니다.

### 3. Band #1 접근 의미

Band #1 접근은 `정상 확장` 또는 `첫 과열 구간`에 더 가깝습니다.

- 추세가 막 붙기 시작하는 구간에서도 자주 나옵니다.
- 따라서 Band #1 자체를 반전 신호로 보기보다 `확장 확인`으로 읽는 편이 좋습니다.

요약하면, Band #1은 `추세 확장 확인선`에 가깝습니다.

### 4. Band #2 접근 의미

Band #2 접근이나 이탈은 `강한 확장`, `과열 / 과매도`, `과확장 감시` 의미가 더 큽니다.

- 강한 추세에서는 바깥 체류가 이어질 수 있습니다.
- 그래서 Band #2는 바로 역방향 진입보다 `속도 조절`, `관리`, `후속 확인` 쪽에 더 유리합니다.

요약하면, Band #2는 `공격`보다 `과확장 감시`의 의미가 강합니다.

## 내부 로직

### 1. 기준 구간

`Anchor Mode`에 따라 기준이 바뀝니다.

`Session`
- 현재 세션만 보여줍니다.
- 이전 세션 선은 남기지 않습니다.

`Recent Trading Days`
- 최근 `N`거래일을 하나의 누적 구간처럼 계산합니다.
- 새 거래일이 열리면 가장 오래된 하루가 빠지고 최근 하루가 추가됩니다.

`Recent Bars`
- 최근 `N`봉을 하나의 이동 창처럼 계산합니다.
- 세션 경계를 무시하고 차트에 보이는 현재 타임프레임 기준 봉 수로 바로 계산합니다.

### 2. VWAP 계산

현재 코드는 아래 값을 누적해서 씁니다.

- `src * volume`
- `volume`
- `src^2 * volume`

이 누적값으로:
- `VWAP`
- 거래량 가중 분산
- 표준편차 기반 밴드

를 계산합니다.

누적값 자체는 window 길이와 무관한 공통값이므로, `computeVwapSet(barsLen)` 헬퍼가 window 길이(`Recent Bars` 모드의 rolling length)만 파라미터로 받아 **두 세트를 독립 계산**합니다: 기본 화면용(`length=50`)과 변곡 지점용(`length=22`). 밴드·밴드2도 각 세트의 window에서 산출되며, 화면 표시는 기본 세트, 변곡 지점 판정은 변곡용 세트의 밴드2를 씁니다. 산식(anchored VWAP + 로그/절대 표준편차 밴드)은 원본과 동일합니다.

### 3. 밴드 계산 방식

`Bands Calculation Mode`는 두 가지입니다.

`Log Standard Deviation`
- 현재 기본값입니다.
- 거래량 가중 `log(price)` 표준편차를 사용합니다.
- 밴드는 `VWAP * exp(± sigma_log * multiplier)` 형태로 계산되어, 절대 가격보다 비율 변화에 더 민감합니다.
- 가격이 0 이하인 자산에는 적합하지 않습니다.

`Absolute Standard Deviation`
- 거래량 가중 표준편차를 절대 가격 단위로 계산해 밴드 폭으로 사용합니다.

즉 이 지표는 `VWAP 기준선 + 확장 폭 방식`을 바꿔가며 읽는 구조입니다.

> **상태판 없음:** 이 지표는 우측 상단 상태 테이블을 두지 않습니다(CEO 지시로 완전 제거). 국면 판단은 이치모쿠 메인 지표의 `VWAP 상태` 셀과, 이 지표의 **변곡 지점 마커**(밴드2 선이 청산선 안으로 수렴)로 읽습니다.

## 샹들리에 청산선 + 변곡 지점(Inflection Point) 마커

이 지표는 이치모쿠 메인 지표(`advanced-ichimoku.pine`)의 **샹들리에 청산선(스윕/청산) 로직을 그대로 이식**해 값을 계산합니다. 계산은 `computeClearLines` 헬퍼 함수로 통일했고, **두 세트**로 나눕니다(CEO 지시: 변곡 전용 청산선 세트 재도입).

- **(a) 기본 청산선 세트**(`스윕/청산` 그룹) — 차트에 청산선(직선)을 직접 `plot`으로 그립니다(CEO 지시로 청산선 페인팅이 이치모쿠에서 이 지표로 이동). **화면 페인팅 전용**입니다.
- **(b) 변곡 지점용 청산선 세트**(`변곡 지점` 그룹, `ip*` 파라미터) — 화면에 청산선을 **그리지 않고** 밴드2 변곡 지점(마커) 판정에만 씁니다. 기본값은 (a)와 동일하며 독립 조정 가능합니다.

산식·파라미터 기본값은 이치모쿠와 동일합니다. **스윕라인(점선, `longCE`/`shortCE`)은 이동하지 않았습니다**(이치모쿠 전략의 진입 트리거 전용) — 이 지표는 청산선(직선)과 변곡 지점 마커만 그립니다.

- **스윕(샹들리에 트레일)**: `longCE = 최근 N봉 고가 − ATR×배수`(가격 아래, 상향 트레일), `shortCE = 최근 N봉 저가 + ATR×배수`(가격 위, 하향 트레일).
- **스윕 가격 모드(`ceMode`)**: 청산선의 계산 방식(선 종류)을 이치모쿠와 동일하게 선택합니다. **기본값은 `기본`**입니다.
  - `기본`: 표준 샹들리에(룩백 고가/저가 − ATR).
  - `거래량 가중`: 고가/저가의 **거래량 가중 이동평균(VWMA)**을 앵커로 사용 → 거래가 몰린 봉에 가중.
  - `최근봉 가중`: 고가/저가의 **선형 가중 이동평균(WMA)**을 앵커로 사용 → 최근봉에 가중(휩쏘 급등/급락에 빠르게).
  - `거래량+최근봉`: **거래량 × 선형 최근봉 가중**(`f_volRecencyMa`)을 앵커로 사용 → 두 가중을 함께.
  - 가중 모드는 앵커를 `가중 트레일 길이`(기본 10)·`가중 트레일 ATR 배수`(기본 1.5)로 계산하며, **형성 중인 현재봉을 제외하고 직전 봉(`[1]`) 기준**으로 잡아 확정값 성격을 유지합니다(신규 리페인팅 없음). 청산선(`longClear`/`shortClear`)은 이 모드로 계산된 `longCE`/`shortCE`에서 그대로 파생됩니다. **청산선 값만 모드에 따라 바뀔 뿐, 밴드2 위치·색은 절대 따라가지 않습니다.**
- **청산선(스윕 너머)**: `longClear`(longCE보다 더 아래), `shortClear`(shortCE보다 더 위). ATR 버퍼 + 옵션으로 가장 가까운 스윙 풀(피벗) 너머까지 확장하며, 스윕과의 최대 간격을 ATR 배수로 상한합니다.
- **기본 파라미터**: `스윕 가격 모드 = 기본`, `스윕 룩백 길이 = 22`, `스윕 ATR 길이 = 22`, `스윕 ATR 배수 = 3.0`, `가중 트레일 길이 = 10`, `가중 트레일 ATR 배수 = 1.5`, `청산 버퍼 ATR 배수 = 0.4`, `스윙 풀 좌우 봉수 = 5`, `청산-스윕 최대 간격 ATR 배수 = 1.0`(모두 이치모쿠와 동일).

변곡 지점(Inflection Point) 판정은 아래와 같습니다. 판정은 **변곡용 VWAP 세트(length=22)의 밴드2**(`ipUpperBandValue2`/`ipLowerBandValue2`)와 **변곡용 청산선**(`ipLongClear`/`ipShortClear`)을 쓰며, 여기에 **변곡점용 베이스 라인(이치모쿠 22선) vs 변곡 VWAP 위치 관계를 AND 게이트**로 결합합니다(둘 다 성립할 때만 마커 표시). **화면 밴드2 선의 위치·색은 언제나 밴드2 원값(VWAP±2σ) 위치 + 기본색(`activeLineColor`)에 고정**되고, 변곡 여부는 **마커로만** 표기합니다(밴드 선 위치·색 변경 없음). **상단·하단 각각 독립**입니다.

- **상단(빨강)**: 변곡용 상단 밴드(`ipUpperBandValue2`)가 위쪽 청산선 안으로 들어옴(`ipUpperBandValue2 < ipShortClear`) **AND** `ipBaseLine > ipVwapValue` → **변곡용 상단 밴드2 값 위치에 숏 청산선 색 마커**.
- **하단(녹색)**: 변곡용 하단 밴드(`ipLowerBandValue2`)가 아래쪽 청산선 안으로 들어옴(`ipLowerBandValue2 > ipLongClear`) **AND** `ipBaseLine < ipVwapValue` → **변곡용 하단 밴드2 값 위치에 롱 청산선 색 마커**.

AND 게이트의 목적은 **변곡점 마커 표시의 정확성 측정**입니다(베이스라인이 변곡 VWAP의 어느 쪽에 있는지로 방향 필터). 즉 마커는 "밴드2의 과확장 여력이 샹들리에 청산선 안으로 수렴했고, 베이스라인 위치도 정합한다(변곡 지점)"는 시각 신호일 뿐, 화면 밴드 선은 이에 따라 위치·색이 전혀 바뀌지 않습니다. 청산선은 확정값·과거참조 기반이라 신규 리페인팅이 없습니다. 마커는 `plotshape`의 `location.absolute` + `size.tiny`로 **변곡용 밴드2 값 좌표**에 그립니다.

> **밴드2 선 원값·기본색 보존:** 변곡 지점 판정은 **마커(`plotshape`)에만** 영향을 주며, 화면 밴드2 선의 plot 값은 언제나 기본 세트 원값(`upperBandValue2`/`lowerBandValue2`)이고 색은 언제나 기본색(`activeLineColor`)입니다. 이 원값은 VWAP·밴드 간격 % 라벨 등에 그대로 쓰이고, 밴드2 채움(fill, 현재 비표시)도 원값 선(`upperBand_2`/`lowerBand_2`)에 직접 붙습니다.

### 청산선(직선) 차트 표시 (이치모쿠에서 이동)

- **롱/숏 청산선(`longClear`/`shortClear`)을 `plot`으로 전 히스토리 봉에 그립니다.** ⚠️ 최근 몇 봉만 그리는 `line.new` 방식이 아니라 **전 구간 봉마다 청산선이 그려지도록**(CEO 요구: "봉 전체 다 나올 수 있어야") `plot`을 씁니다.
- **리페인팅 없음:** 청산선은 확정 성격 값(`ta.*`·과거참조 `[1]` 트레일)에서만 파생되므로, `line.new`에서 `plot`으로 그리기 방식을 바꿔도 값 자체는 불변이라 신규 리페인팅이 없습니다.
- **표시 옵션(스윕/청산 그룹, 이치모쿠에서 이식):** `청산선 표시(전 히스토리 봉)`(ON)·`청산 선 두께`(**1px**, CEO 지시)·`롱 청산선 색`(녹색 #00C853)·`숏 청산선 색`(크림슨 #FF1744). 이 청산선은 **화면 plot 전용**입니다(변곡 지점 판정은 별도 변곡용 청산선 세트를 씀).
- **스윕라인은 그리지 않습니다.** 스윕라인(점선)은 이치모쿠 전략의 진입 트리거로만 쓰이며 이 지표로 옮기지 않았습니다.

### 변곡 지점(Inflection Point) 마커 (변곡용 세트 + 베이스라인 AND 게이트)

- **밴드2 마커 판정은 변곡용 청산선 세트(`변곡 지점` 그룹, `ip*`)를 씁니다**(CEO 지시: 변곡 전용 청산선 세트 재도입). `computeClearLines` 헬퍼를 변곡용 파라미터로 별도 호출한 `ipLongClear`/`ipShortClear`로 판정하며, 밴드2도 변곡용 VWAP 세트(`ipRecentBars=22`)의 `ipUpperBandValue2`/`ipLowerBandValue2`를 씁니다.
- **베이스라인 AND 게이트**: 위 판정에 변곡점용 베이스 라인(`ipBaseLine`, 이치모쿠 22선, 화면 미표시) vs 변곡 VWAP(`ipVwapValue`) 위치 관계를 AND로 결합합니다. 상단 마커는 `ipBaseLine > ipVwapValue`, 하단 마커는 `ipBaseLine < ipVwapValue`일 때만 표시합니다.
- **마커 색 = 변곡용 청산선 색**(`[변곡] 롱/숏 청산선 색`, 기본 롱 #00C853·숏 #FF1744)입니다. 상단선 변곡 → 숏 색, 하단선 변곡 → 롱 색.
- **마커 모양 옵션(`변곡 지점 마커 모양`)** 은 `십자가`(기본)/`원`/`삼각형`/`다이아몬드`/`네모` 중 선택합니다. `plotshape`의 `style`이 `const string`만 받으므로 삼항을 `style` 인자에 직접 인라인해 컴파일 안전을 확보했습니다.
- **마커는 `location.absolute` + `size.tiny`로 변곡용 밴드2 값 좌표에 그립니다.** `location.top`/`location.bottom`은 첫 인자 값을 무시하고 차트 창 상/하단에 고정돼 선택한 모양이 원하는 자리에 나오지 않으므로 쓰지 않습니다.

## 주의사항

- 이 지표는 거래량 데이터가 있어야 정상 동작합니다. 코드에는 거래량이 없을 때 `runtime.error`도 들어 있습니다.
- `Session` 모드는 의도적으로 현재 세션만 보여줍니다.
- `Recent Trading Days`는 달력 일수가 아니라 실제 거래일 기준입니다.
- `Recent Bars`는 차트 타임프레임의 봉 수 기준이라, 타임프레임을 바꾸면 같은 숫자여도 의미가 달라집니다.
- 저유동 종목에서는 VWAP과 밴드 반응이 더 거칠 수 있습니다.
