# Auto VWAP

트레이딩뷰용 Pine Script 오버레이 지표 설명서입니다.

대상 스크립트:
- [`auto-vwap.pine`](./auto-vwap.pine)

이 지표는 `현재 세션 기준 VWAP`, `최근 N거래일 기준 VWAP`, `최근 N봉 기준 VWAP`를 한 스크립트에서 바꿔 볼 수 있게 만든 버전입니다. 핵심은 `평균 체결 단가가 어디인지`, 그리고 `현재 가격이 그 기준에서 얼마나 멀리 벗어났는지`를 빠르게 읽는 데 있습니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| VWAP | 선택한 기준 구간의 거래량 가중 평균 단가입니다. |
| Band #1 | VWAP 대비 1차 확장 구간입니다. |
| Band #2 | VWAP 대비 2차 확장 구간으로, 과확장 감시에 더 가까운 선입니다. |
| 밴드 계산 타입 | `Log Standard Deviation`, `Absolute Standard Deviation` 중 선택합니다. |
| Session 모드 | 현재 세션만 표시하고 이전 세션 선은 남기지 않습니다. |
| Recent Trading Days 모드 | 최근 `N`거래일 누적 구간을 하나로 묶어 계산합니다. |
| Recent Bars 모드 | 최근 `N`봉 누적 구간을 하나의 창처럼 계산합니다. |
| 선 굵기·가격축 | VWAP 선은 기본 **2px**이며, **가격축(price scale) 라벨은 VWAP 선만** 표시합니다(밴드는 차트엔 그리되 가격축 라벨 제외). |
| VWAP 현재값 수평선 | 트레일과 별개로, **현재 봉부터 미래 20봉까지 1px 흰색 수평선**을 VWAP 현재값에 그립니다. |
| 선 색상 변화 | 기본은 흰색이고, `Enable Dynamic Line Color`를 켜면 `Lower Band #2` 아래는 녹색, `Upper Band #2` 위는 빨간색으로 읽습니다. |
| 끝 라벨 | `VWAP = 0`, 각 밴드는 VWAP 대비 `+/- %` 차이를 표시합니다. |

참고:
- 기본값은 `Anchor Mode = Recent Bars`, `Recent Bars = 50`, `Bands Calculation Mode = Log Standard Deviation`입니다. 즉 기본 상태는 **최근 50봉 기준 VWAP + 로그 표준편차 밴드**입니다.
- `Session`은 오늘 평균 단가, `Recent Trading Days`는 최근 비용대, `Recent Bars`는 최근 리듬 기준선에 더 가깝습니다.
- `Band #2` 바깥 자체가 바로 반전은 아니고, 강한 추세에서는 바깥 체류가 이어질 수 있습니다.

## 트레이딩 의미 핵심 4가지

### 1. 가격이 VWAP 위 / 아래에 있는 의미

가격이 VWAP 위에 있으면 `선택한 기준 구간 평균 단가보다 위에서 거래 중`, 아래에 있으면 `평균 단가보다 아래에서 거래 중`이라고 읽습니다.

- 방향 예측선이라기보다 현재 가격의 `비용대 기준 위치`를 보는 도구입니다.
- 실전에서는 `회복 후 유지`, `이탈 후 재진입 실패` 쪽이 더 중요합니다.

요약하면, VWAP은 `평균 단가 기준선`입니다.

### 2. VWAP 근처 왕복 의미

VWAP 근처를 계속 왕복하면 `방향성 추세`보다 `평균 단가 공방`이 진행 중일 가능성이 큽니다.

- 이 구간에서는 추격 진입보다 방향 확정 대기가 더 낫습니다.
- 다른 지표가 좋아 보여도 VWAP 공방이 길면 후속이 약할 수 있습니다.

요약하면, VWAP 공방은 `판단 유보 구간`으로 보는 편이 안전합니다.

### 3. Band #1 접근 의미

Band #1 접근은 `정상 확장` 또는 `첫 과열 구간`에 더 가깝습니다.

- 추세가 막 붙기 시작하는 구간에서도 자주 나옵니다.
- 따라서 Band #1 자체를 반전 신호로 보기보다 `확장 확인`으로 읽는 편이 좋습니다.

요약하면, Band #1은 `추세 확장 확인선`에 가깝습니다.

### 4. Band #2 접근 의미

Band #2 접근이나 이탈은 `강한 확장`, `과열 / 과매도`, `과확장 감시` 의미가 더 큽니다.

- 강한 추세에서는 바깥 체류가 이어질 수 있습니다.
- 그래서 Band #2는 바로 역방향 진입보다 `속도 조절`, `관리`, `후속 확인` 쪽에 더 유리합니다.

요약하면, Band #2는 `공격`보다 `과확장 감시`의 의미가 강합니다.

## 내부 로직

### 1. 기준 구간

`Anchor Mode`에 따라 기준이 바뀝니다.

`Session`
- 현재 세션만 보여줍니다.
- 이전 세션 선은 남기지 않습니다.

`Recent Trading Days`
- 최근 `N`거래일을 하나의 누적 구간처럼 계산합니다.
- 새 거래일이 열리면 가장 오래된 하루가 빠지고 최근 하루가 추가됩니다.

`Recent Bars`
- 최근 `N`봉을 하나의 이동 창처럼 계산합니다.
- 세션 경계를 무시하고 차트에 보이는 현재 타임프레임 기준 봉 수로 바로 계산합니다.

### 2. VWAP 계산

현재 코드는 아래 값을 누적해서 씁니다.

- `src * volume`
- `volume`
- `src^2 * volume`

이 누적값으로:
- `VWAP`
- 거래량 가중 분산
- 표준편차 기반 밴드

를 계산합니다.

### 3. 밴드 계산 방식

`Bands Calculation Mode`는 두 가지입니다.

`Log Standard Deviation`
- 현재 기본값입니다.
- 거래량 가중 `log(price)` 표준편차를 사용합니다.
- 밴드는 `VWAP * exp(± sigma_log * multiplier)` 형태로 계산되어, 절대 가격보다 비율 변화에 더 민감합니다.
- 가격이 0 이하인 자산에는 적합하지 않습니다.

`Absolute Standard Deviation`
- 거래량 가중 표준편차를 절대 가격 단위로 계산해 밴드 폭으로 사용합니다.

즉 이 지표는 `VWAP 기준선 + 확장 폭 방식`을 바꿔가며 읽는 구조입니다.

## 우측 상단 상태판

| 행 | 내용 |
| --- | --- |
| 위치 | `VWAP 위` / `VWAP 아래` + VWAP 대비 % |
| VWAP 기울기 | `상승` / `하락` / `평탄(중심)` (각도) |
| 밴드폭 | `확장` / `수축` / `유지` |
| 밴드 상태 | `하단/상단 밴드워킹` / `하단 지지` / `상단 저항` / `중앙(공방)` |
| 포지션 | `롱 유리` / `숏 유리` / `중립` (사유 포함) |

`포지션` 판정:
- **밴드워킹(추세 추종)**
  - 진입: 수축/저변동 구간에서 **양봉이 상단 밴드(Band #2) 종가 돌파** → `롱 유리(상단 밴드워킹)`, **음봉이 하단 밴드 종가 돌파** → `숏 유리(하단 밴드워킹)`.
  - 종료: **최하단선(Lower Band #2)의 각도가 위로 꺾이면** 하단 밴드워킹 중지, **최상단선(Upper Band #2)의 각도가 아래로 꺾이면** 상단 밴드워킹 중지(=수축 간주).
  - 지속: 밴드폭 확장 + VWAP 기울기 방향 + 가격이 해당 밴드.
- **지지/저항(평균회귀 되돌림)**: 밴드 수축이거나 VWAP 평탄(중심)일 때, 하단을 받으면 `롱 유리(하단 지지)`, 상단에 막히면 `숏 유리(상단 저항)`.
- `중앙(공방)`: VWAP 평탄·가격이 밴드 중앙 → 휩쏘 위험, 관망.

## 자주 만지는 설정

| 설정 | 언제 조정하나 |
| --- | --- |
| `Anchor Mode` | 당일 기준, 최근 거래일 기준, 최근 봉수 기준을 바꾸고 싶을 때 |
| `Recent Trading Days` | 스윙 기준 범위를 넓히거나 줄일 때 |
| `Recent Bars` | 현재 타임프레임에서 최근 몇 봉을 기준으로 볼지 바꾸고 싶을 때 |
| `Source` | 기준 가격을 `hlc3`, `close` 등으로 바꾸고 싶을 때 |
| `Bands Calculation Mode` | 로그 기준 밴드와 절대가격 표준편차 밴드를 바꾸고 싶을 때 |
| `Bands Multiplier #1`, `#2` | 확장 폭을 더 넓게 또는 좁게 보고 싶을 때 |
| `Enable Dynamic Line Color` | Band #2 이탈 시 선 색상 자동 변경을 켜거나 끄고 싶을 때 |
| `VWAP·밴드 간격 % 라벨 표시` | VWAP/밴드 라인 끝에 붙는 **VWAP 대비 % 간격 라벨**(예 `+1.00%`)을 켜고 끌 때. **기본 꺼짐** |
| `Hide VWAP on 1D or Above` | 일봉 이상에서 선을 숨기고 싶을 때 |

## 주의사항

- 이 지표는 거래량 데이터가 있어야 정상 동작합니다. 코드에는 거래량이 없을 때 `runtime.error`도 들어 있습니다.
- `Session` 모드는 의도적으로 현재 세션만 보여줍니다.
- `Recent Trading Days`는 달력 일수가 아니라 실제 거래일 기준입니다.
- `Recent Bars`는 차트 타임프레임의 봉 수 기준이라, 타임프레임을 바꾸면 같은 숫자여도 의미가 달라집니다.
- 저유동 종목에서는 VWAP과 밴드 반응이 더 거칠 수 있습니다.
