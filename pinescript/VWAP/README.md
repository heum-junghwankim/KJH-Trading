# Auto VWAP

트레이딩뷰용 Pine Script 오버레이 지표 설명서입니다.

대상 스크립트:
- [`auto-vwap.pine`](./auto-vwap.pine)

이 지표는 `현재 세션 기준 VWAP`, `최근 N거래일 기준 VWAP`, `최근 N봉 기준 VWAP`를 한 스크립트에서 바꿔 볼 수 있게 만든 버전입니다. 핵심은 `평균 체결 단가가 어디인지`, 그리고 `현재 가격이 그 기준에서 얼마나 멀리 벗어났는지`를 빠르게 읽는 데 있습니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| VWAP | 선택한 기준 구간의 거래량 가중 평균 단가입니다. |
| Band #1 | VWAP 대비 1차 확장 구간입니다. |
| Band #2 | VWAP 대비 2차 확장 구간으로, 과확장 감시에 더 가까운 선입니다. |
| 밴드 계산 타입 | `Log Standard Deviation`, `Absolute Standard Deviation` 중 선택합니다. |
| Session 모드 | 현재 세션만 표시하고 이전 세션 선은 남기지 않습니다. |
| Recent Trading Days 모드 | 최근 `N`거래일 누적 구간을 하나로 묶어 계산합니다. |
| Recent Bars 모드 | 최근 `N`봉 누적 구간을 하나의 창처럼 계산합니다. |
| VWAP 세트(세 세트) | VWAP·밴드·밴드2는 **세 세트**로 계산합니다(CEO 지시: 변곡 전용 세트 재도입 + 밴드워킹 판정 전용 세트 분리). (1) **기본 화면용** — window 길이 `Recent Bars = 50`(기본), 소스 `Source`(기본 `hlc3`), 화면 plot·밴드·라벨·마커 y좌표의 기준. ⚠️ **`Recent Bars`(화면 VWAP)는 화면 표시 전용**이라, 이 값을 바꿔도 변곡 마커의 **표시 여부는 변하지 않습니다**(마커 y위치·화면 밴드·VWAP 선에만 관여). (2) **변곡 지점용(숏/롱 2세트)** — window 길이 `[변곡] Recent Bars = 14`(기본, N선, 숏·롱 공유), 소스는 **숏 변곡 = `[변곡] 숏 Source`(기본 `low`)·롱 변곡 = `[변곡] 롱 Source`(기본 `high`)** 로 분리(아래 참고), 기본 화면 세트와 **완전 독립**이며 변곡 지점 마커 판정(후보·밴드2 진입)에만 씁니다(숏 마커는 숏 세트 상단 밴드2, 롱 마커는 롱 세트 하단 밴드2). (3) **밴드워킹 판정 전용** — window 길이 `[밴드워킹] Recent Bars = 14`(기본), 소스 = 화면 `Source`(기본 `hlc3`), 방향성 밴드워킹 억제(`추세 방향 반대 마커 억제`, 기본 **OFF**) 판정에만 씁니다. 화면 세트·변곡 세트와 **모두 독립**입니다(오염 버그 교정, 아래 참고). `Recent Bars` 앵커 모드에서만 length가 유효하며, `Session`/`Recent Trading Days` 모드는 앵커가 세션/거래일 기준이라 length 개념이 없습니다(anchored VWAP 특성상의 해석). |
| 변곡 VWAP 소스(숏/롱 분리) | 변곡 지점용 VWAP를 **숏 변곡 세트·롱 변곡 세트 2개로 분리**합니다(CEO 지시). **숏 변곡 VWAP 소스 = `[변곡] 숏 Source`(기본 `low` 저가)**, **롱 변곡 VWAP 소스 = `[변곡] 롱 Source`(기본 `high` 고가)** — 두 소스를 각각 입력으로 두어 유연성을 유지합니다(`close`·`hlc3`·`hl2`·`ohlc4` 등 어떤 series든 선택 가능). **window 길이(`[변곡] Recent Bars`)·밴드 표준편차 배수·밴드 계산 모드 등 소스 외 나머지 파라미터는 두 세트가 동일**합니다. 숏 변곡 마커(상단·빨강) 판정은 **숏 변곡 세트(저가)의 상단 밴드2(`ipUpperBandValue2`)**, 롱 변곡 마커(하단·녹색) 판정은 **롱 변곡 세트(고가)의 하단 밴드2(`ipLowerBandValue2`)**를 씁니다(숏/롱 1:1 매핑, 혼선 없음). VWAP 계산 헬퍼(`computeVwapSet`)가 window 길이와 **소스를 함께 인자로 받아** 화면 세트·숏 변곡 세트·롱 변곡 세트를 각자 자기 소스로 독립 누적하므로(호출부별 `var` 누적 독립), 소스를 바꿔도 리페인팅이 발생하지 않습니다(확정봉 기준 누적). 화면 세트 소스는 기존 `Source`를 그대로 씁니다. |
| 선 굵기·가격축 | VWAP 중앙선은 **2px**이며(CEO 지시 — 지지/저항 색 규칙), **가격축(price scale) 라벨은 VWAP 선만** 표시합니다(밴드는 차트엔 그리되 가격축 라벨 제외). **VWAP 중앙선의 우측 가격축 라벨은 선색과 무관하게 항상 흰색으로 고정**됩니다(CEO 지시): 선은 동적 색(녹/빨강/흰)으로 그리되 가격축 라벨만 흰색으로 분리해 표시합니다. 지지/저항 색 옵션 ON/OFF와 무관하게 가격축 라벨은 흰색입니다. 구현은 VWAP 값을 두 plot으로 나눠, 동적 색 선은 `display.all − display.price_scale`(pane·상태줄·데이터윈도우만), 흰색 라벨은 별도 plot으로 `display.price_scale`(가격축만)에 그립니다(선은 pane에 1회만 그려져 중복선 없음). |
| VWAP 선 지지/저항 색 | `VWAP 선 지지/저항 색(구간 종가·꼬리 걸침 이탈)` 옵션(기본 **ON**)이 켜져 있으면 화면 VWAP 중앙선(`vwapValue`)의 색을 **봉 단위**로 칠합니다(CEO 확정 규칙). **"near 구간"** = VWAP와 밴드1 사이의 **동적 %(pct) 범위**입니다(밴드1 전체가 아님). 하단 near `[lowerNear, vwap]`, 상단 near `[vwap, upperNear]`. **`upperNear = vwap + pct×(upperBand1 − vwap)`**, **`lowerNear = vwap − pct×(vwap − lowerBand1)`** — 저항(하단)·지지(상단) **동일 pct 대칭**. **`pct`(구간 비율)는 밴드폭 반비례 동적**입니다: 밴드폭이 **좁을 때(저변동성) 최대 0.60(60%)**, **넓을 때(고변동성) 최소 0.20(20%)**로 수렴합니다. 산식은 `bandWidth1 = upperBand1 − lowerBand1`, `r = bandWidth1 / vwap`(가격 대비 밴드폭 비율, 종목·가격 무관), `t = clamp((r − rLow)/(rHigh − rLow), 0, 1)`, `nearPct = maxPct − t×(maxPct − minPct)`, `pct = clamp(nearPct, 0, 100)/100`입니다 — `r ≤ rLow`(저변동성·좁은밴드)면 `maxPct`(넓은 near), `r ≥ rHigh`(고변동성·넓은밴드)면 `minPct`(좁은 near), 사이는 선형보간(반비례). 기본값은 `VWAP 근처 구간 최소 %(넓을 때)=20`·`최대 %(좁을 때)=60`·`저변동성 임계=0.5%`·`고변동성 임계=2.0%`입니다(임계 뒤집힘·`vwap==0`이면 `t=0` → 최대% 폴백). 색 규약: **지지=녹 / 저항=빨강 / 중립=흰**. 규칙은 다음과 같습니다 — **저항=빨강**(`UPPER_BREAK_LINE_COLOR`) = **① `lowerNear ≤ close ≤ vwap`**(하단 near 안에 종가 마감 = VWAP이 저항) **OR ② `close < lowerNear` AND `high > lowerNear`**(종가는 하단 near 아래로 이탈했지만 **고가**가 하단 near 최저 위 = 꼬리 걸침 되밀림); **지지=녹**(`LOWER_BREAK_LINE_COLOR`) = **③ `vwap ≤ close ≤ upperNear`**(상단 near 안에 종가 마감 = VWAP이 지지) **OR ④ `close > upperNear` AND `low < upperNear`**(종가는 상단 near 위로 이탈했지만 **저가**가 상단 near 최고 아래 = 꼬리 걸침 되밀림); **그 외**(완전 이탈: `close`가 near 구간 밖이고 꼬리도 near에 안 걸침 / 워밍업 `na`) → **중립=흰**(`DEFAULT_LINE_COLOR`). **`close == vwap` 경계**: ①은 `close ≤ vwap`, ③은 `close ≥ vwap`이라 `close == vwap`이 양쪽에 걸립니다 — **저항을 먼저 평가**(코드에서 `if 저항 → else if 지지`)해 `close == vwap`이면 **저항=빨강**으로 확정합니다(부동소수라 실제 동일값은 드묾). **상호배타·미정의 구멍 없음**: ②(`close<lowerNear`)와 ①(`close≥lowerNear`) 배타, ④(`close>upperNear`)와 ③(`close≤upperNear`) 배타, 저항 계열(`close≤vwap` or `close<lowerNear`)과 지지 계열(`close≥vwap` or `close>upperNear`)은 `close==vwap` 외엔 배타이며 `close==vwap`은 위 우선순위로 저항 단일 확정 → 전 조합이 정확히 한 색으로 결정됩니다. **②④의 의미**: ②는 "종가가 하단 near 아래로 이탈했지만 아래꼬리가 아니라 **위꼬리(고가)가 하단 near에 걸쳐 되밀린** 봉"을 저항으로, ④는 "종가가 상단 near 위로 이탈했지만 **저가가 상단 near에 걸쳐 되밀린** 봉"을 지지로 인정합니다(순수 종가 이탈이라도 꼬리가 near를 관통했으면 지지/저항 신호로 봄). **엣지케이스**: 밴드1 편차 0(완전 도지)이면 `bandWidth1=0` → `r=0(≤rLow)`이라 pct=최대나 `upperNear=lowerNear=vwap`로 near 구간이 소멸 — 종가가 그 값이면 ① 해당(저항), 아니면 꼬리(high/low)로 ②④ 판정. 이 색은 봉 단위 series color라 **과거 봉은 값이 확정돼 리페인팅이 없고**, 형성 중 현재봉만 종가/고저 확정 전 색이 바뀔 수 있다가 봉 마감 시 확정됩니다(과거 변조 아님). 판정에 쓰는 `close`·`high`·`low`·`vwap`·밴드1·near는 모두 확정봉 세트값입니다. 색 우선순위: 지지/저항 규칙이 ON이면 이 규칙이 VWAP 선 색을 담당하고, OFF이면 아래 `Enable Dynamic Line Color` 거동(밴드2 이탈 녹/빨강, 기본 흰색)을 따릅니다. |
| 선 색상 변화(밴드2 이탈) | `Enable Dynamic Line Color`를 켜면 `Band #2` **이탈 시** 하단 아래 녹색·상단 위 빨간색으로 읽습니다(기본 흰색). 이 옵션은 `Band #1`에 적용되며, VWAP 선에는 위 `VWAP 선 지지/저항 색`이 **OFF일 때만** 적용됩니다(ON이면 지지/저항 규칙이 우선). 두 색 규칙은 서로 독립입니다. |
| 횡보 압축 배경(밴드2 회색) | `횡보 압축 배경(밴드2 회색) 표시` 옵션(기본 **ON**)이 켜져 있으면, **이평 리본이 좁아져 '횡보 압축'이 발생한 봉부터 다음 분출 전까지** 화면 VWAP **밴드2 상단2~하단2(`upperBandValue2`~`lowerBandValue2`) 사이 영역**을 **반투명 배경색**(기본 회색·투명도 78 = `color.new(color.gray, 78)`, "비정상 매수/매도" 배경 톤; 배경색·투명도는 `횡보 압축 배경색`·`횡보 압축 배경 투명도` 옵션으로 조정)으로 채웁니다. 이 색은 **두 선 사이 영역의 바탕(fill)에만** 칠하며 **밴드2 선(plot) 색은 변경하지 않습니다**. **전체 화면 배경이 아니라 밴드2 영역 한정**입니다. **표시 조건(즉시 ON + 분출까지 유지, ON/OFF 정합 재작성)**: 회색은 **유지 래치(`var bool chopBgOn`)** 로 켭니다. `chopBgOn`은 **압축 상태(`compNow = aCompressed`)가 성립한 봉에 즉시 ON**되고(진입 엣지 `compNow and not compNow[1]`이 아니라 압축 상태 자체로 set → **1봉 지연 없음**), **회색이 켜진 뒤(`chopBgOn`) 압축이 풀리고(`not compNow`) 밴드 폭이 다시 확장(`aExpanding`)되며 방향(`bullNow`/`bearNow`)이 잡히는 = 분출** 봉에서 OFF됩니다(`rBullExpansion`/`rBearExpansion`). 그 외(압축도 분출도 아닌 봉)에는 **직전 상태를 유지**하므로, 압축이 잠깐 풀려도 분출 전까지 회색이 유지되어 **깜빡임이 없습니다**. ⚠️ **버그 수정(회색 무한 지속)**: 직전엔 OFF 트리거를 **엣지 기반 상태머신(`rArmed`, `rChopEnter=compNow and not compNow[1]`로만 무장)** 에 의존했는데, ON은 `compNow` 레벨로 켜지는 반면 `rArmed`는 압축 도중 `aExpanding`이 한 번이라도 참이면 소진되어 **실제 분출 봉에서 분출 판정이 성립하지 않아 회색이 분출 후에도 무한 유지**되는 결함이 있었습니다. 이를 **"회색이 켜져 있다(`chopBgOn`)"는 사실 자체를 무장 상태로 쓰는 방식**으로 재작성해, ON 기준(`compNow` 레벨)과 OFF 기준(`chopBgOn && not compNow && aExpanding && 방향`)을 동일 레벨 위에서 **정합**시켰습니다. `aExpanding`은 압축이 풀린 뒤 평가되므로 무장 소진 문제가 원천 제거되어 **① 압축 봉 즉시 ON ② 분출 전까지 유지·깜빡임 없음 ③ 분출 시 확실히 OFF ④ 무한 유지 없음**을 만족합니다. 압축·분출 판정은 SUPER-ICHIMOKU 메인 지표의 리본 로직을 이식한 것입니다(아래 **횡보 압축(리본)** 참고). 확정봉(`barstate.isconfirmed`) 게이트 없이 **매 봉(형성 중 현재봉 포함) 실시간 판정**하므로 압축 봉에 **즉시 회색**·분출 봉에 **즉시 꺼짐**입니다. 미마감봉에서 회색이 켜졌다 꺼졌다 하거나 과거봉이 재계산되는 **리페인팅은 허용**합니다(CEO 명시). `showBand_2`가 꺼졌거나 밴드2가 `na`인 봉은 채우지 않습니다. **표시 전용**(변곡 마커·게이트·밴드·청산선 판정과 무관). |
| 최외곽 선(`Band #2`) 색·두께 | `Upper/Lower Band #2`(±2σ, 최외곽 선)는 **위치가 항상 밴드2 원값(VWAP±2σ)에 고정**됩니다(이치모쿠 밴드처럼 위치 불변). 색은 기본 단색(`activeLineColor`)이되, `변곡 마커 봉 밴드2 강조 색` 옵션(기본 ON)이 켜져 있으면 **변곡 마커가 뜨면서 동시에 해당 방향 기본(화면) 밴드2가 실제로 청산선 안으로 들어온 봉에서만** 그 밴드2를 마커색으로 강조합니다(상단=숏 마커색 `upperMarkerColor`, 하단=롱 마커색 `lowerMarkerColor` — **마커와 동일한 OBV 에너지 채도**: 강=선명, 약=둔탁; 그 봉만 강조·나머지 봉은 기본색, 옵션 OFF·조건 미충족이면 항상 기본색). | 비정상 매수/매도 배경 | `비정상 매수/매도 배경 표시` 옵션(기본 **ON**)이 켜져 있으면, **과매수/과매도 구간을 화면 전체 세로 배경(`bgcolor`)** 으로 칠합니다(SUPER-ICHIMOKU 메인 지표 `advanced-ichimoku.pine`의 "비정상 캔들" 판정 이식). **판정 = cRSI 과매수/과매도(도미넌트 사이클) + 볼린저 밴드 이탈(고가/저가 기준)** 두 조건 AND. **비정상 매수(=과매수)** = `cRSI ≥ 절대 과매수(70)` **AND** 고가가 볼린저 상단 밴드를 초과(초과폭 ÷ 캔들 범위 ≥ `비정상 밴드 초과 비율`) → **빨강 배경**. **비정상 매도(=과매도)** = `cRSI ≤ 절대 과매도(30)` **AND** 저가가 볼린저 하단 밴드를 이탈(이탈폭 ÷ 캔들 범위 ≥ 비율) → **파랑 배경**(CEO 명시: 매도=파랑·매수=빨강, 방향 직관과 반대지만 지시대로). 상태는 **래치**되어 다음 비정상 봉까지 유지되며, **이격도(Disparity)가 중립(되돌림 주의·확실 모두 해소)으로 복귀하면 리셋**됩니다(원본과 동일한 종료 조건). ⚠️ **자기완결 이식**: 원본은 기준 길이를 이치모쿠 베이스 라인(`basePeriods`)에 연동했으나 auto-vwap엔 그 input이 없어 **전용 input `비정상 기준 길이(베이스 라인 대응)`(기본 22, 원본 basePeriods 기본값과 동일)** 로 자기완결화했고(cRSI 사이클·볼린저·ATR·RSI·추적 lookback이 모두 이 값에 연동), 리셋 트리거인 **이격도 계산도 이치모쿠 원본 산식·input(`이격도` 그룹: MA 종류·길이·임계·통계 구간·σ 배수) 그대로 이식**했습니다(이격도는 비정상 상태 래치 리셋에만 쓰이며 테이블·별도 표시를 새로 만들지 않음). 그 외 판정 파라미터(cRSI Vibration·절대 과매수/과매도·볼린저 배수·밴드 초과 비율)·배경색·투명도(기본 82)는 원본 명칭/기본값과 동일합니다. 배경(bgcolor)은 확정 성격 래치 기반이라 원본과 동일한 리페인팅 특성입니다(형성 중 현재봉만 실시간 변동, 과거봉 확정). **횡보 압축 배경(밴드2 영역 한정 fill, 회색)과 시각 충돌 없음**: 비정상 배경은 화면 전체 세로 스트립이며 반투명(기본 82로 더 옅음)이라 밴드2 회색 fill과 겹쳐도 각자 반투명 합성될 뿐 덮어쓰지 않고, 색(회색 vs 파랑/빨강)으로도 구분됩니다. **표시 전용**(변곡 마커·게이트·밴드·청산선 판정, 매매 전략과 무관). |
| 과매수/과매도 수평선 | `과매수/과매도 수평선 표시` 옵션(기본 **ON**)이 켜져 있으면, **가장 최근 2개의 비정상 이벤트(구간) 방향 조합**으로 매수선/매도선을 표시합니다(CEO 지시). **최근 2개 이벤트에 매도(과매도)가 포함되면 매도선**을, **매수(과매수)가 포함되면 매수선**을 그립니다 — 즉 **둘 다 매도면 매도선 1개, 둘 다 매수면 매수선 1개, 매도 1 + 매수 1(순서 무관)이면 두 선 모두(총 2개)**. 각 선은 그 방향 **가장 최근 1건 구간**의 극값입니다: **매수선**(`과매수 수평선 색(매수)`, 기본 빨강)은 가장 최근 과매수 구간의 최고가(`high` 최댓값 `abnBuyRunHigh`), **매도선**(`과매도 수평선 색(매도)`, 기본 파랑)은 가장 최근 과매도 구간의 최저가(`low` 최솟값 `abnSellRunLow`). **선 색은 수평선 전용 옵션**(`abnLevelBuyColor`/`abnLevelSellColor`)이며 **비정상 배경색**(`비정상 매수/매도 배경 색`)과 **독립적으로 조정**할 수 있습니다(기본값은 배경색과 동일 색이라 기존 동작 그대로). **굵기**는 `과매수/과매도 수평선 굵기`(`abnLevelWidth`, 기본 **1**, 1~5), **선 종류**는 `과매수/과매도 수평선 점선`(`abnLevelDashed`, 기본 **ON=점선** `line.style_dotted` / OFF=실선 `line.style_solid`)으로 조정하며, 이 굵기·점선·색 옵션이 **두 선에 각각** 적용됩니다. 선은 **화면 전체 가로**(`extend=extend.both`)입니다. **가격축 라벨**: 표시되는 각 선의 값을 우측 **가격축 라벨로도 표기**합니다 — `display.price_scale` 전용 `plot`(제목 `과매수선(매수)` / `과매도선(매도)`) 2개라 차트엔 선을 다시 그리지 않고(수평선은 `abnBuyLine`/`abnSellLine` line 오브젝트가 이미 그림 → **중복 선 없음**) 가격축에 숫자만 띄우며, 라벨 색은 각 방향 전용 색과 일치합니다. **표시되는 선만 라벨**이 뜨고, 표시 토글 OFF·해당 방향 미포함·워밍업(`na`)이면 그 방향 라벨은 뜨지 않습니다. **이벤트 방향 히스토리**: 비정상 매수/매도가 **연속되는 봉을 한 "이벤트(구간)"** 로 보고, 그 방향으로 **새로 진입하는 봉(진입 엣지)** 을 새 이벤트의 시작으로 간주해 방향 히스토리를 shift합니다(`curEvtDir`=최근 이벤트 방향, `prevEvtDir`=직전 이벤트 방향; 같은 방향 연속 봉은 shift하지 않고, 방향 전환·중립 복귀 후 재진입은 각각 새 이벤트). **구간 극값 추적**: 진입 엣지에서 그 방향 누적 극값을 그 봉 값으로 초기화한 뒤 같은 방향이 지속되는 동안 `high` 최댓값(과매수) / `low` 최솟값(과매도)을 누적하므로, **진행 중인 구간도 최신 극값으로 갱신**되고 방향별 극값 변수(`abnBuyRunHigh`/`abnSellRunLow`)는 각각 그 방향 "가장 최근 1건"을 유지합니다(혼합이면 두 값이 각자 최근 매수·매도 구간). 라인 오브젝트는 **방향별 1개씩**(`var line abnBuyLine`/`abnSellLine`, 최대 2개)만 유지하며 최초 1회만 `line.new` 후 setter(`set_xy1/2`·`set_color`·`set_style`·`set_width`)로 갱신하므로 누적되지 않습니다(`max_lines_count` 무관). **표시 조건 미충족 방향**(또는 표시 OFF)은 그 선을 `line.delete`로 지우고 `na`로 되돌려 **잔상이 남지 않습니다**. 확정 성격 래치(진입 엣지·이벤트 방향 히스토리) 기반이라 신규 리페인팅이 없습니다(배경과 동일 특성). **표시 전용**입니다. |

참고:
- 기본값은 `Anchor Mode = Recent Bars`, `Recent Bars = 50`, `Bands Calculation Mode = Log Standard Deviation`입니다. 즉 기본 화면 상태는 **최근 50봉 기준 VWAP + 로그 표준편차 밴드**이며, 변곡 지점 판정은 **변곡용 VWAP 세트(`[변곡] Recent Bars = 14`)의 밴드2**를 씁니다(기본 화면 세트와 독립).
- `Session`은 오늘 평균 단가, `Recent Trading Days`는 최근 비용대, `Recent Bars`는 최근 리듬 기준선에 더 가깝습니다.
- `Band #2` 바깥 자체가 바로 반전은 아니고, 강한 추세에서는 바깥 체류가 이어질 수 있습니다.

## 트레이딩 의미 핵심 4가지

### 1. 가격이 VWAP 위 / 아래에 있는 의미

가격이 VWAP 위에 있으면 `선택한 기준 구간 평균 단가보다 위에서 거래 중`, 아래에 있으면 `평균 단가보다 아래에서 거래 중`이라고 읽습니다.

- 방향 예측선이라기보다 현재 가격의 `비용대 기준 위치`를 보는 도구입니다.
- 실전에서는 `회복 후 유지`, `이탈 후 재진입 실패` 쪽이 더 중요합니다.

요약하면, VWAP은 `평균 단가 기준선`입니다.

### 2. VWAP 근처 왕복 의미

VWAP 근처를 계속 왕복하면 `방향성 추세`보다 `평균 단가 공방`이 진행 중일 가능성이 큽니다.

- 이 구간에서는 추격 진입보다 방향 확정 대기가 더 낫습니다.
- 다른 지표가 좋아 보여도 VWAP 공방이 길면 후속이 약할 수 있습니다.

요약하면, VWAP 공방은 `판단 유보 구간`으로 보는 편이 안전합니다.

### 3. Band #1 접근 의미

Band #1 접근은 `정상 확장` 또는 `첫 과열 구간`에 더 가깝습니다.

- 추세가 막 붙기 시작하는 구간에서도 자주 나옵니다.
- 따라서 Band #1 자체를 반전 신호로 보기보다 `확장 확인`으로 읽는 편이 좋습니다.

요약하면, Band #1은 `추세 확장 확인선`에 가깝습니다.

### 4. Band #2 접근 의미

Band #2 접근이나 이탈은 `강한 확장`, `과열 / 과매도`, `과확장 감시` 의미가 더 큽니다.

- 강한 추세에서는 바깥 체류가 이어질 수 있습니다.
- 그래서 Band #2는 바로 역방향 진입보다 `속도 조절`, `관리`, `후속 확인` 쪽에 더 유리합니다.

요약하면, Band #2는 `공격`보다 `과확장 감시`의 의미가 강합니다.

- 이전 세션 선은 남기지 않습니다.

`Recent Trading Days`
- 최근 `N`거래일을 하나의 누적 구간처럼 계산합니다.
- 새 거래일이 열리면 가장 오래된 하루가 빠지고 최근 하루가 추가됩니다.

`Recent Bars`
- 최근 `N`봉을 하나의 이동 창처럼 계산합니다.
- 세션 경계를 무시하고 차트에 보이는 현재 타임프레임 기준 봉 수로 바로 계산합니다.

### 2. VWAP 계산

현재 코드는 아래 값을 누적해서 씁니다.

- `src * volume`
- `volume`
- `src^2 * volume`

이 누적값으로:
- `VWAP`
- 거래량 가중 분산
- 표준편차 기반 밴드

를 계산합니다.

`computeVwapSet(barsLen, srcArg)` 헬퍼가 window 길이(`Recent Bars` 모드의 rolling length)와 **소스**를 파라미터로 받아 **두 세트를 독립 계산**합니다: 기본 화면용(`length=50`, `src`)과 변곡 지점용(`length=14`, `[변곡] Source`). 소스별로 서로 다른 누적이 필요하므로 누적값(`cumulative*`)·세션 앵커 배열을 **함수-로컬 `var`로 내부화**했습니다 — Pine v6에서 함수 내부 `var`는 호출부마다 독립 상태를 가지므로 두 호출이 각자 자기 소스로 독립 누적합니다. 소스가 같으면(기본값 둘 다 `hlc3`) 두 인스턴스가 같은 누적 궤적을 재현하므로 예전(전역 단일 누적) 대비 값이 동일합니다(거동 불변). 밴드·밴드2도 각 세트의 window에서 산출되며, 화면 표시는 기본 세트, 변곡 지점 판정은 변곡용 세트의 밴드2를 씁니다. 산식(anchored VWAP + 로그/절대 표준편차 밴드)은 원본과 동일합니다.

### 3. 밴드 계산 방식

`Bands Calculation Mode`는 두 가지입니다.

`Log Standard Deviation`
- 현재 기본값입니다.
- 거래량 가중 `log(price)` 표준편차를 사용합니다.
- 밴드는 `VWAP * exp(± sigma_log * multiplier)` 형태로 계산되어, 절대 가격보다 비율 변화에 더 민감합니다.
- 가격이 0 이하인 자산에는 적합하지 않습니다.

`Absolute Standard Deviation`
- 거래량 가중 표준편차를 절대 가격 단위로 계산해 밴드 폭으로 사용합니다.

즉 이 지표는 `VWAP 기준선 + 확장 폭 방식`을 바꿔가며 읽는 구조입니다.

> **상태판 없음:** 이 지표는 우측 상단 상태 테이블을 두지 않습니다(CEO 지시로 완전 제거). 국면 판단은 이치모쿠 메인 지표의 `VWAP 상태` 셀과, 이 지표의 **변곡 지점 마커**(밴드2 선이 청산선 안으로 수렴)로 읽습니다. 아래 **비정상 매수/매도**는 배경색·수평선으로만 표시하며 테이블을 두지 않습니다(이격도도 래치 리셋 판정에만 쓰고 셀로 표기하지 않음).

### 4. 비정상 매수/매도(과매수/과매도) — 이치모쿠 이식

SUPER-ICHIMOKU 메인 지표(`advanced-ichimoku.pine`)의 **"비정상 캔들"** 판정을 그대로 이식했습니다. 목적은 **평균 회귀가 필요한 극단 구간(과매수/과매도)** 을 배경색과 수평선으로 한눈에 드러내는 것입니다.

- **cRSI(도미넌트 사이클 RSI)**: 사이클 길이 = `비정상 기준 길이 ÷ 2`(기본 22÷2=11)의 RSI를 Vibration(기본 10)으로 위상 보정·평활한 값. `≥ 절대 과매수(70)`면 과매수, `≤ 절대 과매도(30)`면 과매도.
- **볼린저 밴드 이탈**: `SMA(close, 기준 길이) ± stdev × 배수(2.0)`. 종가가 아닌 **고가/저가**로 판정합니다(종가 실시간 변동이 커서). 상단 초과폭(또는 하단 이탈폭) ÷ 캔들 범위가 `비정상 밴드 초과 비율(0.3)` 이상이면 이탈로 인정.
- **비정상 판정(AND)**: 비정상 매수(=과매수) = cRSI 과매수 **AND** 고가 상단 이탈, 비정상 매도(=과매도) = cRSI 과매도 **AND** 저가 하단 이탈.
- **상태 래치·리셋**: 비정상 봉이 나오면 그 방향 상태를 래치해 다음 비정상 봉까지 유지하고, **이격도(Disparity)가 중립으로 복귀하면 리셋**합니다. 이격도 = `종가 ÷ 이평(기본 SMA 20) × 100`으로, 고정 임계(±10%)와 적응형(자기 표준편차 σ×1.5) 두 기준의 충족 개수로 되돌림을 판정하며, **둘 다 미충족(중립)이면 리셋 트리거**입니다.

⚠️ **자기완결 이식 근거**: 원본은 기준 길이를 이치모쿠 베이스 라인(`basePeriods`)에, 리셋을 이치모쿠 이격도 블록에 의존했습니다. auto-vwap엔 그 둘이 없으므로 (1) 기준 길이를 **전용 input(`비정상 기준 길이`, 기본 22 = 원본 basePeriods 기본값)** 로, (2) 이격도를 **원본 산식·input 그대로 이식**해 자기완결화했습니다. 소스가 같은 기본값이면 원본과 동일한 값을 재현합니다. 매매 신호가 아니라 **표시 전용 국면 표식**이며, 임계·기간은 전략성 파라미터라 임의 변경 시 수학자 리뷰 대상입니다.

## 샹들리에 청산선 + 변곡 지점(Inflection Point) 마커

이 지표는 이치모쿠 메인 지표(`advanced-ichimoku.pine`)의 **샹들리에 청산선(스윕/청산) 로직을 그대로 이식**해 값을 계산합니다. 계산은 `computeClearLines` 헬퍼 함수로 통일했고, **파라미터 세트는 `스윕/청산` 그룹(기본 청산 설정) 하나뿐**입니다(CEO 지시 #4: 변곡 전용 청산선 세트 삭제).

- **화면 페인팅**: 청산 설정의 가격 모드(`ceMode`)로 계산한 청산선(`longClear`/`shortClear`)을 차트에 직접 `plot`으로 그립니다(청산선 페인팅이 이치모쿠에서 이 지표로 이동). **화면에 그려지는 청산선은 이 mode 1개뿐**입니다.
- **변곡 마커 후보 판정**: 같은 파라미터로 **가격 모드만 `기본`·`거래량 가중`·`최근봉 가중` 3가지로 바꿔 각각 계산**한 청산선(`shortClear_base/vol/recent`, `longClear_base/vol/recent`)을 **OR**로 확인합니다(1개 이상 성립 시 후보). mode 외 파라미터(길이·ATR·배수·가중·버퍼·피벗·최대간격)는 모두 기본 청산 설정 공통값입니다. 이 3세트는 화면에 그리지 않고 판정에만 씁니다.

산식·파라미터 기본값은 이치모쿠와 동일합니다. **스윕라인(점선, `longCE`/`shortCE`)은 이동하지 않았습니다**(이치모쿠 전략의 진입 트리거 전용) — 이 지표는 청산선(직선)과 변곡 지점 마커만 그립니다.

- **스윕(샹들리에 트레일)**: `longCE = 최근 N봉 고가 − ATR×배수`(가격 아래, 상향 트레일), `shortCE = 최근 N봉 저가 + ATR×배수`(가격 위, 하향 트레일).
- **스윕 가격 모드(`ceMode`)**: 청산선의 계산 방식(선 종류)을 이치모쿠와 동일하게 선택합니다. **기본값은 `기본`**입니다. ⚠️ `ceMode`는 **화면 페인팅 전용**이라 어떤 값을 골라도 변곡 마커 후보(3모드 OR)에는 영향이 없습니다.
  - `기본`: 표준 샹들리에(룩백 고가/저가 − ATR).
  - `거래량 가중`: 고가/저가의 **거래량 가중 이동평균(VWMA)**을 앵커로 사용 → 거래가 몰린 봉에 가중.
  - `최근봉 가중`: 고가/저가의 **선형 가중 이동평균(WMA)**을 앵커로 사용 → 최근봉에 가중(휩쏘 급등/급락에 빠르게).
  - `거래량+최근봉`: **거래량 × 선형 최근봉 가중**(`f_volRecencyMa`)을 앵커로 사용 → 두 가중을 함께.
  - 가중 모드는 앵커를 `가중 트레일 길이`(기본 10)·`가중 트레일 ATR 배수`(기본 2.0)로 계산하며, **형성 중인 현재봉을 제외하고 직전 봉(`[1]`) 기준**으로 잡아 확정값 성격을 유지합니다(신규 리페인팅 없음). 청산선(`longClear`/`shortClear`)은 이 모드로 계산된 `longCE`/`shortCE`에서 그대로 파생됩니다. **청산선 값만 모드에 따라 바뀔 뿐, 밴드2 위치·색은 절대 따라가지 않습니다.**
- **청산선(스윕 너머)**: `longClear`(longCE보다 더 아래), `shortClear`(shortCE보다 더 위). ATR 버퍼 + 옵션으로 가장 가까운 스윙 풀(피벗) 너머까지 확장하며, 스윕과의 최대 간격을 ATR 배수로 상한합니다.
- **기본 파라미터**: `스윕 가격 모드 = 기본`, `스윕 룩백 길이 = 20`, `스윕 ATR 길이 = 20`, `스윕 ATR 배수 = 3.0`, `가중 트레일 길이 = 10`, `가중 트레일 ATR 배수 = 2.0`, `청산 버퍼 ATR 배수 = 0.5`, `스윙 풀 좌우 봉수 = 5`, `청산-스윕 최대 간격 ATR 배수 = 1.0`. 산식·로직은 이치모쿠와 동일하나 일부 기본값은 이 지표에 맞게 조정돼 있습니다(`스윕 룩백/ATR 길이`는 이치모쿠 22 대비 **20**, `청산 버퍼 ATR 배수`는 **0.5**). ⚠️ `청산 버퍼 ATR 배수`(`sweepBufferMult`)는 **후보 판정용 3모드 청산선 모두에 적용**되므로 값 변경(예 1.0→0.5)은 변곡 마커 발동에 영향을 줍니다. `스윕 가격 모드`는 페인팅 전용이라 무관합니다.

변곡 지점(Inflection Point) 판정은 아래와 같습니다. 판정은 **변곡용 VWAP 세트(length=14)의 밴드2**(`ipUpperBandValue2`/`ipLowerBandValue2`)와 **기본 청산선 3모드**(`shortClear_base/vol/recent`, `longClear_base/vol/recent`)를 씁니다(밴드2가 3모드 청산선 중 1개 이상의 안으로 진입하면 후보). 여기에 아래 **변곡 시 청산선 수평 필터**, **OBV DI 우위 게이트**, **VWAP 선색 게이트**, **방향성 밴드워킹 억제**, **캔들 형태 게이트(옵션, 기본 OFF)**가 AND로 결합됩니다. **화면 밴드2 선의 위치·색은 언제나 밴드2 원값(VWAP±2σ) 위치 + 기본색(`activeLineColor`)에 고정**되고, 변곡 여부는 **마커로만** 표기합니다(밴드 선 위치·색 변경 없음). **상단·하단 각각 독립**입니다.

- **상단(빨강)**: 변곡용 상단 밴드(`ipUpperBandValue2`)가 위쪽 청산선 안으로 들어옴 — `OR_m ( ipUpperBandValue2 < shortClear_m )`, m∈{기본,거래량,최근봉} AND **DI- 우위**(`diMinus > diPlus`) AND **VWAP 선색 ≠ 녹색**(흰/빨강만 허용) AND **캔들 형태 게이트**(`shortShapeOk` — 옵션 ON일 때만 음봉 + 긴아래꼬리 반전형 제외, **OFF(기본)면 항상 통과**) → **화면 숏 청산선(`shortClear`) 위치에 숏 마커색 마커**(숏 에너지 강=선명 빨강 `shortClearLineColor`/약=둔탁 빨강 `#A57A7A`).
- **하단(녹색)**: 변곡용 하단 밴드(`ipLowerBandValue2`)가 아래쪽 청산선 안으로 들어옴 — `OR_m ( ipLowerBandValue2 > longClear_m )` AND **DI+ 우위**(`diPlus > diMinus`) AND **VWAP 선색 ≠ 빨간색**(흰/녹만 허용) AND **캔들 형태 게이트**(`longShapeOk` — 옵션 ON일 때만 양봉 + 긴위꼬리 반전형 제외, **OFF(기본)면 항상 통과**) → **화면 롱 청산선(`longClear`) 위치에 롱 마커색 마커**(롱 에너지 강=선명 녹 `longClearLineColor`/약=둔탁 녹 `#6E8F72`).

**OBV DI 우위 게이트**: 후보일 때, **OBV 기반 DI+/DI-**(OBV-ADX 보조지표 `obv-adx-extreme-background.pine`의 산식을 v6로 이식, 값 동일)로 방향 에너지 우위를 확인합니다 — 숏=`diMinus > diPlus`(DI- 우위), 롱=`diPlus > diMinus`(DI+ 우위), `na` 가드 포함(DI 값이 `na`면 억제). DI는 OBV 누적(`ta.cum(sign(Δclose)·volume)`)의 방향성 이동(`plusDM`/`minusDM`)을 OBV 표준편차의 RMA(`trur`)로 정규화한 `100·EMA(±DM, len)/trur`이며, `len = DI Length`(기본 **5**, OBV-ADX 원본 20보다 짧아 DI 우위 반응이 빠름). `ADX Smoothing`(기본 **5**, OBV-ADX 원본 `lensig` 20에 대응) input은 OBV-ADX 원본과의 옵션 정합용으로 유지하되(계산값 `obvAdx` 산출), 게이트는 DI 우위만 씁니다. 이전의 "3봉 hlc3 평균 게이트"는 이 DI 우위 게이트로 **대체(제거)** 되었습니다. ⚠️ 표시 게이트 전용 — 매매 전략(진입·청산·임계·산식)과 무관합니다.

**OBV 에너지 채도(마커 색 강/약)**: 마커 색은 OBV 방향 에너지 강도로 채도가 갈립니다(추세 강도 시각화). `diEnergy = |diPlus − diMinus|`(=OBV-ADX 상태판의 `|diDiff|`와 동일), `diEnergyAvg = sma(diEnergy, OBV 에너지 평균 길이)`, `energyFading = na … or diEnergy < diEnergyAvg or diEnergy < diEnergy[3]`. `energyStrong = not energyFading`이면 **강(선명색)**, 아니면 **약(둔탁색)** 입니다(`na`는 보수적으로 약=둔탁). 평균 창(`diEnergyAvg`)은 `OBV 에너지 평균 길이`(auto-vwap 기본 **5**, obvDiLen과 결이 맞게 짧게)로 조정하고, `diEnergy[3]`(3봉 전 대비 모멘텀)은 평균 창과 무관한 **독립 고정 룩백**(OBV-ADX 상태판과 동일한 상수 3)이라 평균이 짧아져도 시간축이 어긋나지 않습니다. 선명색은 마커 기존 색(`longClearLineColor`/`shortClearLineColor`), 둔탁색은 저장소 시그니처 팔레트(둔탁 녹 `#6E8F72`/둔탁 빨강 `#A57A7A`, OBV-ADX 상태판과 동일)입니다. 이 산식은 **OBV-ADX 보조지표 상태판(`energyFading`)의 구조를 이식**해 두 지표가 같은 방식으로 강/약을 판정합니다(평균 창 기본값만 다름). `diEnergy`는 `obvAdx` 계산에서 이미 산출한 `|diPlus − diMinus|`를 **재사용**합니다(중복 계산 제거). ⚠️ **색 채도만 바꾸며 마커 발동·모양·y좌표는 불변**이고, 확정 성격 값(`ema`/`sma`/과거참조 `[3]`)만 써 신규 리페인팅이 없습니다. 채도 강/약 판정 평균 봉 수는 `OBV 에너지 평균 길이`(기본 5) input으로 조정합니다.

**캔들 형태 게이트**(옵션 `ipShapeFilter`, 기본 **OFF** — CEO 지시): **OFF(기본)면 게이트가 항상 통과**해 원 변곡 마커가 **방향·형태 무관하게 발동**(도지 포함)합니다. **ON일 때만** 원 변곡 마커 발동에 **현재봉 방향(양봉/음봉) + 몸통↔꼬리 형태 제외**를 AND로 결합합니다. 방향 — 양봉 `ipBull = close > open`, 음봉 `ipBear = close < open`. 몸통·꼬리 — `body = math.abs(close − open)`, `upperWick = high − math.max(close, open)`, `lowerWick = math.min(close, open) − low`. **형태 반전 필터**(공통 boolean으로 추출 — **원 변곡에만** 적용): `shortRejectShape = upperWick < body*0.25 and lowerWick > body`(하방 반전형), `longRejectShape = lowerWick < body*0.25 and upperWick > body`(상방 반전형). 게이트는 flat/선색 게이트와 동일한 옵션 패턴으로 씁니다 — **숏 변곡 유효** `shortShapeOk = not ipShapeFilter or (ipBear and not shortRejectShape)`(ON이면 음봉이되 "위꼬리가 몸통의 25% 미만이고 아래꼬리가 몸통보다 긴" 하방 반전형 제외), **롱 변곡 유효** `longShapeOk = not ipShapeFilter or (ipBull and not longRejectShape)`(ON이면 양봉이되 "아래꼬리가 몸통의 25% 미만이고 위꼬리가 몸통보다 긴" 상방 반전형 제외). 이 두 조건을 각각 `upper2Marker`/`lower2Marker` 발동식에 AND로 얹습니다. **게이트 ON일 때** **도지(`close==open`)** 는 `ipBull`·`ipBear` 모두 false → 숏·롱 **양쪽 제외**(방향 미확정, 보수적 제외로 오탐 방지·자본 보존); 또한 `body=0`이면 `body*0.25=0`이라 `upperWick<0`/`lowerWick<0`이 불성립하여 형태 제외조건 자체도 미성립(안전). **OFF(기본)면 도지도 방향·형태 무관하게 통과**해 원 변곡이 발동합니다. ⚠️ **방향 게이트(양봉/음봉)와 형태 반전 필터(`shortRejectShape`/`longRejectShape`)는 원 변곡에만** 적용합니다. **연장(`longExt`/`shortExt`)에는 반전형 필터 대신 단독 꼬리 조건**을 결합합니다(CEO 지시): 숏 연장은 `lowerWick <= body`(아래꼬리가 몸통보다 길면 억제), 롱 연장은 `upperWick <= body`(위꼬리가 몸통보다 길면 억제) — **위꼬리 25% 조건 없이 반대 꼬리 vs 몸통 단독 비교**만 봅니다("더 길면 금지"=strict `>`, 코드는 `<= body`로 통과 판정=꼬리와 몸통이 같으면 표시). 연장은 고·저점이 **둘 다** 이동(롱=둘 다 상승, 숏=둘 다 하락)하는 추세 지속에서 현재봉이 반대 방향(예: 롱 연장 자리인데 음봉)이거나 반대 원 변곡 자리여도 **방향 무관하게** 마커를 넣지만, **아래꼬리가 몸통보다 긴 봉에는 숏 연장을·위꼬리가 몸통보다 긴 봉에는 롱 연장을 붙이지 않습니다**(아래 연장 항목 참고). 표시식(`showShortMarker`/`showLongMarker`)·연장 색 상속·캔들색(`barcolor`)은 이 표시식 한 곳으로 자동 정합됩니다. ⚠️ 표시 게이트 전용 — 매매 전략과 무관하며, 확정봉 OHLC 기반이라 신규 리페인팅이 없습니다.

**변곡 마커 캔들 색**(CEO 지시, `변곡 마커 캔들 색` 옵션 기본 **ON**): `barcolor()`로 **마커가 표시된 봉의 캔들을 마커색으로** 칠합니다. 조건·색 모두 마커 `plotshape`와 동일 변수 재사용 — 롱은 `showLongMarker → longMarkerColorFinal`, 숏은 `showShortMarker → shortMarkerColorFinal`(연장 봉은 상속색까지 마커와 일치). 롱·숏은 우선순위 억제(`not shortExt`/`not longExt`)로 이미 상호배타지만 `barcolor` 단일 인자이므로 방어적으로 **롱을 먼저 평가**(롱 우선, 규칙 방향 일치)하고, 마커가 없으면 `na` → **원래 캔들 색 유지**. `plotshape`은 마커 y좌표(청산선 `shortClear`/`longClear`)가 `na`면 그 봉 마커를 미표시하므로, `barcolor`에도 **청산선 `not na` 가드**를 AND로 얹어 "실제로 마커가 그려진 봉"에만 칠합니다(청산선 `na` 봉에서 캔들만 칠해지는 불일치 방지). auto-vwap에 기존 `barcolor`가 없어 신규 도입이라 다른 색 로직과 충돌이 없습니다. ⚠️ 표시 전용 — 마커 발동·y좌표·채도·계산 불변, 확정봉 마커 조건 기반이라 리페인팅이 없습니다.

마커는 "밴드2의 과확장 여력이 샹들리에 청산선 안으로 수렴했다(변곡 지점)"는 시각 신호일 뿐, 화면 밴드 선은 이에 따라 위치·색이 전혀 바뀌지 않습니다. 청산선은 확정값·과거참조 기반이라 신규 리페인팅이 없습니다. 마커의 **발동 판정은 변곡용 밴드2 + 기본 청산선 3모드(+수평 필터·OBV DI 우위 게이트·밴드워킹 억제)으로만** 하며, 기본 화면 세트의 값·표시 토글(`showBand_2` 등)은 발동 조건에 **개입하지 않습니다**. 따라서 **기본 밴드2 선을 화면에서 꺼도(`Bands Multiplier #2` 토글 OFF) 마커 판정은 독립적으로 성립**합니다. 마커가 **그려지는 y좌표는 화면에 페인트되는 기본 밴드2 값(`upperBandValue2`/`lowerBandValue2`)**이며, `plotshape`의 `location.absolute` + `size.tiny`로 기본 밴드2 선 좌표에 얹어 그립니다. **na 정합성 보정**: 특정 봉에서 기본 밴드2 값이 `na`(기본 세트 shouldPlot=false 등)이면, 조건이 참일 때 마커가 사라지지 않도록 **변곡용 밴드2 값으로 폴백**해 항상 안정적으로 찍습니다(y위치는 기본 밴드2 우선).

> **밴드2 선 원값 보존:** 변곡 지점 판정은 화면 밴드2 선의 **plot 값(위치)** 을 바꾸지 않으며, 값은 언제나 기본 세트 원값(`upperBandValue2`/`lowerBandValue2`)입니다. 이 원값은 VWAP·밴드 간격 % 라벨 등에 그대로 쓰이고, 밴드2 채움(fill, 현재 비표시)도 원값 선(`upperBand_2`/`lowerBand_2`)에 직접 붙습니다. **색**은 기본색(`activeLineColor`)이 기본이되, `변곡 마커 봉 밴드2 강조 색`(기본 ON)이 켜지면 **변곡 마커가 뜨면서 동시에 해당 방향 화면 밴드2가 청산선 안(상단 `upperBandValue2 < shortClear`, 하단 `lowerBandValue2 > longClear`)인 봉에 한해** 그 밴드2를 마커색으로 강조합니다(상단=`upperMarkerColor`, 하단=`lowerMarkerColor` — **마커와 동일한 OBV 에너지 채도 반영**: 강=선명, 약=둔탁). 마커 발동은 변곡용 밴드2(변곡 세트, window `ipRecentBars` 기본 14)를 3모드 청산선과 비교하는 반면 이 강조 게이트는 **화면 세트 밴드2**(window `recentBars` 기본 50)를 화면 청산선과 비교하므로, 마커만으로는 강조가 걸리지 않고 화면 밴드2까지 청산선 안이어야 강조됩니다. 이는 마커 `plotshape`와 **동일 색 변수(`upper/lowerMarkerColor`)를 재사용**한 봉 단위 색이라 강조색·마커색이 항상 일치하고(같은 봉에서 마커=둔탁·밴드2=선명 불일치 방지), 확정 성격 값(마커·밴드2·청산선·에너지)만 따르므로 리페인팅이 없습니다(옵션 OFF·조건 미충족이면 항상 기본색).

## 횡보 압축(리본) 이식 + 밴드2 회색 배경

SUPER-ICHIMOKU 메인 지표(`advanced-ichimoku.pine`)의 **리본 "수축/확장(압축·분출)" 판정**을 auto-vwap에 이식해, 압축 국면 동안 밴드2 영역을 회색으로 칠합니다(표시 전용).

### 압축·분출 판정 (이치모쿠 리본 A모드 이식, 자기완결)

- **자기완결 이식 — A모드(이평 리본) 단독 고정.** 이치모쿠 원본 리본은 A(이평 리본)·B(구름 두께)·A and B 3모드였으나, auto-vwap엔 구름(선행스팬 A/B)이 없으므로 **구름 의존 B모드를 제거하고 A모드만 고정**해 이식했습니다. A모드는 구름과 무관하게 이평 리본 폭만 쓰므로 **압축·분출 의미가 그대로 보존**됩니다.
- **리본 구성.** `리본 빠른 길이(전환선 대응, 기본 8)`~`리본 느린 길이(리딩B 대응, 기본 44)` 사이를 6등분한 **7개 이평**(`ribbonMa`, 종류는 `리본 이동평균 종류` 기본 SMA)으로 리본을 만들고, 그 최고~최저 폭 `aHeight = max(aTop − aBottom, mintick)`을 씁니다. 원본은 이치모쿠 input(`conversionPeriods`/`leadingSpanBPeriods`)에 길이를 맞췄으나, auto-vwap엔 그 input이 없어 **리본 전용 input(`ribbonConvPeriods`/`ribbonSpanBPeriods`, 동일 기본값 8/44)** 을 새로 선언했습니다.
- **압축(수축)** = `aHeight < rAtr × 수축 판정 ATR 배수`(기본 0.8), `rAtr = ta.atr(폭 ATR 기준, 기본 14)`. **분출(확장)** = `aHeight > aHeight[확장 판정 폭 증가 봉수]`(기본 1)이면서 방향(빠른선≥느린선=상승, 아니면 하락)이 정해질 때. 파라미터(이평 종류·ATR 기준·수축 배수·확장 봉수)는 **이치모쿠 원본과 동일 명칭/기본값**입니다.
- **압축/분출 원자값.** `compNow = aCompressed`(압축 상태), `aExpanding = aHeight > aHeight[확장 봉수]`(폭 재확장), 방향 `bullNow`(빠른선≥느린선)/`bearNow`. 회색 배경 래치는 이 원자값만 조합해 판정하며, **별도 var 상태머신(`rArmed`/`rLastExpDir`)을 두지 않습니다**(무한 유지 버그 수정으로 제거).
- **실시간 반영(CEO 지시).** 원본은 `봉마감 후 확정` 옵션(끄면 실시간)이었으나, 본 이식은 지시대로 **확정봉(`barstate.isconfirmed`) 게이트 없이 매 봉(형성 중 현재봉 포함) 상태를 갱신**합니다. 현재봉에서 압축 즉시 회색·분출 즉시 해제되며, 미마감봉 켜짐/꺼짐·과거봉 재계산 등 **리페인팅은 허용**합니다(CEO 명시).

### 밴드2 회색 배경 (즉시 ON + 분출까지 유지, ON/OFF 정합)

- **유지 래치 즉시 ON + 분출까지 유지.** 회색은 **유지 래치(`var bool chopBgOn`)** 로 켭니다. `chopBgOn`은 압축 상태(`compNow = aCompressed`)가 성립한 봉에 **즉시 ON**되고(진입 엣지 `compNow and not compNow[1]`이 아니라 **압축 상태 자체(`if compNow`)로 set** → 1봉 지연 없음), 상승/하락 분출(`rBullExpansion`/`rBearExpansion`) 봉에서 **OFF**됩니다. 그 외(압축도 분출도 아닌 봉)에는 **직전 상태를 유지**하므로, 압축이 잠깐 풀려도 분출 전까지 회색이 유지되어 **깜빡임이 없습니다**.
- **⚠️ 버그 수정(회색 무한 지속) — ON/OFF 정합.** 직전 방식은 OFF 트리거 `rBullExpansion`/`rBearExpansion`을 **엣지 기반 상태머신**(`rChopEnter = compNow and not compNow[1]`로만 무장되는 `var rArmed`)이 산출하게 했습니다. 그런데 **ON은 `compNow`(레벨)로 켜지는 반면 OFF 무장은 진입 엣지 1회에만 걸려**, 압축이 여러 봉 유지되는 동안 `aExpanding`이 한 번이라도 참이면 `rArmed`가 소진되어(`rArmed:=false`) **실제 분출 봉에서 `rArmed=false`·같은 방향이면 분출 판정이 영영 성립하지 않았습니다** → 분출 후에도 회색이 무한 유지. 수정은 **"회색이 켜져 있다(`chopBgOn`)"는 사실 자체를 무장 상태로** 사용합니다: `rBullExpansion = chopBgOn and not compNow and aExpanding and bullNow`, `rBearExpansion = chopBgOn and not compNow and aExpanding and bearNow`. 즉 **회색이 켜진 뒤 압축이 풀리고(`not compNow`) 밴드 폭이 다시 확장되며 방향이 잡히면** 분출로 보고 OFF합니다. `aExpanding`은 압축이 풀린 뒤 평가되므로 무장 소진 문제가 원천 제거되고, ON 기준과 OFF 기준이 동일한 `chopBgOn`/`compNow` 레벨 위에서 **정합**됩니다. 결과: **① 압축 봉 즉시 ON(지연 없음) ② 분출 전까지 유지·깜빡임 없음 ③ 분출 시 확실히 OFF ④ 무한 유지 없음.** `rBullExpansion`/`rBearExpansion`은 방향 표기용(회색 배경 전용)이며 다른 신호 로직에서 참조되지 않습니다.
- **밴드2 영역 한정 채움.** `fill(upperBand_2, lowerBand_2, …)`을 하나 더 두어(기존 `Bands Fill #2`는 그대로 `display.none` 유지), `showChopBg and chopBgOn and showBand_2 and not na(밴드2)`인 봉에서만 색을 **반투명 배경색**(`chopBgColor = color.new(chopBgBaseColor, chopBgTransp)`, 기본 회색·투명도 78 → 불투명 약 22%)으로, 아니면 `na`(미채움)로 지정합니다. **전체 화면 `bgcolor`가 아니라 밴드2 상단2~하단2 사이만** 칠합니다.
- **배경색·투명도 옵션.** 바탕색은 `횡보 압축 배경색`(`input.color`, 기본 회색), 투명도는 `횡보 압축 배경 투명도`(`input.int` 0~100, 기본 78)로 조정합니다. 둘은 `color.new(배경색, 투명도)`로 합쳐지며(표시 전용), 두 input 모두 `active=showChopBg`로 배경 표시가 켜졌을 때만 활성화됩니다.
- **밴드2 선(plot) 색은 절대 변경하지 않습니다.** 배경색은 두 선 '사이 영역의 바탕(fill)'만 칠하며, `upperBand_2`/`lowerBand_2`의 `color`(변곡 마커 강조/기본색 규칙)는 압축과 무관하게 그대로입니다. 기본 투명도 78은 밴드2 자체 fill(투명도 92)보다 진하게 잡아 압축 구간이 배경으로 뚜렷이 보이게 하며, 너무 옅어 선처럼 오인되지 않게 한 하한선 성격의 값입니다.
- **리페인팅 허용(CEO 명시).** `compNow`·분출 판정·`chopBgOn` 래치는 매 봉(형성 중 현재봉 포함) 실시간 갱신되므로, 압축 봉에 즉시 회색·분출 봉에 즉시 꺼집니다. 미마감봉에서 회색이 켜졌다 꺼졌다 하거나 과거봉이 재계산되는 리페인팅은 실시간 반영을 위해 허용합니다. `showBand_2`가 꺼졌거나 밴드2가 `na`(워밍업 등)인 봉은 채우지 않습니다(na 가드 유지).
- **표시 전용.** 변곡 마커·게이트·밴드·청산선의 판정·위치·색과 완전히 무관합니다.

### 청산선(직선) 차트 표시 (이치모쿠에서 이동)

- **롱/숏 청산선(`longClear`/`shortClear`)을 `plot`으로 전 히스토리 봉에 그립니다.** ⚠️ 최근 몇 봉만 그리는 `line.new` 방식이 아니라 **전 구간 봉마다 청산선이 그려지도록**(CEO 요구: "봉 전체 다 나올 수 있어야") `plot`을 씁니다.
- **리페인팅 없음:** 청산선은 확정 성격 값(`ta.*`·과거참조 `[1]` 트레일)에서만 파생되므로, `line.new`에서 `plot`으로 그리기 방식을 바꿔도 값 자체는 불변이라 신규 리페인팅이 없습니다.
- **표시 옵션(스윕/청산 그룹, 이치모쿠에서 이식):** `청산선 표시(전 히스토리 봉)`(ON)·`청산 선 두께`(**1px**, CEO 지시)·`롱 청산선 색`(녹색 #00C853)·`숏 청산선 색`(크림슨 #FF1744). 이 청산선은 **화면 plot 전용**입니다(화면에 그려지는 mode 1개). 변곡 지점 후보 판정은 같은 파라미터의 3모드 청산선을 별도 계산해 씁니다.
- **가격축 라벨 없음(CEO 지시):** 롱/숏 청산선 plot은 `display.all − display.price_scale`로 그려, **차트 선은 그대로 두되 우측 가격축의 숫자 라벨만 끕니다**(선·색·굵기·스타일 불변). 가격축 라벨은 아래 "과매수/과매도 수평선"의 값이 대신 표기됩니다.
- **스윕라인은 그리지 않습니다.** 스윕라인(점선)은 이치모쿠 전략의 진입 트리거로만 쓰이며 이 지표로 옮기지 않았습니다.

### 변곡 지점(Inflection Point) 마커 (3모드 OR 후보 + 청산선 수평 필터 + OBV DI 우위 게이트 + 밴드워킹 억제 + 캔들 형태 게이트)

- **밴드2 마커 판정은 기본 청산 설정(`스윕/청산` 그룹)을 씁니다**(CEO 지시 #4: 변곡 전용 청산선 세트 삭제). 후보 판정은 `computeClearLines` 헬퍼를 mode만 `기본`·`거래량 가중`·`최근봉 가중` 3가지로 바꿔 호출한 `shortClear_base/vol/recent`·`longClear_base/vol/recent`로 **OR** 판정하며, 밴드2는 변곡용 VWAP 세트(`ipRecentBars=14`)의 `ipUpperBandValue2`(**숏 변곡 세트, 소스=`low`**)·`ipLowerBandValue2`(**롱 변곡 세트, 소스=`high`**)를 씁니다 — 숏/롱 소스 분리(CEO 지시, 나머지 파라미터 동일).
- **변곡 시 청산선 수평 필터**(`ipUpperFlatOk`/`ipLowerFlatOk`, 대상=후보에 기여한 모드의 3모드 청산선 `shortClear_*`/`longClear_*` — `ceMode` 선택과 무관)·**OBV DI 우위 게이트**(`shortDiGate = diMinus > diPlus`/`longDiGate = diPlus > diMinus`, OBV-ADX 이식·`na` 가드)·**VWAP 선색 게이트**(`srcGateShort = vwapSRColor ≠ 녹색`/`srcGateLong = vwapSRColor ≠ 빨강`, 구간 종가+꼬리 걸침 규칙[①②저항/③④지지]의 최종 선색 기준·`enableVwapSRColor` OFF이면 미적용)·**방향성 밴드워킹 억제**(`upperBwOk`/`lowerBwOk`)·**캔들 형태 게이트**(옵션 `ipShapeFilter`, **기본 OFF**; `shortShapeOk = not ipShapeFilter or (음봉 + not shortRejectShape)`/`longShapeOk = not ipShapeFilter or (양봉 + not longRejectShape)` — **OFF면 항상 통과**, ON이면 도지 양쪽 제외·방향 게이트와 반전형 필터는 **원 변곡에만** 적용, **연장에는 옵션과 무관하게 단독 꼬리 조건**(숏=`lowerWick<=body`/롱=`upperWick<=body`) 결합)가 후보 판정에 AND로 결합됩니다. 세부 동작은 위 "지표가 보여주는 것" 표의 해당 항목을 참고하세요.
- **마커 색 = OBV 에너지 채도 반영**: 상단선 변곡 → 숏 마커색(강=선명 빨강 `shortClearLineColor` #FF1744/약=둔탁 빨강 `#A57A7A`), 하단선 변곡 → 롱 마커색(강=선명 녹 `longClearLineColor` #00C853/약=둔탁 녹 `#6E8F72`). 강/약은 `|diPlus−diMinus|` vs 그 평균(`OBV 에너지 평균 길이`)·3봉 전 대비로 판정(OBV-ADX 상태판과 동일 기준). 색 채도만 갈리며 발동·모양·y좌표는 불변.
- **마커 모양 옵션(`변곡 지점 마커 모양`)** 은 `십자가`/`원`(기본)/`삼각형`/`다이아몬드`/`네모` 중 선택합니다. `plotshape`의 `style`이 `const string`만 받으므로 삼항을 `style` 인자에 직접 인라인해 컴파일 안전을 확보했습니다.
- **마커는 `location.absolute` + `size.tiny`로 기본 화면 밴드2 값(`upperBandValue2`/`lowerBandValue2`) 좌표에 그립니다**(발동 판정은 변곡용 밴드2 + 기본 청산선 3모드, y위치는 화면 밴드2). `location.top`/`location.bottom`은 첫 인자 값을 무시하고 차트 창 상/하단에 고정돼 선택한 모양이 원하는 자리에 나오지 않으므로 쓰지 않습니다.
- **변곡 마커 연장(override)** (CEO 신규 규칙): 원 변곡 마커가 뜬 **다음 봉**에서 고·저점이 함께 이동하면(추세 지속) 동일 색 마커를 이어 표시합니다. **방향(양봉/음봉) 게이트와는 무관한 override**이지만, **단독 꼬리 조건**(원 변곡의 반전형 필터가 아닌, 반대 꼬리 vs 몸통 단독 비교)이 연장에 AND로 결합됩니다(CEO 지시 — 반대 꼬리가 몸통보다 긴 봉에는 그 방향 마커 금지, 위꼬리 25% 조건 없음).
  - **롱 연장** `longExt = lower2Marker[1] and low > low[1] and high > high[1] and upperWick <= body` — 이전봉에 롱 변곡 마커가 있었고 현재봉 **저점>이전저점 AND 고점>이전고점**(고·저점 둘 다 상승 = 추세 지속)이면, 현재봉이 **음봉이든 숏 변곡 자리든 무관하게** 롱 마커를 넣습니다(CEO: "숏 변곡 자리든 롱 넣는다"). **단 위꼬리가 몸통보다 길면(`upperWick > body`) 롱 연장 금지**(위꼬리≤몸통이면 표시, 같으면 표시).
  - **숏 연장** `shortExt = upper2Marker[1] and high < high[1] and low < low[1] and lowerWick <= body` — 이전봉에 숏 변곡 마커가 있었고 현재봉 **고점<이전고점 AND 저점<이전저점**(고·저점 둘 다 하락 = 추세 지속)이면, 현재봉이 **양봉이든 롱 변곡 자리든 무관하게** 숏 마커를 넣습니다. **단 아래꼬리가 몸통보다 길면(`lowerWick > body`) 숏 연장 금지**(아래꼬리≤몸통이면 표시, 같으면 표시). ⚠️ 도지(`body=0`): 아래꼬리>0이면 `lowerWick<=0` 불성립 → 숏 억제, 위꼬리>0이면 `upperWick<=0` 불성립 → 롱 억제(아래꼬리·위꼬리 모두 0인 4-price 도지만 통과, 극히 드묾).
  - **override 우선순위**: 연장은 같은 봉의 **반대 원 변곡보다 우선**합니다. 예로 `longExt`가 성립한 봉에 (음봉이라) `upper2Marker`(숏 원 변곡)가 동시에 떠도, 표시식의 `and not longExt`로 숏이 억제되고 롱만 그려집니다(대칭으로 `shortExt` 봉의 `lower2Marker`는 `and not shortExt`로 억제). 방향 게이트가 없어졌으므로 이 override 억제가 "숏 자리든 롱 넣는다"를 실현합니다.
  - **동시 성립 불가**: `longExt`는 고·저점 둘 다 **상승**, `shortExt`는 둘 다 **하락**을 요구하므로 같은 봉에서 동시에 참일 수 없습니다(상호 배타). 우선순위 충돌 케이스가 원천적으로 없습니다.
  - **최종 표시**: 롱 = `showLongMarker = (lower2Marker or longExt) and not shortExt and not longDownMove and not vwapResistance`, 숏 = `showShortMarker = (upper2Marker or shortExt) and not longExt and not shortUpMove and not vwapSupport`.
  - **방향-역행 억제 게이트**(CEO 신규 지시): 표시식 **최종단**에 결합해 화면에 그려지는 숏/롱 마커(원 변곡+연장) 전반에 일관 작동시킵니다. `shortUpMove = high > high[1] and low > low[1]`(고·저점 **둘 다 상승**) → **숏 마커 억제**, `longDownMove = low < low[1] and high < high[1]`(고·저점 **둘 다 하락**) → **롱 마커 억제**. 비교는 strict(`>`/`<`)라 고점 또는 저점이 이전봉과 **같으면** "둘 다 상승/하락"이 아니므로 억제하지 않습니다(경계 통과). 취지: 상승 추세 봉에서 뜨는 숏, 하락 추세 봉에서 뜨는 롱을 걸러 방향-역행 오탐을 줄입니다(자본 보존). ⚠️ **연장과 무충돌**: `shortExt`는 이미 `high<high[1] and low<low[1]`을, `longExt`는 `low>low[1] and high>high[1]`을 요구하므로 각각 `shortUpMove`/`longDownMove`와 **상호 배타** → 연장 마커는 이 게이트에 걸리지 않습니다. 따라서 실질 영향은 **원 변곡 마커에만** 미치고 override(연장 3·4) 취지를 훼손하지 않습니다. `barcolor`는 `showShort/LongMarker`를 경유하므로 캔들색도 자동 정합됩니다. ⚠️ 확정봉 OHLC·`[1]` 참조 → 신규 리페인팅 없음.
  - **VWAP 선 지지/저항 무조건 억제**(CEO 신규 지시): 표시식 **최종단**에 `and not vwapResistance`(롱)·`and not vwapSupport`(숏)를 결합해, **VWAP 선이 지지면 숏을·저항이면 롱을 무조건 억제**합니다. `vwapSupport`/`vwapResistance`는 선색 규칙과 동일한 near 구간 종가+꼬리 걸침 산식으로 판정하되 **`enableVwapSRColor`(선색 옵션)와 무관하게 항상 계산**합니다(색칠만 옵션 종속) — 그래서 옵션 OFF여도 "무조건"이 성립합니다. 최종단에 얹었으므로 **원 변곡·연장·캔들색(`barcolor`)까지 일괄 억제**되고, 밴드2 강조(`upperBand2LineColor`/`lowerBand2LineColor`)에도 각각 `and not vwapSupport`/`and not vwapResistance`를 얹어 정합시켰습니다. `close==vwap` 경계는 저항 우선 확정이라 지지/저항이 **상호배타**(방향 억제 충돌 없음). **중립**(완전 이탈·워밍업 `na`)이면 둘 다 false → 억제 없음. 기존 #6(`srcGate*`, 옵션 종속·원 변곡 단계)은 방향 매핑이 동일한 하위집합이라 **유지**했습니다(임의 제거 지양; ON에서 최종단이 항상 함께 억제하므로 중복이나 무해, OFF 원 변곡 거동 보존). ⚠️ 확정봉 close/high/low·vwap·밴드1 기반 → 신규 리페인팅 없음.
  - **연장 마커 색 = "이전봉과 동일한 색"**: 연장 봉의 색은 이전봉 마커색(`lowerMarkerColor[1]`/`upperMarkerColor[1]`)을 그대로 상속합니다(OBV 에너지 채도까지 이전봉 확정값). 현재봉 재계산이 아니라 `[1]` 참조라 이전봉과 색이 정확히 일치합니다. 원 변곡 봉은 기존대로 현재봉 색을 씁니다.
  - **연쇄(chain) 불허**: 연장 기준은 **원 변곡 마커(`lower2Marker[1]`/`upper2Marker[1]`)만** 봅니다. 연장 마커는 다음 봉 연장의 "이전봉 변곡"으로 카운트하지 **않으므로**, 연장은 원 변곡 직후 **1봉만** 이어집니다. (연장까지 트리거로 삼으면 게이트(DI·수평·밴드워킹)가 모두 꺼진 뒤에도 저점/고점만 갱신되면 무한 연쇄되어 추세 소멸 후에도 마커가 붙습니다 — 자본 보존·오탐 방지 목적의 설계 판단.)
  - ⚠️ **표시 전용**: 마커 발동 판정(`shortCand`/`longCand`·게이트·채도)·청산선·계산은 불변입니다. 연장 봉의 y좌표도 기존과 동일하게 화면 청산선(`shortClear`/`longClear`)에 얹으며, 청산선이 `na`인 봉은 미표시(기존 정책 유지). `low`/`high`·마커`[1]`은 확정봉 값이라 신규 리페인팅이 없습니다(현재봉 실시간 `low`/`high` 변동 성격은 기존 마커와 동일).

## 주의사항

- 이 지표는 거래량 데이터가 있어야 정상 동작합니다. 코드에는 거래량이 없을 때 `runtime.error`도 들어 있습니다.
- `Session` 모드는 의도적으로 현재 세션만 보여줍니다.
- `Recent Trading Days`는 달력 일수가 아니라 실제 거래일 기준입니다.
- `Recent Bars`는 차트 타임프레임의 봉 수 기준이라, 타임프레임을 바꾸면 같은 숫자여도 의미가 달라집니다.
- 저유동 종목에서는 VWAP과 밴드 반응이 더 거칠 수 있습니다.
