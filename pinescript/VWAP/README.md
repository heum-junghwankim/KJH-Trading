# Auto VWAP

트레이딩뷰용 Pine Script 오버레이 지표 설명서입니다.

대상 스크립트:
- [`auto-vwap.pine`](./auto-vwap.pine)

이 지표는 `현재 세션 기준 VWAP`, `최근 N거래일 기준 VWAP`, `최근 N봉 기준 VWAP`를 한 스크립트에서 바꿔 볼 수 있게 만든 버전입니다. 핵심은 `평균 체결 단가가 어디인지`, 그리고 `현재 가격이 그 기준에서 얼마나 멀리 벗어났는지`를 빠르게 읽는 데 있습니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| VWAP | 선택한 기준 구간의 거래량 가중 평균 단가입니다. |
| Band #1 | VWAP 대비 1차 확장 구간입니다. |
| Band #2 | VWAP 대비 2차 확장 구간으로, 과확장 감시에 더 가까운 선입니다. |
| 밴드 계산 타입 | `Log Standard Deviation`, `Absolute Standard Deviation` 중 선택합니다. |
| Session 모드 | 현재 세션만 표시하고 이전 세션 선은 남기지 않습니다. |
| Recent Trading Days 모드 | 최근 `N`거래일 누적 구간을 하나로 묶어 계산합니다. |
| Recent Bars 모드 | 최근 `N`봉 누적 구간을 하나의 창처럼 계산합니다. |
| VWAP 세트(세 세트) | VWAP·밴드·밴드2는 **세 세트**로 계산합니다(CEO 지시: 변곡 전용 세트 재도입 + 밴드워킹 판정 전용 세트 분리). (1) **기본 화면용** — window 길이 `Recent Bars = 50`(기본), 소스 `Source`(기본 `hlc3`), 화면 plot·밴드·라벨·마커 y좌표의 기준. ⚠️ **`Recent Bars`(화면 VWAP)는 화면 표시 전용**이라, 이 값을 바꿔도 변곡 마커의 **표시 여부는 변하지 않습니다**(마커 y위치·화면 밴드·VWAP 선에만 관여). (2) **변곡 지점용** — window 길이 `[변곡] Recent Bars = 20`(기본, N선), 소스 `[변곡] Source`(기본 `hlc3`, 아래 참고), 기본 화면 세트와 **완전 독립**이며 변곡 지점 마커 판정(후보·밴드2 진입)에만 씁니다. (3) **밴드워킹 판정 전용** — window 길이 `[밴드워킹] Recent Bars = 50`(기본), 소스 = 화면 `Source`(기본 `hlc3`), 방향성 밴드워킹 억제(`추세 방향 반대 마커 억제`) 판정에만 씁니다. 화면 세트·변곡 세트와 **모두 독립**입니다(오염 버그 교정, 아래 참고). `Recent Bars` 앵커 모드에서만 length가 유효하며, `Session`/`Recent Trading Days` 모드는 앵커가 세션/거래일 기준이라 length 개념이 없습니다(anchored VWAP 특성상의 해석). |
| 변곡 VWAP 소스 선택 | 변곡 지점용 VWAP 세트의 소스를 `[변곡] Source` 입력으로 **독립 선택**합니다(CEO 지시). 기본값은 `hlc3`(고저종÷3)로, 기존 동작과 **완전히 동일**합니다(거동 불변). `close`(종가)·`hl2`·`ohlc4` 등 어떤 series든 고를 수 있습니다. VWAP 계산 헬퍼(`computeVwapSet`)가 window 길이와 **소스를 함께 인자로 받아** 화면 세트와 변곡 세트를 각자 자기 소스로 독립 누적하므로, 소스를 바꿔도 리페인팅이 발생하지 않습니다(확정봉 기준 누적). 화면 세트 소스는 기존 `Source`를 그대로 씁니다. |
| 선 굵기·가격축 | VWAP 중앙선은 **2px**이며(CEO 지시 — 지지/저항 색 규칙), **가격축(price scale) 라벨은 VWAP 선만** 표시합니다(밴드는 차트엔 그리되 가격축 라벨 제외). **VWAP 중앙선의 우측 가격축 라벨은 선색과 무관하게 항상 흰색으로 고정**됩니다(CEO 지시): 선은 동적 색(녹/빨강/흰)으로 그리되 가격축 라벨만 흰색으로 분리해 표시합니다. 지지/저항 색 옵션 ON/OFF와 무관하게 가격축 라벨은 흰색입니다. 구현은 VWAP 값을 두 plot으로 나눠, 동적 색 선은 `display.all − display.price_scale`(pane·상태줄·데이터윈도우만), 흰색 라벨은 별도 plot으로 `display.price_scale`(가격축만)에 그립니다(선은 pane에 1회만 그려져 중복선 없음). |
| VWAP 선 지지/저항 색 | `VWAP 선 지지/저항 색(구간 종가·꼬리 걸침 이탈)` 옵션(기본 **ON**)이 켜져 있으면 화면 VWAP 중앙선(`vwapValue`)의 색을 **봉 단위**로 칠합니다(CEO 확정 규칙). **"near 구간"** = VWAP와 밴드1 사이의 **동적 %(pct) 범위**입니다(밴드1 전체가 아님). 하단 near `[lowerNear, vwap]`, 상단 near `[vwap, upperNear]`. **`upperNear = vwap + pct×(upperBand1 − vwap)`**, **`lowerNear = vwap − pct×(vwap − lowerBand1)`** — 저항(하단)·지지(상단) **동일 pct 대칭**. **`pct`(구간 비율)는 밴드폭 반비례 동적**입니다: 밴드폭이 **좁을 때(저변동성) 최대 0.60(60%)**, **넓을 때(고변동성) 최소 0.20(20%)**로 수렴합니다. 산식은 `bandWidth1 = upperBand1 − lowerBand1`, `r = bandWidth1 / vwap`(가격 대비 밴드폭 비율, 종목·가격 무관), `t = clamp((r − rLow)/(rHigh − rLow), 0, 1)`, `nearPct = maxPct − t×(maxPct − minPct)`, `pct = clamp(nearPct, 0, 100)/100`입니다 — `r ≤ rLow`(저변동성·좁은밴드)면 `maxPct`(넓은 near), `r ≥ rHigh`(고변동성·넓은밴드)면 `minPct`(좁은 near), 사이는 선형보간(반비례). 기본값은 `VWAP 근처 구간 최소 %(넓을 때)=20`·`최대 %(좁을 때)=60`·`저변동성 임계=0.5%`·`고변동성 임계=2.0%`입니다(임계 뒤집힘·`vwap==0`이면 `t=0` → 최대% 폴백). 색 규약: **지지=녹 / 저항=빨강 / 중립=흰**. 규칙은 다음과 같습니다 — **저항=빨강**(`UPPER_BREAK_LINE_COLOR`) = **① `lowerNear ≤ close ≤ vwap`**(하단 near 안에 종가 마감 = VWAP이 저항) **OR ② `close < lowerNear` AND `high > lowerNear`**(종가는 하단 near 아래로 이탈했지만 **고가**가 하단 near 최저 위 = 꼬리 걸침 되밀림); **지지=녹**(`LOWER_BREAK_LINE_COLOR`) = **③ `vwap ≤ close ≤ upperNear`**(상단 near 안에 종가 마감 = VWAP이 지지) **OR ④ `close > upperNear` AND `low < upperNear`**(종가는 상단 near 위로 이탈했지만 **저가**가 상단 near 최고 아래 = 꼬리 걸침 되밀림); **그 외**(완전 이탈: `close`가 near 구간 밖이고 꼬리도 near에 안 걸침 / 워밍업 `na`) → **중립=흰**(`DEFAULT_LINE_COLOR`). **`close == vwap` 경계**: ①은 `close ≤ vwap`, ③은 `close ≥ vwap`이라 `close == vwap`이 양쪽에 걸립니다 — **저항을 먼저 평가**(코드에서 `if 저항 → else if 지지`)해 `close == vwap`이면 **저항=빨강**으로 확정합니다(부동소수라 실제 동일값은 드묾). **상호배타·미정의 구멍 없음**: ②(`close<lowerNear`)와 ①(`close≥lowerNear`) 배타, ④(`close>upperNear`)와 ③(`close≤upperNear`) 배타, 저항 계열(`close≤vwap` or `close<lowerNear`)과 지지 계열(`close≥vwap` or `close>upperNear`)은 `close==vwap` 외엔 배타이며 `close==vwap`은 위 우선순위로 저항 단일 확정 → 전 조합이 정확히 한 색으로 결정됩니다. **②④의 의미**: ②는 "종가가 하단 near 아래로 이탈했지만 아래꼬리가 아니라 **위꼬리(고가)가 하단 near에 걸쳐 되밀린** 봉"을 저항으로, ④는 "종가가 상단 near 위로 이탈했지만 **저가가 상단 near에 걸쳐 되밀린** 봉"을 지지로 인정합니다(순수 종가 이탈이라도 꼬리가 near를 관통했으면 지지/저항 신호로 봄). **엣지케이스**: 밴드1 편차 0(완전 도지)이면 `bandWidth1=0` → `r=0(≤rLow)`이라 pct=최대나 `upperNear=lowerNear=vwap`로 near 구간이 소멸 — 종가가 그 값이면 ① 해당(저항), 아니면 꼬리(high/low)로 ②④ 판정. 이 색은 봉 단위 series color라 **과거 봉은 값이 확정돼 리페인팅이 없고**, 형성 중 현재봉만 종가/고저 확정 전 색이 바뀔 수 있다가 봉 마감 시 확정됩니다(과거 변조 아님). 판정에 쓰는 `close`·`high`·`low`·`vwap`·밴드1·near는 모두 확정봉 세트값입니다. 색 우선순위: 지지/저항 규칙이 ON이면 이 규칙이 VWAP 선 색을 담당하고, OFF이면 아래 `Enable Dynamic Line Color` 거동(밴드2 이탈 녹/빨강, 기본 흰색)을 따릅니다. |
| 선 색상 변화(밴드2 이탈) | `Enable Dynamic Line Color`를 켜면 `Band #2` **이탈 시** 하단 아래 녹색·상단 위 빨간색으로 읽습니다(기본 흰색). 이 옵션은 `Band #1`에 적용되며, VWAP 선에는 위 `VWAP 선 지지/저항 색`이 **OFF일 때만** 적용됩니다(ON이면 지지/저항 규칙이 우선). 두 색 규칙은 서로 독립입니다. |
| 최외곽 선(`Band #2`) 색·두께 | `Upper/Lower Band #2`(±2σ, 최외곽 선)는 **위치가 항상 밴드2 원값(VWAP±2σ)에 고정**됩니다(이치모쿠 밴드처럼 위치 불변). 색은 기본 단색(`activeLineColor`)이되, `변곡 마커 봉 밴드2 강조 색` 옵션(기본 ON)이 켜져 있으면 **변곡 마커가 뜨면서 동시에 해당 방향 기본(화면) 밴드2가 실제로 청산선 안으로 들어온 봉에서만** 그 밴드2를 마커색으로 강조합니다(상단=숏 마커색 `upperMarkerColor`, 하단=롱 마커색 `lowerMarkerColor` — **마커와 동일한 OBV 에너지 채도**: 강=선명, 약=둔탁; 그 봉만 강조·나머지 봉은 기본색, 옵션 OFF·조건 미충족이면 항상 기본색). "청산선 안" 판정은 **화면에 그려지는 청산선**(`shortClear`/`longClear`) 기준으로 상단 `upperBandValue2 < shortClear`, 하단 `lowerBandValue2 > longClear`입니다(마커 후보의 부등호와 동일). 마커 발동 판정은 **변곡용 밴드2**(변곡 세트, window `ipRecentBars` 기본 10)를 3모드 청산선과 비교하는 반면 강조 게이트는 **화면 밴드2**(화면 세트, window `recentBars` 기본 50)를 화면 청산선과 비교하므로, **마커는 떴지만 화면 밴드2는 청산선 안이 아닌 봉**에서는 강조가 걸리지 않습니다(아래 상세 참고). 예전의 "청산선 안 진입 시 상단 빨강/하단 녹색 상시 색변경"은 제거되었고(CEO 지시), 이 강조는 **조건을 만족한 확정봉에 한정된 봉 단위 색**이라 리페인팅이 없습니다. 그 "변곡 지점" 자체는 아래 **변곡 지점 마커**로도 표시합니다. 선 두께는 **기본 1px 대시(dashed)**입니다. 위 `Enable Dynamic Line Color`(녹/빨강 이탈 표시)는 VWAP 선·`Band #1`에 적용됩니다. |
| 변곡 지점(Inflection Point) 마커 | **변곡용 VWAP 세트(length=20)의 밴드2** 상/하단 선이 **기본 청산선**(`스윕/청산` 그룹, `shortClear`/`longClear`) 안으로 들어오는 순간을 마커로 표시합니다. **후보 판정은 3모드 OR**입니다(CEO 지시 #1·#2): 청산 설정의 가격 모드를 `기본`·`거래량 가중`·`최근봉 가중` 3가지로 각각 계산한 청산선 중 **1개 이상**을 밴드2가 뚫으면 후보. **상단(빨강)**: `OR_m ( ipUpperBandValue2 < shortClear_m )` → 기본 화면 상단 밴드2 값 위치에 숏 청산선 색 마커. **하단(녹색)**: `OR_m ( ipLowerBandValue2 > longClear_m )` → 기본 화면 하단 밴드2 값 위치에 롱 청산선 색 마커. 여기에 아래 **변곡 시 청산선 수평 필터**, **OBV DI 우위 게이트**, **방향성 밴드워킹 억제**가 AND로 결합됩니다. 마커 색은 **OBV 에너지 강도에 따라 채도(강/약)로 나뉩니다**: 롱(하단) 마커는 롱 에너지 **강 → 선명 녹(`longClearLineColor`)**, 약 → **둔탁 녹(`#6E8F72`)**; 숏(상단) 마커는 숏 에너지 **강 → 선명 빨강(`shortClearLineColor`)**, 약 → **둔탁 빨강(`#A57A7A`)**. 강/약 판정은 **OBV-ADX 보조지표 상태판과 동일 산식** — OBV 방향 격차 `|diPlus−diMinus|`가 그 평균(`OBV 에너지 평균 길이`, auto-vwap 기본 5·OBV-ADX 기본 14) 이상이고 3봉 전보다 줄지 않았으면 **강(선명)**, 아니면 **약(둔탁)**(평균 창만 짧게 두고, 3봉 전 `[3]` 모멘텀 룩백은 평균과 무관한 동일 고정 상수). 채도는 색만 바꾸며 **마커 발동·모양·y좌표는 불변**입니다. 모양은 옵션(`변곡 지점 마커 모양`, 기본값 원)으로 조정합니다. 상단·하단은 각각 독립 판정입니다. 화면에 페인팅되는 청산선은 기존대로 청산 설정의 mode(`ceMode`)로 1개만 그리며(표시용), 후보 판정만 3모드 OR로 계산합니다. 마커가 **그려지는 y좌표는 화면에 페인트되는 기본 밴드2 값(`upperBandValue2`/`lowerBandValue2`) 좌표(`location.absolute` + `size.tiny`)** 이며, 화면에 그려지는 밴드2 선의 위치·색은 절대 바꾸지 않습니다(마커만 얹음). |
| 변곡 시 청산선 수평 필터(밴드워킹 억제) | `변곡 시 청산선 수평 필터` 옵션(기본 ON)을 켜면, 변곡 시점 **청산선의 기울기가 근사 수평일 때만** 마커를 표시합니다. 청산선이 추세로 기울어 **밴드워킹 중**(위/아래로 워킹)이면 변곡 조건이 성립해도 마커를 **억제**합니다. 판정은 **최근 N봉 사이 청산선 값 변화량을 ATR로 정규화한 비율**(=ATR 대비 배수)이 임계 이하이면 수평으로 간주합니다(절대 가격차가 아닌 ATR 정규화라 종목·타임프레임 무관). flat 판정은 **후보에 기여한 모드의 청산선** 기준입니다(CEO 정정 지시) — 즉 **후보 3모드(`기본`·`거래량 가중`·`최근봉 가중`) 중 밴드2가 실제로 안으로 들어온(진입한) 모드의 청산선이 수평일 때** 그 방향 수평 통과로 봅니다. 상단 flatOk = `OR_m ( 진입_m AND flat(shortClear_m) )`, 하단 flatOk = `OR_m ( 진입_m AND flat(longClear_m) )`(m∈{기본,거래량,최근봉}). ⚠️ **스윕 가격 모드(`ceMode`) 선택은 이 판정에 관여하지 않습니다** — flat 대상은 mode 리터럴로 고정 호출한 3모드 청산선(`shortClear_*`/`longClear_*`)이며, 화면 페인팅용(`ceMode`) 청산선 `shortClear`/`longClear`는 flat 판정에 쓰지 않습니다(사양: 스윕 모드는 페인팅 전용, 마커에 영향 금지). ATR은 mode 무관 공통값이라 기본 스윕 ATR 길이(`ceAtrLen`)에 맞춥니다. `[변곡] 수평 판정 lookback 봉수`(기본 3), `[변곡] 수평 허용 기울기(ATR 대비 배수)`(기본 0.15)로 조정합니다. 임계를 키우면 더 기울어도 표시, 줄이면 더 엄격히 수평만 표시. 필터를 끄면 기존처럼 기울기 무관하게 표시합니다. |
| OBV DI 우위 게이트 | 후보일 때, **OBV 기반 DI+/DI-**(OBV-ADX 보조지표 `obv-adx-extreme-background.pine`의 산식 이식)로 방향 에너지 우위를 확인합니다(마커 발동에 AND 결합). 숏 변곡: **DI- 우위**(`diMinus > diPlus`). 롱 변곡: **DI+ 우위**(`diPlus > diMinus`). `na` 가드 포함(DI 값이 `na`면 억제). | VWAP 선색 마커 게이트 | **최종 VWAP 선색**(위 `VWAP 선 지지/저항 색` 새 규칙의 결과 `vwapSRColor`)을 마커 발동에 AND로 결합합니다(CEO 규칙 A). **롱(녹색·하단, `lower2Marker`)**: 선색이 **흰색 또는 녹색일 때만** 표시 — **빨간색이면 억제**. **숏(빨강·상단, `upper2Marker`)**: 선색이 **흰색 또는 빨간색일 때만** 표시 — **녹색이면 억제**. 즉 롱은 `vwapSRColor ≠ 빨강`, 숏은 `vwapSRColor ≠ 녹색`을 요구합니다. 취지: VWAP 선이 저항(빨강)을 나타낼 때 롱 진입을, 지지(녹색)를 나타낼 때 숏 진입을 억제해 선색과 마커 방향을 정렬합니다. ⚠️ `VWAP 선 지지/저항 색(구간 종가·꼬리 걸침 이탈)` 옵션이 **OFF이면 선색 판정이 없으므로 이 게이트는 적용되지 않습니다**(기존 마커 거동 유지). ## 트레이딩 의미 핵심 4가지

### 1. 가격이 VWAP 위 / 아래에 있는 의미

가격이 VWAP 위에 있으면 `선택한 기준 구간 평균 단가보다 위에서 거래 중`, 아래에 있으면 `평균 단가보다 아래에서 거래 중`이라고 읽습니다.

- 방향 예측선이라기보다 현재 가격의 `비용대 기준 위치`를 보는 도구입니다.
- 실전에서는 `회복 후 유지`, `이탈 후 재진입 실패` 쪽이 더 중요합니다.

요약하면, VWAP은 `평균 단가 기준선`입니다.

### 2. VWAP 근처 왕복 의미

VWAP 근처를 계속 왕복하면 `방향성 추세`보다 `평균 단가 공방`이 진행 중일 가능성이 큽니다.

- 이 구간에서는 추격 진입보다 방향 확정 대기가 더 낫습니다.
- 다른 지표가 좋아 보여도 VWAP 공방이 길면 후속이 약할 수 있습니다.

요약하면, VWAP 공방은 `판단 유보 구간`으로 보는 편이 안전합니다.

### 3. Band #1 접근 의미

Band #1 접근은 `정상 확장` 또는 `첫 과열 구간`에 더 가깝습니다.

- 추세가 막 붙기 시작하는 구간에서도 자주 나옵니다.
- 따라서 Band #1 자체를 반전 신호로 보기보다 `확장 확인`으로 읽는 편이 좋습니다.

요약하면, Band #1은 `추세 확장 확인선`에 가깝습니다.

### 4. Band #2 접근 의미

Band #2 접근이나 이탈은 `강한 확장`, `과열 / 과매도`, `과확장 감시` 의미가 더 큽니다.

- 강한 추세에서는 바깥 체류가 이어질 수 있습니다.
- 그래서 Band #2는 바로 역방향 진입보다 `속도 조절`, `관리`, `후속 확인` 쪽에 더 유리합니다.

요약하면, Band #2는 `공격`보다 `과확장 감시`의 의미가 강합니다.

## 내부 로직

### 1. 기준 구간

`Anchor Mode`에 따라 기준이 바뀝니다.

`Session`
- 현재 세션만 보여줍니다.
- 이전 세션 선은 남기지 않습니다.

`Recent Trading Days`
- 최근 `N`거래일을 하나의 누적 구간처럼 계산합니다.
- 새 거래일이 열리면 가장 오래된 하루가 빠지고 최근 하루가 추가됩니다.

`Recent Bars`
- 최근 `N`봉을 하나의 이동 창처럼 계산합니다.
- 세션 경계를 무시하고 차트에 보이는 현재 타임프레임 기준 봉 수로 바로 계산합니다.

### 2. VWAP 계산

현재 코드는 아래 값을 누적해서 씁니다.

- `src * volume`
- `volume`
- `src^2 * volume`

이 누적값으로:
- `VWAP`
- 거래량 가중 분산
- 표준편차 기반 밴드

를 계산합니다.

`computeVwapSet(barsLen, srcArg)` 헬퍼가 window 길이(`Recent Bars` 모드의 rolling length)와 **소스**를 파라미터로 받아 **두 세트를 독립 계산**합니다: 기본 화면용(`length=50`, `src`)과 변곡 지점용(`length=20`, `[변곡] Source`). 소스별로 서로 다른 누적이 필요하므로 누적값(`cumulative*`)·세션 앵커 배열을 **함수-로컬 `var`로 내부화**했습니다 — Pine v6에서 함수 내부 `var`는 호출부마다 독립 상태를 가지므로 두 호출이 각자 자기 소스로 독립 누적합니다. 소스가 같으면(기본값 둘 다 `hlc3`) 두 인스턴스가 같은 누적 궤적을 재현하므로 예전(전역 단일 누적) 대비 값이 동일합니다(거동 불변). 밴드·밴드2도 각 세트의 window에서 산출되며, 화면 표시는 기본 세트, 변곡 지점 판정은 변곡용 세트의 밴드2를 씁니다. 산식(anchored VWAP + 로그/절대 표준편차 밴드)은 원본과 동일합니다.

### 3. 밴드 계산 방식

`Bands Calculation Mode`는 두 가지입니다.

`Log Standard Deviation`
- 현재 기본값입니다.
- 거래량 가중 `log(price)` 표준편차를 사용합니다.
- 밴드는 `VWAP * exp(± sigma_log * multiplier)` 형태로 계산되어, 절대 가격보다 비율 변화에 더 민감합니다.
- 가격이 0 이하인 자산에는 적합하지 않습니다.

`Absolute Standard Deviation`
- 거래량 가중 표준편차를 절대 가격 단위로 계산해 밴드 폭으로 사용합니다.

즉 이 지표는 `VWAP 기준선 + 확장 폭 방식`을 바꿔가며 읽는 구조입니다.

> **상태판 없음:** 이 지표는 우측 상단 상태 테이블을 두지 않습니다(CEO 지시로 완전 제거). 국면 판단은 이치모쿠 메인 지표의 `VWAP 상태` 셀과, 이 지표의 **변곡 지점 마커**(밴드2 선이 청산선 안으로 수렴)로 읽습니다.

## 샹들리에 청산선 + 변곡 지점(Inflection Point) 마커

이 지표는 이치모쿠 메인 지표(`advanced-ichimoku.pine`)의 **샹들리에 청산선(스윕/청산) 로직을 그대로 이식**해 값을 계산합니다. 계산은 `computeClearLines` 헬퍼 함수로 통일했고, **파라미터 세트는 `스윕/청산` 그룹(기본 청산 설정) 하나뿐**입니다(CEO 지시 #4: 변곡 전용 청산선 세트 삭제).

- **화면 페인팅**: 청산 설정의 가격 모드(`ceMode`)로 계산한 청산선(`longClear`/`shortClear`)을 차트에 직접 `plot`으로 그립니다(청산선 페인팅이 이치모쿠에서 이 지표로 이동). **화면에 그려지는 청산선은 이 mode 1개뿐**입니다.
- **변곡 마커 후보 판정**: 같은 파라미터로 **가격 모드만 `기본`·`거래량 가중`·`최근봉 가중` 3가지로 바꿔 각각 계산**한 청산선(`shortClear_base/vol/recent`, `longClear_base/vol/recent`)을 **OR**로 확인합니다(1개 이상 성립 시 후보). mode 외 파라미터(길이·ATR·배수·가중·버퍼·피벗·최대간격)는 모두 기본 청산 설정 공통값입니다. 이 3세트는 화면에 그리지 않고 판정에만 씁니다.

산식·파라미터 기본값은 이치모쿠와 동일합니다. **스윕라인(점선, `longCE`/`shortCE`)은 이동하지 않았습니다**(이치모쿠 전략의 진입 트리거 전용) — 이 지표는 청산선(직선)과 변곡 지점 마커만 그립니다.

- **스윕(샹들리에 트레일)**: `longCE = 최근 N봉 고가 − ATR×배수`(가격 아래, 상향 트레일), `shortCE = 최근 N봉 저가 + ATR×배수`(가격 위, 하향 트레일).
- **스윕 가격 모드(`ceMode`)**: 청산선의 계산 방식(선 종류)을 이치모쿠와 동일하게 선택합니다. **기본값은 `기본`**입니다. ⚠️ `ceMode`는 **화면 페인팅 전용**이라 어떤 값을 골라도 변곡 마커 후보(3모드 OR)에는 영향이 없습니다.
  - `기본`: 표준 샹들리에(룩백 고가/저가 − ATR).
  - `거래량 가중`: 고가/저가의 **거래량 가중 이동평균(VWMA)**을 앵커로 사용 → 거래가 몰린 봉에 가중.
  - `최근봉 가중`: 고가/저가의 **선형 가중 이동평균(WMA)**을 앵커로 사용 → 최근봉에 가중(휩쏘 급등/급락에 빠르게).
  - `거래량+최근봉`: **거래량 × 선형 최근봉 가중**(`f_volRecencyMa`)을 앵커로 사용 → 두 가중을 함께.
  - 가중 모드는 앵커를 `가중 트레일 길이`(기본 10)·`가중 트레일 ATR 배수`(기본 2.0)로 계산하며, **형성 중인 현재봉을 제외하고 직전 봉(`[1]`) 기준**으로 잡아 확정값 성격을 유지합니다(신규 리페인팅 없음). 청산선(`longClear`/`shortClear`)은 이 모드로 계산된 `longCE`/`shortCE`에서 그대로 파생됩니다. **청산선 값만 모드에 따라 바뀔 뿐, 밴드2 위치·색은 절대 따라가지 않습니다.**
- **청산선(스윕 너머)**: `longClear`(longCE보다 더 아래), `shortClear`(shortCE보다 더 위). ATR 버퍼 + 옵션으로 가장 가까운 스윙 풀(피벗) 너머까지 확장하며, 스윕과의 최대 간격을 ATR 배수로 상한합니다.
- **기본 파라미터**: `스윕 가격 모드 = 기본`, `스윕 룩백 길이 = 20`, `스윕 ATR 길이 = 20`, `스윕 ATR 배수 = 3.0`, `가중 트레일 길이 = 10`, `가중 트레일 ATR 배수 = 2.0`, `청산 버퍼 ATR 배수 = 0.5`, `스윙 풀 좌우 봉수 = 5`, `청산-스윕 최대 간격 ATR 배수 = 1.0`. 산식·로직은 이치모쿠와 동일하나 일부 기본값은 이 지표에 맞게 조정돼 있습니다(`스윕 룩백/ATR 길이`는 이치모쿠 22 대비 **20**, `청산 버퍼 ATR 배수`는 **0.5**). ⚠️ `청산 버퍼 ATR 배수`(`sweepBufferMult`)는 **후보 판정용 3모드 청산선 모두에 적용**되므로 값 변경(예 1.0→0.5)은 변곡 마커 발동에 영향을 줍니다. `스윕 가격 모드`는 페인팅 전용이라 무관합니다.

변곡 지점(Inflection Point) 판정은 아래와 같습니다. 판정은 **변곡용 VWAP 세트(length=20)의 밴드2**(`ipUpperBandValue2`/`ipLowerBandValue2`)와 **기본 청산선 3모드**(`shortClear_base/vol/recent`, `longClear_base/vol/recent`)를 씁니다(밴드2가 3모드 청산선 중 1개 이상의 안으로 진입하면 후보). 여기에 아래 **변곡 시 청산선 수평 필터**, **OBV DI 우위 게이트**, **VWAP 선색 게이트**, **방향성 밴드워킹 억제**가 AND로 결합됩니다. **화면 밴드2 선의 위치·색은 언제나 밴드2 원값(VWAP±2σ) 위치 + 기본색(`activeLineColor`)에 고정**되고, 변곡 여부는 **마커로만** 표기합니다(밴드 선 위치·색 변경 없음). **상단·하단 각각 독립**입니다.

- **상단(빨강)**: 변곡용 상단 밴드(`ipUpperBandValue2`)가 위쪽 청산선 안으로 들어옴 — `OR_m ( ipUpperBandValue2 < shortClear_m )`, m∈{기본,거래량,최근봉} AND **DI- 우위**(`diMinus > diPlus`) AND **VWAP 선색 ≠ 녹색**(흰/빨강만 허용) → **화면 숏 청산선(`shortClear`) 위치에 숏 마커색 마커**(숏 에너지 강=선명 빨강 `shortClearLineColor`/약=둔탁 빨강 `#A57A7A`).
- **하단(녹색)**: 변곡용 하단 밴드(`ipLowerBandValue2`)가 아래쪽 청산선 안으로 들어옴 — `OR_m ( ipLowerBandValue2 > longClear_m )` AND **DI+ 우위**(`diPlus > diMinus`) AND **VWAP 선색 ≠ 빨간색**(흰/녹만 허용) → **화면 롱 청산선(`longClear`) 위치에 롱 마커색 마커**(롱 에너지 강=선명 녹 `longClearLineColor`/약=둔탁 녹 `#6E8F72`).

**OBV DI 우위 게이트**: 후보일 때, **OBV 기반 DI+/DI-**(OBV-ADX 보조지표 `obv-adx-extreme-background.pine`의 산식을 v6로 이식, 값 동일)로 방향 에너지 우위를 확인합니다 — 숏=`diMinus > diPlus`(DI- 우위), 롱=`diPlus > diMinus`(DI+ 우위), `na` 가드 포함(DI 값이 `na`면 억제). DI는 OBV 누적(`ta.cum(sign(Δclose)·volume)`)의 방향성 이동(`plusDM`/`minusDM`)을 OBV 표준편차의 RMA(`trur`)로 정규화한 `100·EMA(±DM, len)/trur`이며, `len = DI Length`(기본 **5**, OBV-ADX 원본 20보다 짧아 DI 우위 반응이 빠름). `ADX Smoothing`(기본 **5**, OBV-ADX 원본 `lensig` 20에 대응) input은 OBV-ADX 원본과의 옵션 정합용으로 유지하되(계산값 `obvAdx` 산출), 게이트는 DI 우위만 씁니다. 이전의 "3봉 hlc3 평균 게이트"는 이 DI 우위 게이트로 **대체(제거)** 되었습니다. ⚠️ 표시 게이트 전용 — 매매 전략(진입·청산·임계·산식)과 무관합니다.

**OBV 에너지 채도(마커 색 강/약)**: 마커 색은 OBV 방향 에너지 강도로 채도가 갈립니다(추세 강도 시각화). `diEnergy = |diPlus − diMinus|`(=OBV-ADX 상태판의 `|diDiff|`와 동일), `diEnergyAvg = sma(diEnergy, OBV 에너지 평균 길이)`, `energyFading = na … or diEnergy < diEnergyAvg or diEnergy < diEnergy[3]`. `energyStrong = not energyFading`이면 **강(선명색)**, 아니면 **약(둔탁색)** 입니다(`na`는 보수적으로 약=둔탁). 평균 창(`diEnergyAvg`)은 `OBV 에너지 평균 길이`(auto-vwap 기본 **5**, obvDiLen과 결이 맞게 짧게)로 조정하고, `diEnergy[3]`(3봉 전 대비 모멘텀)은 평균 창과 무관한 **독립 고정 룩백**(OBV-ADX 상태판과 동일한 상수 3)이라 평균이 짧아져도 시간축이 어긋나지 않습니다. 선명색은 마커 기존 색(`longClearLineColor`/`shortClearLineColor`), 둔탁색은 저장소 시그니처 팔레트(둔탁 녹 `#6E8F72`/둔탁 빨강 `#A57A7A`, OBV-ADX 상태판과 동일)입니다. 이 산식은 **OBV-ADX 보조지표 상태판(`energyFading`)의 구조를 이식**해 두 지표가 같은 방식으로 강/약을 판정합니다(평균 창 기본값만 다름). `diEnergy`는 `obvAdx` 계산에서 이미 산출한 `|diPlus − diMinus|`를 **재사용**합니다(중복 계산 제거). ⚠️ **색 채도만 바꾸며 마커 발동·모양·y좌표는 불변**이고, 확정 성격 값(`ema`/`sma`/과거참조 `[3]`)만 써 신규 리페인팅이 없습니다. 채도 강/약 판정 평균 봉 수는 `OBV 에너지 평균 길이`(기본 5) input으로 조정합니다.

마커는 "밴드2의 과확장 여력이 샹들리에 청산선 안으로 수렴했다(변곡 지점)"는 시각 신호일 뿐, 화면 밴드 선은 이에 따라 위치·색이 전혀 바뀌지 않습니다. 청산선은 확정값·과거참조 기반이라 신규 리페인팅이 없습니다. 마커의 **발동 판정은 변곡용 밴드2 + 기본 청산선 3모드(+수평 필터·OBV DI 우위 게이트·밴드워킹 억제)으로만** 하며, 기본 화면 세트의 값·표시 토글(`showBand_2` 등)은 발동 조건에 **개입하지 않습니다**. 따라서 **기본 밴드2 선을 화면에서 꺼도(`Bands Multiplier #2` 토글 OFF) 마커 판정은 독립적으로 성립**합니다. 마커가 **그려지는 y좌표는 화면에 페인트되는 기본 밴드2 값(`upperBandValue2`/`lowerBandValue2`)**이며, `plotshape`의 `location.absolute` + `size.tiny`로 기본 밴드2 선 좌표에 얹어 그립니다. **na 정합성 보정**: 특정 봉에서 기본 밴드2 값이 `na`(기본 세트 shouldPlot=false 등)이면, 조건이 참일 때 마커가 사라지지 않도록 **변곡용 밴드2 값으로 폴백**해 항상 안정적으로 찍습니다(y위치는 기본 밴드2 우선).

> **밴드2 선 원값 보존:** 변곡 지점 판정은 화면 밴드2 선의 **plot 값(위치)** 을 바꾸지 않으며, 값은 언제나 기본 세트 원값(`upperBandValue2`/`lowerBandValue2`)입니다. 이 원값은 VWAP·밴드 간격 % 라벨 등에 그대로 쓰이고, 밴드2 채움(fill, 현재 비표시)도 원값 선(`upperBand_2`/`lowerBand_2`)에 직접 붙습니다. **색**은 기본색(`activeLineColor`)이 기본이되, `변곡 마커 봉 밴드2 강조 색`(기본 ON)이 켜지면 **변곡 마커가 뜨면서 동시에 해당 방향 화면 밴드2가 청산선 안(상단 `upperBandValue2 < shortClear`, 하단 `lowerBandValue2 > longClear`)인 봉에 한해** 그 밴드2를 마커색으로 강조합니다(상단=`upperMarkerColor`, 하단=`lowerMarkerColor` — **마커와 동일한 OBV 에너지 채도 반영**: 강=선명, 약=둔탁). 마커 발동은 변곡용 밴드2(변곡 세트, window `ipRecentBars` 기본 10)를 3모드 청산선과 비교하는 반면 이 강조 게이트는 **화면 세트 밴드2**(window `recentBars` 기본 50)를 화면 청산선과 비교하므로, 마커만으로는 강조가 걸리지 않고 화면 밴드2까지 청산선 안이어야 강조됩니다. 이는 마커 `plotshape`와 **동일 색 변수(`upper/lowerMarkerColor`)를 재사용**한 봉 단위 색이라 강조색·마커색이 항상 일치하고(같은 봉에서 마커=둔탁·밴드2=선명 불일치 방지), 확정 성격 값(마커·밴드2·청산선·에너지)만 따르므로 리페인팅이 없습니다(옵션 OFF·조건 미충족이면 항상 기본색).

### 청산선(직선) 차트 표시 (이치모쿠에서 이동)

- **롱/숏 청산선(`longClear`/`shortClear`)을 `plot`으로 전 히스토리 봉에 그립니다.** ⚠️ 최근 몇 봉만 그리는 `line.new` 방식이 아니라 **전 구간 봉마다 청산선이 그려지도록**(CEO 요구: "봉 전체 다 나올 수 있어야") `plot`을 씁니다.
- **리페인팅 없음:** 청산선은 확정 성격 값(`ta.*`·과거참조 `[1]` 트레일)에서만 파생되므로, `line.new`에서 `plot`으로 그리기 방식을 바꿔도 값 자체는 불변이라 신규 리페인팅이 없습니다.
- **표시 옵션(스윕/청산 그룹, 이치모쿠에서 이식):** `청산선 표시(전 히스토리 봉)`(ON)·`청산 선 두께`(**1px**, CEO 지시)·`롱 청산선 색`(녹색 #00C853)·`숏 청산선 색`(크림슨 #FF1744). 이 청산선은 **화면 plot 전용**입니다(화면에 그려지는 mode 1개). 변곡 지점 후보 판정은 같은 파라미터의 3모드 청산선을 별도 계산해 씁니다.
- **스윕라인은 그리지 않습니다.** 스윕라인(점선)은 이치모쿠 전략의 진입 트리거로만 쓰이며 이 지표로 옮기지 않았습니다.

### 변곡 지점(Inflection Point) 마커 (3모드 OR 후보 + 청산선 수평 필터 + OBV DI 우위 게이트 + 밴드워킹 억제)

- **밴드2 마커 판정은 기본 청산 설정(`스윕/청산` 그룹)을 씁니다**(CEO 지시 #4: 변곡 전용 청산선 세트 삭제). 후보 판정은 `computeClearLines` 헬퍼를 mode만 `기본`·`거래량 가중`·`최근봉 가중` 3가지로 바꿔 호출한 `shortClear_base/vol/recent`·`longClear_base/vol/recent`로 **OR** 판정하며, 밴드2는 변곡용 VWAP 세트(`ipRecentBars=20`)의 `ipUpperBandValue2`/`ipLowerBandValue2`를 씁니다.
- **변곡 시 청산선 수평 필터**(`ipUpperFlatOk`/`ipLowerFlatOk`, 대상=후보에 기여한 모드의 3모드 청산선 `shortClear_*`/`longClear_*` — `ceMode` 선택과 무관)·**OBV DI 우위 게이트**(`shortDiGate = diMinus > diPlus`/`longDiGate = diPlus > diMinus`, OBV-ADX 이식·`na` 가드)·**VWAP 선색 게이트**(`srcGateShort = vwapSRColor ≠ 녹색`/`srcGateLong = vwapSRColor ≠ 빨강`, 구간 종가+꼬리 걸침 규칙[①②저항/③④지지]의 최종 선색 기준·`enableVwapSRColor` OFF이면 미적용)·**방향성 밴드워킹 억제**(`upperBwOk`/`lowerBwOk`)가 후보 판정에 AND로 결합됩니다. 세부 동작은 위 "지표가 보여주는 것" 표의 해당 항목을 참고하세요.
- **마커 색 = OBV 에너지 채도 반영**: 상단선 변곡 → 숏 마커색(강=선명 빨강 `shortClearLineColor` #FF1744/약=둔탁 빨강 `#A57A7A`), 하단선 변곡 → 롱 마커색(강=선명 녹 `longClearLineColor` #00C853/약=둔탁 녹 `#6E8F72`). 강/약은 `|diPlus−diMinus|` vs 그 평균(`OBV 에너지 평균 길이`)·3봉 전 대비로 판정(OBV-ADX 상태판과 동일 기준). 색 채도만 갈리며 발동·모양·y좌표는 불변.
- **마커 모양 옵션(`변곡 지점 마커 모양`)** 은 `십자가`/`원`(기본)/`삼각형`/`다이아몬드`/`네모` 중 선택합니다. `plotshape`의 `style`이 `const string`만 받으므로 삼항을 `style` 인자에 직접 인라인해 컴파일 안전을 확보했습니다.
- **마커는 `location.absolute` + `size.tiny`로 기본 화면 밴드2 값(`upperBandValue2`/`lowerBandValue2`) 좌표에 그립니다**(발동 판정은 변곡용 밴드2 + 기본 청산선 3모드, y위치는 화면 밴드2). `location.top`/`location.bottom`은 첫 인자 값을 무시하고 차트 창 상/하단에 고정돼 선택한 모양이 원하는 자리에 나오지 않으므로 쓰지 않습니다.

| `[밴드워킹] VWAP 기울기 lookback 봉수`, `[밴드워킹] 밴드2 레일 기울기 lookback 봉수`, `[밴드워킹] VWAP 평탄 임계(밴드폭 대비 비율)` | 방향성 밴드워킹 판정 파라미터. 이치모쿠 원본 기본값과 동일: 5 / 3 / 0.03(=밴드폭의 3%) |
| `VWAP·밴드 간격 % 라벨 표시` | VWAP/밴드 라인 끝에 붙는 **VWAP 대비 % 간격 라벨**(예 `+1.00%`)을 켜고 끌 때. **기본 꺼짐** |
| `Hide VWAP on 1D or Above` | 일봉 이상에서 선을 숨기고 싶을 때 |

## 주의사항

- 이 지표는 거래량 데이터가 있어야 정상 동작합니다. 코드에는 거래량이 없을 때 `runtime.error`도 들어 있습니다.
- `Session` 모드는 의도적으로 현재 세션만 보여줍니다.
- `Recent Trading Days`는 달력 일수가 아니라 실제 거래일 기준입니다.
- `Recent Bars`는 차트 타임프레임의 봉 수 기준이라, 타임프레임을 바꾸면 같은 숫자여도 의미가 달라집니다.
- 저유동 종목에서는 VWAP과 밴드 반응이 더 거칠 수 있습니다.
