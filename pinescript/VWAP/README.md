# Advanced VWAP

트레이딩뷰에서 사용할 수 있는 Pine Script 지표 설명서입니다.

대상 스크립트:
- [`advanced-vwap.pine`](./advanced-vwap.pine)

## 개요

이 지표는 특정 기준 기간이 바뀔 때마다 다시 계산되는 `Anchored VWAP`를 차트 위에 표시하는 오버레이 스크립트입니다.

기본 `Session` 기준뿐 아니라 `Week`, `Month`, `Quarter`, `Year`, `Decade`, `Century` 같은 시간 기반 앵커와 `Earnings`, `Dividends`, `Splits` 같은 이벤트 기반 앵커까지 선택할 수 있어서, 단순 일중 평균 단가가 아니라 `지금 어떤 기준 기간의 평균 체결 단가 위/아래에 있는가`를 함께 볼 수 있습니다.

여기에 3단계 밴드와 `표준편차 / 퍼센트` 기준 밴드 계산까지 함께 넣어서, 추세 추종과 평균 회귀 둘 다 해석할 수 있도록 만든 버전입니다.

현재 스크립트는 기준 기간이 넘어갈 때 이전 구간의 마지막 점과 새 구간의 시작점이 서로 이어지지 않도록 보정되어 있습니다.

## 사용 지표

이 스크립트는 아래 요소를 조합해서 기준 가격대를 봅니다.

- `Anchored VWAP`
- `Session / Week / Month / Quarter / Year / Decade / Century` 기준 리셋
- `Earnings / Dividends / Splits` 이벤트 기준 리셋
- `표준편차 밴드`
- `퍼센트 밴드`

짧게 보면:

- `VWAP`는 평균 체결 단가
- `Anchor`는 평균을 새로 계산할 기준
- `Band`는 평균에서 얼마나 이격됐는지

## 트레이딩뷰 적용 방법

1. 트레이딩뷰에서 `Pine Editor`를 엽니다.
2. [`advanced-vwap.pine`](./advanced-vwap.pine) 파일의 전체 내용을 복사합니다.
3. Pine Editor에 붙여넣습니다.
4. `Add to chart` 또는 `차트에 추가`를 클릭합니다.
5. 필요하면 `Save`로 개인 스크립트로 저장합니다.

## 기본 정보

- Pine Script 버전: `@version=6`
- 표시 위치: `overlay=true`
- 지표명: `Volume Weighted Average Price`
- 기본 앵커: `Session`
- 기본 소스: `hlc3`
- 기본 밴드: `Band #1`만 활성화
- 기본 밴드 계산: `Standard Deviation`

## 주요 기능

| 기능 | 핵심 의미 | 주요 설정/조건 |
| --- | --- | --- |
| 앵커 기준 VWAP | 선택한 기준 기간의 평균 체결 단가를 추적합니다. | `Anchor Period` |
| 이벤트 리셋 | 실적/배당/분할 같은 이벤트 발생 시 기준을 새로 잡습니다. | `Earnings`, `Dividends`, `Splits` |
| 밴드 3단계 | 평균 단가 주변의 이격 구간을 1~3단계로 봅니다. | `Bands Multiplier #1~#3` |
| 밴드 계산 모드 | 표준편차 기반 또는 퍼센트 기반으로 이격 폭을 계산합니다. | `Bands Calculation Mode` |
| 구간 분리 표시 | 기간 전환 지점에서 선이 이어지지 않도록 표시합니다. | 기준 기간 변경 시 색상 분리 |
| 상위 타임프레임 숨김 | 일봉 이상에서는 VWAP를 숨길 수 있습니다. | `Hide VWAP on 1D or Above` |

## 설정값을 어떻게 쓰면 좋은지

### 1. `Anchor Period`

- `Session`: 데이트레이딩에서 가장 기본입니다.
- `Week`: 주간 평균 단가를 기준으로 추세 지속과 되돌림을 보기 좋습니다.
- `Month`, `Quarter`, `Year`: 스윙이나 포지션 관점에서 큰 기준 가격을 볼 때 좋습니다.
- `Earnings`, `Dividends`, `Splits`: 기업 이벤트 전후로 체결 기준이 어떻게 바뀌는지 볼 때 좋습니다.

핵심은 `내가 보는 매매 기간과 앵커 기간을 맞추는 것`입니다.

- 단타면 `Session`
- 며칠 스윙이면 `Week`
- 몇 주 이상이면 `Month` 이상

### 2. `Bands Calculation Mode`

- `Standard Deviation`: 실제 변동성에 따라 밴드 폭이 달라집니다.
- `Percentage`: 가격 대비 고정 비율로 밴드를 봅니다.

해석은 보통 아래처럼 단순하게 가져가면 됩니다.

- 변동성 적응형을 원하면 `Standard Deviation`
- 일정 퍼센트 이격만 보고 싶으면 `Percentage`

### 3. `Bands Multiplier`

- `1`: 가장 자주 닿는 기본 이격 구간
- `2`: 과열/과매도 해석이 강해지는 구간
- `3`: 추격보다 차익실현이나 역추세 경계가 필요한 구간

## 추천 사용 흐름

1. 먼저 `Anchor Period`를 현재 매매 주기에 맞게 고릅니다.
2. 가격이 VWAP 위에 있는지 아래에 있는지 봅니다.
3. VWAP 기울기가 올라가는지 내려가는지 확인합니다.
4. 밴드 바깥으로 과도하게 이격됐는지 봅니다.
5. 눌림이 VWAP 또는 밴드 안쪽으로 복귀하는지 확인한 뒤 진입을 판단합니다.

실전에서는 보통 아래처럼 해석합니다.

- `가격 > VWAP`이면 매수 우위
- `가격 < VWAP`이면 매도 우위
- `VWAP가 평평`하면 추세보다 횡보 가능성을 우선 봅니다.
- `밴드 바깥 과도한 이격`은 추격보다 되돌림 가능성을 먼저 봅니다.

## POC와 비교해서 보는 법

`VWAP`와 `POC`는 둘 다 공정가치를 보지만 역할이 다릅니다.

- `VWAP`는 해당 기간의 `평균 체결 단가`
- `POC`는 해당 프로파일 구간의 `가장 많이 거래된 단일 가격`

실전 해석은 이렇게 가져가면 단순합니다.

- `VWAP`는 방향과 평균 가격 기준
- `POC`는 거래가 가장 많이 모인 자리 기준
- `가격이 VWAP와 POC 위`면 롱 우위
- `가격이 VWAP와 POC 아래`면 숏 우위
- `가격이 VWAP와 POC 사이`면 전환 구간이라 애매한 경우가 많습니다.

이 스크립트는 `VWAP 기준선`을 보는 역할이므로, `POC`는 Volume Profile 계열 도구와 같이 보는 편이 좋습니다.

## 실전 해석 예시

### 롱 쪽으로 보기 좋은 상황

- `Session VWAP` 위에서 가격이 유지됩니다.
- VWAP 기울기가 완만하게라도 상승합니다.
- 눌림이 `VWAP` 또는 `Band #1` 부근에서 멈춥니다.
- 다음 봉에서 다시 VWAP 위로 종가가 회복됩니다.

이런 구조는 `평균 단가 위에서 눌림 매수` 시나리오로 보기 좋습니다.

### 숏 쪽으로 보기 좋은 상황

- `Session VWAP` 아래에서 가격이 유지됩니다.
- VWAP 기울기가 하락합니다.
- 반등이 `VWAP` 또는 `Band #1` 근처에서 막힙니다.
- 다음 봉에서 다시 VWAP 아래로 밀립니다.

이런 구조는 `평균 단가 아래에서 반등 매도` 시나리오로 보기 좋습니다.

### 평균 회귀로 보기 좋은 상황

- VWAP가 거의 평평합니다.
- 가격이 `Band #2` 또는 `Band #3`까지 과하게 벌어집니다.
- 직전 고점/저점 돌파가 실패하거나 꼬리가 길게 나옵니다.

이때는 추세 추종보다 `VWAP 복귀`를 목표로 보는 쪽이 더 자연스러울 수 있습니다.

## 이 스크립트를 같이 보면 좋은 경우

- [`추세 추적`](../%EC%B6%94%EC%84%B8%20%EC%B6%94%EC%A0%81/README.md): 큰 방향을 먼저 정할 때
- [`거래량 압력 추적`](../%EA%B1%B0%EB%9E%98%EB%9F%89%20%EC%95%95%EB%A0%A5%20%EC%B6%94%EC%A0%81/README.md): 눌림/반등 자리에 실제 체결 압력이 붙는지 확인할 때
- [`MACD 다이버전스 추적`](../MACD/README.md): 평균 단가 기준 위아래에서 모멘텀 재가속을 볼 때

즉:

- `추세 추적`은 방향
- `Advanced VWAP`는 평균 단가 기준
- `거래량`은 실제 힘
- `MACD`는 힘 재개

## 주의사항

- `VWAP`는 거래량 데이터가 필요하므로, 거래량이 없는 종목에서는 동작하지 않습니다.
- `Session VWAP`는 데이트레이딩에 좋지만, 스윙 관점에서는 너무 짧게 느껴질 수 있습니다.
- 기간이 바뀌는 지점에서 선이 끊어지도록 처리되어 있으므로, 새 구간 시작을 시각적으로 더 명확하게 볼 수 있습니다.
- `Hide VWAP on 1D or Above`를 켜면 일봉 이상에서는 보이지 않습니다.
- `Earnings`, `Dividends`, `Splits` 앵커는 종목 데이터 제공 여부에 따라 표시가 달라질 수 있습니다.
- 이 지표 하나만으로 진입을 확정하기보다, 추세 구조와 거래량 반응을 같이 보는 편이 더 안전합니다.
