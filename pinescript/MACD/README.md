# MACD 다이버전스 추적

트레이딩뷰용 Pine Script 보조지표 설명서입니다.

대상 스크립트:
- [`macd-divergence-tracker.pine`](./macd-divergence-tracker.pine)

이 지표는 기본 `MACD / Signal / Histogram` 구조에 `가격 이평 기울기`와 `ATR 필터`를 얹어서, 일반 MACD보다 `히스토그램 다이버전스`와 `모멘텀 재개`를 더 직관적으로 읽기 쉽게 만든 버전입니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| MACD 선 | `Fast Length`와 `Slow Length` 차이로 현재 방향을 보는 기준선입니다. |
| Signal 선 | MACD 위에 얹는 기준선으로, 실제 교차 타이밍 판단에 씁니다. |
| Histogram | `MACD - Signal`을 triple EMA로 한 번 더 부드럽게 만든 막대입니다. |
| 다이버전스 강조색 | `가격 MA 기울기`와 `히스토그램 방향`이 어긋날 때만 초록/빨강 계열로 색을 바꿉니다. |
| 크로스 원 | 옵션을 켜면 MACD와 Signal의 실제 교차 시점을 원으로 표시합니다. |

참고:
- 이 지표에서 말하는 다이버전스는 `클래식 피벗 다이버전스`가 아니라 `가격 이평 기울기`와 `히스토그램 방향`의 어긋남입니다.
- `Histogram Scale`은 시각 크기만 바꾸고 계산 로직은 바꾸지 않습니다.

## 트레이딩 의미 핵심 4가지

### 1. MACD와 Signal 선 관계 의미

`MACD > Signal`이면 기본 모멘텀이 위쪽, `MACD < Signal`이면 기본 모멘텀이 아래쪽이라고 읽습니다.

- 가장 단순한 방향 판단은 선 관계에서 시작합니다.
- 다만 이 관계만으로는 `힘이 다시 붙는 중인지`, `힘이 약해지는 중인지`는 다 담기지 않습니다.

요약하면, 선 관계는 `기본 방향`, 히스토그램은 `힘의 상태`를 보완합니다.

### 2. 초록 계열 히스토그램 강조 의미

초록 계열 강조는 `가격 이평은 눌리는데 히스토그램은 먼저 살아나는 상태`를 뜻합니다.

- 눌림 뒤 재상승 후보를 빠르게 포착할 때 유용합니다.
- 메인 자리 신호가 살아 있는 구간과 겹치면 `재가속 후보` 의미가 커집니다.

요약하면, 초록 강조는 `가격은 주춤하지만 내부 힘은 먼저 복구되는 상태`입니다.

### 3. 빨강 계열 히스토그램 강조 의미

빨강 계열 강조는 `가격 이평은 반등하는데 히스토그램은 먼저 식는 상태`를 뜻합니다.

- 반등 뒤 재하락 후보를 볼 때 유용합니다.
- 상승 말단이나 저항 부근과 겹치면 `상단 피로`, `힘 약화` 쪽 해석이 강해집니다.

요약하면, 빨강 강조는 `가격은 버티지만 내부 힘은 먼저 식는 상태`입니다.

### 4. 크로스 원 의미

크로스 원은 `실제 교차 시점 확인용`입니다.

- 진입 버튼처럼 보기보다 `히스토그램 강조와 선 관계가 이미 갖춰졌는지`를 확인하는 마지막 표시에 가깝습니다.
- 원 하나보다 `강조색 + 선 관계 + 유지 시간`을 같이 보는 편이 더 안정적입니다.

## 내부 로직

### 1. 기본 계산

- `MACD = fast MA - slow MA`
- `Signal = MACD의 이동평균`
- `Histogram = MACD - Signal`을 triple EMA smoothing 한 값

즉 MACD와 Signal은 원형 구조를 유지하고, 히스토그램만 더 부드럽게 만든 형태입니다.

### 2. 히스토그램 다이버전스 후보 조건

현재 코드는 `가격 이동평균 기울기`와 `히스토그램 방향`이 어긋날 때만 다이버전스 후보 색을 켭니다.

강세 후보:
- 가격 MA가 `Confirm Bars` 동안 실제로 하락
- 하락 폭이 `ATR Filter` 이상
- 히스토그램이 `Confirm Bars` 동안 실제로 상승

약세 후보:
- 가격 MA가 `Confirm Bars` 동안 실제로 상승
- 상승 폭이 `ATR Filter` 이상
- 히스토그램이 `Confirm Bars` 동안 실제로 하락

즉 이 지표는 `가격은 아직 반대인데 내부 힘은 먼저 돌아서는가`를 색으로 보여주는 구조입니다.

### 3. 크로스 조건

기본 크로스:
- `MACD`가 `Signal` 위로 돌파하면 상승 크로스
- `MACD`가 `Signal` 아래로 이탈하면 하락 크로스

추가 필터 크로스:
- 상승 크로스이면서 `MACD > 0`
- 하락 크로스이면서 `MACD < 0`

코드 안에는 이 조건으로 `alert()` 호출도 들어 있습니다.

## 자주 만지는 설정

| 설정 | 언제 조정하나 |
| --- | --- |
| `Fast Length`, `Slow Length`, `Signal Smoothing` | MACD 자체 반응 속도를 바꾸고 싶을 때 |
| `Price MA Length` | 가격 기울기 기준을 더 짧게 또는 길게 보고 싶을 때 |
| `Histogram TRIX Smoothing` | 히스토그램을 더 부드럽게 또는 더 민감하게 만들 때 |
| `Divergence Confirm Bars` | 다이버전스 색이 너무 자주 튈 때 |
| `Divergence ATR Filter` | 미세한 기울기 흔들림을 더 강하게 걸러내고 싶을 때 |
| `Histogram Scale` | 히스토그램을 더 크게 또는 작게 보고 싶을 때 |
| `Show Circle on Cross` | 원형 타이밍 표시를 켜거나 끌 때 |
| `Indicator TimeFrame` | 다른 타임프레임 기준 MACD를 보고 싶을 때 |

## 주의사항

- 여기서 말하는 다이버전스는 `가격 MA 기울기`와 `히스토그램 방향`의 어긋남입니다.
- 히스토그램은 triple EMA smoothing이 들어가므로 일반 MACD보다 반응이 약간 늦을 수 있습니다.
- 멀티타임프레임을 쓰면 현재 차트 봉과 계산 봉의 체감이 달라질 수 있습니다.
- MACD 방향이 좋아 보여도 가격 구조나 거래량이 따라오지 않으면 신뢰도는 낮아집니다.
