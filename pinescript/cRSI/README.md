# cRSI (RSI Cyclic Smoothed)

트레이딩뷰용 Pine Script 보조 지표 설명서입니다.

대상 스크립트:
- [`rsi-cyclic-smoothed.pine`](./rsi-cyclic-smoothed.pine)

> Pine Script **v6** 기준입니다(v4에서 마이그레이션, 신호·계산·파라미터 동작은 동일하게 보존).

이 지표는 기본 `RSI`에 **지배 사이클(Dominant Cycle)** 기반의 평활화와 **동적 과매수/과매도 밴드**를 더한 버전입니다. 고정된 `30 / 70` 기준 대신, 최근 사이클 구간에서 cRSI가 실제로 머문 범위를 기준으로 상·하단 밴드를 계산해 줍니다. 원작은 Lars von Thienen의 *Decoding The Hidden Market Rhythm*(2017) 4장에서 소개된 cRSI입니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| CRSI (기본 노란 선) | 사이클로 평활화한 RSI 본선입니다. 다크 차트와 보라/주황 배경에서 눈에 띄도록 기본 노란색(`#FDD835`)으로 표시하며, `선 색상` 그룹의 `cRSI 선 색` 입력으로 원하는 색을 고를 수 있습니다. |
| cMFI (기본 파란 선) | cRSI와 **동일 주기·동일 평활 파이프라인**에 입력만 RSI 대신 MFI(Money Flow Index)를 넣은 순환 평활 MFI 선입니다(기본 파란색 `#2962FF`, `선 색상` 그룹의 `cMFI 선 색` 입력으로 변경 가능). 가격 흐름(cRSI)과 자금 흐름(cMFI)을 같은 간격으로 겹쳐 봅니다. |
| HighBand (기본 빨간 선) / LowBand (기본 초록 선) | 최근 사이클 메모리 구간에서 동적으로 계산한 과매수/과매도 밴드입니다. HighBand(과매수 경계)는 기본 빨강(`#EF5350`), LowBand(과매도 경계)는 기본 초록/청록(`#26A69A`)으로 구분하며, `선 색상` 그룹의 `상단밴드(HighBand) 색`·`하단밴드(LowBand) 색` 입력으로 각각 원하는 색을 고를 수 있습니다. **밴드 상·하단(HighBand/LowBand) 값은 우측 프라이스 스케일(가격 축) 라벨에 표시됩니다.** |
| 30 / 70 점선 (은색) | 참고용 고정 기준선입니다. |
| Band Width Background | 밴드폭이 좁으면 진한 주황색(수축), 넓으면 보라색(확장) 배경으로 강조합니다. 불투명성 기본 15(0=투명, 100=진함)로 연하게 깔립니다. |

참고(축 라벨 표시):
- 우측 **프라이스 스케일(가격 축) 라벨에는 CRSI 본선 값과 함께 밴드 상·하단(HighBand/LowBand) 값**이 표시됩니다. 현재 과매수/과매도 경계가 축에서 바로 읽힙니다. cMFI만 패널에 선으로는 계속 그려지되 우측 축 라벨에는 뜨지 않습니다(`display = display.all - display.price_scale`).

참고:
- 기본값은 `Dominant Cycle Length = 20`이며, 내부적으로 RSI 길이 `cyclelen = 10`, 사이클 메모리 `cyclicmemory = 40`으로 환산됩니다.
- `vibration = 10`, `leveling = 10.0`은 코드 내부 상수로 고정되어 있습니다(입력값 아님).
- 동적 밴드는 사이클 메모리(`40봉`) 구간에서 cRSI 값의 분포를 백분위로 훑어, 하위 `10%`(LowBand)·상위 `10%`(HighBand) 경계를 잡습니다.
- 밴드폭 배경은 `bar_index < cyclicmemory`(초기 구간)에서는 표시되지 않습니다.

## cRSI를 쓰는 이유

일반 RSI는 `30 / 70` 같은 고정 기준을 쓰지만, 종목·국면에 따라 RSI가 실제로 도는 범위가 다릅니다.

- 추세장에서는 RSI가 한쪽에 쏠려 `70`을 계속 넘겨도 과매수로 보기 애매합니다.
- 횡보장에서는 `30 / 70`에 닿기 전에 이미 반전하기도 합니다.

cRSI는 사이클로 평활화해 노이즈를 줄이고, **최근 구간에서 cRSI가 실제로 머문 범위**로 밴드를 그려 "지금 국면 기준으로 과한지"를 봅니다.

## 트레이딩 의미 핵심

### 1. cRSI가 LowBand 아래일 때

`최근 사이클 기준으로도 과하게 눌린 구간`을 뜻합니다.

- 고정 `30`보다 그 국면에 맞는 과매도 경계라서, 횡보·추세 양쪽에서 더 현실적인 자리로 봅니다.
- 단독 진입보다는 `되돌림 감시 자리`로 보는 편이 맞습니다.

### 2. cRSI가 HighBand 위일 때

`최근 사이클 기준으로도 과하게 올라간 구간`을 뜻합니다.

- 급등 뒤 상승 피로를 보는 데 유용합니다.
- 메인 신호가 약해지는 자리와 겹치면 `후행 롱 자제`, `숏 감시` 의미가 커집니다.

### 3. 밴드폭 배경(진한 주황색 / 보라)

밴드폭(`HighBand - LowBand`)은 변동성/사이클 진폭의 대리 지표입니다.

- **진한 주황색(좁은 밴드)**: cRSI가 좁은 범위에서만 움직이는 `수축 구간`. 큰 움직임 직전의 눌림일 수 있어, 밴드 돌파 방향을 주시합니다.
- **보라(넓은 밴드)**: cRSI가 크게 출렁이는 `확장 구간`. 신호가 자주 극단에 닿으므로 과매수/과매도 신호의 신뢰도를 한 번 더 거릅니다.

## 내부 로직

1. `cyclelen = domcycle / 2` 길이로 기본 RSI를 계산합니다.
2. `torque`, `phasingLag`(vibration 기반)로 cRSI를 사이클 평활화합니다.
   - `crsi := torque * (2*rsi - rsi[phasingLag]) + (1 - torque) * crsi[1]`
3. 최근 `cyclicmemory(= domcycle * 2)`봉 구간에서 cRSI의 최댓값/최솟값을 찾습니다.
4. 그 범위를 100단계로 나눠, 분포의 하위 `leveling%`(LowBand)·상위 `leveling%`(HighBand) 경계를 백분위로 잡습니다.
5. 밴드폭으로 배경색(수축/확장)을 칠합니다.

즉 고정 임계선이 아니라, `현재 국면에서 cRSI가 실제로 돈 범위`를 기준으로 과매수/과매도를 판단하는 방식입니다.

## 순환 평활 MFI (cMFI, 파란 선)

cRSI가 **가격(RSI)** 을 사이클 평활한 선이라면, cMFI는 **자금 흐름(MFI, Money Flow Index)** 에 **같은 파이프라인을 그대로** 적용한 선입니다.

- **동일 주기**: MFI 계산 길이를 cRSI와 같은 `cyclelen(= domcycle / 2)`으로 씁니다(`mfi(hlc3, cyclelen)`). 별도 파라미터가 없어 cRSI와 언제나 같은 간격입니다.
- **동일 평활**: cRSI가 쓰는 `torque`·`phasingLag`(둘 다 `vibration` 기반)를 그대로 재사용해 같은 재귀식으로 평활합니다.
  - `cmfi := torque * (2*mfiRaw - mfiRaw[phasingLag]) + (1 - torque) * cmfi[1]`
- **읽는 법**: 두 선이 같은 방향으로 정렬(가격과 자금 흐름 동조)이면 신호 신뢰도가 올라가고, cRSI는 오르는데 cMFI가 따라오지 못하면(또는 반대) 자금이 뒷받침되지 않는 움직임으로 봅니다. 단독 신호가 아니라 **cRSI와의 정렬 확인용 교차 검증선**입니다.
- MFI는 거래량을 반영하므로, 거래량이 없거나 불안정한 심볼에서는 cMFI가 왜곡될 수 있습니다.

## 우측 상단 상태판

맨 윗줄 제목 칸(`cRSI`)에 현재 `포지션` 값이 색상과 함께 표시됩니다.

| 행 | 내용 |
| --- | --- |
| cRSI (제목) | `포지션`: `롱 유리` / `숏 유리` / `중립` (사유 포함) |
| cRSI 값 | cRSI 값 + 괄호 안에 밴드폭 값·상태 — 예: `34.0 (40.0 중립)`. 밴드폭 상태는 **셀 배경색**으로 표시: `수축`=진한 주황색, `확장`=보라, `중립`=회색(텍스트엔 색 이름을 넣지 않음) |
| 밴드 위치 | `LowBand(과매도)` / `HighBand(과열)` / `중립` |

`포지션` 판정 우선순위:
1. **수축 돌파 지속**: 수축(진한 주황색)에서 하단 돌파 → `숏 유리(하단돌파 지속)`, 상단 돌파 → `롱 유리(상단돌파 지속)`. cRSI가 밴드 안으로 복귀할 때까지 유지(돌파 후 밴드가 확장돼도 지속).
2. **확장 다이버전스**: 확장(보라)에서 상승 다이버전스 → `롱 유리(상승 다이버전스)`, 하락 다이버전스 → `숏 유리(하락 다이버전스)`. 의도한 방향으로 밴드를 돌파한 뒤 다시 밴드 안으로 복귀하면 종료. (차트에 ▲▼ 마커로 표시)
3. **과매수/과매도 되돌림**: `HighBand` → `숏 유리(과매수)`, `LowBand` → `롱 유리(과매도)`.
4. 그 외 `중립`(수축 중 밴드 안이면 `중립(수축)`).

## 자주 만지는 설정

| 설정 | 언제 조정하나 |
| --- | --- |
| `Dominant Cycle Length` | 종목/타임프레임의 지배 사이클에 맞춰 밴드 민감도를 바꿀 때 (책의 사이클 도출 방법 참고) |
| `Narrow / Wide Band Width Threshold` | 수축/확장 배경이 켜지는 밴드폭 기준을 바꿀 때 |
| `Show Band Width Background` | 배경 강조를 끄고 선만 보고 싶을 때 |
| `밴드 배경 불투명성 (0=투명, 100=진함)` | 배경 진하기를 조절할 때. **0이면 완전 투명(안 보임)**, 100이면 가장 진함. 기본 15(연한 편) |

## 주의사항

- `Dominant Cycle Length`는 임의값이 아니라, 원서(4장)에서 설명한 방식대로 **실제 지배 사이클을 도출해 넣어야** 의도대로 동작합니다.
- 초기 `cyclicmemory`봉 구간에서는 밴드와 배경이 안정화되지 않습니다.
- 단독 진입 근거보다, 자리·거래량·추세 지표와 함께 보는 보조 확인용에 가깝습니다.
- `vibration`, `leveling`은 현재 입력값이 아니라 코드 상수입니다. 바꾸려면 소스를 직접 수정해야 합니다.
