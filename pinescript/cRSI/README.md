# cRSI (RSI Cyclic Smoothed)

트레이딩뷰용 Pine Script 보조 지표 설명서입니다.

대상 스크립트:
- [`rsi-cyclic-smoothed.pine`](./rsi-cyclic-smoothed.pine)

> Pine Script **v6** 기준입니다(v4에서 마이그레이션, 신호·계산·파라미터 동작은 동일하게 보존).

이 지표는 기본 `RSI`에 **지배 사이클(Dominant Cycle)** 기반의 평활화와 **동적 과매수/과매도 밴드**를 더한 버전입니다. 고정된 `30 / 70` 기준 대신, 최근 사이클 구간에서 cRSI가 실제로 머문 범위를 기준으로 상·하단 밴드를 계산해 줍니다. 원작은 Lars von Thienen의 *Decoding The Hidden Market Rhythm*(2017) 4장에서 소개된 cRSI입니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| CRSI (기본 차분한 골드 선) | 사이클로 평활화한 RSI 본선입니다. 하늘색 밴드, 녹·적 원 마커와 대비되도록 기본 차분한 골드(`#E6B800`, 밝은 노랑보다 명도·채도를 낮춘 골드 톤)로 표시하며, `선 색상` 그룹의 `cRSI 선 색` 입력으로 원하는 색을 고를 수 있습니다. 노랑 계열은 주황(수축) 배경과 색상이 인접하므로, 수축 구간에서는 배경 대비가 다소 약해질 수 있습니다(밴드폭 상태는 상태판 셀 배경색으로도 확인 가능). |
| cMFI (기본 차분한 골드 점선, 기본 미표시) | cRSI와 **동일 주기·동일 평활 파이프라인**에 입력만 RSI 대신 MFI(Money Flow Index)를 넣은 순환 평활 MFI 선입니다(기본 차분한 골드 `#E6B800` — cRSI와 동일 색, `선 색상` 그룹의 `cMFI 선 색` 입력으로 변경 가능). 본선(cRSI 골드 실선)과 색은 같지만 **모양으로 구분**되도록 **점선 느낌으로 표시**합니다 — Pine v6 `plot()`은 dashed 라인 스타일을 직접 지원하지 않아, 동적 series에 맞게 `plot.style_circles`(작은 원 연속)로 점선을 근사합니다. **기본적으로 표시하지 않으며**, `선 색상` 그룹의 `cMFI 표시` 토글을 켜면 나타납니다. 켜면 가격 흐름(cRSI)과 자금 흐름(cMFI)을 같은 간격으로 겹쳐 봅니다. cRSI와 색이 같으므로 실선(cRSI) vs 점선 원(cMFI) 스타일 차이로 구분합니다. |
| HighBand / LowBand (기본 밝은 하늘색/시안 선) | 최근 사이클 메모리 구간에서 동적으로 계산한 과매수/과매도 밴드입니다. 상·하단 모두 기본 밝은 하늘색/시안(`#4DD0E1`, Material Cyan 300 계열)으로 표시하며, `선 색상` 그룹의 `상단밴드(HighBand) 색`·`하단밴드(LowBand) 색` 입력으로 각각 원하는 색을 고를 수 있습니다. **밴드 상·하단(HighBand/LowBand) 값은 우측 프라이스 스케일(가격 축) 라벨에 표시됩니다.** |
| 과매도/과매수 참고 점선 (은색) | 참고용 고정 기준선입니다. 기본 `30`(과매도)·`70`(과매수)이며, `수축 돌파 마커` 그룹의 `과매도 참고 레벨`·`과매수 참고 레벨` 입력으로 조정할 수 있습니다(이 값은 아래 수축 돌파 원 마커의 "인근" 판정에도 함께 쓰입니다). |
| Band Width Background | 밴드폭이 좁으면 진한 주황색(수축), 넓으면 보라색(확장) 배경으로 강조합니다. 불투명성 기본 15(0=투명, 100=진함)로 연하게 깔립니다. |
| 수축 돌파 원 마커 (녹색/빨강) | 밴드폭 수축(주황) 구간에서 cRSI가 밴드를 돌파할 때 **돌파되는 밴드선 위치**에 표시하는 원 마커입니다. **녹색 원**(상단밴드 위치)=골든크로스(하단밴드가 과매도 인근인 상태에서 cRSI가 상단밴드를 상향 돌파), **빨강 원**(하단밴드 위치)=데드크로스(상단밴드가 과매수 인근인 상태에서 cRSI가 하단밴드를 하향 돌파). 기본 표시(`수축 돌파 원 마커 표시` 토글). 자세한 조건은 아래 "수축 돌파 원 마커" 항목 참고. |
| 확장 돌파 원 마커 (녹색/빨강) | 밴드폭 확장(보라) 구간에서 cRSI가 밴드를 돌파할 때 **돌파되는 밴드선 위치**에 표시하는 원(●) 마커입니다. **녹색 원**(하단밴드 위치)=골든크로스(cRSI가 하단밴드 LowBand를 상향 돌파), **빨강 원**(상단밴드 위치)=데드크로스(cRSI가 상단밴드 HighBand를 하향 돌파). 과매도/과매수 인근 조건 없음. 수축 마커(원)와 모양·크기는 동일(원, `linewidth=4`)하며 **배경색(주황 수축 / 보라 확장)으로 구분**합니다. 기본 표시(`확장 돌파 원 마커 표시` 토글). 자세한 조건은 아래 "확장 돌파 원 마커" 항목 참고. |

참고(축 라벨 표시):
- 우측 **프라이스 스케일(가격 축) 라벨에는 CRSI 본선 값과 함께 밴드 상·하단(HighBand/LowBand) 값**이 표시됩니다. 현재 과매수/과매도 경계가 축에서 바로 읽힙니다. cMFI만 패널에 선으로는 계속 그려지되 우측 축 라벨에는 뜨지 않습니다(`display = display.all - display.price_scale`).

참고:
- 기본값은 `Dominant Cycle Length = 20`이며, 내부적으로 RSI 길이 `cyclelen = 10`, 사이클 메모리 `cyclicmemory = 40`으로 환산됩니다.
- `vibration = 10`, `leveling = 10.0`은 코드 내부 상수로 고정되어 있습니다(입력값 아님).
- 동적 밴드는 사이클 메모리(`40봉`) 구간에서 cRSI 값의 분포를 백분위로 훑어, 하위 `10%`(LowBand)·상위 `10%`(HighBand) 경계를 잡습니다.
- 밴드폭 배경은 `bar_index < cyclicmemory`(초기 구간)에서는 표시되지 않습니다.

## cRSI를 쓰는 이유

일반 RSI는 `30 / 70` 같은 고정 기준을 쓰지만, 종목·국면에 따라 RSI가 실제로 도는 범위가 다릅니다.

- 추세장에서는 RSI가 한쪽에 쏠려 `70`을 계속 넘겨도 과매수로 보기 애매합니다.
- 횡보장에서는 `30 / 70`에 닿기 전에 이미 반전하기도 합니다.

cRSI는 사이클로 평활화해 노이즈를 줄이고, **최근 구간에서 cRSI가 실제로 머문 범위**로 밴드를 그려 "지금 국면 기준으로 과한지"를 봅니다.

## 트레이딩 의미 핵심

### 1. cRSI가 LowBand 아래일 때

`최근 사이클 기준으로도 과하게 눌린 구간`을 뜻합니다.

- 고정 `30`보다 그 국면에 맞는 과매도 경계라서, 횡보·추세 양쪽에서 더 현실적인 자리로 봅니다.
- 단독 진입보다는 `되돌림 감시 자리`로 보는 편이 맞습니다.

### 2. cRSI가 HighBand 위일 때

`최근 사이클 기준으로도 과하게 올라간 구간`을 뜻합니다.

- 급등 뒤 상승 피로를 보는 데 유용합니다.
- 메인 신호가 약해지는 자리와 겹치면 `후행 롱 자제`, `숏 감시` 의미가 커집니다.

### 3. 밴드폭 배경(진한 주황색 / 보라)

밴드폭(`HighBand - LowBand`)은 변동성/사이클 진폭의 대리 지표입니다.

- **진한 주황색(좁은 밴드)**: cRSI가 좁은 범위에서만 움직이는 `수축 구간`. 큰 움직임 직전의 눌림일 수 있어, 밴드 돌파 방향을 주시합니다.
- **보라(넓은 밴드)**: cRSI가 크게 출렁이는 `확장 구간`. 신호가 자주 극단에 닿으므로 과매수/과매도 신호의 신뢰도를 한 번 더 거릅니다.

## 내부 로직

1. `cyclelen = domcycle / 2` 길이로 기본 RSI를 계산합니다.
2. `torque`, `phasingLag`(vibration 기반)로 cRSI를 사이클 평활화합니다.
   - `crsi := torque * (2*rsi - rsi[phasingLag]) + (1 - torque) * crsi[1]`
3. 최근 `cyclicmemory(= domcycle * 2)`봉 구간에서 cRSI의 최댓값/최솟값을 찾습니다.
4. 그 범위를 100단계로 나눠, 분포의 하위 `leveling%`(LowBand)·상위 `leveling%`(HighBand) 경계를 백분위로 잡습니다.
5. 밴드폭으로 배경색(수축/확장)을 칠합니다.

즉 고정 임계선이 아니라, `현재 국면에서 cRSI가 실제로 돈 범위`를 기준으로 과매수/과매도를 판단하는 방식입니다.

## 순환 평활 MFI (cMFI, 차분한 골드 점선)

cRSI가 **가격(RSI)** 을 사이클 평활한 선이라면, cMFI는 **자금 흐름(MFI, Money Flow Index)** 에 **같은 파이프라인을 그대로** 적용한 선입니다.

- **동일 주기**: MFI 계산 길이를 cRSI와 같은 `cyclelen(= domcycle / 2)`으로 씁니다(`mfi(hlc3, cyclelen)`). 별도 파라미터가 없어 cRSI와 언제나 같은 간격입니다.
- **동일 평활**: cRSI가 쓰는 `torque`·`phasingLag`(둘 다 `vibration` 기반)를 그대로 재사용해 같은 재귀식으로 평활합니다.
  - `cmfi := torque * (2*mfiRaw - mfiRaw[phasingLag]) + (1 - torque) * cmfi[1]`
- **읽는 법**: 두 선이 같은 방향으로 정렬(가격과 자금 흐름 동조)이면 신호 신뢰도가 올라가고, cRSI는 오르는데 cMFI가 따라오지 못하면(또는 반대) 자금이 뒷받침되지 않는 움직임으로 봅니다. 단독 신호가 아니라 **cRSI와의 정렬 확인용 교차 검증선**입니다.
- MFI는 거래량을 반영하므로, 거래량이 없거나 불안정한 심볼에서는 cMFI가 왜곡될 수 있습니다.

## 우측 상단 상태판

맨 윗줄 제목 칸(`cRSI`)에 현재 `포지션` 값이 색상과 함께 표시됩니다.

| 행 | 내용 |
| --- | --- |
| cRSI (제목) | `포지션`: `롱 유리` / `숏 유리` / `중립` (사유 포함) |
| cRSI 값 | cRSI 값 + 괄호 안에 밴드폭 값·상태 — 예: `34.0 (40.0 중립)`. 밴드폭 상태는 **셀 배경색**으로 표시: `수축`=진한 주황색, `확장`=보라, `중립`=회색(텍스트엔 색 이름을 넣지 않음) |
| 모멘텀 | cRSI 기울기·위치 기반 5분류: `상승 강` / `상승` / `중립` / `하락` / `하락 강` (아래 "모멘텀 행" 참고). 이전의 `밴드 위치` 행(`LowBand(과매도)` / `HighBand(과열)` / `중립`)을 대체합니다 — 밴드 위치 정보는 0행 `포지션` 판정에 이미 반영되어 있습니다. |

### 모멘텀 행

3행은 cRSI의 **모멘텀(기울기)**을 상태로만 서술합니다(매매 지시어 없이 상승/하락/중립). 기울기는 `crsiSlope = crsi − crsi[phasingLag]`(현재 cRSI − `phasingLag`봉 전 cRSI, `phasingLag`는 평활 지연값 = `int((vibration−1)/2) = 4`봉)로 계산하며, 확정된 과거 봉만 참조하므로 리페인팅이 없습니다. 임계값 `slopeEps = 2.0`(코드 상수)로 유의미한 기울기 여부를 가릅니다.

우선순위는 **크로스 지속(`crsiBreak`) > 위치+기울기 조합**입니다. 판정 소스는 수축 돌파 **지속 래치**(`crsiBreak`, 밴드 복귀 전까지 유지)이며, 1회성 원 마커 이벤트(골든/데드)는 판정에 쓰지 않습니다(원 마커로 이미 별도 표현).

| 상태 | 조건 | 셀 배경색 |
| --- | --- | --- |
| **상승 강** | 상단 돌파 지속(`crsiBreak == 1`) **또는** (cRSI가 과매도(`LowBand`)·밴드 안 위치 **이면서** 기울기 > `slopeEps`) | 녹색(진함) |
| **상승** | 크로스 지속 없음 **이고** 기울기 > `slopeEps` (강 게이트 미충족) | 탁한 녹색 `#6E8F72` |
| **중립** | 기울기 절댓값 ≤ `slopeEps` | 회색 |
| **하락** | 크로스 지속 없음 **이고** 기울기 < −`slopeEps` (강 게이트 미충족) | 탁한 적색 `#A57A7A` |
| **하락 강** | 하단 돌파 지속(`crsiBreak == -1`) **또는** (cRSI가 과매수(`HighBand`)·밴드 안 위치 **이면서** 기울기 < −`slopeEps`) | 적색(진함) |

- **"강" 게이트**: 수축 돌파 지속(`crsiBreak`) 상태면 무조건 "강". 크로스 지속이 없을 때는, 위치가 극단에서 반전하거나(과매도에서 상승 / 과매수에서 하락) 밴드 안(중립 위치)에서 방향이 붙을 때 "강", 반대 극단(과매수에서 상승 / 과매도에서 하락)이면 일반 "상승/하락"으로 낮춥니다.

`포지션` 판정 우선순위:
1. **수축 돌파 지속**: 수축(진한 주황색)에서 하단 돌파 → `숏 유리(하단돌파)`, 상단 돌파 → `롱 유리(상단돌파)`. cRSI가 밴드 안으로 복귀할 때까지 유지(돌파 후 밴드가 확장돼도 지속).
2. **확장 다이버전스**: 확장(보라)에서 상승 다이버전스 → `롱 유리(상승 다이버전스)`, 하락 다이버전스 → `숏 유리(하락 다이버전스)`. 의도한 방향으로 밴드를 돌파한 뒤 다시 밴드 안으로 복귀하면 종료. (차트에 ▲▼ 마커로 표시)
3. **과매수/과매도 되돌림**: `HighBand` → `숏 유리(과매수)`, `LowBand` → `롱 유리(과매도)`.
4. 그 외 `중립`(수축 중 밴드 안이면 `중립(수축)`).

## 수축 돌파 원 마커

밴드폭이 **수축(진한 주황색 배경)** 인 구간은 큰 움직임 직전의 눌림일 수 있어, 밴드 돌파 방향을 주시합니다. 이때 cRSI가 밴드를 뚫고 나가면 **돌파되는 밴드선 위치**(골든=상단밴드 ub, 데드=하단밴드 db)에 원을 찍어 눈에 띄게 합니다.

| 마커 | 조건 | 의미 |
| --- | --- | --- |
| 녹색 원 (골든크로스, 상단밴드 위치) | 당봉 수축(현재 봉이 주황 배경) **AND** 하단밴드가 과매도 인근(`과매도 레벨 − 마진 ≤ LowBand ≤ 과매도 레벨 + 마진`) **AND** cRSI가 상단밴드(HighBand)를 **상향 돌파**(`ta.crossover(crsi, ub)`) | 눌림 국면(하단밴드가 과매도 부근)에서 cRSI가 상단 과매수 밴드를 뚫고 반등 시작 → 롱 관점 |
| 빨강 원 (데드크로스, 하단밴드 위치) | 당봉 수축(현재 봉이 주황 배경) **AND** 상단밴드가 과매수 인근(`과매수 레벨 − 마진 ≤ HighBand ≤ 과매수 레벨 + 마진`) **AND** cRSI가 하단밴드(LowBand)를 **하향 돌파**(`ta.crossunder(crsi, db)`) | 과열 국면(상단밴드가 과매수 부근)에서 cRSI가 하단 과매도 밴드를 하향 이탈 → 숏 관점 |

- **"수축 구간" 정의(마커)**: 원 마커는 **당봉 수축**(`narrowBw` — 현재 봉이 주황 배경일 때)만 인정합니다. CEO 정의("주황색 배경일 때")에 맞춰, 돌파 봉 자체가 주황이어야 원을 찍습니다(직전 봉 수축은 불인정). 참고로 상태판의 돌파 지속 래치는 `narrowRecently`(현재 또는 직전 2봉 내 수축)를 그대로 사용하며, 마커와 판정 기준이 다릅니다.
- **"과매도/과매수 인근" 정의(양방향 ±마진)**: 이 지표에는 고정 과매도/과매수 임계 변수가 없고(밴드 자체가 동적 경계), 참고 점선 `30 / 70`이 전통적 과매도/과매수 라인입니다. 이를 재사용해 **밴드가 해당 레벨의 위·아래 근처(±마진) 범위 안에 있을 때만** "인근"으로 봅니다 — 하단밴드가 `과매도 레벨(기본 30) ± 마진` 범위, 상단밴드가 `과매수 레벨(기본 70) ± 마진` 범위. 즉 "밴드 하단이 과매도 부근 = 과매도 선 위나 아래 근처"(과매수도 동일)라는 양방향 정의입니다. 마진 기본값은 `5.0`(cRSI 0~100 스케일에서 밴드가 전통 과매도/과매수 영역에 실질적으로 들어와 있음을 뜻하는 보수적 여유값). 세 값 모두 `수축 돌파 마커` 그룹의 입력으로 조정할 수 있습니다.
- **확정봉 기준**: 크로스는 `barstate.isconfirmed`가 참인 **마감(확정) 봉**에서만 원을 찍습니다. 진행 중(미마감) 봉에서 값이 출렁여 원이 나타났다 사라지는 점멸을 막습니다.
- **리페인팅 주의**: cRSI와 동적 밴드(db/ub)는 최근 사이클 메모리 구간의 재계산에 기반하므로, **이론적으로는** 새 봉이 만들어질 때 과거 밴드 값이 미세하게 변동해 확정봉이라도 원 위치가 약간 흔들릴 수 있습니다(밴드 지표 특성상 완전 무리페인팅은 아님). 다만 확정봉 게이트로 미마감봉 점멸은 제거했습니다. 단독 진입 근거가 아니라 이치모쿠·테이블·타 보조지표 정렬과 함께 보는 확인용으로 씁니다.

## 확장 돌파 원 마커

밴드폭이 **확장(보라 배경)** 인 구간은 cRSI가 크게 출렁이는 국면입니다. 이때 cRSI가 동적 밴드를 뚫고 나가면 **돌파되는 밴드선 위치**(골든=하단밴드 db, 데드=상단밴드 ub)에 원(●)을 찍어 눈에 띄게 합니다. 수축 구간의 원 마커와 달리 **과매도/과매수 인근 조건이 없습니다**(CEO 신규 규칙).

| 마커 | 조건 | 의미 |
| --- | --- | --- |
| 녹색 원 (골든크로스, 하단밴드 위치) | 당봉 확장(현재 봉이 보라 배경, `wideBw`) **AND** cRSI가 하단밴드(LowBand)를 **상향 돌파**(`ta.crossover(crsi, db)`) | 확장 국면에서 cRSI가 하단 밴드를 위로 뚫고 반등 시작 → 롱 관점 |
| 빨강 원 (데드크로스, 상단밴드 위치) | 당봉 확장(현재 봉이 보라 배경, `wideBw`) **AND** cRSI가 상단밴드(HighBand)를 **하향 돌파**(`ta.crossunder(crsi, ub)`) | 확장 국면에서 cRSI가 상단 밴드를 아래로 이탈 → 숏 관점 |

- **"확장 구간" 정의(마커)**: 당봉 확장(`wideBw` — 현재 봉이 보라 배경, `bandWidth >= wideLimit`)일 때만 인정합니다. 배경 보라 조건과 동일 변수를 사용하므로, 원이 찍히는 봉은 반드시 보라 배경입니다.
- **마진 조건 없음**: 수축 원 마커와 달리 과매도/과매수 인근(±마진) 판정을 두지 않습니다. 확장 국면에서는 밴드 자체가 이미 크게 벌어져 있어, 돌파 방향만 봅니다(CEO 신규 규칙).
- **수축 원 마커와의 구분**: 배경(주황↔보라)이 배타적이라 두 종류 마커가 같은 봉에 동시에 나오지는 않습니다. 마커 모양·크기는 **수축·확장 모두 원(●, `plot.style_circles`, `linewidth=4`)으로 동일**하며, 구분은 **배경색(주황 수축 / 보라 확장)** 으로만 합니다. 색(녹=골든/적=데드)도 두 종류가 동일합니다.
- **확정봉 기준**: 크로스는 `barstate.isconfirmed`가 참인 **마감(확정) 봉**에서만 원을 찍습니다. 미마감 봉 점멸을 막습니다.
- **리페인팅 주의**: 수축 원 마커와 동일합니다 — cRSI·동적 밴드(db/ub)는 사이클 메모리 재계산 기반이라 완전 무리페인팅은 아니며, 확정봉 게이트로 미마감봉 점멸만 제거했습니다. 단독 진입 근거가 아니라 이치모쿠·테이블·타 보조지표 정렬과 함께 보는 확인용으로 씁니다.

## 자주 만지는 설정

| 설정 | 언제 조정하나 |
| --- | --- |
| `Dominant Cycle Length` | 종목/타임프레임의 지배 사이클에 맞춰 밴드 민감도를 바꿀 때 (책의 사이클 도출 방법 참고) |
| `Narrow / Wide Band Width Threshold` | 수축/확장 배경이 켜지는 밴드폭 기준을 바꿀 때 |
| `Show Band Width Background` | 배경 강조를 끄고 선만 보고 싶을 때 |
| `밴드 배경 불투명성 (0=투명, 100=진함)` | 배경 진하기를 조절할 때. **0이면 완전 투명(안 보임)**, 100이면 가장 진함. 기본 15(연한 편) |
| `cMFI 표시` | 자금 흐름(cMFI) 교차 검증선을 켜고 싶을 때. **기본 꺼짐**(cRSI 본선만 보고 싶을 때 유지) |
| `수축 돌파 원 마커 표시` | 수축 구간 밴드 돌파 원(녹색/빨강)을 끄고 싶을 때. 기본 켜짐 |
| `확장 돌파 원 마커 표시` | 확장 구간 밴드 돌파 원(녹색/빨강)을 끄고 싶을 때. 기본 켜짐 |
| `과매도 / 과매수 참고 레벨` | 참고 점선과 수축 돌파 마커의 "인근" 판정 기준(기본 30 / 70)을 바꿀 때 |
| `과매도/과매수 인근 마진` | 밴드가 과매도/과매수 레벨의 위·아래 얼마 범위(±마진) 안에 있어야 돌파 마커를 인정할지 여유폭(기본 5.0)을 바꿀 때 |

## 주의사항

- `Dominant Cycle Length`는 임의값이 아니라, 원서(4장)에서 설명한 방식대로 **실제 지배 사이클을 도출해 넣어야** 의도대로 동작합니다.
- 초기 `cyclicmemory`봉 구간에서는 밴드와 배경이 안정화되지 않습니다.
- 단독 진입 근거보다, 자리·거래량·추세 지표와 함께 보는 보조 확인용에 가깝습니다.
- `vibration`, `leveling`은 현재 입력값이 아니라 코드 상수입니다. 바꾸려면 소스를 직접 수정해야 합니다.
