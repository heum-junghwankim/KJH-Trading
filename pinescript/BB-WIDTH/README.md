# BBW (Bollinger Band Width, 볼린저밴드 폭)

트레이딩뷰용 Pine Script **보조 지표** 설명서입니다.

대상 스크립트:
- [`bollinger-band-width.pine`](./bollinger-band-width.pine)

> Pine Script **v6** 기준. 별도 패널(`overlay=false`, 기본)에 그려지는 **변동성(국면) 보조 지표**입니다.

CEO 제공 "Bollinger Bands %b" 소스코드를 기반으로 재구성했습니다. %b(가격의 밴드 내 상대 위치)가 아니라, **볼린저밴드가 얼마나 벌어졌는지(밴드 폭 = 변동성)** 를 산출·표시합니다.

## ⚠️ 이 지표는 방향성 신호가 아닙니다

BBW는 **변동성(국면)** 만 알려줍니다. "폭이 넓다/좁다"는 방향(롱·숏)을 말하지 않습니다.
- **수축(좁은 폭)**: 변동성이 눌린 구간. 큰 움직임 직전의 눌림일 수 있으나, **어느 방향으로 터질지는 BBW가 답하지 않습니다.**
- **확장(넓은 폭)**: 변동성이 커진 구간. 추세가 진행 중이거나 급변 중일 수 있습니다.

방향은 반드시 이치모쿠(메인) + 테이블 + 다른 보조지표 정렬로 교차 검증하세요. BBW는 "지금 시장이 조용한가 시끄러운가"를 보는 필터입니다.

## BBW 정의 · 표시 스케일

**표준 정의**를 사용합니다:

```
BBW = (upper − lower) / basis
    upper = basis + mult × stdev(src, length)
    lower = basis − mult × stdev(src, length)
```

- `basis` = 중심 이동평균(SMA/WMA/VWMA/EMA 중 선택, 아래 참조)
- `stdev` = 표준편차(`ta.stdev`) — **MA 종류와 무관하게 표준대로 유지**
- `basis = 0` 또는 `na`이면 정의 불가 → `na` 가드(0으로 나누기 방지)

**표시 스케일: ×100(%) 표기를 채택했습니다.**
- 근거: `(upper−lower)/basis`의 원값은 보통 0.02~0.15 같은 작은 소수라 차트에서 눈금이 뭉칩니다. 관례적으로 ×100(%)로 표기하면 `2.00`, `15.00`처럼 읽기 쉬워 트레이더가 폭 변화를 3초 안에 파악하기 좋습니다.
- 정의 자체(표준 (upper−lower)/basis)는 그대로 두고, **표시 값만** ×100 했습니다. 상태판·플롯 모두 % 스케일입니다.

## 밴드 폭 라인 표시 모드 (히스토그램=세로 막대 / 선)

메인 밴드 폭(BBW) 값의 표시 방식을 `밴드 폭 라인 표시 모드` 입력으로 고릅니다. **기본값은 히스토그램(세로 막대)**입니다.

- **히스토그램(세로 막대)**: `plot.style_columns`(세로 막대/컬럼). **비정상 거래량 지표(VOLUME-PRESSURE)·MACD 히스토그램과 동일한 컬럼 표현**입니다. 봉마다 독립된 세로 막대라 폭의 절대 크기·확장/수축을 한눈에 보기 좋습니다.
- **선**: 꺾은선(`plot.style_line`). 폭의 추세·기울기를 보기 좋습니다.

> CEO 피드백 반영: 기존 `plot.style_histogram`(0~값 채움 형태)이 낯설다는 의견에 따라 **세로 막대(컬럼)** 로 변경했습니다.

**평균선(폭 이동평균)은 이 옵션과 무관하게 항상 '선'으로 표시**됩니다(변경되지 않음).

구현 참고: Pine v6에서 `plot()`의 `style`은 컴파일 타임 상수라 입력으로 직접 바꿀 수 없습니다. 그래서 막대용·선용 **plot 2개**를 두고, 선택되지 않은 쪽에는 값으로 `na`를 넘겨 그려지지 않게 합니다(표준 패턴). 두 plot은 굵기가 같고, **색은 봉마다 VWAP 상태 색**(아래 참조)을 series로 받아 동일하게 칠합니다.

## 막대/선 색 = 이치모쿠 'VWAP 상태' 색 규칙 (동일 규칙 이식)

CEO 지시로, BBW 세로 막대(및 선) 색을 이치모쿠(`advanced-ichimoku.pine`)의 **'VWAP 상태' 셀 색과 동일한 규칙**으로 칠합니다. `막대/선에 VWAP 상태 색 적용` 입력으로 켜고 끕니다(**기본 켜짐**). 끄면 기본 선 색(`#2962FF` 파랑) 단색으로 표시합니다.

이를 위해 BBW 지표 안에서도 **Auto VWAP를 계산하고 VWAP 상태를 판정**합니다(이치모쿠와 동일한 계산·상태 로직 이식). VWAP선·밴드는 차트에 그리지 않고(별도 패널), 오직 막대색 결정에만 사용합니다.

**색 결정 규칙 (이치모쿠 `vwBandColor`와 100% 동일 — 방향 + 밴드워킹 + 지지/저항 조합):**

| VWAP 상태 | 색 | 색값 |
| --- | --- | --- |
| 상승 + 밴드워킹(상·하단) | 선명 초록 | `color.green` |
| 상승 + 그 외 | 둔탁 녹 | `#6E8F72` |
| 하락 + 밴드워킹(상·하단) | 선명 빨강 | `color.red` |
| 하락 + 그 외 | 둔탁 빨강 | `#A57A7A` |
| 평탄(중심) + 상단 밴드워킹 | 둔탁 녹 | `#6E8F72` |
| 평탄(중심) + 하단 밴드워킹 | 둔탁 빨강 | `#A57A7A` |
| 평탄(중심) + 하단 지지 | 둔탁 녹 | `#6E8F72` |
| 평탄(중심) + 상단 저항 | 둔탁 빨강 | `#A57A7A` |
| 평탄(중심) + 중앙(공방) | 회색 | `color.gray`(투명 30) |
| VWAP 계산 불가 | 회색 | `color.gray`(투명 40) |

**상태 판정 요소(이치모쿠와 동일):**
- **방향(기울기)**: VWAP 5봉 전 대비 변화가 밴드폭(2σ)의 ±3% 초과면 상승/하락, 이내면 평탄(중심).
- **밴드폭 확장/수축**: 2σ 밴드폭이 5봉 전 대비 확대/축소.
- **밴드워킹**: 수축·평탄 뒤 방향 봉이 2σ 밴드를 돌파하면 진입, 반대편 밴드가 되돌려질 때까지 유지(상태 변수 `var int vwWalk`).
- **지지/저항**: 밴드워킹이 아니면서 수축/평탄 + 종가가 1σ 밴드 밖(하단=지지, 상단=저항).

이 규칙은 이치모쿠 우측 테이블 'VWAP 상태' 셀 색과 봉 단위로 동일하게 움직이므로, BBW 패널만 봐도 **이치모쿠 VWAP 국면과 색으로 교차 검증**할 수 있습니다.

**VWAP 계산 기본값**: 앵커 `Recent Bars` / 최근 `20`봉 / 소스 `hlc3` / `Log Standard Deviation` / 밴드 배수 `1.0`·`2.0`. 세부 입력은 `VWAP 상태 색` 그룹에 있으며 대부분 `display.none`(설정창에만 노출, 차트 축 비노출)입니다.

> **최근 봉 수(`vwapRecentBars`)는 BBW에서 기본 `20`으로, 이치모쿠 원본(`50`)과 다릅니다.** BBW 패널의 색 국면을 더 최근 흐름에 민감하게 반응시키기 위한 설정입니다. 나머지 VWAP 계산 기본값은 이치모쿠와 동일합니다.

## 이동평균(basis) 4종 선택

`이동평균 종류(MA Type)` 입력(`input.string`, options)으로 basis 계산에 쓸 이동평균을 고릅니다. **표준편차 항은 종류와 무관하게 항상 `ta.stdev`로 계산**되며, MA 종류는 오직 `basis`(중심선)에만 반영됩니다.

| 옵션 | 함수 | 성격 |
| --- | --- | --- |
| **SMA** (기본) | `ta.sma` | 단순이동평균. 원본 볼린저밴드 관례 기본값. |
| **WMA** | `ta.wma` | 가중이동평균. 최근 봉에 더 큰 가중치. |
| **VWMA** | `ta.vwma` | 거래량 가중이동평균. 거래량 많은 봉을 더 반영. |
| **EMA** | `ta.ema` | 지수이동평균. 최근 값에 지수적으로 반응. |

**구현 방식** (헬퍼 `maSelect`):
```pine
maSelect(source, len, typ) =>
    typ == "EMA"  ? ta.ema(source, len) :
     typ == "WMA"  ? ta.wma(source, len) :
     typ == "VWMA" ? (na(volume) ? ta.sma(source, len) : ta.vwma(source, len)) :
     ta.sma(source, len)
```

**VWMA의 volume na 처리:** VWMA는 거래량에 의존합니다. FX·일부 CFD·지수 등 **volume이 없는(na) 심볼**에서는 `ta.vwma`가 `na`를 반환해 밴드 전체가 사라질 수 있습니다. 이를 막기 위해 `na(volume)`이면 **SMA로 폴백**합니다(밴드가 없어지지 않도록). 즉 거래량 없는 심볼에서 VWMA를 골라도 지표는 SMA로 안전하게 동작합니다.

**폭 이동평균선도 같은 MA 종류를 따릅니다:** BBW 자체의 이동평균선(`폭 이동평균선`)도 basis와 **동일한 `maType`(SMA/WMA/VWMA/EMA)** 로 계산합니다(`maSelect(bbw, maLen, maType)`). 이로써 basis와 폭 MA의 기준이 일관됩니다. 길이는 폭 MA 전용 `폭 이동평균 길이`(`maLen`)를 쓰며 basis의 `length`와는 별개입니다. VWMA를 고르면 폭 값(`bbw`)에 volume 가중이 적용되고, volume 없는 심볼에서는 위와 동일하게 **SMA로 폴백**됩니다.

## 넣은 보조 요소

핵심은 폭 라인이며, 국면 파악을 돕는 최소한의 보조 요소만 추가했습니다.

1. **폭 이동평균선(BBW의 MA)**: 현재 폭이 평소보다 넓은지/좁은지 한눈에 비교. BBW가 이 선 위면 확장, 아래면 수축 경향. basis와 동일한 MA 종류(maType)를 따릅니다. (우측 가격 축 라벨에는 표시하지 않아 축을 어지럽히지 않음)
2. **스퀴즈(최저폭) 배경 강조**: 현재 폭이 최근 N봉 최저폭에 허용치(%) 이내로 근접하면 배경으로 강조. 큰 변동 직전 눌림 후보를 시각적으로 표시. `ta.lowest`로 확정된 과거 값만 사용해 **리페인팅 없음**.
3. **폭 확장 배경 강조(보라)**: 확장 국면(BBW가 폭 MA 위, 상태판 '확장'과 동일 정의)일 때 배경을 보라로 강조. 스퀴즈 배경과 **배타적**으로 동작합니다(둘 다 성립 가능한 봉에서는 스퀴즈를 우선 표시). 불투명도는 `opacity`(클수록 진함)로 입력받아 `transp = 100 − opacity`로 변환해 적용합니다(기본 opacity 30 → 옅은 보라).

## 리페인팅 / 안전성

- 전 계산이 확정 `ta.*`(sma/ema/wma/vwma/stdev/lowest)로 이뤄지며 **미래참조·`lookahead` 없음**. 봉 마감 후 값이 바뀌지 않습니다.
- **VWAP 상태 이식도 `request.security`/`lookahead` 없음**: `var` 누적 + 확정 과거참조(`[n]`)만 사용하며, 이치모쿠 원본과 **동일한 성격**입니다. 봉 마감 후 값이 바뀌지 않습니다. 단, **실시간(미마감) 봉**에서는 진행 중 `close`/`volume`에 따라 VWAP·상태·막대색이 갱신될 수 있습니다(정상 — 이치모쿠 VWAP와 동일).
- VWAP 이식으로 과거참조 인덱싱(`vwCumPV[최근 봉 수]` 등)이 생겨 헤더에 `max_bars_back = 5000`을 두었습니다(이치모쿠와 동일). `timeframe` 인자는 넣지 않습니다(CE10080 방지).
- `basis = 0/na`, `volume = na`(VWMA), 초기 `bar_index < squeezeLen` 구간 모두 가드 처리.
- 별도 패널(비-overlay) 지표라 가격 차트를 가리지 않습니다.

## 주의사항

- **방향성 신호가 아님**(위 경고 참조). 변동성 필터로만 사용하세요.
- 파라미터(length·mult·MA 종류)를 바꾸는 것은 **변동성 민감도 변경**입니다. 신호 임계값 성격이므로 팀 합의 없이 임의 변경을 지양합니다.
- 스퀴즈 허용치·룩백은 종목/타임프레임에 따라 조정이 필요할 수 있습니다.
