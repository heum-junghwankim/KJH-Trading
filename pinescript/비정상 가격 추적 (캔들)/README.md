# 비정상 가격 추적 (캔들)

트레이딩뷰용 Pine Script 오버레이 지표 설명서입니다.

대상 스크립트:
- [`abnormal-price-tracker-candles.pine`](./abnormal-price-tracker-candles.pine)

문서 생성 스크립트:
- [`generate_candle_guide_pdf.py`](./generate_candle_guide_pdf.py): `FPDF`로 초보자용 시각 가이드 PDF를 생성합니다.
- [`generate_candle_guide_weasy.py`](./generate_candle_guide_weasy.py): `WeasyPrint`로 같은 가이드를 HTML 기반 PDF로 생성합니다.

이 지표는 `자리 -> 감시 -> 기준 -> 후보 -> 스윕` 흐름과 리포트 이벤트 체계를 한 장에서 읽기 위한 도구입니다. 모든 신호, 알림, 리포트 값은 `봉 마감 후 종가 확정` 기준으로만 갱신됩니다.

## 현재 소스 기준으로 보이는 것

| 요소 | 현재 표시 방식 |
| --- | --- |
| 0단계 비정상 캔들 | 과매수는 주황 계열, 과매도는 하늘 계열로 봉 색을 바꿉니다. |
| 1단계 다이버전스 | 강세/약세 다이버전스를 삼각형으로 표시하고, 옵션이 켜져 있으면 기준봉 존도 함께 그립니다. |
| 2단계 ATR 배경 / ATR 강화 | 자리 이후 실제 후속 이동이 붙으면 배경이 켜지고, 더 강하면 강화 색으로 바뀝니다. |
| 3단계 기준 봉 | 롱/숏 통합 점수가 조건을 넘은 첫 봉에만 원형 시그널과 기준봉 박스를 만듭니다. |
| 4단계 스윕 | 기준 봉 박스를 실제로 쓸고 간 봉은 밝은 봉 색으로, 다음 봉 재확인은 다이아몬드로 표시합니다. |
| 볼륨 풋프린트 | 옵션을 켜면 확정봉별 매수/매도 추정 볼륨, 델타, 하위봉 수를 라벨로 표시합니다. |
| 리포트 | 롱/숏 이벤트 누적, 시간, 포지션을 오른쪽 표로 보여줍니다. |

참고:
- 이벤트 누적 행은 `비정상 시작 -> 비정상 유지 -> 다이버전스 -> ATR 배경 -> ATR 강화 -> 기준 발생 -> 스윕 확인` 순서로 현재 흐름을 쌓아 갑니다.
- 리포트 포지션은 `OFF -> 관찰 -> 기준 -> 후보 -> 스윕` 다섯 상태만 사용합니다.
- `기준`은 실제 `3단계 기준 봉`이 발생한 현재 봉이고, `후보`는 그 뒤 마지막 `기준 봉 박스` 안에서 현재 봉이 유지되는 상태입니다.

## 트레이딩 의미 핵심 5가지

### 1. 비정상 캔들 의미

비정상 캔들은 `반전 확정`이 아니라 `자리 경고`입니다.

- cRSI 극단과 볼린저 몸통 이탈이 겹친 봉만 먼저 잡습니다.
- 즉 `지금 바로 뒤집힌다`보다 `여기서부터 반응을 감시할 만하다`에 가깝습니다.

요약하면, 0단계는 `자리 후보 시작`입니다.

### 2. 다이버전스 의미

다이버전스는 `강세/약세 효율 저하 힌트`입니다.

- 피벗 확정 뒤에 나오므로 약간 늦지만, 그만큼 무작정 빠른 신호는 줄입니다.
- 자리 후보 뒤에 붙으면 `반전 가능성 감시` 의미가 커집니다.

요약하면, 1단계는 `힌트`이고 아직 실행 단계는 아닙니다.

### 3. ATR 배경과 ATR 강화 의미

ATR 배경은 `자리 이후 실제 후속 이동이 붙었는지`, ATR 강화는 `그 후속 이동이 더 강한지`를 뜻합니다.

- 다이버전스나 비정상 캔들만으로 끝나지 않고 실제 움직임이 이어지는지 확인하는 단계입니다.
- 감시가 본격적으로 살아나는 구간은 보통 ATR 배경부터입니다.

요약하면, 2단계는 `감시 시작과 감시 강화`입니다.

### 4. 기준 봉 의미

기준 봉은 `점수 + 필터`를 모두 통과한 첫 실행 후보 봉입니다.

- 단순 배경이나 색상보다 우선순위가 높습니다.
- 실제로 기준 봉이 뜬 현재 봉만 `기준` 포지션으로 표시됩니다.

요약하면, 3단계는 `실행 후보가 처음 확인된 시점`입니다.

### 5. 스윕과 리포트 포지션 의미

스윕은 `기준 봉 이후 실제 유동성 확인`, 리포트 포지션은 `현재 행동 단계`를 뜻합니다.

- `후보`는 마지막 기준 박스 안에서 유지되는 상태입니다.
- `스윕`은 실제 스윕 확인 뒤에도 스윕 박스 안에서 흐름이 유지되는 상태입니다.

요약하면, 4단계는 `유동성 확인 이후 유지 상태`까지 보는 단계입니다.

## 내부 로직

### 1. 볼륨 풋프린트

`볼륨 풋프린트 표시`를 켜면 각 확정봉에 아래 값을 라벨로 표시합니다.

- `B`: 매수 우세로 추정한 거래량
- `S`: 매도 우세로 추정한 거래량
- `Δ`: `B - S` 델타와 델타 비율
- `LTF n`: 하위 타임프레임 봉 `n`개로 계산했다는 뜻

계산 방식은 두 단계입니다.

1. `하위 타임프레임으로 추정`이 켜져 있고 유효한 하위봉 데이터가 있으면, 지정한 하위 타임프레임의 각 봉을 종가 위치 기준으로 매수/매도 볼륨에 배분해 합산합니다.
2. 하위봉 데이터가 없거나 현재 차트보다 낮은 타임프레임을 쓸 수 없으면, 현재 봉의 `고가~저가` 범위 안에서 종가 위치를 기준으로 매수/매도 볼륨을 추정합니다.

주의: TradingView Pine Script는 실제 bid/ask 체결량이나 틱 단위 호가 풋프린트를 직접 제공하지 않습니다. 따라서 이 값은 실제 거래소 풋프린트가 아니라 OHLCV 기반 추정치로 보고, 비정상 캔들/ATR 배경/거래량 압력 확인을 보조하는 용도로 쓰는 편이 좋습니다.

### 2. 기준 봉 로직

#### 점수 구조

`기준 봉`은 `세팅 점수 + 확인 점수`를 합친 `통합 점수`로 계산합니다.

세팅 점수:

| 항목 | 기본 점수 |
| --- | --- |
| 최근 비정상 캔들 | `+2` |
| 최근 RSI 다이버전스 | `+2` |
| 현재 ATR 배경 | `+1` |
| 현재 ATR 강화 | `+1` |
| 최근 보조 신호 | `+1` |
| 최근 ATR 강화 근접 보너스 | `+2` 기본값 |
| 배경 근접 비정상 주문량 보너스 | `+2` 기본값 |
| ATR 강화 연속 눌림 보너스 | 가변값 |

`ATR 강화 연속 눌림 보너스`는 현재 코드에서 아래처럼 움직입니다.

- `ATR 강화`가 `최소 연속 봉 수` 이상 이어져야 후보가 됩니다.
- 롱은 `음봉 + 저가 갱신 + 눌림 깊이`, 숏은 `양봉 + 고가 갱신 + 반등 깊이`를 봅니다.
- 보너스는 `2~4봉` 구간에서 증가하고, `5~6봉`은 유지, `7봉 이상`은 감점됩니다.
- 같은 눌림 보너스는 한 번 실제 후보에 사용되면 소진됩니다.

확인 점수:

| 항목 | 기본 점수 |
| --- | --- |
| 거래량 확인 | `+2` |
| 봉 방향 일치 | `+1` |
| 직전 종가 대비 진행 | `+1` |
| cRSI 방향 일치 | `+1` |
| MFI 방향 일치 | `+1` |

#### 기준 봉 최소 조건

현재 코드는 아래를 모두 만족해야 `롱/숏 기준 봉`이 됩니다.

- `세팅 점수 >= 3`
- `확인 점수 >= 4`
- `통합 점수 >= 최소 진입 점수` 기본값 `8`
- 현재 방향 점수가 반대 방향 점수보다 큼
- `총 거래량 MA 미만 신호 차단` 조건 통과 시 현재 거래량이 총 거래량 평균 위
- `비정상 주문량 발생 필수` 사용 시 최근 비정상 주문량이 근처에 있어야 함
- 현재 봉에 반대 방향 `ATR 강화`가 켜져 있지 않아야 함

추가로 아래 필터가 켜져 있으면 함께 작동합니다.

- `이전 봉 배경 확인 필수`
- `반대 배경 전환 후 방향 전환`
- `동일 방향 신호 쿨다운`
- `강한 추세 연장 추가 진입 차단`

리포트 `포지션`은 점수 컷으로 `기준/후보`를 나누지 않습니다. 실제 `기준 봉` 시그널이 뜬 봉은 `기준`, 그 뒤 마지막 `기준 봉 박스` 안에서 현재 봉 `고가/저가`가 모두 유지될 때 `후보`로 읽습니다.

### 3. 스윕 로직

현재 스윕은 `다이버전스 박스`가 아니라 `기준 봉 박스`를 기준으로만 동작합니다.

#### 기준 봉 박스

- 롱 박스: `저가 ~ 종가`
- 숏 박스: `고가 ~ 종가`
- 리포트 `포지션`은 실제 `기준 봉` 시그널이 뜬 그 봉에서는 `기준`으로 표시합니다.
- 그 다음 봉부터는 현재 봉 `고가/저가`가 마지막 `기준 봉 박스` 안에 모두 있으면 `후보`로 표시합니다.
- 마지막 `기준 봉 박스`를 이탈하면 리포트 `포지션`은 `관찰`로 돌아가고, `포지션 이유`는 `기준 박스 이탈`로 표시합니다.
- 박스 유지 판정은 설정에서 `윅 포함 / 몸통만 / 종가만` 중 하나로 바꿀 수 있고, 추가 허용 틱도 줄 수 있습니다.

#### 스윕 박스 유지

- `스윕 확인`이 발생하면 마지막 스윕 확인 봉 기준으로 `스윕 박스`를 만듭니다.
- 그 뒤 현재 봉 `고가/저가`가 `스윕 박스` 안에 모두 있으면 리포트 `포지션`은 계속 `스윕`으로 유지됩니다.
- `스윕 박스`를 이탈하면 `포지션 이유`는 `스윕 박스 이탈`로 바뀌고 리포트 `포지션`은 `관찰`로 내려갑니다.
- 박스 유효 시간이 끝나면 `기준 박스 만료` 또는 `스윕 박스 만료`로 안내한 뒤 해당 박스 상태를 종료합니다.

#### 스윕 봉 조건

롱 쪽:
- 기존 롱 박스 하단을 `저가`로 이탈
- 현재 봉이 `음봉`
- `vpAbnormalVolume` 충족
- `sellVolume > buyVolume`

숏 쪽:
- 기존 숏 박스 상단을 `고가`로 이탈
- 현재 봉이 `양봉`
- `vpAbnormalVolume` 충족
- `buyVolume > sellVolume`

#### 다음 봉 다이아몬드 조건

롱 쪽:
- 바로 다음 봉 `양봉`
- `buyVolume > sellVolume`
- `현재 봉 거래량 >= 이전 스윕 봉 거래량 * 스윕 확인 최소 거래량 비율`

숏 쪽:
- 바로 다음 봉 `음봉`
- `sellVolume > buyVolume`
- `현재 봉 거래량 >= 이전 스윕 봉 거래량 * 스윕 확인 최소 거래량 비율`

기본값은 `0.5`입니다.

### 4. 리포트와 포지션 계산

리포트는 `시그널 | 롱 반등 감시 | 숏 반등 감시` 3열 구조입니다. 각 방향 리포트는 첫 이벤트가 들어오면 시작하고, `반대 방향 ATR 강화`가 나오면 종료됩니다.

#### 리포트 항목

| 항목 | 의미 |
| --- | --- |
| 비정상 시작 | 비정상 과매수/과매도 시작 이벤트 누적 수 |
| 비정상 유지 | 비정상 캔들 유지 누적 수 |
| 다이버전스 | 강세/약세 다이버전스 누적 수 |
| ATR 배경 | 감시 배경 누적 수 |
| ATR 강화 | 강한 감시 배경 누적 수 |
| 기준 발생 | 실제 `3단계 기준 봉` 발생 누적 수 |
| 스윕 확인 | 실제 `4단계 스윕 확인` 누적 수 |
| 시작 시간 | 해당 방향 리포트의 첫 이벤트 시간 |
| 종료 시간 | 반대 방향 `ATR 강화`로 꺼진 시간 |
| 포지션 | 현재 신호 기준 행동형 상태 + 내부 통합 점수 |
| 포지션 이유 | 현재 포지션 유지 또는 직전 이탈 이유 |

#### 포지션

현재 코드에서 실제로 계산되는 포지션은 아래 다섯 가지입니다.

| 포지션 | 현재 기준 |
| --- | --- |
| OFF | 리포트 비활성 |
| 관찰 | 리포트는 살아 있지만 아직 후보 전 |
| 기준 | 실제 `기준 봉` 시그널이 발생한 현재 봉 |
| 후보 | 마지막 `기준 봉 박스` 안에서 현재 봉 `고가/저가`가 모두 유지되는 상태 |
| 스윕 | `스윕 확인` 발생 후 마지막 `스윕 박스` 안에서 현재 봉 `고가/저가`가 모두 유지되는 상태 |

표시 예시는 아래처럼 읽으면 됩니다.

- `기준 (8)`: 실제 기준 봉 시그널이 발생한 봉
- `후보 (8)`: 최근 기준 봉 박스 안에서 다음 봉 움직임이 유지된 상태
- `스윕 (11)`: 스윕 확인이 발생했고, 이후에도 스윕 박스 안에서 움직임이 유지되는 상태
- `OFF`: 해당 방향 리포트가 꺼졌거나 아직 시작되지 않음

리포트 표 아래 둘째 줄은 포지션 단계가 아니라, `점수 해석 구간`을 안내합니다.

- 현재 문구는 `최소 진입 점수`, `점수 강화 기준` 설정에 따라 함께 바뀝니다. 기본값 기준으로는 `1~4점 약 / 5~7점 형성 / 8~9점 기준 / 10점+ 강화`입니다.
- 이 줄은 `관찰 / 기준 / 후보 / 스윕` 같은 포지션 단계가 아니라, 현재 점수 강도를 빠르게 읽는 참고선입니다.
- `8~9점`은 현재 코드에서 후보 최소 기준에 근접하거나 막 통과한 구간, `10점+`는 강한 점수 구간으로 읽으면 됩니다.

참고:
- 표 아래 첫 줄 문구도 `OFF / 관찰 / 기준 / 후보 / 스윕` 기준으로 맞춰졌습니다.

## 자주 만지는 설정

| 설정 | 언제 조정하나 |
| --- | --- |
| `최소 진입 점수` | 후보 수를 늘리거나 줄일 때 |
| `점수 강화 기준` | 현재 리포트 포지션 계산에는 직접 쓰지 않으므로, 기준/후보 해석보다 점수 감각을 볼 때 참고 |
| `포지션 박스 유지 판정` | 기준 박스와 스윕 박스를 `윅/몸통/종가` 중 어떤 기준으로 지킬지 바꿀 때 |
| `포지션 박스 허용 틱 수` | 박스 이탈 판정을 조금 느슨하게 할 때 |
| `동일 방향 신호 쿨다운` | 같은 방향 후보 반복을 줄일 때 |
| `비정상 주문량 발생 필수` | 주문량 필터를 강하게 유지할지 결정할 때 |
| `총 거래량 MA 미만 신호 차단` | 한산한 봉 후보를 막고 싶을 때 |
| `ATR 강화 연속 최소 봉 수` | 눌림 보너스를 더 빨리/늦게 붙이고 싶을 때 |
| `눌림 최소 ATR 비율` | 눌림 깊이 기준을 조정할 때 |
| `강한 추세 연장 추가 진입 차단` | 추세 후반 후보 남발을 줄일 때 |
| `스윕 확인 최소 거래량 비율` | 다이아몬드 확인을 더 엄격하게 할 때 |
| `리포트 표시`, `리포트 표시 위치`, `리포트 글자 크기` | 리포트 가독성 조정 |
| `볼륨 풋프린트 표시` | 확정봉의 매수/매도 추정 볼륨과 델타를 차트 위에서 바로 볼 때 |
| `풋프린트 하위 타임프레임` | 봉 내부 흐름을 더 잘게 추정하고 싶을 때 |
| `표시 대상` | 최근 확정봉 전체, 비정상 캔들만, ATR 배경 봉만 중에서 라벨 밀도를 조절할 때 |
| `최소 거래량 배수` | 평균 거래량보다 작은 봉의 풋프린트 라벨을 줄이고 싶을 때 |
| `경량 렌더링 모드` | 차트 렌더링이 느릴 때 과거 박스 수와 박스 길이를 자동으로 줄일 때 |
| `경량 모드 최대 박스 수`, `경량 모드 최대 박스 길이` | 경량 모드에서 화면 객체를 더 줄이거나 조금 늘리고 싶을 때 |

## 주의사항

- 자동매매 전략이 아니라 해석 보조 지표입니다.
- 모든 신호는 `봉 마감 후 종가 확정` 기준입니다.
- 다이버전스는 피벗 확정 뒤에 표시되므로 지연이 있습니다.
- 스윕은 `vpAbnormalVolume` 조건까지 포함하므로, 단순 고가/저가 이탈만으로는 확인되지 않습니다.
- 과거 박스 유지 옵션을 켜면 예전 박스도 남지만, TradingView 오브젝트 한계 안에서 관리됩니다.
