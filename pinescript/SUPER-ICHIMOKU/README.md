# Advanced Ichimoku

트레이딩뷰용 Pine Script Ichimoku 확장 버전입니다. 정통 이치모쿠 계산 위에 **멀티스케일 신호·트리거 테이블·휩쏘 필터·이격도**를 얹은 인디케이터입니다.

| 스크립트 | 용도 |
| --- | --- |
| [`advanced-ichimoku.pine`](./advanced-ichimoku.pine) | 인디케이터 — 멀티스케일 신호 마커 + 우측 상단 테이블 |

> 백테스트 전략(`advanced-ichimoku-strategy.pine`)은 추후 다시 추가될 예정입니다.

## 핵심 기능

| 기능 | 설명 |
| --- | --- |
| 매수/매도 신호 | 래깅 스팬이 기준선 돌파 **AND** 컨버전 라인이 기준선 돌파. 트리거 충족 후 **다음 봉 확정 시 발화**(휩쏘 필터·리페인팅 방지·1봉 지연) |
| 멀티스케일 표시 | 같은 신호를 래깅 설정값의 **0.5× / 1× / 2×** 스케일로 계산해 **삼각형 크기**(가장 작음/작음/보통)로 동시 표시 |
| 신호 모드 | `확정`/`예측` 선택 — 예측 모드는 다음 봉을 가정해 신호를 1봉 빨리 포착(기본=확정) |
| 거래량 가중(VWMA) 신호 | 일반 신호와 함께, 컨버전·기준선을 VWMA로 바꾼 버전의 신호를 **반투명**으로 동시 표시 |
| 신호 순서 제한 | 같은 방향 신호는 반대 신호가 한 번 나온 뒤에만 다시 발생(중복 방지) |
| 우측 상단 테이블 | **4개 섹션 병합 헤더**로 묶어 표시 — **[추세 확인]**(상승/하락 전환·리본 추세) · **[캔들 및 되돌림 확인]**(캔들 상태·비정상 매수/매도·이격도·되돌림 가격) · **[지지/저항 확인]**(구름대·VWAP 지지/저항·VWAP 상태·장기 추세·장기 지지/저항) · **[손익비 타입 및 계산 확인]**(`Cha 롱/숏`=청산선·`VWAP 롱/숏`=밴드2·`Ret 롱숏`=되돌림) |
| 스윕/청산 2선 | **스윕(점선)** = 샹들리에 트레일(세력 스톱헌트 자리), **청산(직선)** = 그 너머(ATR 버퍼+스윙 풀). 둘 다 **트레일+수평선**으로 차트에 표시(테이블 행은 없음 — 손익비 계산엔 청산선 사용) |
| 수축/확장 (휩쏘 필터) | `횡보 압축 → 상승/하락 분출`을 **`+` 모양(플러스)**으로 표시(압축=보라·상승=파랑·하락=빨강, **신호 삼각형 뒤 레이어**). 판정 기준 **이평 리본 / 구름 두께 / 리본+구름** 모드 선택(기본 리본+구름) |
| 이격도 | 선택 이평선(기본 SMA 20) 대비 `종가÷MA×100`(기준 100). 고정 임계+적응형(σ) 2기준 중 1개=`되돌림 주의`(둔탁)·2개=`되돌림 확실`(선명)로 테이블 표시 |
| 장기 추세 | 이평선 리본 통합 — **MA 100/200 선 + backfill**. MA 종류 4종(기본 거래량 가중 VWMA), MA100=파랑·MA200=보라, backfill 우상향=파랑·우하향=보라(투명도 60). 테이블에 강/약 추세·골든/데드 크로스 표시 |

## 신호 로직 + 멀티스케일

`래깅 스팬 = 현재 종가(close)`. 차트상 래깅 스팬이 닿는 위치의 라인은 `[displacement-1]` 값으로 비교합니다.

| 구분 | 조건① (래깅 스팬) | 조건② (컨버전 라인) | 추가 조건 |
| --- | --- | --- | --- |
| 매수 신호 | `close > baseLine[disp-1]` 또는 상향 크로스 | `conversionLine > baseLine` 또는 골든크로스 | 중복 방지 |
| 매도 신호 | `close < baseLine[disp-1]` 또는 하향 크로스 | `conversionLine < baseLine` 또는 데드크로스 | 중복 방지 |

- **신호 순서 제한**: 매도 신호 후에만 매수 신호가 다시, 매수 신호 후에만 매도 신호가 다시 나옵니다(스케일·신호 종류별로 독립). 같은 방향이 유지돼도 첫 봉만 1회. 이 순서·중복 제어는 **상위 게이트(`dir!=방향`)와 확정봉 `dir` 잠금**이 단일 소스로 담당하며, **방향이 전환되면(예: 상승→하락) 반대 방향 첫 신호는 그 봉에서 정상 발화**합니다(원시 신호 계산 함수 내부에는 별도 방향 래치가 없습니다).
- **다음 봉 확정 발화(휩쏘 필터·1봉 지연)**: 캐스케이드 게이트가 봉 N에서 충족돼도 **그 봉에서 즉시 발화하지 않습니다**. 그 봉은 **대기(pending)** 상태가 되고, **다음 봉 N+1(확정봉)에서 확정 조건이 유지될 때만 그 봉에서 신호가 발화**합니다(유지 안 되면 취소 — 신호 없음). **확정 조건 = 대기 시점의 그 스케일 트리거 가격을 다음 봉 종가가 유지**(매수 `close ≥ 대기 buyTrig`, 매도 `close ≤ 대기 sellTrig`). 대기 시점 트리거 값을 래치해 다음 봉 종가와 비교하므로 **미래참조 없이 확정봉 종가 기준**(리페인팅 없음). 전 스케일(0.5×/1×/2×)·매수·매도 모두 대칭. 대기 중 반대 방향 게이트가 새로 뜨면 그쪽으로 대기가 전환됩니다. **삼각형·알림(`alertcondition`)·`dir` 추세 잠금은 모두 확정 발화(`fBuyXX`/`fSellXX`) 단일 소스로 이 확정봉에서 동기화**되어(트리거 봉이 아니라 확정봉에 찍힘) 신호·테이블이 어긋나지 않습니다. **한 트리거 이벤트당 신호는 확정봉에 정확히 1개**만 찍힙니다(확정된 봉에서는 같은 방향 재대기를 막아, 확정봉이 곧바로 다음 트리거 봉이 되어 다음 봉에서 재확정되는 중복 발화를 차단). 트리거 충족 후 신호가 **1봉 지연**되는 대신, 다음 봉으로 이어지지 않는 속임수 돌파(휩쏘)를 걸러 신뢰도를 높입니다.
- **스케일 캐스케이드(하위 추세 상태 전제)**: 상위 스케일 신호는 **하위 스케일이 해당 방향 추세 상태일 때** 발화합니다. **1× 매수는 0.5×가 `상승 추세`(`dir05==1`)인 동안**, **2× 매수는 0.5×·1×가 둘 다 `상승 추세`(`dir05==1 and dir10==1`)인 동안** 그 스케일의 자기 트리거가 충족되면 발화합니다(매도는 `하락 추세` 기준으로 대칭). 하위 추세 상태는 **전제**일 뿐, 각 스케일의 자기 트리거(컨버전/래깅 크로스 조건)는 그대로 요구됩니다. `dir` 추세 잠금은 위 **다음 봉 확정 시점**에 이뤄지므로, 하위가 확정돼 `dir`이 잠긴 뒤에야 상위 스케일이 대기 등록됩니다(봉 순서상 정상). 반대 신호가 그 스케일에 확정되면 `dir` 부호가 뒤집혀 상위 스케일 전제가 자연히 풀립니다(반대도 동일). 알림(1×)도 이 게이트를 따릅니다.
    - **버그 수정(지속 추세 미발화 해소)**: 이전에는 캐스케이드 전제가 "하위 스케일이 **방금 발화**"(구 `buyStage`/`sellStage` 카운터)여서, 지속 상승장에서 1×가 이미 `상승 추세`로 잠기면(dir10==1) **새 0.5× 신호가 1×로 승격 못 하고 2×도 안 뜨는** 문제가 있었습니다. 전제를 "하위 스케일이 **추세 상태**"(`dir`)로 바꿔 해소했습니다(구 카운터 제거).
  - **트리거 스킵(즉시 승격)**: 상위 배수(1× 또는 2×)의 **트리거 가격이 이미 현재 봉에서 충족**(매수=현재 종가 ≥ 트리거, 매도=현재 종가 ≤ 트리거)되고 **하위 배수가 같은 방향 추세 상태**(1× 스킵=0.5× 상승/하락 추세, 2× 스킵=0.5×·1× 둘 다)면, 원시 신호를 기다리지 않고 상위 배수 신호를 **현재 봉에 바로 표시**합니다. 예) `1× 매수` 상승 추세 중 현재가 5000, `2× 매수 트리거 4000` → 현재 봉에 `2× 매수` 표시. 매도도 동일.
  - **순차성 강제(중복 방지)**: 게이트가 모두 **직전 봉까지의 `dir`(하위 추세 상태)** 을 참조하고 `dir`은 다음 봉 확정 시점에 잠기므로, 한 봉 안에서 `0.5×→1×→2×`가 동시에 발화하지 않고 **한 봉엔 한 배수만** 발생합니다. 반대 방향 신호가 확정되면 그 스케일 `dir`이 뒤집혀 상위 전제가 풀리므로 **모멘텀·정상 모드와 무관하게 순차성이 강제**됩니다.
  - **같은 방향 재발화 차단**: 각 스케일은 **이미 그 방향이면(예: 2× 이미 `상승 추세`) 그 스케일에 반대 신호가 나오기 전까지 같은 방향 신호를 다시 표시하지 않습니다**(`dir!=방향` 기준, `dir`은 확정봉에서 잠김). 상위 배수가 같은 방향으로 중복 발화하던 것 방지 — 롱숏·전 스케일 공통.
- **모멘텀 모드(상태 진입 시 신호 가속)**: `정상 캔들`이면 네 스케일 모두 정상 기간(0.5×/1×/2×)으로 계산합니다. **`비정상 매도 캔들`/`롱 반등 감시`(하락 이탈) 또는 `비정상 매수 캔들`/`숏 반등 감시`(상승 이탈) 상태로 진입하면 매수·매도 양방향 모두 모멘텀 모드**가 됩니다(어느 방향으로 진입하든 두 방향 동시 진입). **비정상 판정을 멀티스케일 신호보다 먼저 계산하므로, 비정상 매수/매도 등 신호가 뜬 바로 그 봉(현재 종가 기준)에서 모멘텀 모드가 즉시 적용**됩니다(신호·테이블 모두 1봉 지연 없음). 모멘텀 모드는 **기준선·컨버전·래깅 스팬 기간을 전부 한 단계 빠른 스케일로 시프트**합니다(표시상 `0.5× → 0.25×`, `1× → 0.5×`, `2× → 1×`). 로직 자체는 정상 로직 그대로이고, **적용 기간만 짧아져** 더 빨리 반응합니다.
  - **해제**: 모멘텀 모드는 **정상 이격도(상태 종료 = 이격도 중립 복귀)** 가 되기 전까지 유지되고, 그 시점에 양방향 모두 해제되어 정상 스케일로 복귀합니다. 반대 방향 신호가 나와도 해제되지 않습니다. 재무장은 상태가 한 번 끝났다가 다시 시작(컨텍스트 재진입)할 때만 됩니다.
  - 우측 상단 테이블의 트리거 가격은 모멘텀 모드일 때 **`M ` 접두어**로 표시합니다.

### 멀티스케일 (0.5× / 1× / 2×)

위 신호 로직을 **래깅 설정값(컨버전 8·베이스 22·래깅 22)의 세 배율**로 각각 계산해, 삼각형 **크기**로 어느 시간축에서 나온 신호인지 보여줍니다. (리딩 스팬 B는 신호와 무관하므로 스케일에 안 씀)

| 스케일 | 기간 예시(기본 입력 기준) | 삼각형 크기 |
| --- | --- | --- |
| `0.5×` | 4 · 11 · 11 | 가장 작음 (`tiny`) |
| `1×` | 8 · 22 · 22 | 작음 (`small`) |
| `2×` | 16 · 44 · 44 | 보통 (`normal`) |

- **여러 크기의 삼각형이 같은 봉에 겹치면** 짧은·중간·긴 시간축이 한 방향으로 정렬된 것 → 신뢰도↑.
- 차트엔 **1× 라인/구름만** 그립니다. 0.5×·2×는 신호 계산용입니다.
- 알림(alert)은 **1× 신호** 기준이고, 우측 상단 테이블의 `상승/하락 전환` 트리거 가격은 **0.5× / 1× / 2× 세 스케일을 각각** 표시합니다.

## 신호 모드 (확정 / 예측)

`신호 모드` 입력으로 매수/매도 신호 산출 방식을 고릅니다(세 스케일 모두 같은 모드 적용).

| 모드 | 설명 |
| --- | --- |
| `확정 모드`(기본) | 위 신호 로직 그대로 — 현재 봉 종가로 신호 확정 |
| `예측 모드` | **다음 봉을 가정**해 신호를 1봉 빨리 포착 (조건이 더 빡빡한 모멘텀 게이트 포함) |

예측 모드는 아래 가정/조건으로 신호를 냅니다.

| 항목 | 매수 | 매도 |
| --- | --- | --- |
| 다음 봉 종가 가정 | `(현재 종가 + 고가) / 2` | `(현재 종가 + 저가) / 2` |
| 모멘텀 게이트 | `저가 > 직전 봉 저가` | `고가 < 직전 봉 고가` |
| 래깅 조건 | 가정 종가 > `baseLine[displacement-2]` | 가정 종가 < `baseLine[displacement-2]` |
| 전환선 조건 | `conversionLine > baseLine`(현재 봉 기준선) | `conversionLine < baseLine` |
| 거래량(VWMA) 신호 | **다음 봉 거래량 = 현재 거래량** 가정으로 다음 봉 VWMA 전환·기준선을 재계산해 같은 조건 적용 | 〃 |

- 기본값은 `확정 모드`입니다. 예측 모드는 더 빠른 대신 가정에 기반하므로(확정 전 미리 찍힘) 참고용으로 쓰는 게 안전합니다.

## 거래량 가중(VWMA) 신호

모드 선택이 아니라 일반 신호와 **동시에** 표시됩니다. 컨버전·기준선만 `VWMA = Σ(종가×거래량)/Σ거래량` 으로 바꾼 버전의 **신호 마커만** 반투명으로 그립니다(라인은 안 그림). 세 스케일 모두에 적용됩니다.

| 항목 | 내용 |
| --- | --- |
| 계산 | 컨버전·기준선을 VWMA로 대체, 그 외 로직은 일반과 동일(후행스팬은 실제 종가) |
| 표시 | 일반과 같은 삼각형·크기, 기본 투명도 50% |
| 옵션 | `거래량 가중 신호 표시`(**기본 OFF**), `거래량 가중 투명도`(50) |
| 장점 | 거래량 실린 가격대 반영 → 저거래량 가짜 움직임·휩쏘에 덜 흔들림 |
| 단점 | 거래량 없는/부정확 심볼(일부 FX·지수)에선 `na`/왜곡 · 거래량 스파이크 1봉에 과반응 가능 |
| 활용 | 표준 마커와 **같은 자리 겹치면 신뢰↑**, 거래량만 먼저 뜨면 수급 조기 참고 |

## 캔들 상태 (3) — 최근 3봉 캔들 패턴(표시 전용)

| --- | --- | --- | --- |
| 상승장악형 | `b1` 음봉 → `b0` 양봉이 `b1` 몸통 완전포함(`c0≥o1` and `o0≤c1`) | 강세반전 | 둔한 녹색 `#6E8F72` |
| 하락장악형 | `b1` 양봉 → `b0` 음봉이 `b1` 몸통 완전포함(`c0≤o1` and `o0≥c1`) | 약세반전 | 둔한 빨강 `#A57A7A` |
| 샛별 | `b2` 강한 음봉 → `b1` 작은 몸통 → `b0` 양봉이 `b2` 몸통중앙 이상 회복(`c0≥mid2`) | 강세반전(강) | 녹색 |
| 석별 | `b2` 강한 양봉 → `b1` 작은 몸통 → `b0` 음봉이 `b2` 몸통중앙 이하(`c0≤mid2`) | 약세반전(강) | 빨강 |
| 망치/역망치 | `b0` 작은 몸통 + 한쪽 긴 꼬리(반대 꼬리 짧음), 직전 하락(`c1<c2`) | 강세반전 | 둔한 녹색 `#6E8F72` |
| 교수형/유성 | `b0` 작은 몸통 + 한쪽 긴 꼬리, 직전 상승(`c1>c2`) | 약세반전 | 둔한 빨강 `#A57A7A` |
| 적삼병 | `b2·b1·b0` 3봉 연속 강한 양봉 + 종가 계단상승(`c0>c1>c2`) + 윗꼬리 짧음 | 상승지속 | 녹색 |
| 흑삼병 | `b2·b1·b0` 3봉 연속 강한 음봉 + 종가 계단하락(`c0<c1<c2`) + 아랫꼬리 짧음 | 하락지속 | 빨강 |
| 내부봉 | `b0`가 `b1` 범위에 포함(`h0≤h1` and `l0≥l1`) | 돌파대기 | 회색 |
| 외부봉 | `b0`가 `b1`을 완전 삼킴(`h0>h1` and `l0<l1`, 장악형에 안 걸린 경우) | 변동성확대 | 회색 |
| 도지연속 | `b0·b1` 2봉 몸통비율 ≤ `candleDojiBodyRatio`(0.10) | 관망 | 회색 |
| 혼조 | 위 미매칭, 또는 `range/ATR < candleMinRangeAtr`(0.5), 또는 데이터 부족 | 신호없음 | 회색 |

> 회색 계열(내부봉·외부봉·도지연속·혼조)은 모두 코드상 동일한 fallback 색 `color.new(color.gray, 40)`(회색·투명도 40)입니다.

### 우선순위 판정 순서 (상호배타)

첫 매칭에서 확정하는 `if/else if` 체인이라 한 봉에 **한 패턴만** 나옵니다. 순서:

> **3·4 → 1·2 → 7·8 → 5·6 → 9·10 → 11 → 12**
>
> 즉 샛별·석별 → 상승·하락장악형 → 적삼병·흑삼병 → 망치·교수형 → 내부봉·외부봉 → 도지연속 → 혼조. (데이터 부족 시 즉시 `혼조`.)

강한 반전 신호(샛별/석별)를 장악형보다 먼저, 지속 신호(삼병)를 단봉 반전(망치/교수형)보다 먼저 잡습니다.

### 입력 파라미터

| 입력 | 기본값 | 의미 |
| --- | --- | --- |
| `candleShowState` | `true` | 캔들 상태 (3) 행 표시 토글 |
| `candleConfirmOnClose` | `true` | ON=확정봉 기준(리페인팅 방지) / OFF=현재봉 실시간 |

| `candleLongWickRatio` | `0.60` | 긴 꼬리 비율 한계(망치/유성 판정) |
| `candleSmallBodyRatio` | `0.35` | 작은 몸통 비율 한계(샛별·망치 등) |
| `candleStrongBodyRatio` | `0.60` | 강한 몸통 비율 한계(장악·삼병·샛별) |
| `candleMinRangeAtr` | `0.5` | 최소 범위 ATR 배수(이 미만이면 혼조) |

### ⚠️ 리스크 고지

- **표시 전용입니다.** 어떤 진입/청산/신호 변수에도 연결되지 않으며, 기존 매수/매도 신호 로직과 독립입니다. 순수 관찰용 정보 행입니다.
- **단일/소수 캔들 패턴은 통계적으로 약한 엣지**입니다. 캔들 패턴만으로 진입하지 말고, 반드시 **이치모쿠 메인 + 테이블 + 보조지표 정렬과 교차확인**하세요(펀드 철학: 단일 신호 불신).
- **임계값·예측 방향은 파라미터 후보(제안)** 입니다. 알고리즘 수학자의 백테스트 검증 전에는 관찰용으로만 쓰고, 실전 전략에 반영하지 마세요. (임계·기간·예측 방향 변경은 전략 변경에 해당하며 본부장·알고리즘 수학자 검토가 필요합니다.)
- **갭 기반 패턴(샛별/석별)은 24시간 거래되는 코인·선물에서 갭이 거의 없어** 근접 조건(현 구현은 `b2` 몸통중앙 회복/이탈로 완화) 추가 조정이 필요할 수 있습니다.

## 수축/확장 (휩쏘 필터) — 이평 리본 / 구름 두께 / 리본+구름 모드

이평 수축↔확장을 캔들에 **`+` 모양(플러스)**으로 표시합니다 — `횡보 압축`=보라 `#9C27B0`(상승·하락과 확실히 구분), `상승 분출`=파랑, `하락 분출`=빨강. `+`표식은 **신호 삼각형보다 뒤(아래 레이어)** 에 그려집니다. **판정 기준을 3가지 모드로 선택**합니다(기본 `리본+구름`).

| 모드 | 수축/확장 판정 기준 | 분출 방향 |
| --- | --- | --- |
| `이평 리본` | **입력값 기반 이평 리본**(전환선~리딩B 길이, 7개 MA)의 폭이 ATR 대비 좁으면 수축 | 빠른선(전환선 길이) ≥/< 느린선(리딩B 길이) |
| `구름 두께` | **구름 두께**(선행스팬 A·B 간격)가 ATR 대비 좁으면 수축 | 양운(A≥B)=상승 / 음운=하락 |
| `리본+구름`(기본) | 리본·구름이 **모두** 수축/확장(방향 일치)할 때만 | 두 기준 방향이 같을 때만 분출 |

- **`이평 리본`이 이치모쿠와 정합**: 리본 길이를 `전환선~리딩B`(기본 8~44)에 자동 연동해, 신호와 같은 시간축에서 수축/확장을 판단합니다. 기간 입력을 바꾸면 리본 범위도 따라갑니다.
- **`구름 두께`는 이치모쿠 본연**: 구름 두께 자체가 압축/확장 측정이라 추가 라인 없이 정합적입니다.

| 항목 | 내용 |
| --- | --- |
| 상태머신 | 압축 후 분출(같은 방향 반복은 재압축 필요). **단, 직전 분출과 반대 방향 확장이면 압축 없이 바로 반전 분출**(상승 분출 → 하락 분출, 그 반대도 가능). `리본+구름`은 방향이 엇갈리면 분출 보류(무장 유지) |
| 분출 속도 | 폭이 임계선을 완전히 넘기 전이라도 잡아 빠름. `확장 판정 폭 증가 봉수`↑ = 느리고 확실, 1 = 가장 빠름 |
| 활용 | 신호 + 같은 방향 분출 = 신뢰↑ / `횡보 압축` 구간 신호 = 휩쏘 의심·관망. **진입 트리거 아닌 신뢰도 필터** |
| 옵션(`리본 압축/분출`) | `수축/확장 판정 모드`(리본+구름 기본)·`리본 이동평균 종류(A)`(SMA)·`횡보 압축 표시`·`분출 표시`·`봉마감 후 확정`·`폭 ATR 기준`(14)·`수축 판정 ATR 배수`(0.8)·`확장 판정 폭 증가 봉수`(1) |
