# Advanced Ichimoku

트레이딩뷰용 Pine Script Ichimoku 확장 버전입니다. 정통 이치모쿠 계산 위에 **멀티스케일 신호·트리거 테이블·휩쏘 필터·이격도**를 얹은 인디케이터입니다.

| 스크립트 | 용도 |
| --- | --- |
| [`advanced-ichimoku.pine`](./advanced-ichimoku.pine) | 인디케이터 — 멀티스케일 신호 마커 + 우측 상단 테이블 |

> 백테스트 전략(`advanced-ichimoku-strategy.pine`)은 추후 다시 추가될 예정입니다.

## 핵심 기능

| 기능 | 설명 |
| --- | --- |
| 매수/매도 신호 | 래깅 스팬이 기준선 돌파 **AND** 컨버전 라인이 기준선 돌파. 트리거 충족 후 **다음 봉 확정 시 발화**(휩쏘 필터·리페인팅 방지·1봉 지연) |
| 멀티스케일 표시 | 같은 신호를 래깅 설정값의 **0.5× / 1× / 2×** 스케일로 계산해 **삼각형 크기**(가장 작음/작음/보통)로 동시 표시 |
| 추세 매수/매도 신호 (6모드) | `일반 예측` / `일반 확정` / `거래량 가중 예측` / `거래량 가중 확정` / `혼합 일반` / `혼합 확정` 중 **택일**(기본=`혼합 일반`). 신호 소스(일반 도치안 vs VWMA)와 확정 방식(예측 vs 확정) 조합. **혼합**은 봉이 **비정상 거래량이면 그 봉에서 VWMA 신호를 즉시 발화**(다음봉 확정 우회 — 거래량 가중 전용 모드처럼 그 봉에 바로 삼각형), **정상 봉은 일반 신호를 기존 방식(예측/확정 파이프라인)대로**(장초/뉴스發 비정상 봉 빠른 캐치) |
| 신호 순서 제한 | 같은 방향 신호는 반대 신호가 한 번 나온 뒤에만 다시 발생(중복 방지) |
| 우측 상단 테이블 | **4개 섹션 병합 헤더**로 묶어 표시 — **[추세 확인]**(상승/하락 전환·리본 추세) · **[캔들 및 되돌림 확인]**(캔들 상태·캔들 손절라인·비정상 매수/매도·이격도·되돌림 가격) · **[지지/저항 확인]**(구름대·VWAP 지지/저항·VWAP 상태·장기 추세·장기 지지/저항) · **[손익비 현황 확인]**(`Cha 롱/숏`=청산선·`VWAP 롱/숏`=밴드2·`Ret 롱숏`=되돌림) |
| 스윕/청산 2선 | **스윕(점선)** = 샹들리에 트레일(세력 스톱헌트 자리), **청산(직선)** = 그 너머(ATR 버퍼+스윙 풀). 둘 다 **트레일+수평선**으로 차트에 표시(테이블 행은 없음 — 손익비 계산엔 청산선 사용) |
| 수축/확장 (휩쏘 필터) | `횡보 압축 → 상승/하락 분출`을 **`+` 모양(플러스)**으로 표시(압축=보라·상승=파랑·하락=빨강, **신호 삼각형 뒤 레이어**). 판정 기준 **이평 리본 / 구름 두께 / 리본+구름** 모드 선택(기본 리본+구름) |
| 이격도 | 선택 이평선(기본 SMA 20) 대비 `종가÷MA×100`(기준 100). 고정 임계+적응형(σ) 2기준 중 1개=`되돌림 주의`(둔탁)·2개=`되돌림 확실`(선명)로 테이블 표시 |
| 장기 추세 | 이평선 리본 통합 — **MA 100/200 선 + backfill**. MA 종류 4종(기본 거래량 가중 VWMA), MA100=파랑·MA200=보라, backfill 우상향=파랑·우하향=보라(투명도 60). 테이블에 강/약 추세·골든/데드 크로스 표시 |

## 신호 로직 + 멀티스케일

`래깅 스팬 = 현재 종가(close)`. 차트상 래깅 스팬이 닿는 위치의 라인은 `[displacement-1]` 값으로 비교합니다.

| 구분 | 조건① (래깅 스팬) | 조건② (컨버전 라인) | 추가 조건 |
| --- | --- | --- | --- |
| 매수 신호 | `close > baseLine[disp-1]` 또는 상향 크로스 | `conversionLine > baseLine` 또는 골든크로스 | 중복 방지 |
| 매도 신호 | `close < baseLine[disp-1]` 또는 하향 크로스 | `conversionLine < baseLine` 또는 데드크로스 | 중복 방지 |

- **신호 순서 제한**: 매도 신호 후에만 매수 신호가 다시, 매수 신호 후에만 매도 신호가 다시 나옵니다(스케일·신호 종류별로 독립). 같은 방향이 유지돼도 첫 봉만 1회. 이 순서·중복 제어는 **상위 게이트(`dir!=방향`)와 확정봉 `dir` 잠금**이 단일 소스로 담당하며, **방향이 전환되면(예: 상승→하락) 반대 방향 첫 신호는 그 봉에서 정상 발화**합니다(원시 신호 계산 함수 내부에는 별도 방향 래치가 없습니다).
- **다음 봉 확정 발화(휩쏘 필터·1봉 지연)**: 캐스케이드 게이트가 봉 N에서 충족돼도 **그 봉에서 즉시 발화하지 않습니다**. 그 봉은 **대기(pending)** 상태가 되고, **다음 봉 N+1(확정봉)에서 확정 조건이 유지될 때만 그 봉에서 신호가 발화**합니다(유지 안 되면 취소 — 신호 없음). **확정 조건 = 대기 시점의 그 스케일 트리거 가격을 다음 봉 종가가 유지**(매수 `close ≥ 대기 buyTrig`, 매도 `close ≤ 대기 sellTrig`). 대기 시점 트리거 값을 래치해 다음 봉 종가와 비교하므로 **미래참조 없이 확정봉 종가 기준**(리페인팅 없음). 전 스케일(0.5×/1×/2×)·매수·매도 모두 대칭. 대기 중 반대 방향 게이트가 새로 뜨면 그쪽으로 대기가 전환됩니다. **삼각형·알림(`alertcondition`)·`dir` 추세 잠금은 모두 확정 발화(`fBuyXX`/`fSellXX`) 단일 소스로 이 확정봉에서 동기화**되어(트리거 봉이 아니라 확정봉에 찍힘) 신호·테이블이 어긋나지 않습니다. **한 트리거 이벤트당 신호는 확정봉에 정확히 1개**만 찍힙니다(확정된 봉에서는 같은 방향 재대기를 막아, 확정봉이 곧바로 다음 트리거 봉이 되어 다음 봉에서 재확정되는 중복 발화를 차단). 트리거 충족 후 신호가 **1봉 지연**되는 대신, 다음 봉으로 이어지지 않는 속임수 돌파(휩쏘)를 걸러 신뢰도를 높입니다.
- **스케일 캐스케이드(하위 추세 상태 전제)**: 상위 스케일 신호는 **하위 스케일이 해당 방향 추세 상태일 때** 발화합니다. **1× 매수는 0.5×가 `상승 추세`(`dir05==1`)인 동안**, **2× 매수는 0.5×·1×가 둘 다 `상승 추세`(`dir05==1 and dir10==1`)인 동안** 그 스케일의 자기 트리거가 충족되면 발화합니다(매도는 `하락 추세` 기준으로 대칭). 하위 추세 상태는 **전제**일 뿐, 각 스케일의 자기 트리거(컨버전/래깅 크로스 조건)는 그대로 요구됩니다. `dir` 추세 잠금은 위 **다음 봉 확정 시점**에 이뤄지므로, 하위가 확정돼 `dir`이 잠긴 뒤에야 상위 스케일이 대기 등록됩니다(봉 순서상 정상). 반대 신호가 그 스케일에 확정되면 `dir` 부호가 뒤집혀 상위 스케일 전제가 자연히 풀립니다(반대도 동일). 알림(1×)도 이 게이트를 따릅니다.
    - **버그 수정(지속 추세 미발화 해소)**: 이전에는 캐스케이드 전제가 "하위 스케일이 **방금 발화**"(구 `buyStage`/`sellStage` 카운터)여서, 지속 상승장에서 1×가 이미 `상승 추세`로 잠기면(dir10==1) **새 0.5× 신호가 1×로 승격 못 하고 2×도 안 뜨는** 문제가 있었습니다. 전제를 "하위 스케일이 **추세 상태**"(`dir`)로 바꿔 해소했습니다(구 카운터 제거).
  - **트리거 스킵(즉시 승격)**: 상위 배수(1× 또는 2×)의 **트리거 가격이 이미 현재 봉에서 충족**(매수=현재 종가 ≥ 트리거, 매도=현재 종가 ≤ 트리거)되고 **하위 배수가 같은 방향 추세 상태**(1× 스킵=0.5× 상승/하락 추세, 2× 스킵=0.5×·1× 둘 다)면, 원시 신호를 기다리지 않고 상위 배수 신호를 **현재 봉에 바로 표시**합니다. 예) `1× 매수` 상승 추세 중 현재가 5000, `2× 매수 트리거 4000` → 현재 봉에 `2× 매수` 표시. 매도도 동일.
  - **순차성 강제(중복 방지)**: 게이트가 모두 **직전 봉까지의 `dir`(하위 추세 상태)** 을 참조하고 `dir`은 다음 봉 확정 시점에 잠기므로, 한 봉 안에서 `0.5×→1×→2×`가 동시에 발화하지 않고 **한 봉엔 한 배수만** 발생합니다. 반대 방향 신호가 확정되면 그 스케일 `dir`이 뒤집혀 상위 전제가 풀리므로 **모멘텀·정상 모드와 무관하게 순차성이 강제**됩니다.
  - **같은 방향 재발화 차단**: 각 스케일은 **이미 그 방향이면(예: 2× 이미 `상승 추세`) 그 스케일에 반대 신호가 나오기 전까지 같은 방향 신호를 다시 표시하지 않습니다**(`dir!=방향` 기준, `dir`은 확정봉에서 잠김). 상위 배수가 같은 방향으로 중복 발화하던 것 방지 — 롱숏·전 스케일 공통.
- **모멘텀 모드(상태 진입 시 신호 가속)**: `정상 캔들`이면 네 스케일 모두 정상 기간(0.5×/1×/2×)으로 계산합니다. **`비정상 매도 캔들`/`롱 반등 감시`(하락 이탈) 또는 `비정상 매수 캔들`/`숏 반등 감시`(상승 이탈) 상태로 진입하면 매수·매도 양방향 모두 모멘텀 모드**가 됩니다(어느 방향으로 진입하든 두 방향 동시 진입). **비정상 판정을 멀티스케일 신호보다 먼저 계산하므로, 비정상 매수/매도 등 신호가 뜬 바로 그 봉(현재 종가 기준)에서 모멘텀 모드가 즉시 적용**됩니다(신호·테이블 모두 1봉 지연 없음). 모멘텀 모드는 **기준선·컨버전·래깅 스팬 기간을 전부 한 단계 빠른 스케일로 시프트**합니다(표시상 `0.5× → 0.25×`, `1× → 0.5×`, `2× → 1×`). 로직 자체는 정상 로직 그대로이고, **적용 기간만 짧아져** 더 빨리 반응합니다.
  - **해제**: 모멘텀 모드는 **정상 이격도(상태 종료 = 이격도 중립 복귀)** 가 되기 전까지 유지되고, 그 시점에 양방향 모두 해제되어 정상 스케일로 복귀합니다. 반대 방향 신호가 나와도 해제되지 않습니다. 재무장은 상태가 한 번 끝났다가 다시 시작(컨텍스트 재진입)할 때만 됩니다.
  - 우측 상단 테이블의 트리거 가격은 모멘텀 모드일 때 **`M ` 접두어**로 표시합니다.

### 멀티스케일 (0.5× / 1× / 2×)

위 신호 로직을 **래깅 설정값(컨버전 8·베이스 22·래깅 22)의 세 배율**로 각각 계산해, 삼각형 **크기**로 어느 시간축에서 나온 신호인지 보여줍니다. (리딩 스팬 B는 신호와 무관하므로 스케일에 안 씀)

| 스케일 | 기간 예시(기본 입력 기준) | 삼각형 크기 |
| --- | --- | --- |
| `0.5×` | 4 · 11 · 11 | 가장 작음 (`tiny`) |
| `1×` | 8 · 22 · 22 | 작음 (`small`) |
| `2×` | 16 · 44 · 44 | 보통 (`normal`) |

- **여러 크기의 삼각형이 같은 봉에 겹치면** 짧은·중간·긴 시간축이 한 방향으로 정렬된 것 → 신뢰도↑.
- 차트엔 **1× 라인/구름만** 그립니다. 0.5×·2×는 신호 계산용입니다.
- 알림(alert)은 **1× 신호** 기준이고, 우측 상단 테이블의 `상승/하락 전환` 트리거 가격은 **0.5× / 1× / 2× 세 스케일을 각각** 표시합니다.

## 추세 매수/매도 신호 (6모드 · 택일)

`매매 신호 방식` 그룹의 `추세 진입 신호 방식` 입력 하나로 매수/매도 신호의 **신호 소스**와 **확정 방식**을 함께 고릅니다(세 스케일 모두 같은 모드 적용). 6모드는 **택일**이라 한 번에 한 조합만 표시됩니다.

| 모드 | 신호 소스 | 확정 방식 |
| --- | --- | --- |
| `일반 확정 모드` | 일반 도치안(컨버전·기준선) | 봉마감 확정 — 현재 봉 종가로 신호 확정 |
| `일반 예측 모드` | 일반 도치안 | 예측 — 다음 봉 가정으로 1봉 빨리 포착 |
| `거래량 가중 확정 모드` | VWMA(거래량 가중) | 봉마감 확정 |
| `거래량 가중 예측 모드` | VWMA(거래량 가중) | 예측 |
| `혼합 일반 모드`(**기본**) | **봉별 스위칭** — 비정상 거래량 봉=VWMA(**즉시 발화**), 정상 봉=일반 도치안 | 정상 봉만 예측(모멘텀), 비정상 봉은 즉시 |
| `혼합 확정 모드`(신규) | 〃(봉별 스위칭) | 정상 봉만 봉마감 확정, 비정상 봉은 즉시 |

두 축은 서로 독립입니다.

- **신호 소스** — `일반`은 컨버전·기준선을 그대로 쓰고, `거래량 가중`은 컨버전·기준선을 `VWMA = Σ(종가×거래량)/Σ거래량` 으로 바꿔 계산합니다(그 외 로직은 동일, 후행스팬은 실제 종가). 거래량 가중 모드일 때만 VWMA 신호가 선명한(불투명) 삼각형으로 그려지고 일반 신호는 숨습니다(배타).

### 혼합 모드 (신규 — 장초/뉴스 빠른 캐치)

혼합 모드는 **봉 단위로 신호 소스를 바꿉니다.** 목적은 장초·뉴스發 비정상 거래량 봉에서 거래량 가중 관점으로 추세 변화를 **한 발 빨리 캐치**하는 것입니다.

- **봉별 처리 스위칭 (비정상=즉시 / 정상=방식대로)** — 그 봉이 **비정상 거래량**이면 그 봉의 **VWMA(거래량 가중) 신호를 그 봉에서 즉시 발화**합니다(다음봉 확정 파이프라인을 **우회** — 거래량 가중 전용 모드가 그 봉에 바로 삼각형을 찍는 것과 동일). 비정상이 아니면(정상 봉) **일반 도치안 신호를 기존 방식(예측/확정 파이프라인)** 그대로 처리합니다. 즉 혼합 = `비정상 봉 ? (VWMA 즉시 발화) : (일반 신호 파이프라인)`.
- **비정상 거래량 정의(⚠️ 전략 파라미터)** — `volume > SMA(volume, N) × (1 + 배수)`. 기본값 `N=20` · `배수=1.0`(= 최근 20봉 평균 거래량의 **2배 초과**). 이 정의·기본값은 **VOLUME-PRESSURE 지표와 동일한 펀드 표준**입니다. `비정상 거래량 비교 봉수(N)`·`비정상 거래량 배수(mult)` 입력으로 노출(`매매 신호 방식` 그룹).
- **비정상 봉 즉시 발화(다음봉 확정 우회)** — CEO 요구: 비정상 거래량이 나오면 **바로 그 봉 기준으로** VWMA 매수/매도 신호가 즉시 나와야 합니다. 이전(C-1)에는 혼합 신호를 일반 파이프라인(게이트→pending→**다음봉 확정 대기**)에 태워, 비정상 봉의 VWMA 신호가 그 봉에서 안 찍히고 **다음 봉으로 지연**되는 버그가 있었습니다(예측·확정 공통). 이를 바로잡아 **혼합 모드(예측·확정 공통)에서 비정상 봉은 pending 파이프라인을 우회하고 그 봉에서 즉시 삼각형**을 찍습니다. 즉시 발화 신호 소스는 VWMA **원시신호**(모멘텀 시프트 반영)이며, 발화·`dir` 잠금·삼각형·알림이 모두 그 봉에서 동기화됩니다.
- **확정/예측 구분** — 비정상 봉은 어느 방식이든 **즉시 발화**(모드 무관). 정상 봉만 방식대로: `혼합 일반(예측)`은 예측(모멘텀) 방식으로, `혼합 확정`은 확정봉 기준(다음봉 확정·1봉 지연·리페인팅 없음)으로 정상 봉을 처리합니다. `예측`은 실시간 리페인팅 노출, `확정`은 확정봉 기준.
- **중복방지 정합 (비정상↔정상 경계)** — 즉시 발화라도 **같은 방향 연속 중복 발화는 금지**됩니다. 핵심은 즉시 발화와 정상 봉 파이프라인이 **단일 방향 상태(`dirXX`)를 공유**한다는 점입니다. 거래량 가중 전용 모드의 함수 내부 `vLastDir`(자체 래치)이 아니라 파이프라인의 `dirXX`로 중복방지를 통일해, 정상↔비정상 봉이 바뀌는 경계에서 같은 방향 중복/누락이 **구조적으로** 생기지 않습니다. 즉시 발화 게이트 = `비정상 봉 AND VWMA 원시신호 AND dir≠방향 AND 스케일 캐스케이드 전제(하위 dir)`. 스케일 캐스케이드(1×는 `dir05`, 2×는 `dir05·dir10`이 같은 방향 추세일 때만)·`dir≠방향` 재발화 차단이 정상 봉과 동일 기준으로 적용됩니다. **비정상 봉에서는 pending 등록을 억제**해(정상 봉의 진행 중 대기는 취소) 즉시 발화와 다음봉 확정이 겹치는 이중 발화를 막습니다. CEO 지시대로 "거래량 가중이든 모멘텀이든 일반이든 중복 안 되게 순서대로".
- **트리거 셀 `V` 표시** — 우측 상단 테이블에서 그 봉의 트리거가 **비정상 거래량(VWMA 소스)** 인 경우 해당 트리거 셀 텍스트 맨 앞에 **`V `** 접두를 붙여 소스를 식별합니다(예: `V M ↑ ...`). 정상 봉이면 접두 없이 기존 그대로입니다. 접두 순서는 `V ` → `M `(모멘텀) → 본문.
- **차트-테이블 정합** — 차트 삼각형과 테이블 추세(`dirXX`)가 **같은 최종 발화 소스**(비정상 봉=즉시 발화·정상 봉=확정 발화)를 참조하므로 항상 일치합니다. 삼각형은 일반 모드와 동일한 마커(매수=아쿠아·매도=핑크, 모멘텀 진입 시 청산선 색)로 표시됩니다.
- **비정상 거래량 봉의 진행봉 토글(D-1)** — 비정상 거래량 판정(`abnVolume`)은 **진행 중인 현재 봉의 거래량**을 쓰므로, 봉이 완성되기 전까지 비정상 여부가 **봉 내에서 바뀔 수 있습니다**. 즉시 발화 구조에서는 그 봉이 비정상으로 판정되는 순간 VWMA 신호가 그 봉에 즉시 뜨는데, 이는 **거래량 가중 전용 모드와 동일한 실시간 특성**이며 **CEO의 "즉시 캐치" 취지에 부합하는 의도된 동작**입니다(그래서 `barstate.isconfirmed` 가드를 일부러 걸지 않습니다). 봉 완성 전까지 즉시 신호가 나타났다 사라질 수 있는 실시간 변동(리페인팅 성격)을 완전히 없애려면, 확정 계열(`혼합 확정`)에서 정상 봉은 확정봉 기준으로 처리되지만 **비정상 봉의 즉시 발화 자체는 실시간 판정**이라는 점을 감안하세요(비정상 봉 즉시 캐치와 확정봉 무변동은 개념상 상충 — 즉시성을 우선한 설계).

예측 방식은 아래 가정/조건으로 신호를 냅니다.

| 항목 | 매수 | 매도 |
| --- | --- | --- |
| 다음 봉 종가 가정 | `(현재 종가 + 고가) / 2` | `(현재 종가 + 저가) / 2` |
| 모멘텀 게이트 | `저가 > 직전 봉 저가` | `고가 < 직전 봉 고가` |
| 래깅 조건 | 가정 종가 > `baseLine[displacement-2]` | 가정 종가 < `baseLine[displacement-2]` |
| 전환선 조건 | `conversionLine > baseLine`(현재 봉 기준선) | `conversionLine < baseLine` |
| 거래량(VWMA) 신호 | **다음 봉 거래량 = 현재 거래량** 가정으로 다음 봉 VWMA 전환·기준선을 재계산해 같은 조건 적용 | 〃 |

- 기본값은 `혼합 일반 모드`입니다(장초·뉴스發 비정상 거래량 봉을 거래량 가중 관점으로 빠르게 캐치). 예측 방식은 더 빠른 대신 가정에 기반하므로(확정 전 미리 찍힘) 참고용으로 쓰는 게 안전합니다.
- 거래량 가중 신호 삼각형은 항상 선명하게(불투명) 표시됩니다. 매수/매도 표시 자체는 `매수 신호 표시`·`매도 신호 표시` 토글이 소스와 무관하게 공통 적용됩니다.
- 거래량 가중 소스 장점: 거래량 실린 가격대 반영 → 저거래량 가짜 움직임·휩쏘에 덜 흔들림. 단점: 거래량 없는/부정확 심볼(일부 FX·지수)에선 `na`/왜곡 · 거래량 스파이크 1봉에 과반응 가능.
| 활용 | 표준 마커와 **같은 자리 겹치면 신뢰↑**, 거래량만 먼저 뜨면 수급 조기 참고 |

## 우측 상단 테이블

차트 **우측 상단**에 25행 표를 띄웁니다(`barstate.islast`).

**4개 섹션 병합 헤더 구조**: 표는 좌열=항목(라벨)·우열=그 값/상태의 2열 구조이며, 관련 행을 **4개 섹션**으로 묶습니다. 각 섹션 맨 위에 **좌우 2열 병합 헤더**(딥 네이비 배경 `color.new(#1B2A4A, 0)`·흰 글자·`size.small`, `table.merge_cells`)를 둡니다.

| # | 섹션 헤더(병합) | 포함 행(순서) |

| `이격도 (N)` | (비정상 캔들 아래 배치) 값 = 선택 이평선 대비 종가 위치 `값 (상태)`(% 없이 기준 100 지수). 이격도 = `종가÷MA×100`. **고정 임계(±%) + 적응형(σ)** 중 1개=`되돌림 주의`(둔탁), 2개=`되돌림 확실`(선명), 0개=`중립` | 과열=빨강계·과매도=초록계·확실=선명·주의=둔탁·중립=회색 |

## 캔들 상태 (3) — 최근 3봉 7분류(표시 전용)

우측 상단 테이블 **[캔들 및 되돌림 확인] 섹션 맨 앞 행(비정상 매수/매도 바로 위)** 에 최근 3봉을 **7분류 스코어링**으로 분류해 표시합니다. 좌열 라벨은 `캔들 상태 (3)`, 우열 셀 값은 **`분류명 | 근거패턴`** 형식입니다(근거 패턴이 특정되지 않으면 분류명만). **셀색과 동일한 색으로 현재봉 캔들 바디도 칠합니다**(방향 있는 6개 분류만, 혼조는 원래 봉색 유지 — [기본 표시값](#기본-표시값) 표의 `캔들 상태 바색` 참조).

**7분류**(약세→강세 순): `하락 진행` · `추세형 하락 예측` · `반전형 하락 예측` · `혼조` · `반전형 상승 예측` · `추세형 상승 예측` · `상승 진행`.

- **진행(momentum)**: 3봉이 고점·저점을 한 방향으로 쌓고(HH/HL 또는 LL/LH) **순몸통(netBody)** 이 임계 이상이면 추세 **지속**으로 판정. 적삼병/흑삼병은 이 진행의 강한 형태(3봉 모두 같은 방향)로, 근거 패턴명으로 병기됩니다.
- **반전형 예측(reversal)**: 직전이 하락/상승이던 흐름에서 **반전** 패턴(장악형·망치/역망치·교수형/유성·샛별/석별)이 나오면 방향 **반전을 예측**. **반전형 예측이 진행보다 우선**합니다(추세 꺾임을 먼저 알림).
- **추세형 예측(trend)**: 선행 추세 방향으로 **2봉 극점(고/저) 돌파형** 강봉이 나오면 그 추세 **방향 지속**을 예측. **진행보다 뒤 순위**(진행에 안 걸린 강한 돌파봉을 잡음). 여기에 안 걸려도 **강몸통+순몸통 뚜렷한 방향봉**은 최하단 **강몸통 방향 폴백**으로 추세형 예측에 흡수됩니다(혼조 직전, 근거 패턴 `강몸통`).
- 두 예측 모두 **선행추세 게이트는 종가 전용이 아니라 고점/저점 시도까지 병용**(`priorUpX`/`priorDownX`)해, 종가는 눌렸어도 고점(또는 저점)이 직전 봉을 넘어선 봉에서도 놓치지 않습니다.
- **혼조**: 데이터 부족·3봉 총폭 부족(변동성 없음)·어느 분류에도 안 걸림(도지연속 등 흡수).

- **봉 정의**: `b0`=현재봉·`b1`=직전봉·`b2`=2봉 전. `확정봉 기준`(**기본 OFF=실시간, 리페인팅 감수**)을 켜면 `b0`=마지막 **완성 봉**(offset 1)으로 계산해 **리페인팅을 막고**, 끄면(기본) 형성 중인 현재봉(offset 0)을 `b0`으로 써서 실시간 반영되지만 봉 완성 전까지 값이 바뀔 수 있습니다.
- **정규화**: 몸통·꼬리는 캔들 범위(`high-low`, 0 방어) 대비 비율로, 순몸통·3봉 총폭은 `ATR(candleAtrLen=14)` 대비로 정규화해 심볼·가격대와 무관하게 판정합니다.

### 스코어링 파생값

`b0`=현재봉·`b1`=직전봉·`b2`=2봉 전. `body_i = close - open`(부호 있음), `dir_i = sign(body_i)`(+1/-1/0).

| 파생 | 정의 |
| --- | --- |
| `HH` / `HL` | 고점 계단상승 `h0>h1>h2` / 저점 계단상승 `l0>l1>l2` |
| `LH` / `LL` | 고점 계단하락 `h0<h1<h2` / 저점 계단하락 `l0<l1<l2` |
| `netBody` | `(body0+body1+body2) / ATR` — 3봉 순몸통(방향·강도, ATR 정규화·0 방어) |
| `upCount` / `dnCount` | 3봉 중 양봉 수 / 음봉 수 |
| `range3` | `(max(h0,h1,h2) − min(l0,l1,l2)) / ATR` — 3봉 총폭 |
| `priorDownX` / `priorUpX` | `(c1<c2) or (l1<l2)` / `(c1>c2) or (h1>h2)` — 직전 흐름(종가 **또는** 저점/고점 시도까지 병용) |

**긴 꼬리 파생**: `longLowerWick = (아래꼬리/range)≥candleLongWickRatio and (윗꼬리/range)≤(1−candleLongWickRatio)` · `longUpperWick`은 위/아래 대칭. 망치·역망치·교수형·유성 공통 부품.

**반전형 패턴**(반전형 예측 근거): 상승장악 `dir1==-1 and dir0==1 and c0≥o1 and o0≤c1`(하락장악 대칭) · **망치** `priorDownX and 작은몸통 + longLowerWick`(강세) · **역망치** `priorDownX and 작은몸통 + longUpperWick`(강세) · **교수형** `priorUpX and 작은몸통 + longLowerWick`(약세) · **유성** `priorUpX and 작은몸통 + longUpperWick`(약세) · 샛별 `dir2==-1 and b2강봉 and b1작은몸통 and dir0==1 and c0≥mid2`(석별 대칭).

**추세형 패턴**(추세형 예측 근거): **2봉 고점돌파** `dir0==1 and h0>h1 and h0>h2 and (윗꼬리/range)≤candleShortWickRatio and b0강몸통`(2봉 저점돌파 대칭: `dir0==-1 and l0<l1 and l0<l2 and (아래꼬리/range)≤candleShortWickRatio and b0강몸통`).

> **2봉 극점 돌파형**은 CEO 실전형 케이스입니다 — 직전 **2봉의 고점(또는 저점)을 모두 돌파**하고 **반대쪽 꼬리가 짧아**(강한 마감) **강한 몸통**으로 닫힌 봉입니다. 이전에는 반전 근거로 묶였으나, **선행 추세와 같은 방향의 강한 돌파**라는 성격에 맞춰 **추세형 예측**으로 재분류했습니다(진행 판정 뒤 순위 — 진행 스코어에 안 걸린 강한 돌파봉을 흡수).

### 판정 순서 (상호배타, 첫 매칭 확정)

`if/else if` 체인이라 한 봉에 **한 분류만** 나옵니다. 순위는 **반전형 예측 → 진행 → 추세형 예측**입니다(반전형은 추세 꺾임이라 먼저, 추세형 돌파는 진행에 안 걸린 강봉을 뒤에서 흡수).

| 순서 | 조건 | 분류 |
| --- | --- | --- |
| 1 | `not ready` **또는** `range3 < candleMinRange3Atr` | 혼조 |
| 2 | `priorDownX` AND (샛별 OR 상승장악 OR 망치 OR 역망치) | 반전형 상승 예측 |
| 3 | `priorUpX` AND (석별 OR 하락장악 OR 유성 OR 교수형) | 반전형 하락 예측 |
| 4 | `(HH and HL)·upCount≥2·netBody≥candleNetBodyMin` **또는** `(HH or HL)·upCount==3·netBody≥candleNetBodyStrong` | 상승 진행 |
| 5 | `(LL and LH)·dnCount≥2·netBody≤−candleNetBodyMin` **또는** `(LL or LH)·dnCount==3·netBody≤−candleNetBodyStrong` | 하락 진행 |
| 6 | `priorUpX` AND 2봉고점돌파 | 추세형 상승 예측 |
| 7 | `priorDownX` AND 2봉저점돌파 | 추세형 하락 예측 |
| 8 | 강몸통 방향 폴백: `dir0==1`·`br0≥candleStrongBodyRatio`·`netBody≥candleNetBodyMin` | 추세형 상승 예측 |
| 9 | 강몸통 방향 폴백: `dir0==-1`·`br0≥candleStrongBodyRatio`·`netBody≤−candleNetBodyMin` | 추세형 하락 예측 |
| 10 | 위 미매칭 | 혼조 |

- **강몸통 방향 폴백(순위 최하단, 혼조 직전)**: 2봉 극점 돌파(6·7)·3봉 계단 진행(4·5)에 걸리지 않았지만 **현재봉(b0)이 강몸통(`br0≥candleStrongBodyRatio`)이고 3봉 순몸통(`netBody`)이 뚜렷한** 명백한 강방향봉을 추세형 예측으로 흡수합니다. 저점 미갱신 강음봉(대칭: 고점 미갱신 강양봉)이 else 혼조로 새던 문제를 보완합니다. `dir0` 부호 배타로 상승/하락 폴백은 동시 성립하지 않습니다. **신규 파라미터 없음**(기존 `candleStrongBodyRatio`·`candleNetBodyMin` 재사용).
- 진행 분류에서 3봉이 **모두 같은 방향**(`upCount==3`/`dnCount==3`)이면 근거 패턴을 `적삼병`/`흑삼병`으로 병기합니다. 반전형 예측은 성립 패턴명(`샛별`/`석별`/`장악형`/`망치`/`역망치`/`유성`/`교수형`)을, 추세형 예측은 `2봉고점돌파`/`2봉저점돌파`(폴백은 `강몸통`)를 병기합니다(반전형 우선순위: 샛별·석별 → 장악형 → 망치·역망치 / 유성·교수형).
- **이전 12분류의 4중 AND 브리틀 게이트(적삼병 오분류)는 폐기**했습니다. 예전 적삼병은 "강한 몸통≥0.60 AND 짧은 윗꼬리 AND 종가 계단상승 AND 3봉 모두"를 전부 요구해 정상 강양봉 3연속도 혼조로 떨어졌습니다. 진행은 구조(HH/HL)+순몸통 스코어로 로버스트하게 잡습니다.

### 셀색 7분류 범례

방향 있는 6개 분류는 **현재봉 캔들 바디 색과 동일**합니다(혼조는 캔들 색칠 안 함 → 원래 봉색).

| 색 | 분류 |
| --- | --- |
| 선명 녹색 `color.new(color.green, 0)` | 상승 진행 |
| 둔한 녹색 `color.new(#6E8F72, 0)` | 추세형 상승 예측 |
| 청록 `color.new(#26A69A, 0)` | 반전형 상승 예측 |
| 회색 `color.new(color.gray, 40)` | 혼조 |
| 밝은 빨강 `color.new(#EF5350, 0)` | 반전형 하락 예측 |
| 둔한 빨강 `color.new(#A57A7A, 0)` | 추세형 하락 예측 |

### 캔들 색 표시 조건 — 교차검증 정렬 AND 게이트(표시 전용)

캔들 상태 (3) 7분류로 **현재봉 캔들 바디를 방향색으로 칠하는 것**(`barcolor`)을 **"교차검증 정렬이 될 때만"** 노출하도록 세 조건을 겁니다. **세 조건을 모두 만족(AND)** 할 때만 캔들 바색이 표시되고, **하나라도 미충족이면 색을 칠하지 않습니다**(원래 봉색 유지). **혼조**(방향 없음)는 기존대로 색 없음입니다.

> **표시 전용 게이트**입니다 — 이 조건은 캔들 바색을 **"칠할지 말지"만** 결정합니다. **캔들 7분류 판정·바색 색값·테이블 셀 색**(캔들 상태 (3) 행·캔들 손절라인 행)은 **무변경**이며, **진입/청산·신호·전략 변수 어디에도 연결되지 않습니다.** 테이블 셀 색은 분류 정보로서 게이트와 무관하게 항상 유지됩니다.

**방향 매핑**: 상승 계열(상승 진행·추세형 상승 예측·반전형 상승 예측)=**상승 포지션**, 하락 계열(하락 진행·추세형 하락 예측·반전형 하락 예측)=**하락 포지션**.

> **타임프레임(TF)별 2세트 자동 선택** — 세 조건은 **1D 미만(인트라데이) 세트**와 **1D 이상(스윙) 세트** 두 벌로 나뉘며, **차트 타임프레임에 따라 자동 선택**됩니다. 판정 = `timeframe.in_seconds() < 86400`(1일=86400초). 차트 TF는 컴파일 시점 상수라 룩어헤드·리페인팅·`na` 이슈가 없습니다. 인트라데이는 노이즈가 잦아 **엄격 세트**, 스윙은 봉당 정보량이 커 **완화 세트**를 기본값으로 둡니다. 옵션 패널에서 **`매매 신호 방식` 그룹 바로 아래**에 `캔들 색 표시 조건 (1D 미만)` → `캔들 색 표시 조건 (1D 이상)` 순으로 연속 배치됩니다.

| 옵션 (그룹) | 1D 미만 기본값 | 1D 이상 기본값 | 의미 |
| --- | --- | --- | --- |
| `추세 정렬 최소 신호 수 (0=무관)` | `2` (0~3) | `1` (0~3) | 캔들 방향과 **같은 방향의 전환 신호(0.5×/1×/2×)** 중 **'추세 상태'** 인 개수가 이 값 **이상**일 때만 색 표시. 추세 상태 = 해당 스케일 `dir`이 그 방향(상승=`dir==1`·하락=`dir==-1`, `dir05`/`dir10`/`dir20`). **0이면 이 조건 무시(무관)** |
| `리본 국면 조건` | `반대 포지션 방향 약분출 이상` | `반대 포지션 방향 약분출 이상` | 캔들 방향과 **리본 국면**(`ribbonState`) 정합. **강도 순 4옵션**(`이상`=그 강도부터 더 정합한 쪽 모두 허용). `포지션 방향 분출 이상`=캔들 상승이면 리본 `상승`·하락이면 `하락`(**정합 분출**)일 때만. `횡보 압축 이상`=정합 분출 + 리본 `횡보`(압축)도 허용. `반대 포지션 방향 약분출 이상`=위 + **반대 방향 분출**이더라도 **리본 이동% 절대값**(`ribbonMovePct`)이 아래 **`반대 분출 허용 임계%(약분출)` 이하**(약한 반대 분출)면 허용. `무관`=조건 무시. `대기` 상태는 `무관` 외엔 미통과 |
| `반대 분출 허용 임계% (약분출)` | `1.0` | `2.0` | **`반대 포지션 방향 약분출 이상`** 옵션에서 반대 방향 리본 분출을 '약분출'로 허용할 상한. `ribbonMovePct` 절대값이 이 값 이하면 약한 반대 분출로 보고 캔들 색 허용. 세트별 기본값 = **1D 미만 1.0% / 1D 이상 2.0%**(입력: `candleColorRibbonPctLo`/`candleColorRibbonPctHi`) |
| `되돌림 구간 처리` | `되돌림 확실 시 색 숨김` | `무관` | 이격도 **되돌림 상태**(`disparityState`)에 따라 색 억제. `되돌림 확실 시 색 숨김`=`되돌림 확실`이면 색을 숨김. `되돌림 주의·확실 시 색 숨김`=`되돌림 주의` 또는 `되돌림 확실`이면 색을 숨김. `무관`=조건 무시(항상 표시) |
| `비정상 거래량+신호 동시 캔들색` | `true` | `true` | **OR 오버라이드**(위 3조건 AND 게이트와 별개). 그 봉이 **비정상 거래량**(`abnVolume`)이면서 **매수/매도 신호**(`sBuy05/10/20` 또는 `sSell05/10/20` 중 하나라도)가 함께 나오면, 3조건을 통과하지 못해도 캔들 색을 표기. **신호 방향색**(매수=상승색·매도=하락색). 입력: `candleColorAbnSigLo`/`candleColorAbnSigHi` |

- **조건 1(추세 정렬)**: 캔들 방향과 같은 방향인 스케일 개수(`dir05`/`dir10`/`dir20`)를 세어 임계 이상이면 통과. 캔들 방향과 멀티스케일 전환 추세가 같은 방향으로 **정렬**됐는지 확인.
- **조건 2(리본 국면, 4옵션·강도 순)**: 캔들 방향과 리본 국면의 정합을 **강도 순**으로 게이팅. `정합 분출`(캔들·리본 동방향) → `횡보 압축` 허용 → `반대 방향 분출이라도 이동% 절대값이 허용 임계% 이하`(약한 반대, 임계는 세트별 입력 `반대 분출 허용 임계%(약분출)` = 1D 미만 1.0% / 1D 이상 2.0%)까지 허용 → `무관`. `이상`은 그 강도부터 더 정합한 쪽 모두 통과. 리본 이동%(`ribbonMovePct`)가 `na`면 반대 약분출은 불통과(안전). `대기`는 `무관` 외엔 미통과.
- **조건 3(되돌림 구간)**: 이격도 되돌림 국면(추세 되돌림 위험)에서 방향색을 억제해 **되돌림 중 잘못된 방향 신호 노출을 차단**.
- **OR 오버라이드(비정상 거래량+신호 동시)**: 위 3조건은 AND 게이트지만, 이 옵션(세트별·기본 켜짐)이 켜져 있고 그 봉이 **비정상 거래량이면서 매수/매도 신호가 동시**에 나면 3조건을 **통과 못 해도 캔들 색을 강제 표기**합니다. 최종 게이트 = `3조건 AND 게이트 OR (옵션 AND 비정상거래량 AND (매수신호 OR 매도신호))`. 오버라이드로 칠하는 색은 **신호 방향 우선**(매수 동시=상승색·매도 동시=하락색)으로, 캔들 상태가 혼조(색 없음)이거나 캔들 방향과 신호 방향이 엇갈려도 **색이 `na`로 사라지지 않게 신호 방향색을 보장**합니다(매수·매도가 같은 봉에 함께면 매수/상승색 우선 — `na` 방지용 임의 규칙). AND 게이트로 이미 통과한 봉은 기존 `candleBarColor`를 그대로 쓰되, 그 봉이 혼조(색 `na`)인데 오버라이드도 성립하면 신호 방향색으로 폴백합니다.

- **표시 전용입니다.** 어떤 진입/청산/신호 변수에도 연결되지 않으며, 기존 매수/매도 신호 로직과 독립입니다. 순수 관찰용 정보 행입니다.
- **단일/소수 캔들 패턴은 통계적으로 약한 엣지**입니다. 캔들 패턴만으로 진입하지 말고, 반드시 **이치모쿠 메인 + 테이블 + 보조지표 정렬과 교차확인**하세요(펀드 철학: 단일 신호 불신).
- **임계값·예측 방향은 파라미터 후보(제안)** 입니다. 알고리즘 수학자의 백테스트 검증 전에는 관찰용으로만 쓰고, 실전 전략에 반영하지 마세요. `candleNetBodyMin`이 진행 판정의 핵심 튜닝 노브입니다. (임계·기간·예측 방향 변경은 전략 변경에 해당하며 본부장·알고리즘 수학자 검토가 필요합니다.)

## 수축/확장 (휩쏘 필터) — 이평 리본 / 구름 두께 / 리본+구름 모드

이평 수축↔확장을 캔들에 **`+` 모양(플러스)**으로 표시합니다 — `횡보 압축`=보라 `#9C27B0`(상승·하락과 확실히 구분), `상승 분출`=파랑, `하락 분출`=빨강. `+`표식은 **신호 삼각형보다 뒤(아래 레이어)** 에 그려집니다. **판정 기준을 3가지 모드로 선택**합니다(기본 `리본+구름`).

| 모드 | 수축/확장 판정 기준 | 분출 방향 |
| --- | --- | --- |
| `이평 리본` | **입력값 기반 이평 리본**(전환선~리딩B 길이, 7개 MA)의 폭이 ATR 대비 좁으면 수축 | 빠른선(전환선 길이) ≥/< 느린선(리딩B 길이) |
| `구름 두께` | **구름 두께**(선행스팬 A·B 간격)가 ATR 대비 좁으면 수축 | 양운(A≥B)=상승 / 음운=하락 |
| `리본+구름`(기본) | 리본·구름이 **모두** 수축/확장(방향 일치)할 때만 | 두 기준 방향이 같을 때만 분출 |

- **`이평 리본`이 이치모쿠와 정합**: 리본 길이를 `전환선~리딩B`(기본 8~44)에 자동 연동해, 신호와 같은 시간축에서 수축/확장을 판단합니다. 기간 입력을 바꾸면 리본 범위도 따라갑니다.
- **`구름 두께`는 이치모쿠 본연**: 구름 두께 자체가 압축/확장 측정이라 추가 라인 없이 정합적입니다.

| 항목 | 내용 |
| --- | --- |
| 상태머신 | 압축 후 분출(같은 방향 반복은 재압축 필요). **단, 직전 분출과 반대 방향 확장이면 압축 없이 바로 반전 분출**(상승 분출 → 하락 분출, 그 반대도 가능). `리본+구름`은 방향이 엇갈리면 분출 보류(무장 유지) |
| 분출 속도 | 폭이 임계선을 완전히 넘기 전이라도 잡아 빠름. `확장 판정 폭 증가 봉수`↑ = 느리고 확실, 1 = 가장 빠름 |
| 활용 | 신호 + 같은 방향 분출 = 신뢰↑ / `횡보 압축` 구간 신호 = 휩쏘 의심·관망. **진입 트리거 아닌 신뢰도 필터** |
| 옵션(`리본 압축/분출`) | `수축/확장 판정 모드`(리본+구름 기본)·`리본 이동평균 종류(A)`(SMA)·`횡보 압축 표시`·`분출 표시`·`봉마감 후 확정`·`폭 ATR 기준`(14)·`수축 판정 ATR 배수`(0.8)·`확장 판정 폭 증가 봉수`(1) |
