# Advanced Ichimoku

트레이딩뷰용 Pine Script Ichimoku 확장 버전입니다. 정통 이치모쿠 계산 위에 **멀티스케일 신호·트리거 테이블·휩쏘 필터·이격도**를 얹은 인디케이터입니다.

| 스크립트 | 용도 |
| --- | --- |
| [`advanced-ichimoku.pine`](./advanced-ichimoku.pine) | 인디케이터 — 멀티스케일 신호 마커 + 우측 상단 테이블 |
| ~~`ichimoku-strategy.pine`~~ | **삭제됨 (추후 재작성 예정, CEO 지시)** — 기존 백테스트 전략(`strategy`) 파일은 제거되었습니다. 이 문서는 현재 인디케이터(`advanced-ichimoku.pine`)만 설명합니다. |

> **ℹ️ 전략 파일 안내:** 과거 `ichimoku-strategy.pine`(인디케이터 시각화 이식 + 추세 추종 진입/청산 + 자금관리 백테스트 전략)은 **CEO 지시로 삭제**되었으며 **추후 재작성 예정**입니다. 재작성 시 진입/청산 로직·리페인팅 정책·부분 익절/강제청산 규칙 등을 이 절에 다시 문서화합니다. 아래 문서는 모두 **인디케이터(`advanced-ichimoku.pine`)** 기준입니다.

## 핵심 기능

| 기능 | 설명 |
| --- | --- |
| 매수/매도 신호 | 래깅 스팬이 기준선 돌파 **AND** 컨버전 라인이 기준선 돌파. 트리거 충족 후 **다음 봉 확정 시 발화**(휩쏘 필터·리페인팅 방지·1봉 지연) |
| 멀티스케일 표시 | 같은 신호를 래깅 설정값의 **0.5× / 1× / 2×** 스케일로 계산해 **삼각형 크기**(가장 작음/작음/보통)로 동시 표시 |
| 추세 매수/매도 신호 (6모드) | `일반 예측` / `일반 확정` / `거래량 가중 예측` / `거래량 가중 확정` / `혼합 확정` / `혼합 예측` 중 **택일**(기본=`혼합 확정`). 신호 소스(일반 도치안 vs VWMA)와 확정 방식(예측 vs 확정) 조합. **혼합**은 봉이 **비정상 거래량이면 그 봉에서 VWMA 신호를 즉시 발화**(다음봉 확정 우회 — 거래량 가중 전용 모드처럼 그 봉에 바로 삼각형), **정상 봉은 일반 신호를 기존 방식대로**(장초/뉴스發 비정상 봉 빠른 캐치). ⚠️ **`혼합 확정`은 구 `혼합 일반`을 개명한 것**(로직·isPredict 동일, 옛 순수확정 `혼합 확정`은 폐기). **`혼합 예측`은 `혼합 확정` 경로에 '전환 행 색 예측 발화'(후보 A)를 추가** — 추세 확인 테이블의 **상승/하락 전환(스케일) 행 셀이 둔탁 녹(`#6E8F72`, 롱)/둔탁 빨강(`#A57A7A`, 숏) = 전환 예측·대기 상태로 칠해지고** 진행봉 방향(양봉=롱/음봉=숏)이 일치하면 **그 봉에서 즉시 발화**(원시 도치안 신호 불요, 비정상봉 즉시발화와 병존 — CEO 최종 확정: '보이는 둔적/둔녹이면 방향 맞는 캔들일 때 무조건 발화'). 발화 기준은 **캔들 색이 아니라 전환 행 셀 색**(`f_trigBg`/`f_trigNear`와 동일 소스)이라 화면 셀 색과 신호가 100% 일치합니다. **⚠️ `혼합 예측`은 진행봉 실시간 예측이라 리페인팅 계열이며 전략 매매·백테스트가 크게 변한다**(수학자 리뷰·본부 확인 대상) |
| 신호 순서 제한 | 같은 방향 신호는 반대 신호가 한 번 나온 뒤에만 다시 발생(중복 방지) |
| 우측 상단 테이블 | **3개 섹션 병합 헤더**로 묶어 표시 — **[추세 확인]**(상승/하락 전환·중기 추세(VWAP·100선)·장기 추세) · **[캔들 및 되돌림 확인]**(캔들 상태·캔들 손절라인·비정상 매수/매도·이격도·되돌림 가격) · **[지지/저항 확인]**(구름대·VWAP 상태·50선 지지/저항·100선 지지/저항·200선 지지/저항). **⚠️ 구 [손익비 현황 확인] 섹션(`Cha 롱/숏`·`Ret 롱숏`) 전체는 CEO 지시로 지표에서 제거됨**(표시 전용·매매 미사용). (구 `VWAP 롱/숏` 행은 그 이전에 이미 제거됨.) 손익비를 먹이던 청산선(샹들리에) 값 계산도 지표에서 함께 제거됨 — 매매용 값 계산은 **삭제된 전략 파일(추후 재작성 예정)에서 다룰 예정** |
| 스윕/청산선 | **⚠️ 지표(`advanced-ichimoku.pine`)에서는 스윕·청산선 페인팅에 이어 값 계산까지 CEO 지시로 완전히 제거됨**(손익비 행 제거와 함께 — 지표는 매매를 하지 않으므로 소비처가 없어짐). 청산선(직선) 차트 표시는 [Auto VWAP 지표](../VWAP/README.md)로 이동했습니다(전 히스토리 봉에 `plot`). **스윕라인/청산선 값 계산(진입 트리거·손절 기준선용)은 삭제된 전략 파일(추후 재작성 예정)에서 다룰 예정**입니다 |
| 수축/확장 (휩쏘 필터) — 리본 추세 | **⚠️ 리본 추세 표시는 CEO 지시로 제거됨** — 테이블 `리본 추세` 행 및 차트 `횡보 압축 → 상승/하락 분출` `+` 표식(plotshape)·리본 알림(alertcondition)이 모두 삭제됐습니다. **리본 자리(테이블 [추세 확인] 섹션)에는 `중기 추세`(VWAP·100선) 1행이 들어갑니다.** (**리본 계산부(7개 이평·수축/분출·상태머신)는 성능 리팩토링 ④에서 사멸코드로 제거됨** — 소비처가 0이었던 표시 산출물만 이전에 잘려 나가 계산 전체가 미사용이 됐기 때문. 구름 지지/저항이 재사용하는 `rAtr`(=`ta.atr`)만 남음.) `수축/확장 판정 모드` 등 리본 입력 그룹은 옵션 패널에만 잔존(표시·계산 미연결) |
| 중기 추세 (VWAP·100선) | **([추세 확인] 섹션, 구 `리본 추세` 자리·전환 6행 다음 1행, `장기 추세` 바로 위)** **VWAP(50)과 100선(MA100)의 "배열 상태"로 판정하는 추세** — `장기 추세` 행(MA100 vs MA200)과 **완전히 동일한 로직·포맷·색**을 VWAP(빠른 선) vs 100선(느린 선)에 이식. VWAP > 100선 = **상승 배열**, VWAP < 100선 = **하락 배열**. 두 선 간격(`mtGap`) 확대=`강한`·축소·반대 크로스 임박=`약한`·그 사이=`상승/하락`, 크로스 순간은 `골든 크로스`/`데드 크로스`. 값 포맷은 장기 추세와 동일한 **`상승/하락 (±변화율% \| N봉)`** — 마지막 VWAP×100선 골든/데드 크로스 시점 이후 **VWAP 변화율**과 경과 봉수(예 `강한 상승 (+2.4% \| 18봉)`). VWAP·MA100 워밍업(`vwHasVwap and not na(ltMa1)`) 전엔 `-`(회색). ⚠️ **지지/저항 포맷 아님** — 장기 추세와 동일 포맷 | **방향 계열 색**(장기 추세와 동일): 상승계열=청록 `#0E9C8A`·하락계열=자홍 `#C2185B`. 강함(폭 확대)=선명(0)·약함(축소·크로스 임박)=옅음(55)·추세=중간(30)·크로스 이벤트=선명(0), 워밍업 na=회색 |
| 이격도 | 선택 이평선(기본 SMA 20) 대비 `종가÷MA×100`(기준 100). 고정 임계+적응형(σ) 2기준 중 1개=`되돌림 주의`(둔탁)·2개=`되돌림 확실`(선명)로 테이블 표시 |
| 장기 추세 | 이평선 리본 통합 — **MA 100/200 선 + backfill**. MA 종류 4종(기본 거래량 가중 VWMA), MA100=파랑·MA200=보라, backfill 우상향=파랑·우하향=보라(투명도 60). 테이블에 강/약 추세·골든/데드 크로스 표시 |

## 신호 로직 + 멀티스케일

`래깅 스팬 = 현재 종가(close)`. 차트상 래깅 스팬이 닿는 위치의 라인은 `[displacement-1]` 값으로 비교합니다.

| 구분 | 조건① (래깅 스팬) | 조건② (컨버전 라인) | 추가 조건 |
| --- | --- | --- | --- |
| 매수 신호 | `close > baseLine[disp-1]` 또는 상향 크로스 | `conversionLine > baseLine` 또는 골든크로스 | 중복 방지 |
| 매도 신호 | `close < baseLine[disp-1]` 또는 하향 크로스 | `conversionLine < baseLine` 또는 데드크로스 | 중복 방지 |

- **신호 순서 제한**: 매도 신호 후에만 매수 신호가 다시, 매수 신호 후에만 매도 신호가 다시 나옵니다(스케일·신호 종류별로 독립). 같은 방향이 유지돼도 첫 봉만 1회. 이 순서·중복 제어는 **상위 게이트(`dir!=방향`)와 확정봉 `dir` 잠금**이 단일 소스로 담당하며, **방향이 전환되면(예: 상승→하락) 반대 방향 첫 신호는 그 봉에서 정상 발화**합니다(원시 신호 계산 함수 내부에는 별도 방향 래치가 없습니다).
- **다음 봉 확정 발화(휩쏘 필터·1봉 지연)**: 캐스케이드 게이트가 봉 N에서 충족돼도 **그 봉에서 즉시 발화하지 않습니다**. 그 봉은 **대기(pending)** 상태가 되고, **다음 봉 N+1(확정봉)에서 확정 조건이 유지될 때만 그 봉에서 신호가 발화**합니다(유지 안 되면 취소 — 신호 없음). **확정 조건 = 대기 시점의 그 스케일 트리거 가격을 다음 봉 종가가 유지**(매수 `close ≥ 대기 buyTrig`, 매도 `close ≤ 대기 sellTrig`). 대기 시점 트리거 값을 래치해 다음 봉 종가와 비교하므로 **미래참조 없이 확정봉 종가 기준**(리페인팅 없음). 전 스케일(0.5×/1×/2×)·매수·매도 모두 대칭. 대기 중 반대 방향 게이트가 새로 뜨면 그쪽으로 대기가 전환됩니다. **삼각형·알림(`alertcondition`)·`dir` 추세 잠금은 모두 확정 발화(`fBuyXX`/`fSellXX`) 단일 소스로 이 확정봉에서 동기화**되어(트리거 봉이 아니라 확정봉에 찍힘) 신호·테이블이 어긋나지 않습니다. **한 트리거 이벤트당 신호는 확정봉에 정확히 1개**만 찍힙니다(확정된 봉에서는 같은 방향 재대기를 막아, 확정봉이 곧바로 다음 트리거 봉이 되어 다음 봉에서 재확정되는 중복 발화를 차단). 트리거 충족 후 신호가 **1봉 지연**되는 대신, 다음 봉으로 이어지지 않는 속임수 돌파(휩쏘)를 걸러 신뢰도를 높입니다.
- **스케일 캐스케이드(하위 추세 상태 전제)**: 상위 스케일 신호는 **하위 스케일이 해당 방향 추세 상태일 때** 발화합니다. **1× 매수는 0.5×가 `상승 추세`(`dir05==1`)인 동안**, **2× 매수는 0.5×·1×가 둘 다 `상승 추세`(`dir05==1 and dir10==1`)인 동안** 그 스케일의 자기 트리거가 충족되면 발화합니다(매도는 `하락 추세` 기준으로 대칭). 하위 추세 상태는 **전제**일 뿐, 각 스케일의 자기 트리거(컨버전/래깅 크로스 조건)는 그대로 요구됩니다. `dir` 추세 잠금은 위 **다음 봉 확정 시점**에 이뤄지므로, 하위가 확정돼 `dir`이 잠긴 뒤에야 상위 스케일이 대기 등록됩니다(봉 순서상 정상). 반대 신호가 그 스케일에 확정되면 `dir` 부호가 뒤집혀 상위 스케일 전제가 자연히 풀립니다(반대도 동일). 알림(1×)도 이 게이트를 따릅니다.
    - **버그 수정(지속 추세 미발화 해소)**: 이전에는 캐스케이드 전제가 "하위 스케일이 **방금 발화**"(구 `buyStage`/`sellStage` 카운터)여서, 지속 상승장에서 1×가 이미 `상승 추세`로 잠기면(dir10==1) **새 0.5× 신호가 1×로 승격 못 하고 2×도 안 뜨는** 문제가 있었습니다. 전제를 "하위 스케일이 **추세 상태**"(`dir`)로 바꿔 해소했습니다(구 카운터 제거).
  - **트리거 스킵(즉시 승격)**: 상위 배수(1× 또는 2×)의 **트리거 가격이 이미 현재 봉에서 충족**(매수=현재 종가 ≥ 트리거, 매도=현재 종가 ≤ 트리거)되고 **하위 배수가 같은 방향 추세 상태**(1× 스킵=0.5× 상승/하락 추세, 2× 스킵=0.5×·1× 둘 다)면, 원시 신호를 기다리지 않고 상위 배수 신호를 **현재 봉에 바로 표시**합니다. 예) `1× 매수` 상승 추세 중 현재가 5000, `2× 매수 트리거 4000` → 현재 봉에 `2× 매수` 표시. 매도도 동일.
  - **순차성 강제(중복 방지)**: 게이트가 모두 **직전 봉까지의 `dir`(하위 추세 상태)** 을 참조하고 `dir`은 다음 봉 확정 시점에 잠기므로, 한 봉 안에서 `0.5×→1×→2×`가 동시에 발화하지 않고 **한 봉엔 한 배수만** 발생합니다. 반대 방향 신호가 확정되면 그 스케일 `dir`이 뒤집혀 상위 전제가 풀리므로 **모멘텀·정상 모드와 무관하게 순차성이 강제**됩니다.
  - **같은 방향 재발화 차단**: 각 스케일은 **이미 그 방향이면(예: 2× 이미 `상승 추세`) 그 스케일에 반대 신호가 나오기 전까지 같은 방향 신호를 다시 표시하지 않습니다**(`dir!=방향` 기준, `dir`은 확정봉에서 잠김). 상위 배수가 같은 방향으로 중복 발화하던 것 방지 — 롱숏·전 스케일 공통.
- **모멘텀 모드(상태 진입 시 신호 가속)**: `정상 캔들`이면 네 스케일 모두 정상 기간(0.5×/1×/2×)으로 계산합니다. **`비정상 매도 캔들`/`롱 반등 감시`(하락 이탈) 또는 `비정상 매수 캔들`/`숏 반등 감시`(상승 이탈) 상태로 진입하면 매수·매도 양방향 모두 모멘텀 모드**가 됩니다(어느 방향으로 진입하든 두 방향 동시 진입). **비정상 판정을 멀티스케일 신호보다 먼저 계산하므로, 비정상 매수/매도 등 신호가 뜬 바로 그 봉(현재 종가 기준)에서 모멘텀 모드가 즉시 적용**됩니다(신호·테이블 모두 1봉 지연 없음). 모멘텀 모드는 **기준선·컨버전·래깅 스팬 기간을 전부 한 단계 빠른 스케일로 시프트**합니다(표시상 `0.5× → 0.25×`, `1× → 0.5×`, `2× → 1×`). 로직 자체는 정상 로직 그대로이고, **적용 기간만 짧아져** 더 빨리 반응합니다.
  - **해제**: 모멘텀 모드는 **정상 이격도(상태 종료 = 이격도 중립 복귀)** 가 되기 전까지 유지되고, 그 시점에 양방향 모두 해제되어 정상 스케일로 복귀합니다. 반대 방향 신호가 나와도 해제되지 않습니다. 재무장은 상태가 한 번 끝났다가 다시 시작(컨텍스트 재진입)할 때만 됩니다.
  - 우측 상단 테이블의 트리거 가격은 모멘텀 모드일 때 **`M ` 접두어**로 표시합니다.

### 멀티스케일 (0.5× / 1× / 2×)

위 신호 로직을 **래깅 설정값(컨버전 8·베이스 22·래깅 22)의 세 배율**로 각각 계산해, 삼각형 **크기**로 어느 시간축에서 나온 신호인지 보여줍니다. (리딩 스팬 B는 신호와 무관하므로 스케일에 안 씀)

| 스케일 | 기간 예시(기본 입력 기준) | 삼각형 크기 |
| --- | --- | --- |
| `0.5×` | 4 · 11 · 11 | 가장 작음 (`tiny`) |
| `1×` | 8 · 22 · 22 | 작음 (`small`) |
| `2×` | 16 · 44 · 44 | 보통 (`normal`) |

- **여러 크기의 삼각형이 같은 봉에 겹치면** 짧은·중간·긴 시간축이 한 방향으로 정렬된 것 → 신뢰도↑.
- 차트엔 **1× 라인/구름만** 그립니다. 0.5×·2×는 신호 계산용입니다.
- 알림(alert)은 **1× 신호** 기준이고, 우측 상단 테이블의 `상승/하락 전환` 트리거 가격은 **0.5× / 1× / 2× 세 스케일을 각각** 표시합니다.

## 추세 매수/매도 신호 (6모드 · 택일)

`매매 신호 방식` 그룹의 `추세 진입 신호 방식` 입력 하나로 매수/매도 신호의 **신호 소스**와 **확정 방식**을 함께 고릅니다(세 스케일 모두 같은 모드 적용). 6모드는 **택일**이라 한 번에 한 조합만 표시됩니다.

| 모드 | 신호 소스 | 확정 방식 |
| --- | --- | --- |
| `일반 확정 모드` | 일반 도치안(컨버전·기준선) | 봉마감 확정 — 현재 봉 종가로 신호 확정 |
| `일반 예측 모드` | 일반 도치안 | 예측 — 다음 봉 가정으로 1봉 빨리 포착 |
| `거래량 가중 확정 모드` | VWMA(거래량 가중) | 봉마감 확정 |
| `거래량 가중 예측 모드` | VWMA(거래량 가중) | 예측 |
| `혼합 확정 모드`(**기본** · 구 `혼합 일반` 개명) | **봉별 스위칭** — 비정상 거래량 봉=VWMA(**즉시 발화**), 정상 봉=일반 도치안 | 정상 봉만 예측(모멘텀·`isPredict`), 비정상 봉은 즉시 |
| `혼합 예측 모드`(신규 · 후보 A) | 〃(봉별 스위칭) **+ 전환 행 색 예측 발화** | `혼합 확정`과 동일 경로 + 추세 전환 행 셀 색 기반 예측 발화(리페인팅 계열) |

> ⚠️ **개명 주의(모드 재구성)** — 구 `혼합 일반 모드`가 **`혼합 확정 모드`로 개명**되었습니다(로직·`isPredict` 그대로 — 이름만 바뀜, 여전히 정상 봉은 예측/모멘텀 방식). 옛 순수확정 `혼합 확정 모드`는 **폐기**되었습니다. 신규 **`혼합 예측 모드`** 는 `혼합 확정` 경로에 **전환 행 색 예측 발화(후보 A)** 만 추가한 별도 모드입니다. `sigPredict`(예측/확정 분기)는 `"예측"` 포함 모드 + `혼합 확정`(구 혼합 일반의 예측 로직 보존)에서 true입니다.

두 축은 서로 독립입니다.

- **신호 소스** — `일반`은 컨버전·기준선을 그대로 쓰고, `거래량 가중`은 컨버전·기준선을 `VWMA = Σ(종가×거래량)/Σ거래량` 으로 바꿔 계산합니다(그 외 로직은 동일, 후행스팬은 실제 종가). 거래량 가중 모드일 때만 VWMA 신호가 선명한(불투명) 삼각형으로 그려지고 일반 신호는 숨습니다(배타).

### 혼합 모드 (신규 — 장초/뉴스 빠른 캐치)

혼합 모드는 **봉 단위로 신호 소스를 바꿉니다.** 목적은 장초·뉴스發 비정상 거래량 봉에서 거래량 가중 관점으로 추세 변화를 **한 발 빨리 캐치**하는 것입니다.

- **봉별 처리 스위칭 (비정상=즉시 / 정상=방식대로)** — 그 봉이 **비정상 거래량**이면 그 봉의 **VWMA(거래량 가중) 신호를 그 봉에서 즉시 발화**합니다(다음봉 확정 파이프라인을 **우회** — 거래량 가중 전용 모드가 그 봉에 바로 삼각형을 찍는 것과 동일). 비정상이 아니면(정상 봉) **일반 도치안 신호를 기존 방식(예측/확정 파이프라인)** 그대로 처리합니다. 즉 혼합 = `비정상 봉 ? (VWMA 즉시 발화) : (일반 신호 파이프라인)`.
- **비정상 거래량 정의(⚠️ 전략 파라미터)** — `volume > SMA(volume, N) × (1 + 배수)`. 기본값 `N=20` · `배수=1.0`(= 최근 20봉 평균 거래량의 **2배 초과**). 이 정의·기본값은 **VOLUME-PRESSURE 지표와 동일한 펀드 표준**입니다. `비정상 거래량 비교 봉수(N)`·`비정상 거래량 배수(mult)` 입력으로 노출(`매매 신호 방식` 그룹).
- **비정상 봉 즉시 발화(다음봉 확정 우회)** — CEO 요구: 비정상 거래량이 나오면 **바로 그 봉 기준으로** VWMA 매수/매도 신호가 즉시 나와야 합니다. 이전(C-1)에는 혼합 신호를 일반 파이프라인(게이트→pending→**다음봉 확정 대기**)에 태워, 비정상 봉의 VWMA 신호가 그 봉에서 안 찍히고 **다음 봉으로 지연**되는 버그가 있었습니다(예측·확정 공통). 이를 바로잡아 **혼합 모드(예측·확정 공통)에서 비정상 봉은 pending 파이프라인을 우회하고 그 봉에서 즉시 삼각형**을 찍습니다. 즉시 발화 신호 소스는 VWMA **원시신호**(모멘텀 시프트 반영)이며, 발화·`dir` 잠금·삼각형·알림이 모두 그 봉에서 동기화됩니다.
- **확정/예측 구분** — 비정상 봉은 어느 방식이든 **즉시 발화**(모드 무관). 정상 봉은 `혼합 확정`(구 `혼합 일반`)·`혼합 예측` 둘 다 **동일한 예측/모멘텀 경로**(`isPredict=true`)로 처리됩니다(개명 전 로직 그대로). `혼합 예측`은 여기에 아래 **전환 행 색 예측 발화**만 추가로 얹습니다.
- **중복방지 정합 (비정상↔정상 경계)** — 즉시 발화라도 **같은 방향 연속 중복 발화는 금지**됩니다. 핵심은 즉시 발화와 정상 봉 파이프라인이 **단일 방향 상태(`dirXX`)를 공유**한다는 점입니다. 거래량 가중 전용 모드의 함수 내부 `vLastDir`(자체 래치)이 아니라 파이프라인의 `dirXX`로 중복방지를 통일해, 정상↔비정상 봉이 바뀌는 경계에서 같은 방향 중복/누락이 **구조적으로** 생기지 않습니다. 즉시 발화 게이트 = `비정상 봉 AND VWMA 원시신호 AND dir≠방향 AND 스케일 캐스케이드 전제(하위 dir)`. 스케일 캐스케이드(1×는 `dir05`, 2×는 `dir05·dir10`이 같은 방향 추세일 때만)·`dir≠방향` 재발화 차단이 정상 봉과 동일 기준으로 적용됩니다. **비정상 봉에서는 pending 등록을 억제**해(정상 봉의 진행 중 대기는 취소) 즉시 발화와 다음봉 확정이 겹치는 이중 발화를 막습니다. CEO 지시대로 "거래량 가중이든 모멘텀이든 일반이든 중복 안 되게 순서대로".
- **트리거 셀 `V` 표시** — 우측 상단 테이블에서 그 봉의 트리거가 **비정상 거래량(VWMA 소스)** 인 경우 해당 트리거 셀 텍스트 맨 앞에 **`V `** 접두를 붙여 소스를 식별합니다(예: `V M ↑ ...`). 정상 봉이면 접두 없이 기존 그대로입니다. 접두 순서는 `V ` → `M `(모멘텀) → 본문.
- **차트-테이블 정합** — 차트 삼각형과 테이블 추세(`dirXX`)가 **같은 최종 발화 소스**(비정상 봉=즉시 발화·정상 봉=확정 발화)를 참조하므로 항상 일치합니다. 삼각형은 일반 모드와 동일한 마커(매수=아쿠아·매도=핑크, 모멘텀 진입 시 청산선 색)로 표시됩니다.
- **비정상 거래량 봉의 진행봉 토글(D-1)** — 비정상 거래량 판정(`abnVolume`)은 **진행 중인 현재 봉의 거래량**을 쓰므로, 봉이 완성되기 전까지 비정상 여부가 **봉 내에서 바뀔 수 있습니다**. 즉시 발화 구조에서는 그 봉이 비정상으로 판정되는 순간 VWMA 신호가 그 봉에 즉시 뜨는데, 이는 **거래량 가중 전용 모드와 동일한 실시간 특성**이며 **CEO의 "즉시 캐치" 취지에 부합하는 의도된 동작**입니다(그래서 `barstate.isconfirmed` 가드를 일부러 걸지 않습니다). 봉 완성 전까지 즉시 신호가 나타났다 사라질 수 있는 실시간 변동(리페인팅 성격)을 완전히 없애려면 **확정 계열(`일반 확정`·`거래량 가중 확정`)** 을 쓰세요 — 혼합 계열(`혼합 확정`·`혼합 예측`)은 정상 봉이 예측 경로이고 비정상 봉의 즉시 발화 자체가 실시간 판정이라, 무변동과는 개념상 상충합니다(즉시성 우선 설계).

#### 혼합 예측 모드 — 전환 행 색 예측 발화(후보 A, CEO 확정)

`혼합 예측 모드`는 `혼합 확정`의 모든 동작(봉별 스위칭·비정상 봉 즉시 발화)을 **그대로 포함**하면서, 여기에 **추세 확인 테이블의 '상승/하락 전환(스케일)' 행 셀 색을 이용한 즉시 발화**를 **추가**합니다(병존). 정상 봉의 일반 파이프라인, 비정상 봉의 VWMA 즉시 발화는 손대지 않고, 그 위에 전환 행 색 예측 발화 경로 하나를 더 얹는 구조입니다.

- **발화 기준 = 전환 행 셀 색** — 추세 확인 테이블의 `상승 전환(0.5×/1×/2×)`·`하락 전환(0.5×/1×/2×)` 행 셀 색이 **둔탁 녹(`#6E8F72`)/둔탁 빨강(`#A57A7A`) = '전환 예측·대기' 상태**일 때 발화합니다(CEO 확정 사양). ⚠️ **캔들 상태(3) 행의 캔들 색(barcolor)이 아닙니다** — 과거 캔들 상태 기반 발화는 근본 오류로 폐기됐습니다.
- **셀 색 소스 = `f_trigBg`** — 전환 행 셀 색은 `f_trigBg(blocked, trig, isBuy)`가 결정하며, 둔탁 녹/적 조건은 **`not blocked`(= `dir≠방향`, 확정 추세 아님) AND `f_trigNear(trig, isBuy)`(트리거가 현재가 기준 `nearPct`% 이내)** 입니다(선명 녹/적=확정 추세=`blocked`, 회색=중립). 발화도 **정확히 같은 소스**(`f_trigNear` 함수·동일 스케일 트리거 변수 `buyTrigS`/`sellTrigS`)를 써서, 화면에 보이는 셀 색과 발화가 100% 일치합니다.
- **발화 조건 (스케일 s ∈ {0.5×, 1×, 2×})** — ★ CEO 최종 확정: **원시 도치안 신호(`buyS`/`sellS`) 요구 없음.** 발화 = **테이블 셀이 둔적/둔녹으로 렌더되는 그 불리언 자체 + 방향 캔들**입니다("보이는 둔적/둔녹이면 방향 맞는 캔들일 때 무조건 발화").
  - **롱**: `상승 전환(s)` 행 셀이 **둔탁 녹(`#6E8F72`)** 이 되는 조건(= `dir≠상승`(=`not buyBlockedS`) **AND** `f_trigNear(buyTrigS, true)`) **AND** 진행봉 양봉(`close > open`) → 그 봉에서 즉시 발화. (원시 `buyS` 불요·캐스케이드 전제 없음)
  - **숏**: `하락 전환(s)` 행 셀이 **둔탁 빨강(`#A57A7A`)** 이 되는 조건(= `dir≠하락`(=`not sellBlockedS`) **AND** `f_trigNear(sellTrigS, false)`) **AND** 진행봉 음봉(`close < open`) → 즉시 발화. (원시 `sellS` 불요)
  - 스케일 매핑: `0.5×`→`전환(0.5×)` 행(`buyTrig05`/`sellTrig05`), `1×`→`전환(1×)` 행(`buyTrig10`/`sellTrig10`), `2×`→`전환(2×)` 행(`buyTrig20`/`sellTrig20`).
- **셀 색 조건과 완전 동일 집합** — 발화 불리언 = 테이블 `f_trigBg`가 둔적/둔녹을 렌더하는 불리언(`not blocked AND f_trigNear`) ∩ 방향 캔들입니다(부분집합 아님). ⚠️ `f_trigBg`의 셀 색 조건에는 **스케일 캐스케이드 전제가 없으므로**(단지 `not blocked` + 근접), 발화도 캐스케이드를 걸지 않습니다(셀 색에 내재된 것만 활용). 이전 사양은 게이트 `gBuyXX`/`gSellXX`(원시신호 + 캐스케이드 포함)를 요구해 "둔적/둔녹인데 원시신호 없으면 미발화"였으나, CEO 지시로 **원시신호·캐스케이드 요구를 제거**했습니다.
- **중복 차단·dir 정합·병존** — `dir≠방향`(= `not blocked`)이 같은 방향 재발화를 막습니다(이미 그 방향 추세면 셀이 '선명색'이라 둔적/둔녹 미해당). pending 억제(`not pBuyXX`/`not pSellXX`)로 같은 봉 확정 이중 발화도 차단합니다. 비정상 봉 즉시 발화(`hybImm`)와 전환 행 색 예측 발화가 같은 봉에 겹쳐도 같은 방향이면 `dir`을 한 번만 잠그므로 이중 삼각형이 생기지 않고, pending 등록은 두 경로 모두에서 억제됩니다.
- **⚠️ 리페인팅 계열** — 전환 행 셀 색은 진행봉 종가(`close`) 기준 근접(`f_trigNear`)으로 실시간 재계산되고, 진행봉 방향(`close`/`open`)도 봉 마감 전까지 바뀝니다. 따라서 이 발화 신호는 **봉 완성 전에 나타났다 사라질 수 있습니다**(확정봉 필터 우회). **전략(추후 재작성 예정) 재작성 시 진입·강제청산 타이밍·백테스트에 크게 영향을 줄 특성입니다.** 이는 후보 A의 본질적 특성으로 **의도된 트레이드오프**입니다. 리페인팅 없는 동작을 원하면 `혼합 확정`(또는 다른 확정 계열) 모드를 쓰세요. **전략 파라미터에 준하는 변경이므로 알고리즘 수학자 리뷰·본부 확인 대상**입니다.
- **지표·전략 동일** — 차트 삼각형(지표)과 전략 매매 신호(`sBuyXX`/`sSellXX`)가 동일한 전환 행 색 예측 발화 경로를 참조합니다("차트에 보이는 신호 = 전략이 쓰는 신호" 일원화 유지).

| 모멘텀 모드 트리거 동기화 | 모멘텀 모드면 매수·매도 트리거 모두 **한 단계 빠른 스케일 기간**(0.5×→0.25×, 1×→0.5×, 2×→1×)의 결합가로 리포트하고 값 앞에 **`M `** 표기. 정상 상태는 위 결합가(정상 기간) |
| 래깅 기준가 | 다음 봉 래깅 스팬이 닿는 `baseLine[(disp×배율)-2]`(스케일별) |
| 전환선 전환가 | 다음 봉에 기준선·전환선이 같이 변함 → 상승 `2×다음봉기준선 − 최근(컨버전-1)봉 최저`, 하락 `… − 최고` |
| 진행 중 표시 | 각 스케일의 방향은 **실제로 표시된 삼각형(게이트·스킵·모멘텀 반영)** 기준이라 차트와 항상 일치 — 그 방향이 이미 진행 중이면 상승 전환 칸 `상승 추세 (기준선 이격도)`, 하락 전환 칸 `하락 추세 (기준선 이격도)`. 이격도 = **종가÷그 스케일 기준선×100(기준 100)** → 값이 100에서 많이 벌어질수록 되돌림 위험 |
| 화살표 ↑/↓ | 현재가(`close`) 대비 트리거 가격 위치(높으면 ↑, 낮으면 ↓) |
| N봉 (시간 조정 투영) | 트리거 셀 끝 `\| N봉`. **현재 종가가 계속 유지된다고 가정할 때, 몇 봉 더 지나야 그 스케일의 상승/하락 전환이 성립하는지**의 최소 봉수 투영. **투영 대상 조건 = 핵심 도치안 전환선 vs 기준선 크로스만**(매수 `convProj > baseProj`·매도 `convProj < baseProj`) — 트리거 가격의 래깅 성분·스케일 캐스케이드는 다봉·확률 의존이라 시간 투영이 애매해 **제외**. **가정**: 미래 봉의 고가·저가·종가를 현재 종가로 고정 → k봉 후 도치안 창 = 과거 `(len−k)`봉 실제 고저 + 미래 `k`개 현재 종가. **horizon** = 그 스케일 기준선 길이(baseP)와 22 중 큰 값까지 탐색, 그 안에 성립 안 하면 `-`. **계산 불가(`-`)** = horizon 내 미성립·과거 데이터 부족(`bar_index < len`). **진행 중(상승/하락 추세)**이면 트리거 가격 대신 추세 텍스트라 N봉 **미표기**. 모멘텀 모드면 트리거와 동일하게 conv/base 길이를 한 단계 빠른 기간으로 시프트해 투영. **미래참조 아님**(과거 데이터로 미래 창을 수학적으로 투영)·**리페인팅 없음**(`barstate.islast`에서만 확정봉 종가 기준 1회 계산). ⚠️ **타이밍 투영은 가정(현재가 유지) 기반 참고치** — 실제 미래가는 변하므로 N봉은 근사임. |
| 천 단위 쉼표 | 트리거·청산 가격은 `1,000`처럼 3자리마다 쉼표, 소수 자릿수는 심볼 `mintick` 기준 |
| 구름대 평가 시점 | 구름은 displacement만큼 앞으로 밀리므로 현재 봉 아래 구름 = `[displacement-1]` 과거 선행스팬으로 평가 |
| VWAP×100선 크로스 마커 | 중기 추세 골든/데드를 확정 크로스로 차트에 표시 — 골든(`ta.crossover(vwapValue, ltMa1)`, VWAP(50)가 MA100 상향 돌파)=**100선(MA100) 가격 위치의 녹색 십자**, 데드(`ta.crossunder(vwapValue, ltMa1)`, 하향 돌파)=**100선(MA100) 가격 위치의 붉은 십자**. `plotshape`(차트 상/하단 고정) 대신 **`plot(조건 ? ltMa1 : na, style=plot.style_cross, linewidth=4)`** 로 표시해 크로스가 일어난 실제 가격(100선)에 십자를 찍음(십자 굵기 `linewidth=4` — Pine `plot` 유효 최대) — 조건 미충족 봉은 `na`라 해당 크로스 봉에만 십자 1개 표시(선 연결 없음). VWAP·MA100 워밍업 na 가드(`vwHasVwap and not na(ltMa1)`) 후 확정 크로스만 사용 → **리페인팅 없음**. 동일 조건으로 `alertcondition`(골든/데드) 제공. (⚠️ 구 리본 `+`표식·리본 알림은 제거됨) |
| 스윕/청산선 페인팅 | **⚠️ 제거됨(CEO 지시).** 청산선(직선) 차트 표시는 [Auto VWAP 지표](../VWAP/README.md)로 이동(전 히스토리 봉에 `plot`, 청산선 색·두께·on/off 옵션 포함). 스윕라인(점선, 샹들리에 ATR 트레일)은 이동하지 않았습니다. 이치모쿠 **전략**은 스윕라인/청산선 **값을 매매에 계속 사용**(스윕라인=진입 트리거·청산선=진입봉 손절 기준선)하나 **차트에 그리지는 않습니다.** **지표(`advanced-ichimoku.pine`)는 페인팅에 이어 손익비 행 제거와 함께 스윕/청산선 값 계산·`스윕/청산` 입력 그룹까지 완전히 제거**되었습니다(소비처였던 `Cha 손익비`가 사라짐) |

### 스윕 가격 모드 (샹들리에 기반) — 스윕라인/청산선 값 계산 (전략 전용)

> ⚠️ 아래는 **스윕라인/청산선 값을 계산하는 방식**이며(진입 트리거·손절 기준선용), 참고용 산식 보존입니다. **전략 파일은 삭제되어 추후 재작성 예정**이며, 재작성 시 이 산식을 사용할 수 있습니다. **지표(`advanced-ichimoku.pine`)에서는 손익비 행 제거와 함께 이 값 계산·`스윕/청산` 입력 그룹이 완전히 제거**되었습니다(**차트 페인팅도 이미 제거** — 청산선의 차트 표시는 [Auto VWAP](../VWAP/README.md)에서 동일 산식으로 제공).

| 모드 | 가중(앵커) | 특징 |
| --- | --- | --- |
| `기본` | 가중 없음 (`highest/lowest` 룩백) | 표준 샹들리에 |
| `거래량 가중` | `VWMA(고가/저가)` | 거래량 몰린 봉을 더 반영 → 거래 집중 구간에서 청산가↑ |
| `최근봉 가중` | `WMA(고가/저가)` | 최근봉을 더 반영(이평 최근봉 가중) → **휩쏘 급등/급락 봉에서 청산가를 빠르게·높게** |
| `거래량+최근봉` | 거래량 × 선형 최근봉 가중 | 두 가중을 함께 |

- `기본`은 룩백 극값(`highest/lowest`, 현재봉 포함) 기준, 가중 모드(거래량/최근봉/거래량+최근봉)는 `가중 트레일 길이`·`가중 트레일 ATR 배수`로 계산한 **가중평균 앵커**를 사용합니다(모드마다 청산가 위치가 실제로 달라짐). **가중 모드는 형성 중인 현재봉을 제외하고 직전 봉 기준(`[1]`)으로 앵커를 잡아** 봉 내 반복 변동(리페인트)을 줄이고, 라인·테이블에 그 값을 표시합니다. 가중 모드는 짧은 길이·타이트 배수라 최근/거래량 변화에 더 빠르게 반응합니다.
- **(참고) 구 전략 `스윕/청산` 옵션 기본값**(지표에서는 이 그룹 전체 제거, 전략 파일은 삭제·재작성 예정): `스윕 룩백`(22)·`스윕 ATR`(22)·`스윕 ATR 배수`(3.0)·`청산 스윙 풀 반영`(ON)·`스윙 풀 좌우 봉수`(5)·`청산-스윕 최대 간격 ATR 배수`(1.0)·`스윕 가격 모드`=`기본`·`가중 트레일 길이`(10)·`가중 트레일 ATR 배수`=`1.5`·`청산 버퍼 ATR 배수`=`0.4`.
  - **※ 페인팅 옵션(`트레일 길이`·`스윕·청산선 표시`·선 두께·선 색·`롱/숏 청산선 색`)은 페인팅 제거와 함께 삭제**됨. 청산선 페인팅 옵션은 [Auto VWAP](../VWAP/README.md)에 이식됨(`청산선 표시`·`청산 선 두께`·`롱/숏 청산선 색`).
- **타이트해지는 만큼 되돌림에 조기 청산이 늘 수 있음** — 스파이크에서만 물리게 하려면 `가중 트레일 ATR 배수`↑.

## 캔들 상태 (3) — 최근 3봉 7분류(표시 전용)

우측 상단 테이블 **[캔들 및 되돌림 확인] 섹션 맨 앞 행(비정상 매수/매도 바로 위)** 에 최근 3봉을 **7분류 스코어링**으로 분류해 표시합니다. 좌열 라벨은 `캔들 상태 (3)`, 우열 셀 값은 **`분류명 | 근거패턴`** 형식입니다(근거 패턴이 특정되지 않으면 분류명만). `캔들 색상 표시` 토글(`candleColorOn`, 기본 OFF)을 켜면 **셀색과 동일한 색으로 현재봉 캔들 바디도 칠합니다**(방향 있는 6개 분류만, 혼조는 원래 봉색 유지 — [기본 표시값](#기본-표시값) 표의 `캔들 상태 바색` 참조).

**7분류**(약세→강세 순): `하락 진행` · `추세형 하락 예측` · `반전형 하락 예측` · `혼조` · `반전형 상승 예측` · `추세형 상승 예측` · `상승 진행`.

- **진행(momentum)**: 3봉이 고점·저점을 한 방향으로 쌓고(HH/HL 또는 LL/LH) **순몸통(netBody)** 이 임계 이상이면 추세 **지속**으로 판정. 적삼병/흑삼병은 이 진행의 강한 형태(3봉 모두 같은 방향)로, 근거 패턴명으로 병기됩니다.
- **반전형 예측(reversal)**: 직전이 하락/상승이던 흐름에서 **반전** 패턴(장악형·망치/역망치·교수형/유성·샛별/석별·**반전강봉**)이 나오면 방향 **반전을 예측**. **반전형 예측이 진행보다 우선**합니다(추세 꺾임을 먼저 알림). **반전강봉**은 꼬리형(작은몸통)과 강몸통 폴백 사이 **중간몸통 사각지대**를 메우는 항으로, 선행추세 반대 방향의 중간~강 몸통봉이 직전 봉 극점을 하회/상회 마감할 때(예: 상승세 끝 큰 음봉→반전형 하락) 성립합니다.
- **추세형 예측(trend)**: 선행 추세 방향으로 **2봉 극점(고/저) 돌파형** 강봉이 나오면 그 추세 **방향 지속**을 예측. **진행보다 뒤 순위**(진행에 안 걸린 강한 돌파봉을 잡음). 여기에 안 걸려도 **강몸통+순몸통 뚜렷한 방향봉**은 최하단 **강몸통 방향 폴백**으로 추세형 예측에 흡수됩니다(혼조 직전, 근거 패턴 `강몸통`).
- 두 예측 모두 **선행추세 게이트는 종가 전용이 아니라 고점/저점 시도까지 병용**(`priorUpX`/`priorDownX`)해, 종가는 눌렸어도 고점(또는 저점)이 직전 봉을 넘어선 봉에서도 놓치지 않습니다.
- **혼조**: 데이터 부족·3봉 총폭 부족(변동성 없음)·어느 분류에도 안 걸림(도지연속 등 흡수).

- **봉 정의**: `b0`=현재봉·`b1`=직전봉·`b2`=2봉 전. `확정봉 기준`(**기본 OFF=실시간, 리페인팅 감수**)을 켜면 `b0`=마지막 **완성 봉**(offset 1)으로 계산해 **리페인팅을 막고**, 끄면(기본) 형성 중인 현재봉(offset 0)을 `b0`으로 써서 실시간 반영되지만 봉 완성 전까지 값이 바뀔 수 있습니다.
- **정규화**: 몸통·꼬리는 캔들 범위(`high-low`, 0 방어) 대비 비율로, 순몸통·3봉 총폭은 `ATR(candleAtrLen=14)` 대비로 정규화해 심볼·가격대와 무관하게 판정합니다.

### 스코어링 파생값

`b0`=현재봉·`b1`=직전봉·`b2`=2봉 전. `body_i = close - open`(부호 있음), `dir_i = sign(body_i)`(+1/-1/0).

| 파생 | 정의 |
| --- | --- |
| `HH` / `HL` | 고점 계단상승 `h0>h1>h2` / 저점 계단상승 `l0>l1>l2` |
| `LH` / `LL` | 고점 계단하락 `h0<h1<h2` / 저점 계단하락 `l0<l1<l2` |
| `netBody` | `(body0+body1+body2) / ATR` — 3봉 순몸통(방향·강도, ATR 정규화·0 방어) |
| `upCount` / `dnCount` | 3봉 중 양봉 수 / 음봉 수 |
| `range3` | `(max(h0,h1,h2) − min(l0,l1,l2)) / ATR` — 3봉 총폭 |
| `priorDownX` / `priorUpX` | `(c1<c2) or (l1<l2)` / `(c1>c2) or (h1>h2)` — 직전 흐름(종가 **또는** 저점/고점 시도까지 병용). **추세형 예측(2봉 돌파)에만 사용** |
| `priorDownExcl` / `priorUpExcl` | **반전형 전용 배타 게이트**. `priorNet = ((c1>c2)+(h1>h2)) − ((c1<c2)+(l1<l2))`의 부호로 선행추세 방향을 **하나로 확정** — `priorUpExcl = priorNet>0`, `priorDownExcl = priorNet<0`(동점 0=불명확→반전형 안 뜸). 아래 [반전형 게이트 배타화(버그 수정)](#반전형-게이트-배타화-반전형-하락-예측-미발화-버그-수정) 참조 |

**긴 꼬리 파생**: `longLowerWick = (아래꼬리/range)≥candleLongWickRatio and (윗꼬리/range)≤(1−candleLongWickRatio)` · `longUpperWick`은 위/아래 대칭. 망치·역망치·교수형·유성 공통 부품.

**반전형 패턴**(반전형 예측 근거): 상승장악 `dir1==-1 and dir0==1 and c0≥o1 and o0≤c1`(하락장악 대칭) · **망치** `priorDownExcl and 작은몸통 + longLowerWick`(강세) · **역망치** `priorDownExcl and 작은몸통 + longUpperWick`(강세) · **교수형** `priorUpExcl and 작은몸통 + longLowerWick`(약세) · **유성** `priorUpExcl and 작은몸통 + longUpperWick`(약세) · 샛별 `dir2==-1 and b2강봉 and b1작은몸통 and dir0==1 and c0≥mid2`(석별 대칭) · **반전강봉(하락)** `priorUpExcl and dir0==-1 and br0≥candleRevBodyRatio and c0<l1`(반전강봉(상승) 대칭: `priorDownExcl and dir0==1 and br0≥candleRevBodyRatio and c0>h1`) — 중간몸통 반전, 직전 봉 극점 하회/상회 마감으로 방향 확증(장악형보다 완화). **반전형은 모두 배타 게이트(`priorUpExcl`/`priorDownExcl`)로 방향을 확정**해, 위꼬리 반전형(유성/역망치)·아래꼬리 반전형(망치/교수형)이 동시에 켜지지 않게 한다(아래 버그 수정 참조).

**추세형 패턴**(추세형 예측 근거): **2봉 고점돌파** `dir0==1 and h0>h1 and h0>h2 and (윗꼬리/range)≤candleShortWickRatio and b0강몸통`(2봉 저점돌파 대칭: `dir0==-1 and l0<l1 and l0<l2 and (아래꼬리/range)≤candleShortWickRatio and b0강몸통`).

> **2봉 극점 돌파형**은 CEO 실전형 케이스입니다 — 직전 **2봉의 고점(또는 저점)을 모두 돌파**하고 **반대쪽 꼬리가 짧아**(강한 마감) **강한 몸통**으로 닫힌 봉입니다. 이전에는 반전 근거로 묶였으나, **선행 추세와 같은 방향의 강한 돌파**라는 성격에 맞춰 **추세형 예측**으로 재분류했습니다(진행 판정 뒤 순위 — 진행 스코어에 안 걸린 강한 돌파봉을 흡수).

### 판정 순서 (상호배타, 첫 매칭 확정)

`if/else if` 체인이라 한 봉에 **한 분류만** 나옵니다. 순위는 **반전형 예측 → 진행 → 추세형 예측**입니다(반전형은 추세 꺾임이라 먼저, 추세형 돌파는 진행에 안 걸린 강봉을 뒤에서 흡수).

| 순서 | 조건 | 분류 |
| --- | --- | --- |
| 1 | `not ready` **또는** `range3 < candleMinRange3Atr` | 혼조 |
| 2 | `priorDownExcl` AND (샛별 OR 상승장악 OR 망치 OR 역망치 OR 반전강봉↑) | 반전형 상승 예측 |
| 3 | `priorUpExcl` AND (석별 OR 하락장악 OR 유성 OR 교수형 OR 반전강봉↓) | 반전형 하락 예측 |
| 4 | `(HH and HL)·upCount≥2·netBody≥candleNetBodyMin` **또는** `(HH or HL)·upCount==3·netBody≥candleNetBodyStrong` | 상승 진행 |
| 5 | `(LL and LH)·dnCount≥2·netBody≤−candleNetBodyMin` **또는** `(LL or LH)·dnCount==3·netBody≤−candleNetBodyStrong` | 하락 진행 |
| 6 | `priorUpX` AND 2봉고점돌파 | 추세형 상승 예측 |
| 7 | `priorDownX` AND 2봉저점돌파 | 추세형 하락 예측 |
| 8 | 강몸통 방향 폴백: `dir0==1`·`br0≥candleStrongBodyRatio`·`netBody≥candleNetBodyMin` | 추세형 상승 예측 |
| 9 | 강몸통 방향 폴백: `dir0==-1`·`br0≥candleStrongBodyRatio`·`netBody≤−candleNetBodyMin` | 추세형 하락 예측 |
| 10 | 위 미매칭 | 혼조 |

- **강몸통 방향 폴백(순위 최하단, 혼조 직전)**: 2봉 극점 돌파(6·7)·3봉 계단 진행(4·5)에 걸리지 않았지만 **현재봉(b0)이 강몸통(`br0≥candleStrongBodyRatio`)이고 3봉 순몸통(`netBody`)이 뚜렷한** 명백한 강방향봉을 추세형 예측으로 흡수합니다. 저점 미갱신 강음봉(대칭: 고점 미갱신 강양봉)이 else 혼조로 새던 문제를 보완합니다. `dir0` 부호 배타로 상승/하락 폴백은 동시 성립하지 않습니다. **신규 파라미터 없음**(기존 `candleStrongBodyRatio`·`candleNetBodyMin` 재사용).
- 진행 분류에서 3봉이 **모두 같은 방향**(`upCount==3`/`dnCount==3`)이면 근거 패턴을 `적삼병`/`흑삼병`으로 병기합니다. 반전형 예측은 성립 패턴명(`샛별`/`석별`/`장악형`/`망치`/`역망치`/`유성`/`교수형`/`반전강봉`)을, 추세형 예측은 `2봉고점돌파`/`2봉저점돌파`(폴백은 `강몸통`)를 병기합니다(반전형 우선순위: 샛별·석별 → 장악형 → 망치·역망치 / 유성·교수형 → 반전강봉).
- **반전강봉(반전형 최하단 항)**: 꼬리형 반전(`br0≤candleSmallBodyRatio`)과 강몸통 방향 폴백(`br0≥candleStrongBodyRatio`) 사이 **중간몸통(0.45~)** 사각지대에서, `priorUpExcl`라 추세형·진행에 안 걸리고 else 혼조로 새던 **상승세 끝 중간~강 음봉**(대칭: 하락세 끝 양봉)을 반전형으로 흡수합니다. `c0<l1`(대칭 `c0>h1`)로 방향 확증. `dir0` 부호로 상승/하락 배타이며, `c0<l1`가 `HL`(`l0>l1`)과, `dir0==-1`가 `upCount==3`과 모순이라 **기존 상승 진행 케이스를 가로채지 않습니다**. 신규 노브 `candleRevBodyRatio`(0.45).
- **이전 12분류의 4중 AND 브리틀 게이트(적삼병 오분류)는 폐기**했습니다. 예전 적삼병은 "강한 몸통≥0.60 AND 짧은 윗꼬리 AND 종가 계단상승 AND 3봉 모두"를 전부 요구해 정상 강양봉 3연속도 혼조로 떨어졌습니다. 진행은 구조(HH/HL)+순몸통 스코어로 로버스트하게 잡습니다.

#### 반전형 게이트 배타화 — "반전형 하락 예측" 미발화 버그 수정

**증상**: 위꼬리가 긴 반전 캔들(선행 상승세 끝 유성형)에서 **"반전형 하락 예측"이 뜨지 않았습니다.** "반전형 상승 예측"은 정상 발화.

**원인**: 유성(`priorUpX`)과 역망치(`priorDownX`)는 몸통 형태(`작은몸통 + longUpperWick`)가 **완전히 동일**하고 선행추세 게이트만 다릅니다. 그런데 `priorUpX = (c1>c2) or (h1>h2)`와 `priorDownX = (c1<c2) or (l1<l2)`는 OR가 느슨해 **동시 참**이 흔합니다(예: 종가는 올랐지만 저점을 갱신한 흔들림 봉, 외봉). 동시 참이면 위꼬리 봉에서 역망치·유성이 **둘 다 성립**하고, 판정순서가 `반전형 상승(2) → 반전형 하락(3)`이라 **항상 상승(역망치)이 하락(유성)을 선점** → 하락이 구조적으로 미발화. 아래꼬리 봉(망치/교수형)도 같은 편향으로 항상 상승(망치)이 이겨, 결과적으로 **상승 반전은 정상·하락 반전만 죽는** 비대칭이 나타났습니다.

**수정**: 반전형 전용 배타 게이트 `priorUpExcl`/`priorDownExcl`(선행추세 강도 스코어 `priorNet`의 부호)로 방향을 **하나로 확정**했습니다. `priorNet=0`(상승·하락 신호 동수, 방향 불명확)이면 반전형을 열지 않습니다. 이제 revUp·revDn이 동시에 참일 수 없어 판정순서 선점 편향이 사라집니다. **판정 순서는 그대로 두었습니다**(순서를 뒤집으면 편향 방향만 반대로 이동할 뿐이라 근본 해결이 아님 — 게이트 배타화가 근본 처방).

**무회귀**: `priorDownExcl ⊂ priorDownX`(부분집합)이라 상승 반전 발화가 **늘지 않습니다.** 유일한 변화는 `priorNet=0`인 방향 불명확 봉에서 과거에 편향으로 뜨던 상승/하락 반전이 이제 안 뜨는 것(거짓양성 제거). 추세형·진행 분류는 기존 `priorUpX`/`priorDownX`를 그대로 써 **전혀 영향받지 않습니다.**

### 셀색 7분류 범례

방향 있는 6개 분류는 **현재봉 캔들 바디 색과 동일**합니다(혼조는 캔들 색칠 안 함 → 원래 봉색). 단, 캔들 색칠은 `캔들 색상 표시` 토글(`candleColorOn`, 기본 OFF)이 켜진 경우에만 적용됩니다.

| 색 | 분류 |
| --- | --- |
| 선명 녹색 `color.new(color.green, 0)` | 상승 진행 |
| 둔한 녹색 `color.new(#6E8F72, 0)` | 추세형 상승 예측 |
| 청록 `color.new(#26A69A, 0)` | 반전형 상승 예측 |
| 회색 `color.new(color.gray, 40)` | 혼조 |
| 밝은 빨강 `color.new(#EF5350, 0)` | 반전형 하락 예측 |
| 둔한 빨강 `color.new(#A57A7A, 0)` | 추세형 하락 예측 |
| 선명 빨강 `color.new(color.red, 0)` | 하락 진행 |

### 캔들 색 표시 — 캔들 상태 7분류 기준(`캔들 색상 표시` 토글, 기본 OFF)

캔들 상태 (3) 7분류로 **현재봉 캔들 바디를 방향색으로 칠합니다**(`barcolor`). **캔들 색은 캔들 상태 7분류 방향색(`candleBarColor`)을 그대로 표시**합니다 — 캔들 상태가 방향 있는 계열(상승/하락)이면 그 색, **혼조**(방향 없음)면 색 없음(`na`). 표시 여부는 **`캔들 색상 표시` 토글(`candleColorOn`, 기본 OFF)** 이 켜져야만 색칠합니다. **OFF(기본)면 `na`를 넘겨 캔들은 차트 기본색을 유지**합니다. 이 토글은 캔들 색칠만 제어하며 **테이블의 `캔들 상태 (3)` 행 표시(`candleShowState`)와는 독립**입니다.

> **표시 전용**입니다 — 캔들 색은 캔들 바색을 **"칠하는 것"만** 담당합니다. **캔들 7분류 판정·바색 색값·테이블 셀 색**(캔들 상태 (3) 행·캔들 손절라인 행)·**진입/청산·신호·전략 변수** 어디에도 연결되지 않습니다.

**방향 매핑**: 상승 계열(상승 진행·추세형 상승 예측·반전형 상승 예측)=상승색, 하락 계열(하락 진행·추세형 하락 예측·반전형 하락 예측)=하락색. 혼조는 색 없음.

### ⚠️ 리스크 고지

- **표시 전용입니다.** 어떤 진입/청산/신호 변수에도 연결되지 않으며, 기존 매수/매도 신호 로직과 독립입니다. 순수 관찰용 정보 행입니다.
- **단일/소수 캔들 패턴은 통계적으로 약한 엣지**입니다. 캔들 패턴만으로 진입하지 말고, 반드시 **이치모쿠 메인 + 테이블 + 보조지표 정렬과 교차확인**하세요(펀드 철학: 단일 신호 불신).
- **임계값·예측 방향은 파라미터 후보(제안)** 입니다. 알고리즘 수학자의 백테스트 검증 전에는 관찰용으로만 쓰고, 실전 전략에 반영하지 마세요. `candleNetBodyMin`이 진행 판정의 핵심 튜닝 노브입니다. (임계·기간·예측 방향 변경은 전략 변경에 해당하며 본부장·알고리즘 수학자 검토가 필요합니다.)
- **갭 기반 패턴(샛별/석별)은 24시간 거래되는 코인·선물에서 갭이 거의 없어** 근접 조건(현 구현은 `b2` 몸통중앙 회복/이탈로 완화) 추가 조정이 필요할 수 있습니다.

## 수축/확장 (휩쏘 필터) — 이평 리본 / 구름 두께 / 리본+구름 모드

> **⚠️ 리본 추세 화면 표시(캔들 `+` 표식·테이블 `리본 추세` 행·리본 알림)는 CEO 지시로 제거됐고, 리본 계산부(7개 이평·수축/분출·상태머신)도 성능 리팩토링 ④에서 사멸코드로 제거됐습니다.** 아래 설명은 **제거 이전의 리본 판정 로직**에 대한 히스토리 참고 서술입니다 — 현재 코드엔 구름 지지/저항이 재사용하는 `rAtr`(=`ta.atr`)만 남아 있고, 리본 계산·화면·테이블은 존재하지 않습니다. 리본 입력 그룹(`수축/확장 판정 모드` 등)은 옵션 패널에만 잔존(표시·계산 미연결).

(구) 이평 수축↔확장을 캔들에 **`+` 모양(플러스)**으로 표시했습니다 — `횡보 압축`=보라 `#9C27B0`(상승·하락과 확실히 구분), `상승 분출`=파랑, `하락 분출`=빨강. `+`표식은 **신호 삼각형보다 뒤(아래 레이어)** 에 그려졌습니다. **판정 기준을 3가지 모드로 선택**합니다(기본 `리본+구름`).

| 모드 | 수축/확장 판정 기준 | 분출 방향 |
| --- | --- | --- |
| `이평 리본` | **입력값 기반 이평 리본**(전환선~리딩B 길이, 7개 MA)의 폭이 ATR 대비 좁으면 수축 | 빠른선(전환선 길이) ≥/< 느린선(리딩B 길이) |
| `구름 두께` | **구름 두께**(선행스팬 A·B 간격)가 ATR 대비 좁으면 수축 | 양운(A≥B)=상승 / 음운=하락 |
| `리본+구름`(기본) | 리본·구름이 **모두** 수축/확장(방향 일치)할 때만 | 두 기준 방향이 같을 때만 분출 |

- **`이평 리본`이 이치모쿠와 정합**: 리본 길이를 `전환선~리딩B`(기본 8~44)에 자동 연동해, 신호와 같은 시간축에서 수축/확장을 판단합니다. 기간 입력을 바꾸면 리본 범위도 따라갑니다.
- **`구름 두께`는 이치모쿠 본연**: 구름 두께 자체가 압축/확장 측정이라 추가 라인 없이 정합적입니다.

| 항목 | 내용 |
| --- | --- |
| 상태머신 | 압축 후 분출(같은 방향 반복은 재압축 필요). **단, 직전 분출과 반대 방향 확장이면 압축 없이 바로 반전 분출**(상승 분출 → 하락 분출, 그 반대도 가능). `리본+구름`은 방향이 엇갈리면 분출 보류(무장 유지) |
| 분출 속도 | 폭이 임계선을 완전히 넘기 전이라도 잡아 빠름. `확장 판정 폭 증가 봉수`↑ = 느리고 확실, 1 = 가장 빠름 |

| 옵션(`리본 추세(수축·분출)`) | `수축/확장 판정 모드`(리본+구름 기본)·`리본 이동평균 종류(A)`(SMA)·`횡보 압축 표시`·`분출 표시`·`봉마감 후 확정`·`폭 ATR 기준`(14)·`수축 판정 ATR 배수`(0.8)·`확장 판정 폭 증가 봉수`(1) — **⚠️ 리본 표시·계산부 제거로 이 옵션들은 현재 화면·계산 어디에도 연결되지 않음(패널 UI만 잔존)** |

## 옵션 그룹

> **옵션 패널 순서 = 이치모쿠 상태 테이블 3섹션 순서에 맞춰 배치** — 트레이더가 테이블을 읽는 흐름(추세 확인 → 캔들 및 되돌림 확인 → 지지/저항 확인)과 옵션 패널 그룹 순서를 일치시켜 설정을 빠르게 찾도록 했습니다. (구 4번째 섹션 [손익비 현황 확인]은 CEO 지시로 제거됨.) Pine 옵션 패널은 `input()` **선언 순서**로 그려지므로, `input` 선언(계산 로직은 원위치 유지)을 이 순서로 재배치했습니다. 아래 표의 나열 순서 = 실제 패널 순서. `섹션` 열은 그 그룹이 대응하는 테이블 섹션입니다.
