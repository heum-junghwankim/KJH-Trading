# PineScript 트레이딩 가이드

이 폴더는 현재 저장소에 있는 Pine Script 지표를 `같은 해석 흐름`으로 읽기 위한 상위 안내서입니다.

지금 기준 메인 축은 [`ICHIMOKU`](./ICHIMOKU/README.md)이고, 나머지 지표는 그 판단을 보완하는 역할입니다. 이치모쿠는 그 자체로 VWAP·장기추세·이격도·비정상 캔들·리본을 한 오버레이 안에 통합한 종합 지표이며, 아래 보조 지표들은 그 판단을 다른 각도에서 교차 확인하는 데 씁니다.

## 한눈에 보기

| 지표 | 한 줄 역할 |
| --- | --- |
| [`ICHIMOKU`](./ICHIMOKU/README.md) | 메인 축. 자리·추세·구름·이격도·스윕/청산을 종합해서 봅니다. |
| [`Auto VWAP`](./VWAP/README.md) | 평균 체결 단가와 확장 구간을 봅니다. |
| [`거래량 압력 추적`](./VOLUME-PRESSURE/README.md) | 현재 봉에 실제 압력이 붙는지 봅니다. |
| [`MACD 다이버전스 추적`](./MACD/README.md) | 내부 모멘텀이 다시 붙는지 봅니다. |
| [`CCI 익스트림 크로스 시그널`](./CCI/README.md) | 극단값 이후 재진입 크로스를 봅니다. |
| [`OBV ADX 극단값 배경`](./OBV-ADX/README.md) | OBV 기반 낙폭과대와 과매수를 배경으로 봅니다. |
| [`이평선 리본`](./MA-RIBBON/README.md) | 이동평균 정렬 구조로 추세 방향을 봅니다. |

짧게 줄이면:

- `이치모쿠`는 메인 축(자리·추세·구름)
- `VWAP`은 기준 단가
- `거래량`은 압력
- `MACD`는 힘 재개
- `CCI`는 극단값 뒤 재진입
- `OBV ADX`는 OBV 기반 극단값 배경
- `리본`은 이동평균 정렬 구조

## 추천 읽는 순서

1. [`ICHIMOKU`](./ICHIMOKU/README.md)로 현재 자리·추세·구름과 스윕/청산 흐름을 먼저 봅니다.
2. [`Auto VWAP`](./VWAP/README.md)으로 가격이 기준 단가 위인지 아래인지 확인합니다.
3. [`거래량 압력 추적`](./VOLUME-PRESSURE/README.md)으로 실제 압력이 같은 방향인지 확인합니다.
4. [`CCI 익스트림 크로스 시그널`](./CCI/README.md)로 과열/과매도 뒤 재진입이 나오는지 봅니다.
5. [`MACD 다이버전스 추적`](./MACD/README.md)으로 모멘텀 재개 여부를 봅니다.
6. [`OBV ADX 극단값 배경`](./OBV-ADX/README.md)으로 OBV 기준 낙폭과대 / 과매수 구간을 함께 확인합니다.
7. [`이평선 리본`](./MA-RIBBON/README.md)으로 이동평균 정렬 구조가 방향과 맞는지 큰 그림을 봅니다.

## 가장 단순한 해석

| 상황 | 보통 이렇게 읽습니다 |
| --- | --- |
| 롱 쪽 | 이치모쿠 자리·추세 정렬 + VWAP 회복 + 매수 압력 + MACD 재가속 + 반대 신호 약함 |
| 숏 쪽 | 이치모쿠 자리·추세 정렬 + VWAP 재이탈 + 매도 압력 + MACD 재하락 + 반대 신호 약함 |
| 관망 | 자리, 압력, 모멘텀 중 하나라도 비어 있거나 서로 충돌 |

## 관망이 더 나은 경우

- 메인 이치모쿠 신호는 좋은데 거래량이 약할 때
- VWAP을 중심으로 계속 왕복할 때
- MACD가 아직 애매한데 먼저 진입하고 싶어질 때
- 한두 지표만 좋고 나머지가 비어 있을 때

핵심은 `좋은 진입을 더 찾는 것`보다 `나쁜 진입을 피하는 것`입니다.

## 마지막 원칙

- 메인 지표보다 먼저 보조 지표로 진입하지 않기
- 거래량 없이 모양만 좋은 신호는 한 번 더 의심하기
- 기준 단가와 반대 방향으로 무리하게 추격하지 않기
- 확정봉 기준 신호와 봉중 반응 신호를 구분해서 보기
