# Advanced Ichimoku

트레이딩뷰용 Pine Script Ichimoku 확장 버전입니다. 정통 이치모쿠 계산 위에 **신호·후보·트리거 테이블·휩쏘 필터·백테스트 전략**을 얹었습니다.

| 스크립트 | 용도 |
| --- | --- |
| [`advanced-ichimoku.pine`](./advanced-ichimoku.pine) | 인디케이터 — 신호/후보 마커 + 우측 하단 테이블 |
| [`advanced-ichimoku-strategy.pine`](./advanced-ichimoku-strategy.pine) | 전략 — 동일 로직 + 분할 진입/샹들리에 청산 백테스트 |

## 핵심 기능

| 기능 | 설명 |
| --- | --- |
| 매수/매도 신호 | 래깅 스팬이 기준선 돌파 **AND** 컨버전 라인이 기준선 돌파 |
| 매수/매도 후보 | 신호보다 빠른 조기 경고 (래깅이 *컨버전 라인* 돌파 + 컨버전이 기준선 돌파) |
| 거래량 가중(VWMA) 마커 | 표준 신호와 함께, 컨버전·기준선을 VWMA로 바꾼 버전의 후보/신호를 **반투명**으로 동시 표시 |
| 신호 순서 제한 | 같은 방향 신호는 반대 신호가 한 번 나온 뒤에만 다시 발생(중복 방지) |
| 우측 하단 테이블 | `상승/하락 전환` 트리거 가격 + `구름대`·`이평 리본` 상태 + `롱/숏 청산`가 (인디케이터·전략 공통) |
| 리본 압축/분출 | 20~50 이평 리본의 `횡보 압축 → 상승/하락 분출`을 다이아몬드로 표시(휩쏘 필터) |
| (전략) 분할 진입/청산 | 신호당 자본 5% 분할 진입, 샹들리에 청산가 크로스 시 전량 청산 |

## 기본 기간값 (인디케이터·전략 공통)

| 항목 | 값 | 비고 |
| --- | --- | --- |
| 컨버전 라인 길이 | 8 | 전환선 = `(8봉 최고+최저)/2` (돈키언 중간값) |
| 베이스 라인 길이 | 22 | 기준선 = `(22봉 최고+최저)/2` |
| 리딩 스팬 B 길이 | 44 | 선행 스팬 B |
| 레깅 스팬 길이 (displacement) | 22 | 후행스팬·구름 오프셋 = `displacement-1 = 21` |

| 횡보 압축 ◆ | 다이아몬드 · 캔들 **위·아래 모두** · 회색 |

## 신호 / 후보 로직

`래깅 스팬 = 현재 종가(close)`. 차트상 래깅 스팬이 닿는 위치의 라인은 `[displacement-1]`(=21봉 전) 값으로 비교합니다.

| 구분 | 조건① (래깅 스팬) | 조건② (컨버전 라인) | 추가 조건 |
| --- | --- | --- | --- |
| 매수 신호 | `close > baseLine[21]` 또는 상향 크로스 | `conversionLine > baseLine` 또는 골든크로스 | 중복 방지 |
| 매도 신호 | `close < baseLine[21]` 또는 하향 크로스 | `conversionLine < baseLine` 또는 데드크로스 | 중복 방지 |
| 매수 후보 | `close > conversionLine[21]` 또는 상향 크로스 | `conversionLine > baseLine` | 모멘텀 `close ≥ close[1]` · 신호봉/중복 제외 |
| 매도 후보 | `close < conversionLine[21]` 또는 하향 크로스 | `conversionLine < baseLine` | 모멘텀 `close ≤ close[1]` · 신호봉/중복 제외 |

- **신호 vs 후보의 차이는 조건①뿐**입니다(신호=래깅 vs 기준선, 후보=래깅 vs 컨버전). 컨버전(8)이 기준선(22)보다 빨라 보통 **후보 → 신호** 순으로 진행됩니다.
- **신호 순서 제한**: 매도 신호 후에만 매수 신호가 다시, 매수 신호 후에만 매도 신호가 다시 나옵니다. 같은 방향이 유지돼도 첫 봉만 1회.

## 거래량 가중(VWMA) 마커

모드 선택이 아니라 표준 신호와 **동시에** 표시됩니다. 컨버전·기준선만 `VWMA = Σ(종가×거래량)/Σ거래량` 으로 바꾼 버전의 **후보/신호 마커만** 반투명으로 그립니다(라인은 안 그림).

| 항목 | 내용 |
| --- | --- |
| 계산 | 컨버전·기준선을 VWMA로 대체, 그 외 로직은 표준과 동일(후행스팬은 실제 종가) |
| 표시 | 후보=원·신호=삼각형, 기본 투명도 50% |
| 옵션 | `거래량 가중 신호/후보 표시`(ON), `거래량 가중 투명도`(50) |
| 장점 | 거래량 실린 가격대 반영 → 저거래량 가짜 움직임·휩쏘에 덜 흔들림 |
| 단점 | 거래량 없는/부정확 심볼(일부 FX·지수)에선 `na`/왜곡 · 거래량 스파이크 1봉에 과반응 가능 |
| 활용 | 표준 마커와 **같은 자리 겹치면 신뢰↑**, 거래량만 먼저 뜨면 수급 조기 참고 |

## 우측 하단 테이블 (인디케이터·전략 공통)

차트 **우측 하단**에 7행 표를 띄웁니다(`barstate.islast`).

| 행 | 값 | 셀 색 |
| --- | --- | --- |
| `상승 전환` | 상승 후보·신호 트리거 중 **더 낮은 값**(먼저 닿는 상승 트리거) `↑/↓ 가격` | 초록 |
| `하락 전환` | 하락 후보·신호 트리거 중 **더 높은 값**(먼저 닿는 하락 트리거) `↑/↓ 가격` | 빨강 |
| `구름대` | `위/안/아래·양운/음운 (두께÷ATR)` | 위=초록·아래=빨강·안=회색 |
| `이평 리본` | `횡보 압축/상승 분출/하락 분출 (리본폭÷ATR)` | 압축=회색·상승=파랑·하락=빨강 |
| `롱 청산` | 샹들리에 롱 청산가 `가격 (현재가 대비 %)` — 예 `68,500 (-2.1%)` | 회색 |
| `숏 청산` | 샹들리에 숏 청산가 `가격 (현재가 대비 %)` — 예 `72,000 (+3.0%)` | 회색 |

부가 설명:

| 항목 | 내용 |
| --- | --- |
| 전환 = 후보·신호 결합 | 두 트리거를 한 행으로: 상승은 둘 중 더 낮은(먼저 닿는) 값, 하락은 더 높은 값. 차트 후보 원·신호 삼각형은 그대로 유지 |
| 트리거 = 두 조건 결합가 | 신호 트리거 = `max/min(래깅 기준가, 전환선 전환가)`. 후보는 래깅 기준가를 컨버전 라인으로 + 모멘텀(현재 종가 바닥/천장) 반영 |
| 래깅 기준가 | 다음 봉 래깅 스팬이 닿는 `baseLine[displacement-2]`(신호) / `conversionLine[displacement-2]`(후보) |
| 전환선 전환가 | 다음 봉에 기준선·전환선이 같이 변함 → 상승 `2×다음봉기준선 − 최근(컨버전-1)봉 최저`, 하락 `… − 최고` |
| 화살표 ↑/↓ | 현재가(`close`) 대비 트리거 가격 위치(높으면 ↑, 낮으면 ↓) |
| `-` 표기 | 직전에 같은 방향 신호가 나온 상태(중복 방지)면 그 방향 전환 칸은 `-` |
| 구름대 평가 시점 | 구름은 displacement만큼 앞으로 밀리므로 현재 봉 아래 구름 = `[displacement-1]` 과거 선행스팬으로 평가 |
| 리본 상태 동기화 | 마지막으로 발생한 다이아몬드(상태머신) 기준이라 차트 마커와 항상 일치 |
| 롱/숏 청산가 | 샹들리에 ATR 트레일링(차트엔 안 그림, 표에만). 가격 옆 괄호는 **현재 종가 대비 %**(롱 청산은 아래라 −, 숏 청산은 위라 +). 옵션 `청산(샹들리에)`: 룩백(22)·ATR(22)·배수(3.0) |

## 리본 압축/분출 (휩쏘 필터)

20~50 이동평균 리본의 압축↔분출을 캔들에 다이아몬드로 표시합니다(상태 다이아몬드만, 리본 라인은 안 그림).

| 표시 | 판정 |
| --- | --- |
| `횡보 압축`(회색) | 리본 폭(`MA20~50` 최댓−최솟)이 `ATR × 압축배수`보다 좁아지는 첫 봉 → 다음 분출 무장 |
| `상승 분출`(파랑) | 압축 후 폭이 다시 늘기 시작하는 첫 봉에 `MA20 ≥ MA50` |
| `하락 분출`(빨강) | 압축 후 폭이 다시 늘기 시작하는 첫 봉에 `MA20 < MA50` |

| 항목 | 내용 |
| --- | --- |
| 상태머신 | `압축 → 분출 → 압축 → 분출 …` 순서로만. 분출 후 다시 압축이 와야 다음 분출 허용 |
| 분출 속도 | 폭이 임계선을 완전히 넘기 전이라도 잡아 빠름. `분출 판정 폭 증가 봉수`↑ = 느리고 확실, 1 = 가장 빠름 |
| 활용 | 신호 + 같은 방향 분출 = 신뢰↑ / `횡보 압축` 구간 신호 = 휩쏘 의심·관망. **진입 트리거 아닌 신뢰도 필터** |
| 옵션(`리본 압축/분출`) | `리본 이동평균 종류`(SMA 기본)·`횡보 압축 표시`·`분출 표시`·`봉마감 후 확정`·`리본폭 ATR 기준`(14)·`압축 판정 ATR 배수`(0.8)·`분출 판정 폭 증가 봉수`(1) |

## 전략 진입 / 청산 (`advanced-ichimoku-strategy.pine`)

인디케이터와 동일한 신호/후보 로직 + **분할 진입·샹들리에 청산** 백테스트.

| 항목 | 규칙 |
| --- | --- |
| 포지션 방향 | `롱` 또는 `숏` (한 방향만 거래) |
| 진입 단위 | 신호당 자본 **5%**(`percent_of_equity 5`), **기본·거래량(VWMA) 신호가 각각 5%씩** 누적(`pyramiding=100`) |
| 진입 필터 (`양쪽 청산선` 기본) | 롱은 종가가 롱·숏 청산가 **둘 다 위**, 숏은 **둘 다 아래**일 때만 |
| 진입 필터 (`해당 청산선만`) | 롱은 `longCE` 위, 숏은 `shortCE` 아래일 때만 |
| 진입 필터 (`필터 없음`) | 청산가와 무관하게 신호 시 진입 |
| 청산 (전량) | 롱: 종가가 `longCE` **하향 크로스** → 전량청산 · 숏: 종가가 `shortCE` **상향 크로스** → 전량청산 |
| 청산가 | 샹들리에 ATR 트레일링(테이블 `롱/숏 청산`과 동일). 반대 신호로는 청산하지 않음(청산가 크로스만) |

| 기본 전략 설정 | 값 |
| --- | --- |
| `default_qty_type` / `default_qty_value` | `percent_of_equity` / `5` |
| `pyramiding` | `100` (분할 누적 허용) |
| `initial_capital` | `10000` (※ 한국 주식 등 주가가 크면 5%로 1주도 못 사 거래 0건이 될 수 있음 → 자본 키우기) |
| `margin_long` / `margin_short` | `0` / `0` (증거금 제약 없음) |
| `process_orders_on_close` / `use_bar_magnifier` | `true` / `true` |
| 수수료·슬리피지 | 코드 미가정 → TradingView 전략 속성에서 직접 입력 권장 |

## 적용 방법

1. TradingView Pine Editor에서 스크립트를 엽니다(시그널만 → 인디케이터 / 백테스트 → 전략).
2. 저장 후 차트에 추가합니다.
3. 전략 수익률은 `Strategy Tester` 탭에서, 체결 내역은 `List of Trades` 탭에서 확인합니다.
