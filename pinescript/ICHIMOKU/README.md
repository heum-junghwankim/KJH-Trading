# Advanced Ichimoku

트레이딩뷰용 Pine Script Ichimoku 확장 버전입니다. 정통 이치모쿠 계산 위에 **멀티스케일 신호·트리거 테이블·휩쏘 필터·이격도**를 얹은 인디케이터입니다.

| 스크립트 | 용도 |
| --- | --- |
| [`advanced-ichimoku.pine`](./advanced-ichimoku.pine) | 인디케이터 — 멀티스케일 신호 마커 + 우측 하단 테이블 |

> 백테스트 전략(`advanced-ichimoku-strategy.pine`)은 추후 다시 추가될 예정입니다.

## 핵심 기능

| 기능 | 설명 |
| --- | --- |
| 매수/매도 신호 | 래깅 스팬이 기준선 돌파 **AND** 컨버전 라인이 기준선 돌파 |
| 멀티스케일 표시 | 같은 신호를 래깅 설정값의 **0.5× / 1× / 2×** 스케일로 계산해 **삼각형 크기**(가장 작음/작음/보통)로 동시 표시 |
| 신호 모드 | `확정`/`예측` 선택 — 예측 모드는 다음 봉을 가정해 신호를 1봉 빨리 포착(기본=확정) |
| 거래량 가중(VWMA) 신호 | 일반 신호와 함께, 컨버전·기준선을 VWMA로 바꾼 버전의 신호를 **반투명**으로 동시 표시 |
| 신호 순서 제한 | 같은 방향 신호는 반대 신호가 한 번 나온 뒤에만 다시 발생(중복 방지) |
| 우측 하단 테이블 | `상승/하락 전환` 트리거 가격 + `구름대`·`중기 추세`·`장기 추세`·`이격도`·`되돌림 가격` 상태 + `롱/숏 스윕`·`롱/숏 청산`가 |
| 스윕/청산 2선 | **스윕(점선)** = 샹들리에 트레일(세력 스톱헌트 자리), **청산(직선)** = 그 너머(ATR 버퍼+스윙 풀). 둘 다 트레일+수평선+테이블 표시 |
| 수축/확장 (휩쏘 필터) | `횡보 압축 → 상승/하락 분출`을 **다이아몬드**로 표시(압축=보라·상승=파랑·하락=빨강, **신호 삼각형 뒤 레이어**). 판정 기준 **이평 리본 / 구름 두께 / 리본+구름** 모드 선택(기본 리본+구름) |
| 이격도 | 선택 이평선(기본 SMA 20) 대비 `종가÷MA×100`(기준 100). 고정 임계+적응형(σ) 2기준 중 1개=`되돌림 주의`(둔탁)·2개=`되돌림 확실`(선명)로 테이블 표시 |
| 장기 추세 | 이평선 리본 통합 — **MA 100/200 선 + backfill**. MA 종류 4종(기본 거래량 가중 VWMA), MA100=파랑·MA200=보라, backfill 우상향=파랑·우하향=보라(투명도 60). 테이블에 강/약 추세·골든/데드 크로스 표시 |

## 신호 로직 + 멀티스케일

`래깅 스팬 = 현재 종가(close)`. 차트상 래깅 스팬이 닿는 위치의 라인은 `[displacement-1]` 값으로 비교합니다.

| 구분 | 조건① (래깅 스팬) | 조건② (컨버전 라인) | 추가 조건 |
| --- | --- | --- | --- |
| 매수 신호 | `close > baseLine[disp-1]` 또는 상향 크로스 | `conversionLine > baseLine` 또는 골든크로스 | 중복 방지 |
| 매도 신호 | `close < baseLine[disp-1]` 또는 하향 크로스 | `conversionLine < baseLine` 또는 데드크로스 | 중복 방지 |

- **신호 순서 제한**: 매도 신호 후에만 매수 신호가 다시, 매수 신호 후에만 매도 신호가 다시 나옵니다(스케일·신호 종류별로 독립). 같은 방향이 유지돼도 첫 봉만 1회.

### 멀티스케일 (0.5× / 1× / 2×)

위 신호 로직을 **래깅 설정값(컨버전 8·베이스 22·래깅 22)의 세 배율**로 각각 계산해, 삼각형 **크기**로 어느 시간축에서 나온 신호인지 보여줍니다. (리딩 스팬 B는 신호와 무관하므로 스케일에 안 씀)

| 스케일 | 기간 예시(기본 입력 기준) | 삼각형 크기 |
| --- | --- | --- |
| `0.5×` | 4 · 11 · 11 | 가장 작음 (`tiny`) |
| `1×` | 8 · 22 · 22 | 작음 (`small`) |
| `2×` | 16 · 44 · 44 | 보통 (`normal`) |

- **여러 크기의 삼각형이 같은 봉에 겹치면** 짧은·중간·긴 시간축이 한 방향으로 정렬된 것 → 신뢰도↑.
- 차트엔 **1× 라인/구름만** 그립니다. 0.5×·2×는 신호 계산용입니다.
- 알림(alert)은 **1× 신호** 기준이고, 우측 하단 테이블의 `상승/하락 전환` 트리거 가격은 **0.5× / 1× / 2× 세 스케일을 각각** 표시합니다.

## 신호 모드 (확정 / 예측)

`신호 모드` 입력으로 매수/매도 신호 산출 방식을 고릅니다(세 스케일 모두 같은 모드 적용).

| 모드 | 설명 |
| --- | --- |
| `확정 모드`(기본) | 위 신호 로직 그대로 — 현재 봉 종가로 신호 확정 |
| `예측 모드` | **다음 봉을 가정**해 신호를 1봉 빨리 포착 (조건이 더 빡빡한 모멘텀 게이트 포함) |

예측 모드는 아래 가정/조건으로 신호를 냅니다.

| 항목 | 매수 | 매도 |
| --- | --- | --- |
| 다음 봉 종가 가정 | `(현재 종가 + 고가) / 2` | `(현재 종가 + 저가) / 2` |
| 모멘텀 게이트 | `저가 > 직전 봉 저가` | `고가 < 직전 봉 고가` |
| 래깅 조건 | 가정 종가 > `baseLine[displacement-2]` | 가정 종가 < `baseLine[displacement-2]` |
| 전환선 조건 | `conversionLine > baseLine`(현재 봉 기준선) | `conversionLine < baseLine` |
| 거래량(VWMA) 신호 | **다음 봉 거래량 = 현재 거래량** 가정으로 다음 봉 VWMA 전환·기준선을 재계산해 같은 조건 적용 | 〃 |

- 기본값은 `확정 모드`입니다. 예측 모드는 더 빠른 대신 가정에 기반하므로(확정 전 미리 찍힘) 참고용으로 쓰는 게 안전합니다.

## 거래량 가중(VWMA) 신호

모드 선택이 아니라 일반 신호와 **동시에** 표시됩니다. 컨버전·기준선만 `VWMA = Σ(종가×거래량)/Σ거래량` 으로 바꾼 버전의 **신호 마커만** 반투명으로 그립니다(라인은 안 그림). 세 스케일 모두에 적용됩니다.

| 항목 | 내용 |
| --- | --- |
| 계산 | 컨버전·기준선을 VWMA로 대체, 그 외 로직은 일반과 동일(후행스팬은 실제 종가) |
| 표시 | 일반과 같은 삼각형·크기, 기본 투명도 50% |
| 옵션 | `거래량 가중 신호 표시`(**기본 OFF**), `거래량 가중 투명도`(50) |
| 장점 | 거래량 실린 가격대 반영 → 저거래량 가짜 움직임·휩쏘에 덜 흔들림 |
| 단점 | 거래량 없는/부정확 심볼(일부 FX·지수)에선 `na`/왜곡 · 거래량 스파이크 1봉에 과반응 가능 |
| 활용 | 표준 마커와 **같은 자리 겹치면 신뢰↑**, 거래량만 먼저 뜨면 수급 조기 참고 |

## 우측 하단 테이블

차트 **우측 하단**에 16행 표를 띄웁니다(`barstate.islast`).

| 행 | 값 | 셀 색 |
| --- | --- | --- |
| `상승 전환(0.5×/1×/2×)` | 각 스케일에서 **매수 신호**가 나오려면 다음 봉 종가가 닿아야 할 트리거 가격 `↑/↓ 가격 (현재가 대비 %)` (3행). 트리거가 현재가 **+`근접 강조 임계`(기본 2%) 이내, 또는 이미 현재가 아래(−%)=조건 충족**이면 값 셀 **녹색** 강조 | 초록 |
| `하락 전환(0.5×/1×/2×)` | 각 스케일에서 **매도 신호**가 나오려면 닿아야 할 트리거 가격 `↑/↓ 가격 (현재가 대비 %)` (3행). 트리거가 **−임계 이내, 또는 이미 현재가 위(+%)=충족**이면 **빨강** 강조 | 빨강 |
| `구름대` | **지지/저항 강도**: 지그재그 레그가 구름 안으로 들어오면 — 하락 레그(고점→하락)면 `지지 강함/보통/약함`, 상승 레그(저점→상승)면 `저항 강함/보통/약함`. 강도 = 구름 두께(ATR 대비)↑=강·접근 각도(ATR/봉)↑=약. 레그가 구름 밖이면 `위/안/아래·양운/음운` | 지지=초록·저항=빨강(강함=진함·약함=옅음)·그 외 위=초록/아래=빨강/안=회색 |
| `중기 추세` | (구 이평 리본) `횡보 압축/상승 분출/하락 분출 (이벤트 후 가격 ±%)` — 마지막 압축/분출 **발생 봉 종가 대비 현재 종가**의 변화율 | 압축=보라·상승=파랑·하락=빨강 |
| `장기 추세` | 한 사이클을 **한 단계씩** 세그먼트: **골든/데드 크로스 → (빌드업) 상승·하락 추세 → 확대=강한 → (피크 평탄) 상승·하락 추세 → 축소=약한 → 임박=약한 (데크/골크 주의) → 반대 크로스**. 폭 변화가 `평탄 ATR 배수` 초과 확대=강함/축소=약함/그 사이=평탄. **크로스 직후 `폭 변화 판정 봉수`동안 빌드업(추세)** + **반대 크로스 임박(간격 좁고 확대 아님) 시 약한(주의) 고정**으로 단계 건너뜀·크로스 직전 블립 방지 | 골든=파랑 배경·데드=보라 배경 / 상승=파랑·하락=보라(강함=선명·약함=옅음·추세=중간) |
| `이격도 (N)` | 라벨에 MA 길이 표시(예 `이격도 (20)`). 값 = 선택 이평선 대비 종가 위치 `값% (상태)`. 이격도 = `종가÷MA×100`(기준 100). **고정 임계(±%) + 적응형(σ)** 두 기준 중 **1개 충족=`되돌림 주의`(둔탁색), 2개 모두=`되돌림 확실`(선명색)**, 0개=`중립` | 과열(위)=빨강계·과매도(아래)=초록계 — 확실=선명·주의=둔탁·중립=회색 |
| `되돌림 가격` | 이격도가 **중립으로 복귀하는 단일 목표가** `가격 (현재가 대비 %)`. **과열(상승 중→하락 반등)** 이면 중립 상단(최대값), **과매도(하락 중→상승 반등)** 이면 중립 하단(최소값). 중립이면 `-`. 경계 = `MA × (1 ± 중립폭%)`, 중립폭 = `min(고정 임계, 통계 σ)` | 회색 |
| `롱/숏 스윕` | 샹들리에 트레일가(세력 스톱헌트 자리) `가격 (현재가 대비 %)`. 차트엔 **점선 트레일 + 점선 수평선**(롱=아쿠아·숏=크림슨 레드) | 선 색(옅게) |
| `롱/숏 청산` | 스윕 **너머**(ATR 버퍼+스윙 풀)의 실제 청산가 `가격 (현재가 대비 %)`. 차트엔 **직선 트레일 + 직선 수평선 + 가격축 라벨**(롱/숏 청산가만) | 선 색(진하게) |

부가 설명:

| 항목 | 내용 |
| --- | --- |
| 전환 = 스케일별 신호 트리거 | `상승/하락 전환`을 **0.5× / 1× / 2× 세 스케일**로 각각 표시. 각 값은 그 스케일 매수/매도 신호가 나오려면 다음 봉 종가가 닿아야 할 가격 (후보 제거). 작은 스케일일수록 빨리 반응 |
| 트리거 = 두 조건 결합가 | 신호 트리거 = `max/min(래깅 기준가, 전환선 전환가)`. 매수는 둘 중 더 높은 가격, 매도는 더 낮은 가격. 기간은 스케일별로 적용(0.5×=컨버전 4·베이스 11·래깅 11, 1×=8·22·22, 2×=16·44·44) |
| 래깅 기준가 | 다음 봉 래깅 스팬이 닿는 `baseLine[(disp×배율)-2]`(스케일별) |
| 전환선 전환가 | 다음 봉에 기준선·전환선이 같이 변함 → 상승 `2×다음봉기준선 − 최근(컨버전-1)봉 최저`, 하락 `… − 최고` |
| 진행 중 표시 | 각 스케일 신호 방향이 직전과 같아(중복) 막히면 = 그 방향이 이미 진행 중 → 상승 전환 칸은 `상승 추세`, 하락 전환 칸은 `하락 추세`를 **구름대 셀과 같은 채도**(포지션 색·투명도 30)로 표시 |
| 화살표 ↑/↓ | 현재가(`close`) 대비 트리거 가격 위치(높으면 ↑, 낮으면 ↓) |
| 천 단위 쉼표 | 트리거·청산 가격은 `1,000`처럼 3자리마다 쉼표, 소수 자릿수는 심볼 `mintick` 기준 |
| 구름대 평가 시점 | 구름은 displacement만큼 앞으로 밀리므로 현재 봉 아래 구름 = `[displacement-1]` 과거 선행스팬으로 평가 |
| 리본 상태 동기화 | 마지막으로 발생한 다이아몬드(상태머신) 기준이라 차트 마커와 항상 일치 |
| 스윕(점선) | 샹들리에 ATR 트레일링 = 보이는 손절 군집(세력 스톱헌트 표적). 과거 트레일(점선)+현재 봉 점선 수평선. 가격 모드 4종 선택 |
| 청산(직선) | 스윕 **너머**에 둔 실제 청산선. `청산 = 스윕 − 버퍼ATR`(숏은 +), `스윙 풀 반영` 시 가장 가까운 피벗 저/고점 너머까지 더 바깥. 단 **`청산-스윕 최대 간격 ATR 배수`로 간격 상한**을 둬 스윕과 너무 벌어지지 않게 함. 과거 트레일(직선)+현재 봉 직선 수평선 |
| 공통 | 가격 옆 괄호는 **현재 종가 대비 %**. 두께는 `선 두께(스윕·청산 공통)`, 색은 **롱=아쿠아·숏=오렌지**(변경 가능, `스윕·청산선 표시`로 끄기). 테이블 셀도 선 색으로 칠함(스윕=옅게·청산=진하게). **롱/숏 청산가는 가격축(price scale) 라벨로도 표시**(스윕은 미표시) |

### 스윕 가격 모드 (샹들리에 기반, 기본값 `기본`)

| 모드 | 가중(앵커) | 특징 |
| --- | --- | --- |
| `기본` | 가중 없음 (`highest/lowest` 룩백) | 표준 샹들리에 |
| `거래량 가중` | `VWMA(고가/저가)` | 거래량 몰린 봉을 더 반영 → 거래 집중 구간에서 청산가↑ |
| `최근봉 가중` | `WMA(고가/저가)` | 최근봉을 더 반영(이평 최근봉 가중) → **휩쏘 급등/급락 봉에서 청산가를 빠르게·높게** |
| `거래량+최근봉` | 거래량 × 선형 최근봉 가중 | 두 가중을 함께 |

- `기본`은 룩백 극값(`highest/lowest`) 기준, 가중 모드는 `가중 트레일 길이`·`가중 트레일 ATR 배수`로 계산한 **가중평균 앵커**를 그대로 사용합니다(모드마다 청산가 위치가 실제로 달라짐). 가중 모드는 짧은 길이·타이트 배수라 최근/거래량 변화에 더 빠르게 반응합니다.
- 옵션 `스윕/청산`: `스윕 룩백`(22)·`스윕 ATR`(22)·`스윕 ATR 배수`(3.0)·`스윕 가격 모드`·`가중 트레일 길이`(10)·`가중 트레일 ATR 배수`(1.5)·`청산 버퍼 ATR 배수`(0.4)·`청산 스윙 풀 반영`(ON)·`스윙 풀 좌우 봉수`(5)·`청산-스윕 최대 간격 ATR 배수`(1.0)·`트레일 길이(봉)`(200)·`스윕·청산선 표시`·`선 두께`(스윕·청산 공통, 1)·롱/숏 선 색.
- **타이트해지는 만큼 되돌림에 조기 청산이 늘 수 있음** — 스파이크에서만 물리게 하려면 `가중 트레일 ATR 배수`↑.

## 수축/확장 (휩쏘 필터) — 이평 리본 / 구름 두께 / 리본+구름 모드

이평 수축↔확장을 캔들에 **다이아몬드**로 표시합니다 — `횡보 압축`=보라 `#9C27B0`(상승·하락과 확실히 구분), `상승 분출`=파랑, `하락 분출`=빨강. 다이아몬드는 **신호 삼각형보다 뒤(아래 레이어)** 에 그려집니다. **판정 기준을 3가지 모드로 선택**합니다(기본 `리본+구름`).

| 모드 | 수축/확장 판정 기준 | 분출 방향 |
| --- | --- | --- |
| `이평 리본` | **입력값 기반 이평 리본**(전환선~리딩B 길이, 7개 MA)의 폭이 ATR 대비 좁으면 수축 | 빠른선(전환선 길이) ≥/< 느린선(리딩B 길이) |
| `구름 두께` | **구름 두께**(선행스팬 A·B 간격)가 ATR 대비 좁으면 수축 | 양운(A≥B)=상승 / 음운=하락 |
| `리본+구름`(기본) | 리본·구름이 **모두** 수축/확장(방향 일치)할 때만 | 두 기준 방향이 같을 때만 분출 |

- **`이평 리본`이 이치모쿠와 정합**: 리본 길이를 `전환선~리딩B`(기본 8~44)에 자동 연동해, 신호와 같은 시간축에서 수축/확장을 판단합니다. 기간 입력을 바꾸면 리본 범위도 따라갑니다.
- **`구름 두께`는 이치모쿠 본연**: 구름 두께 자체가 압축/확장 측정이라 추가 라인 없이 정합적입니다.

| 항목 | 내용 |
| --- | --- |
| 상태머신 | 압축 후 분출(같은 방향 반복은 재압축 필요). **단, 직전 분출과 반대 방향 확장이면 압축 없이 바로 반전 분출**(상승 분출 → 하락 분출, 그 반대도 가능). `리본+구름`은 방향이 엇갈리면 분출 보류(무장 유지) |
| 분출 속도 | 폭이 임계선을 완전히 넘기 전이라도 잡아 빠름. `확장 판정 폭 증가 봉수`↑ = 느리고 확실, 1 = 가장 빠름 |
| 활용 | 신호 + 같은 방향 분출 = 신뢰↑ / `횡보 압축` 구간 신호 = 휩쏘 의심·관망. **진입 트리거 아닌 신뢰도 필터** |
| 옵션(`리본 압축/분출`) | `수축/확장 판정 모드`(리본+구름 기본)·`리본 이동평균 종류(A)`(SMA)·`횡보 압축 표시`·`분출 표시`·`봉마감 후 확정`·`폭 ATR 기준`(14)·`수축 판정 ATR 배수`(0.8)·`확장 판정 폭 증가 봉수`(1) |
