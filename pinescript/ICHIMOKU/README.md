# Advanced Ichimoku

트레이딩뷰용 Pine Script Ichimoku 확장 버전입니다.

대상 스크립트:
- [`advanced-ichimoku.pine`](./advanced-ichimoku.pine) — 인디케이터 (신호 + 후보 표시)
- [`advanced-ichimoku-strategy.pine`](./advanced-ichimoku-strategy.pine) — 전략 (백테스트 + 신호)

이 버전은 기본 Ichimoku 계산을 유지하면서 아래를 추가했습니다.

- 이미지 기준 기본 표시 상태를 맞춘 라인/클라우드 구성
- `래깅 스팬 vs 래깅 위치 캔들 종가` + `현재 컨버전 라인 vs 베이스 라인` 조합의 매수/매도 신호
- 같은 방향 신호가 연속으로 찍히지 않게 하는 신호 순서 제한
- 다음 봉에서 신호가 나올 가능성을 미리 보여주는 `매수 후보 / 매도 후보` 표시 (인디케이터·전략 공통)
- 우측 하단에 `다음 봉이 어느 가격에 마감해야 각 신호가 나오는지` 알려주는 트리거 가격 테이블 (인디케이터·전략 공통)
- 이평 20~50 리본의 `횡보 압축 → 상승/하락 분출`을 다이아몬드로 표시하는 휩쏘 필터 (인디케이터·전략 공통)
- 프라이스 스케일(우측 가격축)에는 `베이스 라인` 값만 표시
- (전략) 동일 신호 로직으로 Strategy Tester에서 수익률을 확인하는 백테스트

## 신호 로직

### 매도 신호

매도 신호는 아래 두 묶음이 동시에 만족할 때 발생합니다.

1. `래깅 스팬`이 `래깅 스팬이 존재하는 캔들의 종가`를 하향 이탈하거나, 이미 그 아래에 있음
2. `현재 컨버전 라인`이 `베이스 라인`을 데드 크로스하거나, 이미 그 아래에 있음

현재 봉 기준 계산은 아래처럼 해석했습니다.

- 래깅 스팬: 현재 `close`
- 래깅 위치 캔들 종가: `close[displacement - 1]`
- 래깅 약세 확인: `ta.crossunder(close, close[displacement - 1])` 또는 `close < close[displacement - 1]`
- 전환선 약세 확인: `ta.crossunder(conversionLine, baseLine)` 또는 `conversionLine < baseLine`

### 매수 신호

매수 신호는 위의 정확한 반대입니다.

1. `래깅 스팬`이 `래깅 스팬이 존재하는 캔들의 종가`를 상향 돌파하거나, 이미 그 위에 있음
2. `현재 컨버전 라인`이 `베이스 라인`을 골든 크로스하거나, 이미 그 위에 있음

현재 봉 기준 계산은 아래처럼 해석했습니다.

- 래깅 강세 확인: `ta.crossover(close, close[displacement - 1])` 또는 `close > close[displacement - 1]`
- 전환선 강세 확인: `ta.crossover(conversionLine, baseLine)` 또는 `conversionLine > baseLine`

## 신호 순서 제한

같은 방향 신호가 연속으로 찍히지 않게 제한했습니다.

- 매도 신호가 나온 뒤에만 매수 신호가 다시 나올 수 있습니다.
- 매수 신호가 나온 뒤에만 매도 신호가 다시 나올 수 있습니다.

즉, 같은 방향 상태가 계속 유지돼도 첫 신호만 1번 찍히고, 반대 방향 신호가 나온 뒤에야 다시 같은 방향 신호가 허용됩니다.

## 매수 / 매도 후보 (인디케이터·전략 공통)

두 스크립트 모두 `매수 후보 / 매도 후보` 원형 마커가 있습니다. 이는 확정 신호가 아니라 **다음 봉에서 신호가 발생할 가능성**을 현재 봉에서 미리 보여 주는 표시입니다.

- 래깅 비교 기준을 한 칸 앞당긴 `close[displacement - 2]`(없으면 현재 `close`)로 다음 봉 상태를 추정합니다.
- 전환선 조건은 현재 봉과 동일하게 적용합니다.
- 같은 방향 신호 순서 제한을 그대로 따르며, 같은 봉에서 확정 매수/매도 신호가 나오면 후보는 표시하지 않습니다.

확정 신호와 다음 봉 가격 흐름에 따라 후보가 그대로 실현되지 않을 수 있습니다. 진입 확정용이 아니라 참고용 표시입니다.

## 우측 하단 테이블 (트리거 가격 + 구름대/리본 상태)

두 스크립트 모두 차트 **우측 하단**에 표를 띄웁니다. 위쪽 4행은 다음 봉의 트리거 가격, 아래 2행은 현재 봉의 구름대·리본 상태입니다.

| 다음 봉 | 트리거 가격 |
| --- | --- |
| `상승 후보 ●` | `close[displacement - 3]` 위로 마감 시 |
| `상승 신호 ▲` | `close[displacement - 2]` 위로 마감 시 |
| `하락 후보 ●` | `close[displacement - 3]` 아래로 마감 시 |
| `하락 신호 ▼` | `close[displacement - 2]` 아래로 마감 시 |

- 신호는 `후행스팬 조건`(다음 봉 종가가 과거 종가를 돌파)과 `전환선 vs 기준선` 게이트를 **동시에** 만족해야 발생합니다. 표의 가격은 그중 **가격으로 정해지는 후행스팬 기준값**이며, 전환선 게이트가 그 방향으로 열려 있을 때 그대로 유효합니다.
- **트리거 가격 앞 화살표는 현재 가격(`close`) 대비 트리거 가격의 위치**입니다. 트리거 가격이 현재가보다 높으면 `↑`, 낮으면 `↓`. (예: 현재가 400, 트리거 500 → `↑`)
- 트리거 가격은 후행스팬 기준값이라 `상승/하락 신호`가 같은 가격을 공유하고, 그 가격을 **위로** 마감하면 상승, **아래로** 마감하면 하락 쪽으로 갈립니다(후보는 한 칸 더 최근 종가 기준).
- **신호 순서 제한 반영**: 이미 같은 방향 신호가 나온 상태(중복방지)에서는 그 방향 트리거가가 `-`로 표시됩니다. 예) 직전이 매수 신호면 `상승 후보`·`상승 신호` 칸이 `-`, 직전이 매도 신호면 `하락 후보`·`하락 신호` 칸이 `-`. 다음에 나올 수 있는 방향만 숫자로 보여 헷갈리지 않게 했습니다.
- `displacement` 기본값(22)에선 신호 기준값 = `close[20]`, 후보 기준값 = `close[19]`입니다.

### 구름대 / 리본 상태 행 (괄호 안 숫자값 포함)

표 아래 2행은 현재 봉 기준 상태를 **문자열 + 괄호 안 숫자값**으로 보여 줍니다.

| 행 | 표시 예 | 의미 |
| --- | --- | --- |
| `구름대` | `위·양운 (1.8)` | 가격이 구름대 `위 / 안 / 아래` 어디에 있는지 + `양운(선행A≥B) / 음운` 색, 괄호 숫자 = **구름 두께 ÷ ATR**(두꺼울수록 강한 지지/저항). 셀 색은 위치(위=초록, 아래=빨강, 안=회색) |
| `이평 리본` | `상승 분출 (1.45)` | **마지막으로 발생한 다이아몬드(상태머신) 기준** 상태 `횡보 압축 / 상승 분출 / 하락 분출`(차트 마커와 동기화), 괄호 숫자 = **현재 리본폭 ÷ ATR**(이 값이 `압축 판정 ATR 배수`보다 작으면 압축 구간). 셀 색은 압축=회색, 상승=파랑, 하락=빨강 |

- 구름은 `displacement`만큼 앞으로 밀려 그려지므로, 현재 봉 아래 구름은 `cloudOffset(=displacement-1)` 만큼 과거의 선행스팬 값으로 평가합니다.
- 두 숫자 모두 ATR 배수라, 종목·가격대가 달라도 같은 기준으로 비교할 수 있습니다.

## 리본 압축/분출 (휩쏘 필터)

이치모쿠는 추세추종이라 횡보장 휩쏘에 약합니다. 이를 거르기 위해 **20~50 이동평균 리본의 압축↔분출 상태**를 캔들 아래 다이아몬드로 함께 표시합니다(인디케이터·전략 공통). 리본 라인 자체는 그리지 않고, 상태 다이아몬드만 표시합니다.

| 표시 | 판정 | 모양 |
| --- | --- | --- |
| `횡보 압축` | 리본 폭(`MA20~50` 최댓값−최솟값)이 `ATR × 압축 배수`보다 좁아지는 첫 봉 | 회색 다이아몬드 |
| `상승 분출` | 압축 후 리본 폭이 다시 늘기 시작하는 첫 봉에 `MA20 ≥ MA50` | 파랑 다이아몬드 |
| `하락 분출` | 압축 후 리본 폭이 다시 늘기 시작하는 첫 봉에 `MA20 < MA50` | 빨강 다이아몬드 |

상태머신은 `압축 → 분출 → 압축 → 분출 …` 순서로만 진행합니다.

1. 리본 폭이 `ATR × 압축 배수`보다 좁아지는 첫 봉 = `횡보 압축` — 동시에 다음 분출을 **무장**시킵니다.
2. 이후 리본 폭이 **다시 늘기 시작하는(분출 판정 봉수 기준으로 확장되는) 첫 봉**에 `MA20` vs `MA50` 방향으로 분출 1회(상승/하락), 그리고 무장 해제. 폭이 압축 임계선을 완전히 넘기 전이라도 잡으므로 분출이 더 빨리 나옵니다. `분출 판정 폭 증가 봉수`를 키우면 더 확실히 늘 때만(느리게), 1로 두면 가장 빠르게 잡습니다.
3. 분출 후에는 **다시 `횡보 압축`이 와야** 다음 분출이 허용됩니다(분출 연속 방지).

활용:
- 매수 신호 + `상승 분출` / 매도 신호 + `하락 분출` → 추세 출발과 정렬 → **신뢰↑**.
- `횡보 압축`이 이어지는 구간의 신호 → 휩쏘 의심 → **관망**.
- 진입 트리거가 아니라 **신뢰도 필터**입니다.

옵션(`리본 압축/분출` 그룹): `리본 이동평균 종류(SMA 기본)`, `횡보 압축 표시`, `분출(상승/하락) 표시`, `봉마감 후 확정`, `리본폭 ATR 기준`, `압축 판정 ATR 배수`, `분출 판정 폭 증가 봉수(작을수록 빠름)`.

## 전략 스크립트

[`advanced-ichimoku-strategy.pine`](./advanced-ichimoku-strategy.pine)은 인디케이터와 동일한 `buySignal / sellSignal` 로직을 그대로 사용해서 Strategy Tester에서 수익률을 확인할 수 있게 만든 버전입니다.

진입/청산 방식은 아래와 같습니다.

- 각 `매수 신호`에서는 먼저 숏 포지션을 청산하고, `롱` 모드인 경우 롱 포지션을 엽니다.
- 각 `매도 신호`에서는 먼저 롱 포지션을 청산하고, `숏` 모드인 경우 숏 포지션을 엽니다.
- 차트의 삼각형 마커는 원시 신호가 아니라 `실제 체결 후 포지션 변화` 기준으로만 표시됩니다.
- 그래서 `롱 진입 / 롱 청산 / 숏 진입 / 숏 청산` 마커가 Strategy Tester의 거래 흐름과 직접 대응됩니다.

전략 옵션:

- `포지션 방향`: `롱` 또는 `숏` (한 방향만 선택)

### 진입 방식 / 손익 해석

- **전체 자본 진입**: 매 진입은 전체 자본(`percent_of_equity 100%`)으로 포지션을 잡습니다. 원금은 `initial_capital = 10000`입니다.
- **마진콜(강제청산) 로직 없음**: 기존의 `숏 마진콜 방지 청산`(하드 스탑) 로직은 삭제된 상태입니다. 신호가 아닌 강제 체결이 섞이는 것을 막기 위함입니다.
- **Total P&L / Net Profit %**: `Net Profit`(달러)은 원금에서 늘어난 절대 금액이고, 옆의 `%`는 원금(`10000`) 대비 비율이라 그대로 "원금 대비 수익률"로 읽으면 됩니다.

기본 설정은 `default_qty_type=strategy.percent_of_equity`, `default_qty_value=100`, `pyramiding 0`, `process_orders_on_close=true`, `use_bar_magnifier=true`, `margin_long=0`, `margin_short=0`, `initial_capital=10000`입니다. `margin 0`이라 증거금 제약 없이 가용 자본으로 진입합니다. 마진콜 방지 로직도 없으므로, `숏` 모드에서 가격이 크게 급등하면 강제청산이 발생할 수 있습니다. 수수료와 슬리피지는 코드에서 가정하지 않았습니다. 실제 거래소 기준으로 보려면 TradingView 전략 속성에서 수수료와 슬리피지를 직접 넣어 주세요.

## 옵션 구성

### 인디케이터 입력 탭

- `기본 설정`: 컨버전 / 베이스 / 리딩 스팬 B / 레깅 스팬 길이
- `표시`: 각 라인과 Cloud Fill 표시 여부

- `후보 신호`: `매수 후보 표시`, `매도 후보 표시`

### 전략 입력 탭

- `기본 설정`: 컨버전 / 베이스 / 리딩 스팬 B / 레깅 스팬 길이
- `전략`: `포지션 방향` (진입은 전체 자본 100%, 원금은 `initial_capital=10000`)
- `표시`: 각 라인과 Cloud Fill 표시 여부
- `포지션 표시`: `롱 진입 표시`, `롱 청산 표시`, `숏 진입 표시`, `숏 청산 표시`
- `후보 신호`: `매수 후보 표시`, `매도 후보 표시`

## 적용 방법

1. TradingView Pine Editor를 엽니다.
2. 시그널만 보고 싶으면 [`advanced-ichimoku.pine`](./advanced-ichimoku.pine)을 사용합니다.
3. 수익률 백테스트를 하려면 [`advanced-ichimoku-strategy.pine`](./advanced-ichimoku-strategy.pine)을 사용합니다.
4. 저장 후 차트에 추가합니다.
5. 전략 수익률은 TradingView의 `Strategy Tester` 탭에서 확인합니다.
6. 거래 수가 이상해 보이면 `List of Trades` 탭에서 실제 체결 목록을 함께 확인합니다.
