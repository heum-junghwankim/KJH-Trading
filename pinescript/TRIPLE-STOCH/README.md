## 지표 구조 — 3개 스토캐스틱의 역할

세 개의 서로 다른 기간 스토캐스틱을 **역할 분담**해 웨이브(파동) 진입을 잡는 설계입니다.

| 스토캐스틱 | 기본 (Length/K/D) | 역할 | 색 |
| --- | --- | --- | --- |
| **Slow** | 20 / 9 / 9 | **컨텍스트**(큰 추세·국면) | 파랑 `#1565C0` |
| **Mid** | 10 / 6 / 6 | **리셋**(눌림·되돌림 후 재개) | 청록 `#00897B` |
| **Fast** | 5 / 3 / 3 | **트리거**(진입 타이밍 크로스) | 주황 `#FB8C00` |

각 스토캐스틱은 표준 정의로 계산합니다(`stoch_pair` 헬퍼):
```
raw_k = 100 × (close − 최저저가) / max(최고고가 − 최저저가, mintick)
k     = SMA(raw_k, smooth_k)
d     = SMA(k, smooth_d)
```
- `max(range, syminfo.mintick)`로 0으로 나누기를 방지합니다(횡보·갭 없는 구간 가드).

## 신호 로직 (CEO 확정 — 원본 보존)

**Slow = 컨텍스트, Mid = 리셋, Fast = 트리거** 3단 구조입니다.

- **`slow_bull_trend` / `slow_bear_trend`** (컨텍스트): Slow %K가 %D 위/아래이고, 추세 미드라인(50)을 넘거나 최근 `slow_context_lookback`(8)봉 내 과매도/과매수를 찍은 상태.
- **`mid_bull_reset` / `mid_bear_reset`** (리셋): Mid %K가 %D 위/아래로 2봉 연속 상승/하락(`ta.rising`/`ta.falling`)하고, 최근 `mid_pullback_lookback`(6)봉 내 미드라인까지 되돌림이 있었던 상태(눌림 후 재개).
- **`fast_bull_trigger` / `fast_bear_trigger`** (트리거): Fast %K가 %D를 상향/하향 교차(`ta.crossover`/`ta.crossunder`)하면서 교차 지점이 과매도/과매수 영역인 상태.
- **`bull_wave_background` / `bear_wave_background`** (웨이브 셋업): 컨텍스트 + 리셋이 모두 성립(`slow_*_trend and mid_*_reset`). 배경색으로 강조됩니다.
- **`all_bull_aligned` / `all_bear_aligned`** (3중 정렬): Slow·Mid·Fast **셋 다** %K > %D(또는 셋 다 < %D). alert 조건.

### 플롯·배경·alert (원본)
- 6개 라인(각 스토캐스틱의 %K·%D). %K는 진하게, %D는 반투명·얇게.
- Fast 트리거 성립 봉에 큰 원(circles) 마커.
- 웨이브 셋업 성립 시 배경색(상승=녹, 하락=적, 투명 80).
- 과매수(80)·미드라인(50)·과매도(20) 수평선.
- 3중 정렬 alert 2종.

## ⚠️ 단독 신호 아님 — 교차 검증 전제

이 지표는 **모멘텀·타이밍**을 봅니다. 방향의 최종 판단은 반드시 **이치모쿠(메인) + 우측 테이블 + 다른 보조지표 정렬**로 교차 검증하세요(펀드 철학: 단일 신호 불신). 삼중 정렬·웨이브는 "지금 진입 타이밍이 정렬됐나"를 보는 **확인용**입니다.

### 종합 판정 어휘·색 (저장소 통일)
표시 문구는 **매매 지시 없이 시장 상태를 객관적으로 서술**합니다("상승/하락" 방향 + 국면). 색은 저장소 시그니처 채도 규칙:
- **강(선명, 투명도 0)**: 상승 = `color.green`, 하락 = `color.red`
- **약/셋업(둔탁)**: 상승 = `#6E8F72`, 하락 = `#A57A7A`
- **중립**: `color.gray`(투명 40)

### 종합 판정 우선순위(캐스케이드)
1. 웨이브 셋업 + Fast 트리거 → **상승/하락 웨이브+트리거** [선명]
2. 3중 정렬 → **3중 상승/하락 정렬** [선명]
3. 웨이브 셋업만(트리거 전) → **상승/하락 셋업** [둔탁]
4. 그 외 → **중립** [회색]

### band-walking 완화 (수학자 요구 반영)
강추세(3중 정렬) 중에는 과매수/과매도가 **밴드워킹**(추세가 극단 영역에 붙어 계속 진행)일 수 있어, 반전으로 오해하기 쉽습니다. 그래서 셀2에서:
- **3중 상승 정렬 중 과매수** → `과매수(추세중)` **회색**으로 표기(강추세 구간의 상태 서술).
- **3중 하락 정렬 중 과매도** → `과매도(추세중)` **회색** (대칭).
- 정렬이 아닌 상태의 과매수/과매도만 `과매수(과열)` · `과매도(침체)`로 둔탁색 표기.

## 리페인팅 / 안전성

- 전 계산이 확정 `ta.*`(`highest`/`lowest`/`sma`/`rising`/`falling`/`crossover`/`crossunder`)로 이뤄지며 **`request.security`·`lookahead` 없음**. 미래참조가 없습니다.
- **테이블 판정은 확정봉(`[1]`) 기준**입니다. 미마감(실시간) 봉에서 Fast 크로스가 붙었다 떨어졌다 하며 테이블이 점멸하는 것을 막기 위해, crossover/정렬/셋업 판정을 **직전 확정봉 값**으로 읽습니다.
  - **주의:** 이 때문에 테이블 텍스트는 **차트의 원·배경보다 1봉 지연**되어 갱신됩니다. 원본 플롯·배경·alert(즉시 반영)와 테이블(확정봉)의 성격이 다름을 유의하세요. 지연은 의도된 안정화입니다.
- 원본 플롯·배경·alert는 **원본 그대로**(실시간 봉에서 갱신될 수 있음 — 스토캐스틱 지표의 정상 거동).
- 별도 패널 지표라 가격 차트를 가리지 않습니다.

## 주의사항

- **임계값·기간은 전략 파라미터**입니다(Oversold/Overbought/Midline/Lookback). 바꾸면 신호 성격이 달라지므로, 변경 시 백테스트·과최적화 리스크를 팀(본부장·알고리즘 수학자)과 먼저 검토합니다. 임의 변경 금지.
- 스토캐스틱은 강추세에서 극단 영역에 오래 머무릅니다(밴드워킹). 위 완화 규칙을 참고하되, 최종 방향은 이치모쿠 정렬로 확인하세요.
- TradingView 실측(컴파일·차트 거동)에서 최종 확인이 필요합니다.
