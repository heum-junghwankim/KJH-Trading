# 삼중 Stoch (Triple Stochastic Wave, 삼중 스토캐스틱 웨이브)

트레이딩뷰용 Pine Script **보조 지표** 설명서입니다.

대상 스크립트:
- [`triple-stochastic-wave.pine`](./triple-stochastic-wave.pine)

> Pine Script **v6** 기준. 별도 패널(`overlay` 아님)에 그려지는 **모멘텀·타이밍 보조 지표**입니다.

CEO 확정 제공 "Triple-Stochastic-Wave" 소스코드를 기반으로 합니다. 저장소 톤앤매너의 **우측 상단 상태 테이블**을 추가했으며, **웨이브 배경 조건은 CEO 지시로 스토캐스틱 정배열/역배열(세 %K 서열)로 재정의**했습니다. 그 외 계산·플롯·트리거·alert는 원본 로직을 보존합니다.

> **웨이브 배경 재정의 (CEO 지시):** 기존 배경 조건 `slow_*_trend and mid_*_reset`(컨텍스트+리셋)을 **정확한 정배열/역배열**로 교체했습니다. 상승 배경 = `fast_k > mid_k > slow_k`(정배열), 하락 배경 = `fast_k < mid_k < slow_k`(역배열). 이에 따라 `mid_bull_reset`/`mid_bear_reset` 및 `Mid Pullback Lookback` 입력이 제거되었습니다. 배경 색·투명도·테이블·마커·플롯·alert는 불변입니다.

## 지표 구조 — 3개 스토캐스틱의 역할

세 개의 서로 다른 기간 스토캐스틱을 **역할 분담**해 웨이브(파동) 진입을 잡는 설계입니다.

| 스토캐스틱 | 기본 (Length/K/D) | 역할 | 색 |
| --- | --- | --- | --- |
| **Slow** | 20 / 9 / 9 | **컨텍스트**(큰 추세·국면) | 파랑 `#1565C0` |
| **Mid** | 10 / 6 / 6 | **중간축**(정배열/역배열 서열의 가운데) | 청록 `#00897B` |
| **Fast** | 5 / 3 / 3 | **트리거**(진입 타이밍 크로스) | 주황 `#FB8C00` |

각 스토캐스틱은 표준 정의로 계산합니다(`stoch_pair` 헬퍼):
```
raw_k = 100 × (close − 최저저가) / max(최고고가 − 최저저가, mintick)
k     = SMA(raw_k, smooth_k)
d     = SMA(k, smooth_d)
```
- `max(range, syminfo.mintick)`로 0으로 나누기를 방지합니다(횡보·갭 없는 구간 가드).

## 신호 로직 (CEO 확정 — 원본 보존)

**Slow = 컨텍스트, Fast = 트리거** 구조이며, 웨이브 배경은 세 스토캐스틱 %K의 **정배열/역배열 서열**로 판정합니다.

- **`slow_bull_trend` / `slow_bear_trend`** (컨텍스트): Slow %K가 %D 위/아래이고, 추세 미드라인(50)을 넘거나 최근 `slow_context_lookback`(8)봉 내 과매도/과매수를 찍은 상태. (테이블 정렬/추세 행에서 사용)
- **`fast_bull_trigger` / `fast_bear_trigger`** (트리거): Fast %K가 %D를 상향/하향 교차(`ta.crossover`/`ta.crossunder`)하면서 교차 지점이 과매도/과매수 영역인 상태.
- **`stoch_bull_order` / `stoch_bear_order`** (정배열/역배열 서열): 세 스토캐스틱의 **%K 값 서열**로 판정. 정배열 = `fast_k > mid_k > slow_k`(위에서 아래로 fast→mid→slow, 상승 서열), 역배열 = `fast_k < mid_k < slow_k`(하락 서열). 엄격 부등호로 "정확히" 순서대로 정렬된 상태만 참.
- **`bull_wave_background` / `bear_wave_background`** (웨이브 배경): 스토캐스틱이 **정확히 정배열/역배열**일 때만(`stoch_bull_order` / `stoch_bear_order`). 배경색으로 강조됩니다.
- **`all_bull_aligned` / `all_bear_aligned`** (3중 시그널 정렬): Slow·Mid·Fast **각각** %K > 자기 %D(또는 셋 다 < 자기 %D). alert 조건. *(주의: 각 스토캐스틱의 %K vs 자기 %D 비교이며, 스토캐스틱 간 서열인 `stoch_*_order`와는 다른 개념입니다.)*

### 플롯·배경·alert (원본)
- 6개 라인(각 스토캐스틱의 %K·%D). %K는 진하게, %D는 반투명·얇게.
- Fast 트리거 성립 봉에 큰 원(circles) 마커.
- 스토캐스틱 정배열/역배열 성립 시 배경색(정배열=녹, 역배열=적, 투명 80).
- 과매수(80)·미드라인(50)·과매도(20) 수평선.
- 3중 정렬 alert 2종.

### 1행 = 2·3행의 종합 (포지션 상태 리포트)
1행(종합 판정)은 **자체 조건을 다시 평가하지 않고**, 아래 두 행의 결과만 입력으로 삼아 "지금 시장이 어느 방향으로·얼마나 기울었는지"를 **포지션 관점 상태 서술**로 요약합니다. 매매 지시어(롱/숏/진입/청산)는 쓰지 않고 **방향 우위 + 강도/국면 수식**으로만 표기합니다.

- **2행(정렬/추세)** → 방향(`상승`/`하락`/`중립`)과 강도(`3중 정렬`=강추세 여부)를 제공.
- **3행(모멘텀)** → 방향과 유형(`전환`/`트리거`/`셋업`/`과열·침체`)을 제공. 단, **band-walking 완화**로 `과매수/과매도(추세중)`는 방향·유형을 중립 취급하여 1행에 영향을 주지 않습니다.

### 추세 전환 상태 (최우선 판정 — 수학자 요구 반영)
과매도 바닥에서 Fast 골든크로스(반등 초입)가 나오는데도, 2행 정렬이 **위상만 보고 위치(과매도)를 못 봐서** 1행이 `중립/관망`이나 `하락 우위`로 오안내하던 결함을 교정했습니다. 극단 위치 + Fast 크로스를 **전환 초입**으로 최우선 분류합니다.
- **3행**: `과매도 + Fast 골든크로스` → `상승 전환(반등 임박)` **선명 녹** / `과매수 + Fast 데드크로스` → `하락 전환(하락 임박)` **선명 적**. (유형 `reversal`, 웨이브·트리거보다 **위** 우선)
- **1행**: 3행이 `reversal`이면 종합도 `상승 전환`/`하락 전환`으로 **최우선 확정**. 엇갈림 중립화보다 위에 있어 **2행 정렬이 전환을 덮어쓰지 못합니다.**
- **의도된 해석**: `1행 상승 전환 + 2행 하락 추세`는 모순이 아니라 **하락 추세 바닥에서의 반등 초입**을 뜻합니다(2행은 아직 하락 국면, 3행이 전환 신호를 먼저 포착). 2행 정렬 판정은 불변입니다.
- **가드 없음**: 밴드워킹 가드를 걸지 않은 순수 조건입니다(수학자: slow 위치가 밴드워킹 방향과 자연 배타라 오탐 희소).

### 종합 판정 어휘·색 (저장소 통일)
표시 문구는 **매매 지시 없이 시장 상태를 객관적으로 서술**합니다(방향 우위 + 국면). 색은 저장소 시그니처 채도 규칙:
- **강(선명, 투명도 0)**: 상승 = `color.green`, 하락 = `color.red`
- **약/대기(둔탁)**: 상승 = `#6E8F72`, 하락 = `#A57A7A`
- **중립**: `color.gray`(투명 40)

### 1행 판정 매핑표 (2행 × 3행 → 1행)
| 2행(방향·강도) | 3행(방향·유형) | 1행 종합 | 색 |
| --- | --- | --- | --- |
| 무관 (정렬 불변) | 상승 전환(과매도+Fast 골든크로스) | **상승 전환** | 선명 녹 |
| 무관 (정렬 불변) | 하락 전환(과매수+Fast 데드크로스) | **하락 전환** | 선명 적 |
| 상승 강(3중 정렬) | 상승 트리거 | **강한 상승 우위** | 선명 녹 |
| 상승 강(3중 정렬) | 상승 셋업 | **상승 우위(대기)** | 둔탁 녹 |
| 상승 강(3중 정렬) | 과매수(과열) | **상승 우위(과열 주의)** | 둔탁 녹 |
| 상승 강(3중 정렬) | 없음/추세중 | **상승 우위** | 선명 녹 |
| 상승 약(추세) | 상승 트리거 | **상승 우위** | 둔탁 녹 |
| 상승 약(추세) | 상승 셋업 / 없음 | **상승 우위(대기)** | 둔탁 녹 |
| 중립(엇갈림) | 상승 트리거/셋업 | **상승 우위(대기)** | 둔탁 녹 |
| 하락 (대칭 전체) | 하락 (대칭) | **하락 우위 / 강한 하락 우위 / …(대기)/(침체 주의)** | 적 |
| 방향 상충 (2행↔3행 반대) | | **중립/관망** | 회색 |
| 둘 다 방향 없음 | | **중립/관망** | 회색 |

> 우선순위: **(0) 추세 전환(reversal) = 최우선** → (1) 강추세+같은방향 트리거 = 최강 → (2) 2·3행 방향 상충 = 중립 차단 → (3) 상승 계열 → (4) 하락 계열. 전환이 (2) 중립 차단보다 위에 있어 2행 정렬과 무관하게 확정됩니다. `강한 X 우위`는 2행이 3중 정렬(강추세)이면서 3행이 같은 방향 트리거일 때만 부여됩니다.

### band-walking 완화 (수학자 요구 반영)
강추세(3중 정렬) 중에는 과매수/과매도가 **밴드워킹**(추세가 극단 영역에 붙어 계속 진행)일 수 있어, 반전으로 오해하기 쉽습니다. 그래서 셀2에서:
- **3중 상승 정렬 중 과매수** → `과매수(추세중)` **회색**으로 표기(강추세 구간의 상태 서술).
- **3중 하락 정렬 중 과매도** → `과매도(추세중)` **회색** (대칭).
- 정렬이 아닌 상태의 과매수/과매도만 `과매수(과열)` · `과매도(침체)`로 둔탁색 표기.

## 리페인팅 / 안전성

- 전 계산이 확정 `ta.*`(`highest`/`lowest`/`sma`/`rising`/`falling`/`crossover`/`crossunder`)로 이뤄지며 **`request.security`·`lookahead` 없음**. 미래참조가 없습니다.
- **테이블 판정은 확정봉(`[1]`) 기준**입니다. 미마감(실시간) 봉에서 Fast 크로스가 붙었다 떨어졌다 하며 테이블이 점멸하는 것을 막기 위해, crossover/정렬/셋업 판정을 **직전 확정봉 값**으로 읽습니다.
  - **주의:** 이 때문에 테이블 텍스트는 **차트의 원·배경보다 1봉 지연**되어 갱신됩니다. 원본 플롯·배경·alert(즉시 반영)와 테이블(확정봉)의 성격이 다름을 유의하세요. 지연은 의도된 안정화입니다.
- 원본 플롯·배경·alert는 **원본 그대로**(실시간 봉에서 갱신될 수 있음 — 스토캐스틱 지표의 정상 거동).
- 별도 패널 지표라 가격 차트를 가리지 않습니다.

## 주의사항

- **임계값·기간은 전략 파라미터**입니다(Oversold/Overbought/Midline/Lookback). 바꾸면 신호 성격이 달라지므로, 변경 시 백테스트·과최적화 리스크를 팀(본부장·알고리즘 수학자)과 먼저 검토합니다. 임의 변경 금지.
