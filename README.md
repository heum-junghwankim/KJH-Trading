# KJH-Trading

**슈퍼 이치모쿠(메인·overlay)** 위에 **거래량 압력 · 삼중 스토캐스틱 · cRSI · OBV-ADX** 네 보조지표를 얹어, 하나의 신호가 아니라 **여러 지표가 한 방향으로 정렬됐는지 교차 검증**하는 트레이딩뷰 Pine Script 모음입니다.

핵심 원칙은 단순합니다 — **신호 하나를 믿지 않는다.** 이치모쿠 신호 + 우측 상단 테이블 + 보조지표 포지션이 같은 방향일 때만 자리로 봅니다.

> Auto VWAP은 이치모쿠 인디케이터에 일부 편입돼 테이블 행(`중기 추세`(VWAP·100선)·`VWAP 상태`·`50선 지지/저항`·`장기 추세`·`100선/200선 지지/저항`)으로 내장됩니다. **이평선 리본 추세는 이치모쿠에서 제거**됐고(테이블 `리본 추세` 행·`+` 표식·리본 알림 삭제), 그 자리엔 `중기 추세`(VWAP(50) vs MA100 배열)가 들어갑니다. 리본 기반 **횡보 압축 배경**은 auto-vwap으로 이식돼 밴드2(±2σ) 영역 회색 배경으로 표시됩니다. 보조지표는 각자 별도 스크립트로 각 창에 띄워 상태판을 볼 수도 있습니다.

## 1. 슈퍼 이치모쿠 (메인 · overlay)

정통 이치모쿠 계산(전환선 8 · 기준선 22 · 선행스팬B 44 · 후행 22) 위에 **멀티스케일 신호 · 트리거 테이블 · 휩쏘 필터 · 이격도**를 얹은 메인 인디케이터입니다. 차트 우측 상단에 **4개 섹션으로 묶인 종합 테이블**을 띄워, 추세·캔들·지지/저항·손익비를 한눈에 읽게 합니다. 신호는 단독 트리거가 아니라 **정렬 확인용**으로 설계됩니다.

### 매수/매도 신호 — 멀티스케일 + 확정 발화

두 조건이 **동시에** 충족될 때 신호가 발생합니다.

| 신호 | 조건① (래깅 스팬) | 조건② (컨버전 라인) |
| --- | --- | --- |
| **매수(롱)** ▲ | 종가가 기준선(22) 상향 돌파/이상 | 컨버전(8)이 기준선 상향 돌파/이상 |
| **매도(숏)** ▼ | 위의 정확한 반대 | 위의 정확한 반대 |

| 특성 | 동작 |
| --- | --- |
| 멀티스케일 | 같은 신호를 **0.5× / 1× / 2×** 세 스케일로 계산해 삼각형 **크기**(가장 작음/작음/보통)로 동시 표시. 여러 크기가 한 봉에 겹칠수록 시간축 정렬 → 신뢰↑ |
| 확정 발화(휩쏘 필터) | 트리거 충족 봉에서 즉시 안 찍고 **대기** → **다음 봉 종가가 트리거 가격을 유지할 때만** 그 봉에 발화(유지 못 하면 취소). 확정봉 종가 기준이라 **리페인팅 없음**, 1봉 지연 대신 속임수 돌파(휩쏘)를 거름 |
| 스케일 캐스케이드 | 상위 배수는 하위가 같은 방향 추세일 때만 — **1×는 0.5×가 추세 중**, **2×는 0.5×·1× 둘 다 추세 중**에 발화. 반대 신호가 나면 전제가 풀림 |
| 신호 순서 제한 | 같은 방향 신호는 반대 신호가 한 번 나온 뒤에만 다시 발생(중복 방지) |
| 모멘텀 모드 | 비정상 캔들/반등 감시 진입 시 기간을 한 단계 빠른 스케일로 시프트(0.5×→0.25× 등) → 더 빨리 반응. 이격도 중립 복귀 시 해제. 트리거 셀에 `M ` 접두 |

⚠️ 추세추종이라 **횡보장 휩쏘·후행성**이 약점 → 신호만 믿지 말고 아래 테이블·보조지표로 검증합니다.

### 신호 6모드 (택일 · 기본 `혼합 확정 모드`)

`매매 신호 방식` 그룹의 `추세 진입 신호 방식` 입력 하나로 **신호 소스**(일반 도치안 / VWMA 거래량 가중)와 **확정 방식**(예측 / 확정)을 조합합니다. 한 번에 한 조합만.

| 모드 | 신호 소스 | 확정 방식 |
| --- | --- | --- |
| `일반 확정 모드` | 일반 도치안(컨버전·기준선) | 확정봉 종가 기준(1봉 지연·리페인팅 없음) |
| `일반 예측 모드` | 일반 도치안 | 예측(다음 봉 가정, 1봉 빨리·실시간 리페인팅) |
| `거래량 가중 확정 모드` | VWMA(거래량 가중) | 확정 |
| `거래량 가중 예측 모드` | VWMA | 예측 |
| **`혼합 확정 모드`**(기본 · 구 `혼합 일반` 개명) | **봉별 스위칭** — 비정상 거래량 봉=VWMA **즉시 발화**, 정상 봉=일반 도치안 | 정상 봉만 예측(`isPredict`), 비정상 봉은 즉시 |
| `혼합 예측 모드`(신규 · 후보 A) | 〃(봉별 스위칭) **+ 전환 행 색 예측 발화** | `혼합 확정`과 동일 경로 + 추세 전환 행 셀 색 기반 예측 발화(리페인팅 계열) |

> **혼합 모드의 핵심** — 그 봉이 **비정상 거래량**(`volume > SMA(volume,20) × 2`, VOLUME-PRESSURE와 동일 정의)이면 VWMA 신호를 **다음봉 확정 파이프라인을 우회해 그 봉에서 즉시 발화**합니다(장초·뉴스發 급변 빠른 캐치). 정상 봉은 기존 파이프라인(예측/확정)대로. 즉시 발화도 `dir≠방향`·스케일 캐스케이드를 공유해 **중복 발화는 구조적으로 차단**됩니다. 비정상 봉 트리거는 테이블 셀에 `V ` 접두.
>
> ⚠️ **모드 재구성** — 구 `혼합 일반 모드`가 **`혼합 확정 모드`로 개명**(로직·`isPredict` 그대로), 옛 순수확정 `혼합 확정`은 폐기. 신규 **`혼합 예측 모드`(후보 A)** 는 `혼합 확정` 경로에 **전환 행 색 예측 발화**를 추가한 모드 — 추세 확인 테이블의 `상승/하락 전환(0.5×/1×/2×)` 행 셀 색이 **둔탁 녹(`#6E8F72`)/둔탁 빨강(`#A57A7A`) = 전환 예측·대기 상태**이고 진행봉 방향(양봉=롱/음봉=숏)이 일치하면 기존 게이트를 통과할 때 **그 봉에서 즉시 발화**(비정상봉 즉시발화와 병존). 발화 기준은 **캔들 색이 아니라 전환 행 셀 색**(테이블 `f_trigBg`/`f_trigNear`와 동일 소스)이라 화면 셀 색과 신호가 일치합니다. **⚠️ 진행봉 실시간 예측이라 리페인팅 계열이며 전략 매매·백테스트가 크게 변합니다**(리페인팅 없는 백테스트는 `일반 확정`·`거래량 가중 확정` 사용). 지표·전략 동일 반영.

### 우측 상단 테이블 — 4개 섹션

표는 좌열=항목·우열=값의 2열 구조이며, 관련 행을 **4개 섹션 병합 헤더**(딥 네이비 `#1B2A4A`·흰 글자)로 묶습니다. 아래 소섹션은 테이블 4섹션과 1:1로 대응합니다.

#### 1-1. 추세 확인

| 행 | 값 | 셀 색 |
| --- | --- | --- |
| `상승 전환`(0.5×/1×/2×) | 각 스케일 매수 신호가 나오려면 다음 봉 종가가 닿아야 할 **트리거 가격** `↑/↓ 가격 (현재가 대비 %) \| N봉`. 진행 중이면 `상승 추세 (기준선 이격도)`. 모멘텀=`M `·비정상거래량=`V ` 접두 | 근접(±2%)=둔탁 초록 · 진행 중=선명 초록 |
| `하락 전환`(0.5×/1×/2×) | 위의 매도 대칭. 진행 중이면 `하락 추세 (기준선 이격도)` | 근접=둔탁 빨강 · 진행 중=선명 빨강 |
| `중기 추세` | (구 `리본 추세` 자리) **VWAP(50) vs 100선(MA100) 배열 추세** — `장기 추세`(MA100 vs MA200)와 **동일 로직·포맷·색**. VWAP > 100선=상승 배열·아래=하락 배열, 간격 확대=`강한`·축소/크로스 임박=`약한`, 크로스 순간=`골든/데드 크로스`. 값 `상승/하락 (±변화율% \| N봉)` | 상승계열=청록 `#0E9C8A` · 하락계열=자홍 `#C2185B` · 강함=선명·약함=옅음·크로스=선명 |
| `장기 추세` | (지지/저항 섹션에서 이 자리로 이동) MA100·200 기반 8종 `골든/데드 크로스`·`(강한/약한) 상승·하락` + `(크로스 후 MA100 변화% \| 봉수)` | **상승계열=청록 `#0E9C8A`** · **하락계열=자홍 `#C2185B`** · 강함=선명·약함=옅음·크로스=선명 |

> **⚠️ 리본 추세 제거(CEO 지시)** — 구 `리본 추세` 행·차트 `+` 표식(횡보 압축/상승·하락 분출)·리본 알림은 모두 제거됐고, 그 자리에 `중기 추세`(VWAP·100선)가 들어갑니다. 리본 기반 **횡보 압축 배경**은 [auto-vwap](pinescript/VWAP/README.md)으로 이식돼 밴드2(±2σ) 영역 회색 배경으로 표시됩니다.

> `N봉` = **시간 조정 투영**. 현재가가 유지된다고 가정할 때 그 스케일 전환(도치안 전환선↔기준선 크로스)이 성립하기까지의 최소 봉수. horizon 내 미성립·데이터 부족이면 `-`, 진행 중이면 미표기. 미래참조 아님(과거 데이터로 미래 창을 투영)·리페인팅 없음. **가정 기반 참고치**(실제 미래가는 변함).

#### 1-2. 캔들 및 되돌림 확인

| 행 | 값 | 셀 색 |
| --- | --- | --- |
| `캔들 상태 (3)` | 최근 **3봉을 7분류**로 스코어링해 `분류명 \| 근거패턴`. **표시 전용**(신호 미연결). 현재봉 캔들 바디도 이 색으로 칠함(혼조 제외) — **캔들 색은 캔들 상태 7분류 기준으로 조건 게이트 없이 그대로 표시**(표시 토글만 연동) | 7분류 색(아래) |
| `캔들 손절라인` | 캔들 상태 **바로 아래**. 7분류 방향 재사용 — **하락 계열=3봉 최고점** `S {절대손절가} (+N%)`, **상승 계열=3봉 최저점** `L {절대손절가} (-N%)`, 혼조=`-`. 3봉 극점 기준 절대 스톱 가격(괄호는 현재가 대비 %)·**표시 전용**·진입 로직 무관 | 캔들 상태와 동일(혼조=회색) |
| `비정상 매수/매도` | cRSI 과열 + 볼린저 이탈(고/저가 기준) → `비정상 매수/매도`, 이후 반등 감시 `롱/숏 감시(강)`. 병기 `비정상 매수 \| 숏 감시`. 정상이면 양봉=`정상 매수`·음봉=`정상 매도`. 정상 복귀=이격도 중립 | 롱 감시=초록·강화=파랑 · 숏 감시=주황·강화=빨강 · 비정상 매도=아쿠아·비정상 매수=빨강 · 정상=회색 |
| `이격도 (N)` | 종가÷MA×100(기준 100) `값 (상태)`. 고정 임계+적응형(σ) 중 1개=`되돌림 주의`(둔탁)·2개=`되돌림 확실`(선명)·0개=`중립` | 과열=빨강계·과매도=초록계·확실=선명·주의=둔탁·중립=회색 |
| `되돌림 가격` | 이격도가 중립으로 복귀하는 목표가 `가격 (현재가 대비 %)`. 중립이면 `-` | 회색 |

**캔들 상태 7분류** (약세→강세, 표시 전용):

| 분류 | 의미 | 셀 색 |
| --- | --- | --- |
| `하락 진행` | 3봉이 저점을 쌓으며 순몸통 임계 이상 하락 지속 | 선명 빨강 `color.red` |
| `추세형 하락 예측` | 선행 하락 방향 2봉 저점 돌파형 강봉(지속 예측) | 둔한 빨강 `#A57A7A` |
| `반전형 하락 예측` | 상승 흐름에서 반전 패턴(장악·유성·석별·반전강봉 등) | 밝은 빨강 `#EF5350` |
| `혼조` | 데이터 부족·변동성 부족·미매칭 | 회색 `color.gray` |

> **캔들 색 표시(캔들 상태 7분류 기준 · 조건 게이트 없음)** — 현재봉 캔들 바디 색칠(`barcolor`)은 **캔들 상태 7분류 방향색(`candleBarColor`)을 조건 게이트 없이 그대로 표시**합니다. 캔들 상태가 방향 있는 계열(상승/하락)이면 그 색, **혼조**면 색 없음(`na`). 표시 여부는 **캔들 상태 (3) 표시 토글**만 연동합니다. **표시 전용** — 캔들 7분류 판정·바색 색값·테이블 셀 색은 무변경이며 진입/청산·신호·전략 변수 어디에도 연결되지 않습니다. 자세한 정의는 [SUPER-ICHIMOKU README](pinescript/SUPER-ICHIMOKU/README.md#캔들-색-표시--캔들-상태-7분류-기준조건-게이트-없음) 참조.

## 2. 거래량 압력 지표

현재 봉의 `매수 우세 / 매도 우세`를 **종가 위치**로 추정하고, `비정상 거래량`과 `평균선(SMA)`을 함께 보는 압력 확인 도구입니다.

| 요소 | 계산 · 의미 |
| --- | --- |
| Buy/Sell 추정 | `buyVolume = volume × (close−low)/(high−low)`, sell은 대칭. 체결 델타가 아니라 **종가 위치 기반 추정치** |
| 비정상 거래량 | `volume > SMA(volume, N) × (1 + 배수)`. 기본 `N=20`·`배수=1.0`(=평균의 **2배 초과**) |
| 평균선(SMA) | `SMA(volume, 봉수)` — 현재 거래량이 평소보다 실제로 큰지 확인 |

**우측 상단 포지션**

| 포지션 | 판정 |
| --- | --- |
| `롱 유리`(초록) | 평균선 **상회** + 매수 우세 |
| `숏 유리`(빨강) | 평균선 **상회** + 매도 우세 |
| `중립`(회색) | 평균선 **하회**(방향은 있어도 추진력 약함 → 관망) |

> 비정상 거래량은 포지션을 가르지 않고 `압력 방향` 셀에 녹여 표기합니다 — **비정상이면 셀 색 진하게**, 일반은 연한 색으로 강도 구분.

## 3. 삼중 스토캐스틱 지표 (Triple Stochastic Wave)

서로 다른 기간의 스토캐스틱 3개를 **역할 분담**해 웨이브(파동) 진입 타이밍을 잡는 별도 패널 보조지표입니다. CEO 확정 소스의 **계산·신호·플롯·alert를 원본 보존**하고 저장소 톤앤매너의 **우측 상단 상태 테이블을 추가**했으며, **웨이브 배경은 CEO 지시로 스토캐스틱 정배열/역배열(세 %K 서열)로 재정의**했습니다(그 외 v6 포팅 — 문법만 v6화, 나머지 거동 동일).

| 스토캐스틱 | 기본 (Length/K/D) | 역할 |
| --- | --- | --- |
| **Slow** | 20 / 9 / 9 | **컨텍스트**(큰 추세·국면) |
| **Mid** | 10 / 6 / 6 | **중간축**(정배열/역배열 서열의 가운데) |
| **Fast** | 5 / 3 / 3 | **트리거**(진입 타이밍 크로스) |

각 스토캐스틱은 표준 정의(`raw_k = 100 × (close − 최저저가)/max(범위, mintick)`, `k = SMA(raw_k, K)`, `d = SMA(k, D)`)로 계산하며, `max(range, mintick)`로 0 나누기를 가드합니다.

**신호 로직 (Slow=컨텍스트 · Mid=중간축 · Fast=트리거)**

| 신호 | 의미 |
| --- | --- |
| `slow_*_trend` | Slow %K가 %D 위/아래 + 미드라인(50) 돌파 또는 최근 룩백 내 과매도/과매수 흔적 |
| `fast_*_trigger` | Fast %K가 %D를 과매도/과매수 영역에서 상향/하향 교차 |
| `stoch_*_order` | **세 %K 값 서열** — 정배열 `fast_k > mid_k > slow_k`(상승), 역배열 `fast_k < mid_k < slow_k`(하락). 엄격 부등호 |
| `*_wave_background` | **⚠️ 정배열/역배열 재정의(CEO 지시)** — 세 %K가 **정확히 정배열/역배열**(`stoch_*_order`)일 때만 배경색 강조. 구 `컨텍스트+리셋`(`mid_*_reset`)·`Mid Pullback Lookback` 입력은 제거됨 |
| `all_*_aligned` | Slow·Mid·Fast **각각** %K가 자기 %D 위/아래(셋 다 동일 방향) → 3중 정렬 alert. (스토캐스틱 간 서열 `stoch_*_order`와는 별개 개념) |

**우측 상단 상태 테이블** (추가분 — 종합 판정 캐스케이드)

| 포지션 | 판정 순서 |
| --- | --- |
| `상승/하락 웨이브+트리거`(선명) | 정배열/역배열(웨이브) + Fast 트리거 |
| `3중 상승/하락 정렬`(선명) | Slow·Mid·Fast 각각 %K가 자기 %D 대비 동일 방향 |
| `상승/하락 셋업`(둔탁) | 정배열/역배열만(트리거 전) |
| `중립`(회색) | 그 외 |

> **밴드워킹 완화** — 3중 정렬(강추세) 중의 과매수/과매도는 반전이 아니라 추세 지속일 수 있어 `과매수(추세중)`·`과매도(추세중)`로 **회색** 표기(정렬 아닌 상태의 과열/침체만 둔탁색). **테이블 판정은 확정봉(`[1]`) 기준**이라 원본 플롯·배경보다 1봉 지연(의도된 안정화). 전 계산이 확정 `ta.*`이며 `request.security`·`lookahead` 없음 → **미래참조·리페인팅 없음**.

## 4. cRSI 지표 (RSI Cyclic Smoothed)

기본 RSI에 **지배 사이클 기반 평활**과 **동적 과매수/과매도 밴드**를 더한 버전입니다. 고정 `30/70` 대신, 최근 사이클 구간에서 cRSI가 실제로 머문 범위로 밴드를 그려 "지금 국면 기준으로 과한지"를 봅니다.

| 요소 | 의미 |
| --- | --- |
| cRSI(노랑 선) | 사이클 평활 RSI 본선 |
| cMFI(파랑 선) | 동일 주기·평활 파이프라인에 MFI(자금 흐름) 입력 — cRSI와 정렬 확인용 교차 검증선 |
| HighBand(빨강)/LowBand(초록) | 사이클 메모리 구간에서 백분위(상·하위 10%)로 잡은 동적 과매수/과매도 경계(가격축 라벨 표시) |
| 밴드폭 배경 | **좁음=진한 주황(수축)**, **넓음=보라(확장)** — 변동성/사이클 진폭 대리 |
| 수축 돌파 원 마커(녹/빨강) | **수축(주황) 구간**에서 cRSI가 밴드를 **실제로 돌파**할 때 그 밴드선 위치에 원 표시 — 녹색(골든, `crossover(crsi,ub)`)·빨강(데드, `crossunder(crsi,db)`). **밴드 위치(과매도/과매수) 조건 없음**. **진행방향 게이트**(골든=`crsi>crsi[1]` 상승 중·데드=`crsi<crsi[1]` 하락 중, 밴드 밀림 유사교차 억제) + **한 수축 구간당 각 1회 래치**(구간 새로 시작 시 리셋). **⚠️ 확장(보라) 구간 원 마커는 제거됨**(CEO 지시). 다이버전스 ▲▼ 마커는 유지 |

**우측 상단 포지션** (판정 우선순위)

| 포지션 | 판정 |
| --- | --- |
| `롱 유리` | 수축 후 상단 돌파 지속 / 확장 상승 다이버전스 / `LowBand`(과매도) |
| `숏 유리` | 수축 후 하단 돌파 지속 / 확장 하락 다이버전스 / `HighBand`(과열) |
| `중립` | 수축 중 밴드 안 / 중앙 |

## 5. OBV-ADX 지표

`OBV` 기반 `+DI / −DI / ADX` 구조에, 고정 임계 대신 **`DI Difference`의 z-score 극단값**으로 `낙폭과대`/`초과매수`를 배경으로 보여주는 버전입니다.

| 요소 | 의미 |
| --- | --- |
| `+DI / −DI / ADX` | OBV 변화량 기반 방향성·강도 (기본 `+DI=파랑`·`−DI=주황`·`ADX=빨강`) |
| 낙폭과대(녹 배경) | `diDiffZScore ≤ −Oversold Z` + `ADX ≥ 최소` |
| 초과매수(빨 배경) | `diDiffZScore ≥ Overbought Z` + `ADX ≥ 최소` |

**우측 상단 포지션**

| 포지션 | 판정 |
| --- | --- |
| `롱 유리`(강화/약화) | ADX 충분 + `+DI 우위`. **강화**=OBV 방향 에너지 증가(선명), **약화**=소진 중(둔탁) |
| `숏 유리`(강화/약화) | ADX 충분 + `−DI 우위` |
| `중립(휩쏘)` | ADX 약함(추세 힘 없음 → 관망, 1차 필터) |

> `ADX` 행에 값 + `충분/약함`, 과열이면 `(과열)` 접미. 셀 색은 DI 방향(초록/빨강)·충분=선명·약함=둔탁, **과열 구간은 과열측 색으로 덮어씀**(롱측 `#6E8F72`·숏측 `#A57A7A`). 기본 `DI Length=20`·`ADX Smoothing=20`.

**보조지표 정렬 = 신뢰도** — 거래량 압력·삼중 스토캐스틱·cRSI·OBV-ADX 포지션이 이치모쿠와 같은 방향으로 모일수록 신뢰↑. 엇갈리거나 `중립/휩쏘`가 섞이면 관망.

### 6-2. 매매 1 — 되돌림 매매 (역추세·횡보, ⚠️ 고난이도)

**auto-vwap 기준 추세 반대 또는 횡보**에서 **이격도(20)** 과매도/과매수로 **평균 복귀**에 베팅합니다. 승률보다 손익비로 이깁니다. 두 세부 케이스 — **(a) 지지/저항선 완전 이탈**(고정 %손절), **(b) 근처 지지/저항, 5% 이내 이탈 허용**(전저점/전고점 손절) — 를 롱·숏 각각 전개합니다.

⚠️ **"쌀 조건(과매도)·비쌈 조건(과열)"만으로 진입 금지.** 추세가 그대로 이어지면 계속 물립니다. 반드시 아래 **보조지표 확인**을 통과해야 방아쇠를 당깁니다.

#### 6-2a. 되돌림 롱 (하단 과매도 급락 후 반등)

하단 지지선(**하단 Band #2**·장기선 MA100·200) 부근까지 **과매도 급락** 후 VWAP 중앙선으로의 **반등**에 베팅합니다.

| 항목 | 내용 |
| --- | --- |
| **국면 조건** | auto-vwap 기준 하락/횡보 + 이치모쿠 `이격도(20)` 하단 `되돌림 확실`(과매도). VWAP 상태 셀 `하단 지지` 또는 `하단 이탈` |
| **진입 트리거** | **(a) 완전 이탈**: 종가가 **하단 Band #2**(`lowerBandValue2`)·장기선을 완전 이탈한 뒤 반등 캔들 → 진입 / **(b) 근처 지지**: 하단 지지선 **5% 이내** 이탈까지만 허용하며 지지 확인 후 반등 |
| **손절** | **(a) 고정 −4%**(진입가×0.96) / **(b) 직전 스윙 전저점**(`캔들 손절라인` 하락계열 3봉 최저점 참고) |

| **보조지표 확인** | cRSI **하단돌파 해제/복귀**(`롱 유리(과매도)`·`롱 유리(상승 다이버전스)`) · 삼중 스토 **과매도권(20)에서 Fast 골든크로스 + 정배열 성립**(`상승 웨이브+트리거`) · 이치모쿠 `롱 감시`/`롱 감시 강화` · OBV-ADX `숏 유리(약화)`→`+DI 우위` 전환(하락 에너지 소진) |

#### 6-2b. 되돌림 숏 (상단 과열 급등 후 하락)

상단 저항선(**상단 Band #2**·장기선 MA100·200) 부근까지 **과열 급등** 후 VWAP 중앙선으로의 **하락**에 베팅합니다. (롱의 지지선↔저항선·과매도↔과열·급락↔급등을 실제로 뒤집은 대칭)

| 항목 | 내용 |
| --- | --- |
| **국면 조건** | auto-vwap 기준 상승/횡보 + 이치모쿠 `이격도(20)` 상단 `되돌림 확실`(과열). VWAP 상태 셀 `상단 저항` 또는 `상단 이탈` |
| **진입 트리거** | **(a) 완전 이탈**: 종가가 **상단 Band #2**(`upperBandValue2`)·장기선을 완전 이탈(상방 오버슈트)한 뒤 되돌림 캔들 → 진입 / **(b) 근처 저항**: 상단 저항선 **5% 이내** 이탈까지만 허용하며 저항 확인 후 하락 |
| **손절** | **(a) 고정 +4%**(진입가×1.04) / **(b) 직전 스윙 전고점**(`캔들 손절라인` 상승계열 3봉 최고점 참고) |
| **목표(익절)** | **VWAP 중앙선**(`vwapValue`, 저항=빨강) — 평균 복귀 지점 |
| **손익비(RR)** | (VWAP까지 목표 폭) ÷ (손절 폭). RR 미확보 시 진입 안 함 |
| **보조지표 확인** | cRSI **상단돌파 해제/복귀**(`숏 유리(과매수)`·`숏 유리(하락 다이버전스)`) · 삼중 스토 **과매수권(80)에서 Fast 데드크로스 + 역배열 성립**(`하락 웨이브+트리거`) · 이치모쿠 `숏 감시`/`숏 감시 강화` · OBV-ADX `롱 유리(약화)`→`-DI 우위` 전환(상승 에너지 소진) |

### 6-3. 매매 2 — 추세 변곡 매매 (횡보 → 전환 초입)

**횡보 수축이 끝나고 방향이 전환되는 초입**을 노립니다. 판단은 **auto-vwap 변곡 마커 + 최근 50선 지지/저항 + 장기 추세선(MA100·200) 배열**로 합니다. 손절은 **청산선(`longClear`/`shortClear`) 기준**, 목표는 **청산선 손절 대비 2:1 이상(RR ≥ 2)**.

#### 6-3a. 추세 변곡 롱 (상승 전환 초입)

| 항목 | 내용 |
| --- | --- |
| **국면 조건** | auto-vwap `횡보 압축 배경`(밴드2 회색) 이후 상승 분출 초입. 이치모쿠 `상승 전환(0.5×→1×)` 진행 + `중기 추세` 상승 배열로 전환 |
| **진입 트리거** | auto-vwap **하단 녹색 변곡 마커**(`lower2Marker`, `DI+ 우위` 게이트) 발생 + **최근 VWAP 지지**(`vwapValue` 지지=녹) + **장기 추세선(MA100·200) 배열 정합**(가격이 배열을 상방 회복/정배열 근접) |
| **손절** | **롱 청산선**(`longClear`, 아래 트레일) 이탈 |
| **목표(익절)** | 손절 폭의 **2배 이상**(RR ≥ 2). **가격이 VWAP 100·200 아래**면 → **그 MA100·200 라인을 1차 청산 목표**로 사용. **가격이 VWAP 100·200 위**면 → **이격도(20) 기준 분할 청산**(`되돌림 가격`·`상단 저항` 도달마다 분할) |
| **손익비(RR)** | 청산선 거리로 (목표 폭 ÷ 손절 폭) ≥ 2 확인. 미달 시 진입 안 함 |
| **보조지표 확인** | 삼중 스토 **골든크로스 + 정배열**(`fast_k > mid_k > slow_k`, `3중 상승 정렬`·`상승 웨이브+트리거`) · cRSI **골든크로스**(`crossover(crsi, ub)`, `롱 유리(상단돌파)`) · OBV-ADX **추세 에너지**(`+DI 우위(상승)` + ADX `충분` + `롱 유리(강화)`) |

#### 6-3b. 추세 변곡 숏 (하락 전환 초입)

(롱의 상승 전환↔하락 전환·하단 마커↔상단 마커·지지↔저항·골든↔데드·정배열↔역배열을 실제로 뒤집은 대칭)

| 항목 | 내용 |
| --- | --- |
| **국면 조건** | auto-vwap `횡보 압축 배경`(밴드2 회색) 이후 하락 분출 초입. 이치모쿠 `하락 전환(0.5×→1×)` 진행 + `중기 추세` 하락 배열로 전환 |
| **진입 트리거** | auto-vwap **상단 빨강 변곡 마커**(`upper2Marker`, `DI- 우위` 게이트) 발생 + **최근 VWAP 저항**(`vwapValue` 저항=빨강) + **장기 추세선(MA100·200) 배열 정합**(가격이 배열을 하방 이탈/역배열 근접) |
| **손절** | **숏 청산선**(`shortClear`, 위 트레일) 이탈 |
| **목표(익절)** | 손절 폭의 **2배 이상**(RR ≥ 2). **가격이 VWAP 100·200 위**면 → **그 MA100·200 라인을 1차 청산 목표**로 사용. **가격이 VWAP 100·200 아래**면 → **이격도(20) 기준 분할 청산**(`되돌림 가격`·`하단 지지` 도달마다 분할) |
| **손익비(RR)** | 청산선 거리로 (목표 폭 ÷ 손절 폭) ≥ 2 확인. 미달 시 진입 안 함 |
| **보조지표 확인** | 삼중 스토 **데드크로스 + 역배열**(`fast_k < mid_k < slow_k`, `3중 하락 정렬`·`하락 웨이브+트리거`) · cRSI **데드크로스**(`crossunder(crsi, db)`, `숏 유리(하단돌파)`) · OBV-ADX **추세 에너지**(`-DI 우위(하락)` + ADX `충분` + `숏 유리(강화)`) |

### 6-4. 매매 3 — 추세 추종 매매 (추세 진행 중 눌림/반등)

**추세 방향으로 눌림(롱)·반등(숏)에서 진입**해 추세를 따라갑니다. 진입 근거로 잡은 눌림/반등 선을 **이탈하면 청산**합니다.

#### 6-4a. 추세 추종 롱 (상승 추세 눌림)

| 항목 | 내용 |
| --- | --- |
| **국면 조건** | 상승 추세 진행 — VWAP 상태 `상단 밴드워킹` + 이치모쿠 `중기 추세`·`장기 추세` 상승(청록) + 이격도(20) 아직 `되돌림 확실` 아님 |
| **진입 트리거** | 상승 추세 중 **VWAP 중앙선(`vwapValue`) 또는 상단 Band #1(`upperBandValue1`) + 이치모쿠 기준선** 근처로 **눌림** 후 지지 확인 매수 |
| **손절(청산)** | 진입 근거로 잡은 **눌림 선(VWAP·상단 Band #1·기준선) 이탈 시** 청산 |
| **목표(익절)** | 추세 지속 구간 홀딩(`상단 밴드워킹` 유지 동안). 밴드워킹이 꺾이면(`중앙(공방)`/`하단 이탈`) 청산 |
| **손익비(RR)** | 눌림 선 손절이 얕아 RR이 크게 확보되는 구간. 청산선(`longClear`) 참고 |
| **보조지표 확인** | 삼중 스토 **소파동 과매도권**(정배열 유지하며 과매도(20) 근처 눌림 후 `상승 트리거`) · cRSI **하단 밴드(`db`) 근처**(추세 눌림, `상승` 모멘텀 유지) · OBV-ADX `+DI 우위` + `롱 유리(강화)` 유지 |

#### 6-4b. 추세 추종 숏 (하락 추세 반등)

(롱의 상승↔하락·눌림↔반등·상단 밴드워킹↔하단 밴드워킹·상단 Band #1↔하단 Band #1·과매도권↔과매수권을 실제로 뒤집은 대칭)

| 항목 | 내용 |
| --- | --- |
| **국면 조건** | 하락 추세 진행 — VWAP 상태 `하단 밴드워킹` + 이치모쿠 `중기 추세`·`장기 추세` 하락(자홍) + 이격도(20) 아직 `되돌림 확실` 아님 |
| **진입 트리거** | 하락 추세 중 **VWAP 중앙선(`vwapValue`) 또는 하단 Band #1(`lowerBandValue1`) + 이치모쿠 기준선** 근처로 **반등** 후 저항 확인 매도 |
| **손절(청산)** | 진입 근거로 잡은 **반등 선(VWAP·하단 Band #1·기준선) 이탈(상방 돌파) 시** 청산 |
| **목표(익절)** | 추세 지속 구간 홀딩(`하단 밴드워킹` 유지 동안). 밴드워킹이 꺾이면(`중앙(공방)`/`상단 이탈`) 청산 |
| **손익비(RR)** | 반등 선 손절이 얕아 RR이 크게 확보되는 구간. 청산선(`shortClear`) 참고 |
| **보조지표 확인** | 삼중 스토 **소파동 과매수권**(역배열 유지하며 과매수(80) 근처 반등 후 `하락 트리거`) · cRSI **상단 밴드(`ub`) 근처**(추세 반등, `하락` 모멘텀 유지) · OBV-ADX `-DI 우위` + `숏 유리(강화)` 유지 |

---

> **자본 보존 우선.** 6개 케이스 모두 손익비(R:R)가 확보되지 않으면 진입하지 않습니다. **손절 임계·목표 규칙 값(되돌림 완전이탈 고정 ±4%, 변곡 RR ≥ 2, 이격도 분할 임계 등)은 잠정 전략 값**이며 코드(`.pine`)에 반영돼 있지 않고 알고리즘 수학자의 백테스트로 확정합니다. 되돌림·변곡은 **반드시 손절 기준선(청산선·전저점/전고점·고정 %)을 먼저 정하고** 방아쇠를 당깁니다. 단일 신호는 불신하며, 이치모쿠 + auto-vwap + 보조지표가 **한 방향으로 정렬(교차 검증)** 됐을 때만 진입합니다.

## 7. 매매 철학

| 원칙 | 내용 |
| --- | --- |
| **단일 신호 불신** | 신호 하나를 믿지 않는다 — 이치모쿠 + 테이블 + 보조지표가 한 방향으로 정렬됐는지 교차 검증한다 |
| **자본 보존 우선** | 얼마 벌지보다 얼마 잃을지를 먼저 정한다. 살아남으면 기회는 다시 온다 |
| **국면 규정** | 추세/횡보/역추세 중 지금이 어느 국면인지 먼저 규정하고, 국면에 맞는 방식으로만 매매한다 |
| **R:R 미확보 진입 배제** | 손익비(R:R)가 확보되지 않은 자리엔 진입하지 않는다 |
| **비대칭 베팅** | 확신이 아니라 확률로 베팅한다. 틀리면 작게 잃고 맞으면 크게 먹는다 — 그 비대칭이 전부다 |
| **기다림도 매매** | 자리가 아니면 관망이 최선의 포지션이다. 시장이 시나리오를 부정하면 시장이 아니라 나를 의심하고 물러난다 |

> 이치모쿠 백테스트 전략(자동매매) 스크립트는 추후 다시 추가될 예정입니다.
