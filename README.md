# KJH-Trading

이치모쿠(메인)와 보조지표 5종을 한 화면에 올려, **추세 추종 · 횡보 돌파 · 역추세** 세 국면을 같은 테이블로 판단하는 트레이딩뷰 Pine Script 모음입니다.

핵심은 "신호 하나를 믿는 것"이 아니라, **이치모쿠 신호 + 우측 상단 테이블 + 보조지표 포지션**이 한 방향으로 정렬됐는지 교차 검증하는 데 있습니다.

## 1. 지표 읽는 법

### 1-1. 이치모쿠 메인 신호 (8 / 22 / 44 / 22)

두 조건이 **동시에** 충족될 때 신호가 발생합니다.

| 신호 | 조건 |
| --- | --- |
| **매수(롱)** ▲ | 래깅 스팬(종가)이 기준선(22) 상향 돌파/이상 **+** 컨버전 라인(8)이 기준선 상향 돌파/이상 |
| **매도(숏)** ▼ | 위의 정확한 반대 |

- 같은 신호를 컨버전 **4 / 8 / 16** 세 스케일로 계산해 삼각형 **크기**(작음/중간/보통)로 동시 표시합니다.
- **다음 봉 확정 발화(휩쏘 필터·1봉 지연)**: 트리거가 충족된 봉에서 즉시 찍지 않고 **대기**한 뒤, **다음 봉 종가가 그 트리거 가격을 유지할 때만** 그 봉에 삼각형을 찍습니다(유지 못 하면 취소). 확정봉 종가 기준이라 리페인팅이 없고, 신호가 1봉 늦는 대신 다음 봉으로 안 이어지는 속임수 돌파(휩쏘)를 걸러 신뢰도를 높입니다. 알림·추세 잠금도 이 확정봉에서 동기화됩니다(전 스케일·매수/매도 대칭).
- **스케일 캐스케이드(하위 추세 상태 전제)**: 상위 배수 삼각형은 **하위 배수가 해당 방향 추세 상태일 때** 나옵니다 — **8(1×)은 4(0.5×)가 그 방향 추세 중**, **16(2×)은 4·8이 둘 다 그 방향 추세 중**에 그 배수의 트리거가 충족되면 발화(매도도 대칭). 여러 크기가 한 봉/짧은 구간에 겹칠수록 시간축이 정렬된 것 → **신뢰↑**. 반대 신호가 나오면 그 스케일 방향이 뒤집혀 상위 전제가 풀립니다. *(지속 추세에서 8이 이미 추세로 잠겨 새 4 신호가 8·16으로 승격 못 하던 문제 수정)*
- **트리거 스킵**: 상위 배수(8·16)의 트리거 가격이 이미 현재 봉에서 충족됐고 하위 배수가 같은 방향 추세 상태면, 상위 배수 삼각형을 현재 봉에 **바로 표시**합니다(원시 신호 대기 없이 즉시 승격, 매도도 동일). 게이트가 하위 추세 상태를 전제로 하므로 반대 신호가 나면 전제가 풀려 모멘텀·정상 무관하게 순차성이 유지됩니다(중복 방지).
- ⚠️ 추세추종이라 **횡보장 휩쏘 / 후행성**이 약점 → 신호만 믿지 말고 아래 테이블·보조지표로 검증합니다.

### 1-2. 우측 상단 테이블 — 각 행의 의미와 활용

표는 **4개 섹션 병합 헤더**(딥 네이비 `#1B2A4A`)로 묶입니다 — **[추세 확인]**(상승/하락 전환·리본 추세) → **[캔들 및 되돌림 확인]**(캔들 상태 (3)·비정상 매수/매도·이격도·되돌림 가격) → **[지지/저항 확인]**(구름대·VWAP 지지/저항·VWAP 상태·장기 추세·장기 지지/저항) → **[손익비 타입 및 계산 확인]**(손익비 5행). 아래 표는 행별 의미이며(섹션 순서 무관), `캔들 상태 (3)`은 [캔들 및 되돌림 확인] 섹션 맨 앞에 옵니다.

| 행 | 값·의미 | 실전 해석 |
| --- | --- | --- |
| `상승/하락 전환` | 다음 봉 종가가 닿아야 매수/매도 신호가 되는 **트리거 가격**(4/8/16 세 스케일) `↑/↓ 가격 (%) \| N봉`. 끝의 **`N봉`=시간 조정 투영**(현재가 유지 가정 시 그 스케일 전환=핵심 도치안 크로스가 성립하기까지의 봉수, horizon 내 미성립·데이터 부족이면 `-`, 진행 중이면 미표기). 근접(±2%) 시 **둔탁한** 포지션 색, 진행 중이면 **선명** 색 `상승/하락 추세 (기준선 이격도)`(기준 100) — 100에서 많이 벌어지면 되돌림 위험 | 신호 **직전 대기 가격** + 얼마 남았는지(가격·시간) + 진행 중 늘어난 정도 |
| `리본 추세` | `횡보 수축 / 상승 분출 / 하락 분출 (이벤트 후 가격 % \| 몇 봉째)` — **현재봉 실시간** | **횡보 수축**=휩쏘 경계(관망), **상승 분출/하락 분출**=추세 시작 신호(봉수로 진행 정도) |
| `비정상 매수/매도` | cRSI 과열+볼린저 이탈(고저) `비정상 매수/매도`, 이후 **반등 감시**(`롱/숏 감시(강)`). 둘 다 있으면 `비정상 매수 \| 숏 감시`처럼 병기, 정상이면 양봉=`정상 매수`·음봉=`정상 매도` | **역추세 진입 트리거**. 롱 감시=반등 매수 자리, 숏 감시=하락 반전 자리 |
| `이격도` | 종가÷MA×100(기준 100) · `되돌림 주의/확실/중립` | 과열/과매도 정도. **확실**이면 되돌림 임박 → 역추세 근거 |
| `되돌림 가격` | 이격도가 중립으로 복귀하는 목표가 | 역추세 매매의 **1차 목표가** 참고 |
| `구름대` | 지그재그(스윙) 레그가 구름에 **근접**(경계 ±ATR 배수 이내)할 때만 — 구름 위서 **내려오면 지지**·구름 아래서 **올라오면 저항**, 강도 약·보통·강(구름 두꺼울수록↑·접근 완만할수록↑), 앞에 양운/음운 접두. 지지/저항 아닌 **맥락**은 위치×구름색 방향색(위 양운=녹·위 음운=흐린 녹·아래 음운=빨·아래 양운=흐린 빨·구름 안=회색) | 지지/저항 **강함**이면 그 자리서 반등/저항 기대 → 역추세 근거 |
| `VWAP 지지/저항` | VWAP 위=`지지`·아래=`저항` + `(이격도 \| 유지 봉수)`. 이격도=종가÷VWAP×100(기준 100), ±2σ 밖이면 둔탁색(과이격) | VWAP이 받침(지지)/누름(저항). 이격 과대면 되돌림 경계·봉수로 지속력 판단 |
| `VWAP 상태` | `기울기(상승/하락/평탄) \| 밴드폭(확장/수축/유지) \| 밴드 상태`. 밴드 상태는 밴드워킹이면 `밴드워킹`으로만(방향은 셀 색), 그 외 `하단 지지/상단 저항`·`중앙(공방)` | 셀 색: 상승=녹(밴드워킹 선명·그 외 둔탁)·하락=빨강(밴드워킹 선명·그 외 둔탁), 평탄이면 상단 밴드워킹·하단 지지=둔탁 녹·하단 밴드워킹·상단 저항=둔탁 빨강·중앙=회색. 밴드워킹=추세 매매, 지지/저항·중앙=역추세/관망 |
| `장기 추세` | MA100·200 기반 8종 `골든/데드 크로스`·`상승/강한 상승/약한 상승`·`하락/강한 하락/약한 하락` + `(크로스 후 MA100 변화% \| 봉수)` | 큰 흐름. 추세 매매는 이 방향과 **같은 쪽**만 |
| `장기 지지/저항` | `100선 지지/저항 \| 200선 지지/저항`(몸통이 MA100/200을 완전히 이탈해야 지지/저항, 걸치면 중립) | 셀 배경: 둘 다 지지=파랑·둘 다 저항=빨강·엇갈림=둔탁(글자 흰색) |
| `손익비 타입 및 계산 확인` | (섹션 4 헤더 — 좌우 2열 병합 한 칸, 4개 섹션 헤더와 같은 딥 네이비 배경 `#1B2A4A`) 아래 손익비 5행을 묶는 구분 헤더 | 손익비 블록 시작 표시 |
| 손익비 5행 | 기준별로 **독립 5행**: `Cha 롱/숏`=**청산선 기준**(진입=종가, 목표·손절=반대편/같은편 청산선, 청산선 범위 밖이면 `계산 불가`, **셀 색=유리/중립/불리 판정**) · `VWAP 롱/숏`=**VWAP선·밴드2 기준**(VWAP 위/아래 **양쪽 계산**; regime 기준선 = 위: 하단=VWAP·상단=상단밴드2 / 아래: 하단=하단밴드2·상단=VWAP. 롱=상단기준선 목표/하단기준선 손절, 숏=반대. **밴드2 이탈 시에만** `계산 불가`, **셀 색=Cha와 동일**: ≥1.5 유리 초록/≥1.0 중립 회색/<1 불리 빨강/계산불가 회색) · `Ret 롱숏`=**되돌림 가격 기준**(목표=되돌림 가격%. 손절=**순수 E(이격 바깥 엣지 `disparityNeutralPct×retStopMult`, 초기 2.0·백테스트 확정) 우선 + 계산불가 시 고정손절 폴백** 되돌림 주의 −2%·확실 −4%(본부추천·백테스트 확정), 폴백 시 테이블 괄호 안 손절 자리를 `고정 -X.X%`로 표기(예 `L 3.20 (+6.4% / 고정 -2.0%)`). `L`=과매도 롱·`S`=과열 숏. **셀 색=별도 등급**: RR≥5 녹색/3~5 둔탁녹 `#6E8F72`/<3·계산불가 회색(<3=회색은 본부 기본값·조정 가능). **되돌림 주의/확실이면 항상 계산됨**·`계산 불가`는 이격도 중립뿐) | **진입 타당성의 핵심 숫자**. Cha가 주 손익비. 전략별 손익비 기준(아래)으로 필터 |

### 1-3. 보조지표 `포지션` 상태판

각 보조지표는 창 우측 상단에 **`롱 유리`(초록) / `숏 유리`(빨강) / `중립`(회색)** 을 한 가지로 표시합니다.

| 지표 | `롱 유리` | `숏 유리` | `중립`(관망) |
| --- | --- | --- | --- |
| 거래량 압력 | 평균선 상회 + **매수 우세** | 평균선 상회 + **매도 우세** | 평균선 하회(추진력 약함) |
| cRSI | 확장+과매도 / 상승 다이버전스 / 수축 후 상단 돌파 | 확장+과매수 / 하락 다이버전스 / 수축 후 하단 돌파 | 수축 중 밴드 안 / 중앙 |
| CCI | 과매도 뒤 골든크로스 → 동반 상승 | 과매수 뒤 데드크로스 → 동반 하락 | 무장 / 신호선 이탈 |
| OBV-ADX | ADX 충분 + `+DI 우위`(강화/약화) | ADX 충분 + `-DI 우위` | ADX 약함(휩쏘) |
| Auto VWAP | 상단 밴드워킹 / 하단 지지 | 하단 밴드워킹 / 상단 저항 | 중앙(공방) |

**정렬 = 신뢰도**: 5개 포지션이 모두 같은 방향이면 강한 신호, 거래량·OBV-ADX 포함 3~4개 일치면 보통, 엇갈리거나 `중립/휩쏘`가 섞이면 관망.

## 2. 매매 전략

세 전략은 **서로 다른 시장 국면**을 노립니다. 진입 전 반드시 `VWAP 상태`와 `손익비`로 지금이 어느 국면인지부터 구분하세요.

### 전략 A — VWAP 추세 추종 (손익비 1~2배 이상)

> **이미 방향이 난 추세를, VWAP에 기대어 계속 올라탄다.**

**국면 조건**
- `VWAP 상태` = **밴드워킹**(상단=상승 추세 / 하단=하락 추세)이 진행 중.
- 가격이 VWAP 방향으로 정렬(위=롱 / 아래=숏), `장기 추세`도 같은 방향.

**진입 체크리스트 (추세 방향으로만)**
1. **VWAP 상태** = 밴드워킹 지속 + 기울기 같은 방향. VWAP·밴드로 **되돌림(눌림)** 왔을 때 진입.
2. **이치모쿠 신호**: 진입 방향의 매수/매도 삼각형(0.5×↑). 캐스케이드가 붙으면 신뢰↑.
3. **장기 추세** = 같은 방향. 큰 흐름을 거스르지 않는다.
4. **보조지표 포지션** = 진입 방향 3개 이상(특히 OBV-ADX `강화`).
5. **손익비**: 이 국면(밴드워킹·되돌림)에선 **`VWAP 롱/숏 손익비`**(목표=밴드2 ÷ 손절=VWAP)가 추세 추종용 손익비입니다. **1~2배 이상**이면 진입. 진행 중 추세라 승률로 먹는 자리 → 손익비가 크지 않아도 됩니다. (참고: `Cha 롱/숏 손익비`는 청산선 기준 주 손익비.)

**관리**
- 손절 = **VWAP**(추세 지지선, `VWAP 손익비` 기준) 또는 진입 방향 `청산가`.
- 목표 = 반대편 `청산가`. 밴드워킹이 꺾이면(밴드폭 확장→수축 / 밴드워킹 이탈) 청산.

### 전략 B — 횡보 돌파(스퀴즈) 추세 (손익비 1.5~3배 이상)

> **에너지가 응축(수축/중앙 공방)됐다가 분출하는 순간, 새 추세 방향으로 올라탄다.**

**국면 조건**
- `VWAP 상태` = **중앙(공방)** 또는 밴드폭 = **수축** → 스퀴즈. 이후 **확장** 전환이 분출.
- `리본 추세` = **횡보(압축)** → **상승/하락**(분출).

**진입 체크리스트 (분출 방향으로 정렬될 때만)**
1. **이치모쿠 신호**: 진입 방향의 매수/매도 삼각형. 4→8 캐스케이드가 붙으면 신뢰↑.
2. **VWAP 상태** = 밴드폭 **확장** 전환 + 기울기 방향 일치. **밴드워킹**으로 바뀌면 추세 확정(→ 전략 A로 전환·연장).
3. **장기 추세** = 같은 방향이면 신뢰↑.
4. **보조지표 포지션** = 분출 방향 3개 이상(거래량 압력·OBV-ADX).
5. **손익비**: `Cha 롱/숏 손익비`(청산선 기준) **1.5~3배 이상**. 응축 구간(베이스)에 가깝게 진입해 A보다 손익비가 조금 더 큽니다.

**관리**
- 손절 = 진입 방향 `청산가`(스윕 너머) 또는 응축 구간 반대편.
- 목표 = 반대편 `청산가`. 밴드워킹이 붙으면 전략 A 관리로 이어감.
- **청산 신호**: 반대 방향 신호, 리본 추세 반대 분출, 밴드폭 재수축 중 먼저 오는 것.

### 전략 C — 역추세 매매 (손익비 5~10배 이상)

> **한쪽으로 과하게 쏠린 가격이 평균으로 되돌아오는 걸 크게 노린다. 승률은 낮아도 손익비로 이긴다.**

**국면 조건**
- `VWAP 상태` = **박스권**(상단 저항/하단 지지 반복) 또는 가격이 잡으려는 방향과 **반대로 과이동**(밴드 바깥).
- `이격도` = **되돌림 주의/확실**(과열 또는 과매도).

**진입 체크리스트 (역방향 진입)**
1. **비정상 캔들 / 반등 감시**: `롱 감시`(하단 이탈 후 반등) → 매수, `숏 감시`(상단 이탈 후 반전) → 매도. 진입 시 양방향 **모멘텀 모드**(기간 한 단계 시프트, 트리거·`상승/하락 추세` 텍스트에 `M`), 신호 뜬 봉에서 즉시 적용, 이격도 중립 복귀 시 해제. 모멘텀 삼각형은 청산선 색(매수 파랑/매도 크림슨).
2. **이격도 확실** + `되돌림 가격`을 1차 목표로.
3. **구름대 지지/저항 강함** 또는 **VWAP 상단 저항/하단 지지** 도달 → 반전 벽.
4. **보조지표 다이버전스/극단**: cRSI 과매수·과매도 + 다이버전스, CCI 극단 후 반대 크로스, OBV-ADX `약화`(추세 소진).
5. **손익비**: **5~10배 이상**. 극단 되돌림이면 **`Ret 롱숏 손익비`**(목표=되돌림 가격 ÷ 손절=**순수 E(이격 바깥 엣지 `disparityNeutralPct×retStopMult`, 초기 2.0·백테스트 확정) 우선, 계산불가 시 고정손절 폴백** 주의 −2%·확실 −4%(본부추천·백테스트 확정, 폴백 시 괄호 안 손절 자리를 `고정 -X.X%`로 표기), `L`=과매도 롱·`S`=과열 숏)가 역추세용 손익비입니다. **5배 미만이면 진입하지 않습니다.**

**관리**
- 손절 = 극단 방향 `청산가` 바로 너머(반전 실패 = 추세 지속 → 빠른 손절).
- 목표 = `되돌림 가격` → VWAP → 반대편 밴드로 **분할 청산**.
- **정상 캔들 복귀**(이격도 중립)면 되돌림 종료 → 남은 물량 정리.

### 전략 구분 요약

| 구분 | A. VWAP 추세 추종 | B. 횡보 돌파(스퀴즈) | C. 역추세 |
| --- | --- | --- | --- |
| VWAP 상태 | **밴드워킹** 진행 | 중앙/수축 → **확장** | **박스권 / 반대 과이동** |
| 트리거 | VWAP **되돌림(눌림)** | 압축 → **분출** | **비정상 캔들·반등 감시**·이격도 확실 |
| 장기 추세 | **같은 방향** | 같으면 신뢰↑ | 무관/반대 |
| 손익비 | **1~2배 이상** (`VWAP`) | **1.5~3배 이상** (`Cha`) | **5~10배 이상** (`Ret`) |
| 승률/보상 | 높은 승률·작은 보상 | 중간 | 낮은 승률·큰 보상 |

## 3. 트레이딩 철학

- **예측하지 말고 반응하라.** 미래를 맞히는 게임이 아니다. 유리한 자리가 오면 올라타고, 아니면 기다린다.
- **잃지 않는 것이 먼저다.** 얼마 벌지보다 얼마 잃을지를 먼저 정한다. 살아남으면 기회는 반드시 다시 온다.
- **확신이 아니라 확률로 베팅한다.** 어떤 진입도 틀릴 수 있다. 틀리면 작게 잃고 맞으면 크게 먹는다 — 그 비대칭이 전부다.
- **기다림도 매매다.** 자리가 아니면 관망이 최선의 포지션이다. 지루함을 못 견뎌 하는 매매가 계좌를 깎는다.
- **감정이 아니라 계획이 실행한다.** 진입·손절·목표는 방아쇠를 당기기 전에 정하고, 그다음엔 흔들리지 않고 그대로 따른다.
- **시장은 늘 옳다, 내 생각은 아니다.** 가격이 내 시나리오를 부정하면 시장이 아니라 나를 의심하고 물러난다.

> 이치모쿠 백테스트 전략(자동매매) 스크립트는 추후 다시 추가될 예정입니다.
