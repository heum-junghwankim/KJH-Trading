# 분석 리포트 작성 원칙

## 차트 스크린샷

- `imgs/` 디렉토리에 종목별 차트 스크린샷을 저장한다 (예: `imgs/HPSP.png`, `imgs/한국카본.png`)
- 리포트 생성 시 해당 스크린샷을 PDF에 포함한다 (base64 인라인 임베딩)

## 파일 이름 규칙

- 형식: `{종목}_매매전략_김프로_{날짜}.pdf`
- 날짜: YYMMDD (예: 260326)
- 예시: `HPSP_매매전략_김프로_260326.pdf`, `한국카본_매매전략_김프로_200326.pdf`

## 리포트 구조 (4섹션, 각 섹션은 반드시 1페이지에 수렴)

1. **표지** — 종목명, 현재가, 전일 대비, MA Ribbon, 한줄 요약, 차트 스크린샷
2. **4단계 분석** — 추세 구조 / 비정상 가격 감지 / 거래량 확인 / 추격 위험
3. **매매 전략** — 현재 상황 정리, 눌림 진입 구간(1차/2차 패널 2개), 진입 전 확인 3가지, 목표가(1차/2차 패널 2개), 손절가, R/R 테이블, 무효화 기준, 요약
4. **실전 체크리스트** — 진입 전 체크항목, 절대 하지 말 것, 최종 정리

### 레이아웃 규칙
- 각 섹션은 정확히 1페이지에 들어가야 한다. 내용이 넘치면 글자 크기/여백을 줄여서 맞춘다.
- 섹션 사이에 빈 페이지가 생기지 않도록 한다.
- CSS에서 `page-break-after: always`는 섹션 div에만 적용하고, 섹션 내부 요소에는 사용하지 않는다.
- `page-break-inside: avoid`를 섹션 내부 박스에 적용해 박스가 페이지를 넘어가지 않게 한다.

### 매매 전략 페이지 패널 규칙
- **진입 구간**: 1차 진입(공격적) | 2차 진입(보수적) — 가로 2열 패널로 배치
- **목표가**: 1차 목표(보수적) | 2차 목표(공격적) — 가로 2열 패널로 배치
- **손절가**: 단독 박스 (빨간 테두리)
- **R/R**: 간단한 2~4행 테이블 (진입 기준 / 목표 / 수익-손실 / R/R 비율)
- **무효화 기준**: 단독 박스 (주황 테두리)
- 진입/목표 패널 이외의 데이터를 패널로 만들지 않는다

## 차트에 표시되는 지표 해석 가이드

차트 스크린샷에는 4개 지표가 동시에 표시된다. 리포트 작성 시 아래 해석 기준을 따른다.

### 차트 레이아웃 (위에서 아래로)

| 영역 | 지표 | 역할 |
|------|------|------|
| 맨 위 리본 | MA Ribbon (SMA 50/100/200/400) | 이평선 정렬 상태로 추세 강도 판단. 50=파랑, 100=노랑, 200=빨강, 400=보라 |
| 메인 차트 | 추세 추적 + 비정상 가격 추적 (캔들) | 캔들, 스윙 라벨, VWAP, 일목 구름, 비정상 캔들 색상, 다이버전스, ATR 배경, 진입 후보, 스윕 확인 |
| 중간 패널 | 거래량 압력 추적 | 매수/매도 거래량 분리, 비정상 거래량, 전체 거래량 평균선 |
| 맨 아래 패널 | 비정상 가격 추적 (보조) | cRSI(노란선) + MFI(파란선), Reversal Risk 라벨 |

### 4단계 분석 순서: 추세 -> 자리 -> 거래량 -> 추격 위험

**1단계: 추세 추적 (방향)**
- 스윙 라벨: `HH/HL` = 상승 추세, `LH/LL` = 하락 추세
- 적응형 VWAP (빨강/파랑선): 가격이 위에 있으면 매수 우위
- 일목 기준선/구름: 가격이 구름 위 = 중기 지지 확인
- MA Ribbon 색상: SMA 50(파랑) / SMA 100(노랑) / SMA 200(빨강) / SMA 400(보라)
- MA 정렬: 가격 > SMA50 > SMA100 > SMA200 > SMA400 = 완전 강세
- 정렬 배경: 녹색 계열 = 강세 정렬, 자주 계열 = 약세 정렬

**2단계: 비정상 가격 추적 - 캔들 (자리)**
- 비정상 캔들: `주황` = cRSI 과매수 + 볼린저 상단 이탈 (과열), `하늘` = cRSI 과매도 + 볼린저 하단 이탈 (투매)
- RSI 다이버전스: `청록 삼각형` = 강세(하락 힘 약화), `갈색 삼각형` = 약세(상승 힘 약화)
- ATR 배경: `녹색/파랑` = 강세 후속 이동, `주황/빨강` = 약세 후속 이동
- 진입 후보: `어두운 녹색 동그라미+박스` = 롱, `어두운 빨강 동그라미+박스` = 숏
- 유동성 스윕: `밝은 초록 봉+다이아몬드` = 롱 확인, `밝은 빨강 봉+다이아몬드` = 숏 확인
- 신호 단계: 비정상 캔들(0단계) -> 다이버전스(1) -> ATR 배경(2) -> 진입 후보(3) -> 스윕 확인(4)

**3단계: 거래량 압력 추적 (실행)**
- 거래량 막대: `흰색` = 정상 매수, `회색` = 정상 매도, `연두` = 비정상 매수, `빨강` = 비정상 매도
- 전체 거래량 평균선: 최근 주문량의 기준선을 함께 확인
- 핵심: 눌림 바닥에서 연두색(비정상 매수) 막대가 터지면 매수 힘 확인

**4단계: 비정상 가격 추적 - 보조 (추격 위험)**
- cRSI(노란선) vs MFI(파란선): 둘의 괴리가 크면 가격과 자금 흐름 불일치
- Reversal Risk 라벨: `붉은색 Bearish H/L` = 위에서 꺾일 위험, `녹색 Bullish H/L` = 아래서 되돌릴 위험
- `H` = 강한 경고(추격 자제), `L` = 약한 경고
- Bearish H가 떠 있으면 고점 추격 금지, Bullish H가 나오면 바닥 흡수 시작 신호

### 핵심 한 줄

- `추세` = 방향, `캔들` = 자리, `거래량` = 실행, `보조` = 추격 위험
- 4개가 같은 방향으로 정렬될 때 가장 좋은 진입

## PDF 스타일 가이드

### 테마
- 다크 테마 (#0d1420 배경, #ddd 기본 텍스트)
- 강조색: #00e6c8 (청록 = 체크, 가격, 결론), #ffc83a (금색 = 섹션 제목, R/R)
- 경고색: #ff4444 (빨강 = 손절, 하지 말 것), #ff8800 (주황 = 무효화)

### 글자 크기 (최적화 기준)
- `@page { size: A4; margin: 18mm 16mm; }`
- `body { font-size: 9.5pt; line-height: 1.6; }`
- `h1`: 20pt (페이지 메인 제목)
- `h2`: 14pt (섹션 소제목, 금색)
- `h3`: 11pt (항목 제목)
- 본문: 9.5pt
- 사용 지표 설명: 8.5pt, 회색
- 결론 박스: 9pt, bold
- 푸터/면책: 7.5pt

### 박스 스타일
- **stage-box** (4단계 분석): 테두리 1px solid, 라운드 8px, padding 12px 16px, margin-bottom 10px
  - 1단계: border-color #00a0b4 (청록)
  - 2단계: border-color #44bb44 (녹색)
  - 3단계: border-color #cc8800 (주황)
  - 4단계: border-color #aa44aa (보라)
- **stage-label**: 인라인 블록, 단계색 배경, 흰색 텍스트, 8pt bold, padding 2px 8px, 라운드 4px
- **conclusion**: 배경 #1a3020, 텍스트 #88dd88, padding 6px 12px, 라운드 4px

### 매매 전략 패널
- **entry-box / target-box**: flex:1, border 1px solid #00a0b4, 라운드 8px, padding 12px 16px
  - 태그(공격적/보수적): 인라인 블록, 라운드 3px, 7pt bold, 흰색 텍스트
  - 가격: 16pt bold, #00e6c8
  - 설명: 8.5pt, 회색
- **stop-box**: 배경 #3a1010, border 1px solid #cc3333, 텍스트 중앙, 가격 16pt bold #ff4444
- **rr-table**: width 100%, border-collapse, td padding 4px 8px, border-bottom #1a3050, R/R값 #ffc83a bold
- **invalidation-box**: border 1px solid #cc8800, 라운드 8px, padding 10px
- **summary-box**: 배경 #0a3040, border #2a80b0, 라운드 6px, 텍스트 중앙, #00e6c8 bold

### 체크리스트
- **checklist-item**: flex 레이아웃, 체크박스(14px, border #00a060) + 텍스트, 홀짝 배경 교차 (#16203a / #1a2540), padding 8px 12px
  - 제목: #00e6c8 bold 9.5pt
  - 설명: 기본색 9pt
  - 경고: 빨강 8pt
- **dont-box**: 배경 #3a1010, border #cc3333, 항목 #ff8888
- **final-box**: 배경 #0a2040, border #2a80b0, 텍스트 중앙

### 표지
- 상단 블록: 배경 #16203a, 라운드 12px, padding 20px, 제목 20pt bold
- 현재가/전일대비: flex 2열, border #1a3050, 라운드 8px, 가격 18pt bold #00e6c8
- MA Ribbon: 배경 #0a1830, border #1a3050, 라운드 8px, 제목 #ffc83a bold
- 한줄 요약: 배경 #006644, 라운드 6px, 중앙 정렬, 흰색 bold
- 차트 이미지: width 100%, 라운드 6px

### 공통
- 모든 박스: `page-break-inside: avoid`
- 섹션 div: `page-break-after: always` (마지막 섹션 제외)
- 푸터: 중앙 정렬, #555, 7.5pt, 상단 border-top #1a3050

## PDF 한글 폰트 규칙 (필수)

한글이 포함된 PDF를 생성할 때 반드시 아래 규칙을 따른다.

### 절대 금지
- fpdf2, reportlab 등 Python PDF 라이브러리로 직접 한글 PDF를 만들지 않는다
- .ttc (TrueType Collection) 파일을 PDF 라이브러리에 직접 전달하지 않는다
- CFF/PostScript 아웃라인 폰트(.otf, .otc, OTTO 시그니처)를 PDF에 임베딩하지 않는다
- PDF 생성을 위한 별도 Python 스크립트(.py) 파일을 만들지 않는다. HTML 문자열을 직접 구성해서 WeasyPrint로 변환한다.

### 필수 방식: WeasyPrint (HTML -> PDF)
1. pip install weasyprint --break-system-packages
2. HTML + CSS로 문서를 작성한다
3. font-family: 'Apple SD Gothic Neo', 'Noto Sans CJK KR', sans-serif 를 사용한다
4. weasyprint.HTML(string=html).write_pdf(output_path) 로 변환한다

### 이유
WeasyPrint는 시스템의 Pango/FreeType 엔진으로 렌더링 후
TrueType 서브셋으로 올바르게 임베딩하므로,
모바일, 웹 뷰어, 데스크탑 등 모든 플랫폼에서 한글이 깨지지 않는다.

### 한글 특수문자 주의
유니코드 특수기호가 폰트에 없을 수 있다.
HTML 엔티티나 해당 폰트에 존재하는 문자만 사용한다.
